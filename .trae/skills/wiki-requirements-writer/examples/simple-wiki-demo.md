# BR-001 提升登录转化率

> 下级实现：
> - [[UR-001|新用户快速登录]]

## 1. 目标

- 业务目标：降低新用户首次登录流失率
- 成功指标：
  - 登录转化率从 `32%` 提升到 `45%`
  - 登录流程主路径不超过 `3` 步

## 2. 范围

### In Scope

- 手机号验证码登录
- 验证码发送与校验

### Out of Scope

- 第三方社交登录
- 多设备会话管理

# UR-001 新用户快速登录

> 上级来源：[[BR-001|提升登录转化率]]
>
> 下级实现：
> - [[SR-001|短信验证码登录服务]]

## 1. 用户故事

- 目标用户：首次访问的新用户、忘记密码的老用户
- 用户故事：
  - 作为一个首次访问的新用户，我希望通过手机号验证码快速登录，以便我无需先注册和记忆密码。

## 2. 场景

- 前置条件：用户未登录，已进入登录页
- 主流程：
  1. 用户输入手机号
  2. 系统校验手机号格式
  3. 用户点击发送验证码
  4. 系统发送验证码并开始倒计时
  5. 用户输入验证码并提交
  6. 系统校验成功后创建登录态

## 3. 验收标准

- Given 用户输入合法手机号
  When 用户点击`发送验证码`
  Then 系统应发送验证码并显示 `60` 秒倒计时

- Given 用户输入正确验证码
  When 用户点击`立即登录`
  Then 系统应创建登录态并跳转首页

# SR-001 短信验证码登录服务

> 上级来源：[[UR-001|新用户快速登录]]
>
> 下级实现：
> - [[DR-001|发送验证码规则]]
> - [[DR-002|验证码校验规则]]

## 1. 功能定义

- 功能目标：提供短信验证码登录能力
- 输入：手机号、验证码
- 输出：登录结果、会话信息

## 2. 业务规则

- BR-001：验证码长度固定为 `6` 位数字
- BR-002：验证码有效期为 `60` 秒
- BR-003：同一手机号 `60` 秒内不得重复发送
- BR-004：连续输错 `5` 次后锁定 `15` 分钟

## 3. 状态流转

- `INIT -> CODE_SENT`
- `CODE_SENT -> VERIFIED`
- `CODE_SENT -> EXPIRED`
- `VERIFIED -> LOGGED_IN`

## 4. 非功能要求

- 性能：发送接口 `P95 <= 500ms`
- 可用性：登录服务可用性 `>= 99.9%`
- 安全：验证码不得明文落库，日志中手机号需脱敏

# DR-001 发送验证码规则

> 上级来源：[[SR-001|短信验证码登录服务]]

## 1. 输入输出

### 输入

| 输入项 | 类型 | 必填 | 约束 | 来源 |
|:---|:---|:---|:---|:---|
| mobile | string | 是 | 手机号格式合法 | 登录页 |

### 输出

| 输出项 | 类型 | 约束 | 去向 |
|:---|:---|:---|:---|
| sendResult | boolean | 发送成功或失败 | 前端页面 |

## 2. 处理逻辑

1. 校验手机号格式
2. 检查发送频率限制
3. 生成验证码并加密存储
4. 调用短信网关发送

## 3. 异常与错误码

| 错误码 | 触发条件 | 系统响应 | 用户提示 |
|:---|:---|:---|:---|
| SMS-001 | 手机号格式错误 | 拒绝发送 | 手机号格式不正确 |
| SMS-002 | 发送过于频繁 | 拒绝发送 | 请稍后再试 |
| SMS-003 | 短信网关超时 | 记录日志并返回失败 | 发送失败，请稍后重试 |

---

# DR-002 验证码校验规则

> 上级来源：[[SR-001|短信验证码登录服务]]

## 1. 业务断言

- 验证码过期后不得继续使用
- 校验成功后验证码立即失效

## 2. 验收标准

- Given 验证码已过期
  When 用户点击`立即登录`
  Then 系统应提示`验证码已过期`

- Given 用户连续输错 `5` 次验证码
  When 用户再次尝试提交
  Then 系统应提示当前手机号已被临时锁定
