# apply-retrospective 自查清单

## 复盘前 — 前置校验

- [ ] openspec 模式：`verify.md` 存在
- [ ] openspec 模式：`verify.md` 未被标记为 `❌ FAIL`
- [ ] git log 可识别提交范围（`<base>..HEAD`）
- [ ] 确认输出目录（openspec 模式自动；通用模式默认 `docs/review/`，禁止输出到系统临时目录（如 `/tmp/`、`/var/tmp/`））

## 复盘中 — 生成质量

### §0 Evidence

- [ ] 提交范围和 diff 规模已通过命令获取（非估算）
- [ ] 任务完成率已从 tasks.md 精确计算（非目测）
- [ ] 条目按实际情况动态增删（非机械套用默认清单）
- [ ] 每条数据可被后续章节引用

### §1 Wins

- [ ] 每项声明引用 §0 中的具体证据（commit hash / file / test）
- [ ] 每项 evidence 引用可定位到证明「成功」的具体文件/行（非「a1b2c3d: 代码实现顺利」式空洞引用）
- [ ] 无空洞的自我表扬（如"完成得很好""一切顺利""无特殊问题"）

### §2 Misses

- [ ] 严重程度分级正确（🔴阻塞 / 🟡痛苦 / 📌小问题）
- [ ] 每项引用具体证据
- [ ] 不涉及代码质量评价（对照边界红线表检查）
- [ ] 如无 🔴 无 🟡 项：已出具「零问题声明」并回答以下 3 条自问：
  - 「本次变更真的没有阻塞/痛苦的问题吗？」
  - 「是否有我选择忽视的问题（因为不便记录）？」
  - 「如果有外部观察者复盘，会同意零问题吗？」
- [ ] 所有 📌 项合计不超过 3 个，超过则视为把严重问题降级 → 重新分级

### §3 Plan deviations

- [ ] 对比了 plan.md/tasks.md 与实际执行（openspec 模式）
- [ ] 每个偏差说明了「变化内容」和「原因」
- [ ] 通用模式：如实标注无正式计划

### §4 Skill compliance

- [ ] 仅列出 apply 阶段技能（无 brainstorming/writing-plans 等 plan 阶段技能）
- [ ] 每个 ✗ 填写了 What / Why / How to prevent 三问
- [ ] Why 给出了具体 trigger（引用 git log --stat 输出或 AGENTS.md/CLAUDE.md 规则原文），无模糊理由
- [ ] Why 未使用禁止理由清单中的任一表述（不需要/太小/没时间/纯配置/变更量小/单提交/手动等价/已知模式/没必要）
- [ ] How to prevent 选了具体方案（非空泛承诺）；`n/a — skip justified` 仅当 Why 引用了 AGENTS.md/CLAUDE.md 的明确跳过规则

### §5 Surprises

- [ ] 记录了被证伪的假设（非"一切顺利"）
- [ ] 写入 "(none observed)" 前已检查：「是否有任何我预期会发生但没发生的事？是否有任何我没预期但发生了的事？」
- [ ] 无内容时写了 `(none observed)` 占位

### §6 Promote candidates

- [ ] 每条包含 Why 和 How to apply
- [ ] 晋升目标选择合理
- [ ] 未勾选的 `- [ ]` 可供下周期 grep 提取

## 复盘后 — 输出验证

- [ ] 文件写入正确位置
- [ ] 所有章节均已生成（无遗漏跳过的章节）
- [ ] 满足跳过条件时，写入了明确的跳过理由（含 `git diff --stat` 输出）
- [ ] 中文输出，格式正确

## 越界扫描（必须执行，不可跳过）

- [ ] 全文扫描越界红灯关键词，任一命中 → 复核是否越界：
  - `函数过长` `命名不清晰` `耦合` `抽象层` `覆盖率不够` `commit 太大`
  - `代码质量` `架构问题` `命名差` `格式不一致` `测试不足` `设计缺陷`
- [ ] 所有「看起来像质量评价但归属过程视角」的表述已改写
- [ ] **未替代 code review** — 报告中无代码质量评价
- [ ] **未替代 openspec review** — 报告中无设计文档评价

## 理性化自查（必须执行，不可跳过）

- [ ] 已对照「常见理性化借口对照」表逐条检查，无命中
- [ ] 已对照 Red Flags 列表逐条检查，无命中
- [ ] 如 §2 零 🔴🟡，零问题声明 + 3 条自问已通过

## 边界检查

- [ ] **向前指针就绪** — 如果这是已有 retro 的后续，已检查是否需要 Update 指针
- [ ] **Carry-forward 已执行** — 如果存在历史未勾选 candidate，已逐条处理
