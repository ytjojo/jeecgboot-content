# 用户短信登录 PRD

## 1. 文档信息

- 文档名称：用户短信登录
- 版本：V1.0
- 状态：Draft
- 负责人：产品经理
- 日期：2026-05-03

## 2. 背景与问题

当前用户登录依赖账号密码，注册和登录路径较长，导致首访用户流失率较高。需要提供短信验证码登录能力，降低首次登录门槛。

## 3. 目标与成功指标

- 业务目标：提升新用户登录转化率
- 产品目标：提供手机号 + 验证码快捷登录
- 成功指标：
  - 新用户登录转化率从 `32%` 提升到 `45%`
  - 登录流程操作步骤不超过 `3` 步

## 4. 范围

### In Scope

- 手机号输入
- 验证码发送
- 验证码校验
- 登录成功后建立会话

### Out of Scope

- 第三方社交账号登录
- 多设备登录管理
- 风控策略后台配置

## 5. 用户与场景

- 目标用户：首次访问的新用户、忘记密码的老用户、系统管理员
- 用户故事：
  - 作为一个首次访问的新用户，我希望使用手机号验证码快速登录，以便我不用先注册和记忆密码。
  - 作为一个系统管理员，我希望查看短信发送失败和频控记录，以便排查登录转化异常。

## 6. 需求层级（Epic / Feature / Story）

### EPIC-001 降低用户登录门槛

- 业务目标：提升新用户登录转化率
- Source / Reference：N/A
- Owner：产品经理

#### FEATURE-001 短信验证码登录

- Feature 目标：支持用户通过手机号和验证码完成登录
- 涉及角色：首次访问的新用户、忘记密码的老用户、系统管理员
- 优先级：Must

##### STORY-001 发送短信验证码

- 用户 / 角色：首次访问的新用户
- 用户价值：快速获取验证码并继续登录流程
- 验收标准：
  - Given 用户输入合法手机号
    When 用户点击`发送验证码`
    Then 系统应成功发送验证码并展示 `60` 秒倒计时
- 边界与异常：手机号格式错误、短信网关超时、重复发送

##### STORY-002 管理员查看短信异常

- 用户 / 角色：系统管理员
- 用户价值：定位短信发送失败、频控命中等影响登录转化的问题
- 验收标准：
  - Given 存在短信发送失败记录
    When 系统管理员查看短信异常日志
    Then 系统应展示失败原因、手机号脱敏信息和发生时间
- 边界与异常：无异常记录、权限不足、日志查询超时

## 7. 业务流程

1. 用户点击登录按钮
2. 用户输入手机号
3. 系统校验手机号格式
4. 系统发送验证码
5. 用户输入验证码并提交
6. 系统校验验证码
7. 校验通过后创建登录态并跳转首页

## 8. 功能需求

### FR-001 发送验证码

- 优先级：Must
- 触发方式：用户点击`发送验证码`
- 前置条件：手机号格式校验通过
- 业务规则：
  - 验证码为 `6` 位数字
  - 验证码有效期为 `60` 秒
  - `60` 秒内不可重复发送
  - 单个手机号 `24` 小时内最多发送 `10` 次
- 异常处理：
  - 手机号格式错误时，不允许发送
  - 短信网关超时时，提示`发送失败，请稍后重试`

### FR-002 验证码登录

- 优先级：Must
- 触发方式：用户点击`立即登录`
- 前置条件：用户已输入手机号和验证码
- 业务规则：
  - 验证码错误连续 `5` 次后，当前手机号 `15` 分钟内禁止继续尝试
  - 登录成功后生成用户会话
- 边界条件：
  - 验证码过期
  - 重复点击提交
  - 网络中断

## 9. 交互说明

- 发送验证码后，按钮进入倒计时状态
- 验证中时，登录按钮显示加载中，不可重复点击
- 验证失败时，保留用户已填写的手机号

## 10. 数据与埋点

- 事件：`login_sms_send_click`
- 事件：`login_sms_send_success`
- 事件：`login_sms_submit`
- 事件：`login_sms_success`

## 11. 非功能需求

- 性能：验证码发送接口 `P95 <= 500ms`
- 可用性：登录服务可用性不低于 `99.9%`
- 安全性：验证码不得明文落库，日志中手机号需脱敏

## 12. 验收标准

- Given 用户输入合法手机号
  When 用户点击`发送验证码`
  Then 系统应成功发送验证码并展示 `60` 秒倒计时

- Given 用户输入正确验证码
  When 用户点击`立即登录`
  Then 系统应创建登录态并跳转首页

- Given 用户输入过期验证码
  When 用户点击`立即登录`
  Then 系统应提示`验证码已过期`

## 13. 风险与依赖

- 依赖短信服务商接口可用
- 若短信通道不稳定，可能影响登录转化

## 14. 里程碑 / 发布计划

- MVP 范围：手机号验证码发送、验证码校验、登录态创建、基础异常日志
- 灰度方案：先对 `10%` 新用户开放，观察发送成功率和登录转化率
- 回滚条件：短信发送成功率低于 `95%` 或登录错误率明显上升

## 15. 词汇表 (Glossary)

| 术语 | 定义 | 备注 |
| --- | --- | --- |
| 验证码有效期 | 用户收到验证码后可用于登录的时间窗口 | 本需求为 `60` 秒 |
| 频控 | 为防止滥用，对发送次数或校验次数进行限制 | 按手机号维度统计 |
| 系统管理员 | 负责查看配置、异常日志和运营治理信息的后台角色 | 不等同于普通用户 |

## 16. 开放问题

- 是否需要补充图形验证码防刷
