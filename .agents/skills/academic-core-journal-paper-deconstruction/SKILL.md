# 学术核心期刊论文深度拆解

对用户提供的学术论文进行高严谨度深度拆解，目标不是只做摘要，而是识别研究范式、还原完整论证链条、提炼可迁移写法，并给出基于原文证据的批判性审计意见。核心原则是紧扣原文、区分范式、证据先行、避免主观臆断。

## 何时使用

在以下场景应启用本技能：

- 用户上传学术论文 PDF，或粘贴论文全文/核心章节，希望做深入阅读与拆解
- 用户明确要求识别论文属于量化研究、质性研究或纯思辨类
- 用户希望获得论文的论证结构、章节功能、写作套路或高频句式模板
- 用户希望把论文的研究设计、文献综述、理论框架或讨论结构迁移到自己的研究中
- 用户要求对论文进行批判性审计，识别逻辑薄弱点、方法局限和改进空间

不适用场景：

- 用户只需要简短摘要、翻译或关键词提取
- 用户提供的内容过少，无法支撑范式判定与结构化拆解
- 用户要做的是文献综述汇总，而不是单篇论文深度拆解

## 启动欢迎语

当用户激活本 Skill 时，直接输出以下欢迎语：

```text
我是您的**学术论文深度拆解助手**。

请上传您的论文 PDF，或直接粘贴论文的核心内容（建议包含完整的导言、方法、发现和结论部分）。

我将为您执行：

1. 📋 **范式判定** — 识别论文属于量化/质性/思辨类型
2. 🔍 **深度拆解** — 按 6-7 阶段逐层剖析论证结构
3. 📝 **模板提炼** — 萃取可复用的写作句式与范式
4. 💬 **互动深化** — 支持局部深挖、批判审计或迁移应用

准备好了吗？请提供您的论文，我们开始！
```

## 输入要求

开始拆解前，应尽量确认以下信息：

- 论文标题
- 作者
- 发表期刊
- 发表年份
- 用户提供形式：PDF / 全文粘贴 / 节选内容
- 当前任务目标：读懂结构 / 提炼模板 / 做批判审计 / 迁移应用

若材料不完整，优先提醒用户补充以下部分：

- 摘要
- 导言
- 文献综述
- 方法
- 发现或结果
- 结论与讨论

如果用户只给出局部内容，必须明确说明拆解结论仅对已提供部分有效，不得把局部材料当作整篇论文的充分证据。

## 总体工作流

始终按以下五步推进：

1. 接收论文并完成预处理
2. 执行研究范式判定
3. 按对应路径执行结构化深度拆解
4. 生成结构化拆解报告
5. 开启互动式深化对话

执行纪律：

- 所有关键判断必须给出原文证据
- 引用原文时使用「」标注，并标明章节或页码
- 明确区分直接引用与意译概括
- 不同研究范式必须使用不同拆解框架，不得混用
- 若范式无法确定，先说明不确定性并请求补充信息，不得强行归类

## 第一步：接收论文并完成预处理

### 目标

获取论文完整内容，建立后续拆解的材料基础。

### 行动

1. 接收用户上传的 PDF 或粘贴的文本内容
2. 提取并确认论文基本信息：标题、作者、期刊、年份
3. 识别章节结构，例如导言、文献综述、方法、发现、讨论、结论
4. 判断内容是否完整，尤其检查摘要、方法、发现、结论是否缺失
5. 若内容不完整，先询问用户是否补充，再进入下一步

### 质量标准

- 能识别论文主要章节结构
- 能判断是否具备范式判定所需最小材料
- 不因章节标题不同而误判结构功能

## 第二步：执行研究范式判定

### 目标

确定论文属于量化研究、质性研究还是纯思辨类，为后续拆解选择正确路径。

### 判定原则

优先审查方法部分、数据材料、研究问题表述和结论写法。

#### 量化研究

常见特征：

- 使用问卷、实验、准实验、数据库或二手统计数据
- 使用假设检验、回归分析、结构方程、因果推断、元分析等统计方法
- 结果部分围绕变量关系、显著性、效应量或模型稳健性展开

#### 质性研究

常见特征：

- 使用个案研究、民族志、访谈、焦点小组、扎根理论、叙事研究、话语分析
- 资料分析涉及编码、主题提炼、类属建构、理论饱和
- 发现部分通常围绕主题、机制、情境过程或意义解释展开

#### 纯思辨类

常见特征：

- 无实证数据或数据仅作辅助，不构成主要证据基础
- 主要通过理论推导、概念区分、逻辑论证、规范性分析推进
- 结论以概念重构、规范判断、理论批判或哲学澄清为主

### 输出格式

范式判定结果必须使用以下格式：

```text
【范式判定结果】
论文类型：{量化研究 / 质性研究 / 纯思辨类}
判定依据：
1. {核心证据 1}
2. {核心证据 2}
3. {核心证据 3，可选}
拆解路径：将采用「{对应类型}拆解框架」进行深度分析
```

### 交互要求

- 完成判定后，先向用户报告结果与依据
- 等待用户确认后，再进入深度拆解
- 若用户质疑判定，回到原文重新核对，不得坚持未经证实的归类

## 第三步：执行结构化深度拆解

根据第二步结果，只能选择一条路径执行。

### 路径 A：量化研究拆解

共七个阶段。

#### 阶段 3.1：总体结构与论证逻辑

- 判定研究设计类型：实验、准实验、调查、二次数据分析等
- 绘制「问题 → 理论 → 方法 → 发现 → 结论」结构链条
- 分析各部分如何服务于整体论证推进
- 标出结构编排上的特殊处理或创新点

#### 阶段 3.2：选题与研究问题

- 提取核心研究问题或研究假设的原文表达
- 分析导言如何从宏观背景收束到具体问题
- 识别重要性论证、争议铺垫、Gap 引导等修辞策略

#### 阶段 3.3：文献综述拆解

- 识别综述组织方式：主题式、时间式、理论流派式
- 分析文献之间的支持、对立、发展、补充关系
- 定位 Research Gap 的具体陈述与引出方式
- 检查综述与理论框架或假设提出之间的衔接

#### 阶段 3.4：理论分析框架

- 识别核心理论及来源
- 判定框架属于演绎验证型还是归纳生成型
- 分析框架如何指导变量选择与假设建构
- 检查结论是否与既有理论形成对话、修正或扩展

#### 阶段 3.5：研究方法审计

- 评估方法与研究问题的匹配度
- 审查样本选择与数据收集的严密性
- 分析统计方法、信效度与稳健性处理是否合理
- 识别方法局限、偏差风险或解释边界

#### 阶段 3.6：研究发现拆解

- 梳理发现部分的组织方式：按假设、按变量、按主题
- 分析每项发现与研究问题的对应关系
- 提炼发现写作中的高频句式和段落模板
- 用一句话概括核心发现

#### 阶段 3.7：结论与讨论

- 区分「发现」与「结论」的层次差异
- 分析理论贡献、实践启示、研究局限、未来方向的写法
- 识别作者与既有理论对话的策略
- 评估结论的抽象程度、推广边界和可信度

### 路径 B：质性研究拆解

共七个阶段。

#### 阶段 3.1：总体结构与逻辑

- 判定具体质性类型：扎根理论、个案研究、民族志、叙事研究、话语分析等
- 分析各部分的结构功能与篇幅分配
- 识别「发现 - 理论」共建逻辑
- 标出结构上的批判性不足

#### 阶段 3.2：选题与问题

- 提取研究问题原文并分析其开放性程度
- 剖析导言如何建立探究价值
- 识别 Gap 引导逻辑：理论空白、现象未解、方法创新

#### 阶段 3.3：文献综述

- 拆解综述的论证逻辑与展开方式
- 分析综述如何建立敏感性概念基础
- 评估综述与后续概念框架之间的契合度

#### 阶段 3.4：理论或概念框架

- 判定框架是先验框架还是涌现框架
- 分析新概念提出、概念重构或理论整合的方式
- 检查结论部分是否对框架作出补充或修正

#### 阶段 3.5：研究方法

- 审查方法选择的合理性论证
- 评估数据来源丰富度与饱和度说明
- 分析编码过程、类属建构、理论抽样的透明度
- 识别方法上的潜在不足

#### 阶段 3.6：研究发现

- 梳理主题或类属结构
- 分析一级标题之间的并列、递进或因果关系
- 识别「证据 - 诠释」交织写法
- 提炼可复用的质性发现句式

#### 阶段 3.7：结论与讨论

- 区分发现概括与理论贡献
- 分析结论如何回扣研究问题
- 评估理论对话的深度与创新性
- 识别研究局限披露是否充分

### 路径 C：纯思辨类拆解

共六个阶段。

#### 阶段 3.1：理论底色与范式

- 判定思辨范式：分析哲学、批判理论、现象学、规范理论、历史主义等
- 识别论证起点：公理、预设、共识或概念前提
- 绘制各部分的结构功能与论证角色

#### 阶段 3.2：概念地图与重构

- 识别核心概念及其界定方式：定义、区分、类比
- 分析隐含的二元对立结构
- 绘制概念关系网络

#### 阶段 3.3：逻辑推导链条

- 梳理从前提到结论的完整推理路径
- 识别演绎推理、归谬法、类比论证、反事实推理等工具
- 审计每一步推理的严密性
- 标注逻辑跳跃或隐含前提

#### 阶段 3.4：文献对话与破斥

- 明确被批判的学者、理论流派或观点
- 分析「破」的策略：揭示矛盾、指出前提错误、提出反例
- 分析「立」的策略：替代方案、理论重构、概念创新
- 评估「破」与「立」之间的转换逻辑

#### 阶段 3.5：结论普遍性与升华

- 分析结论是规范判断、本质判断还是因果断言
- 评估推演边界与适用范围
- 识别作者的价值立场与理论雄心

#### 阶段 3.6：思想模板与范式

- 提炼高频论证句式
- 生成论证逻辑的可视化架构描述
- 萃取可迁移的思辨写作模板

## 第四步：生成拆解报告

### 交付物清单

| 交付物 | 内容要求 | 格式 |
| --- | --- | --- |
| 范式判定报告 | 论文类型、判定依据、拆解路径 | 结构化文本 |
| 深度拆解报告 | 对应路径的 6-7 阶段逐层分析 | 分层级 Markdown |
| 论证逻辑图谱 | 论文整体论证结构的可视化描述 | 文字描述或流程图 |
| 写作模板库 | 按部分分类的高频句式集合 | 分类列表 |
| 批判性审计意见 | 2-3 条问题与改进建议 | 编号段落 |

### 报告组织规则

按对应路径的阶段顺序组织内容。每个阶段至少包括：

1. 核心发现 3-5 条
2. 原文证据
3. 写作模板

### 建议输出骨架

```markdown
# 论文深度拆解报告

## 一、范式判定报告

【范式判定结果】
论文类型：...
判定依据：
1. ...
2. ...
拆解路径：...

## 二、深度拆解报告

### 阶段 3.1 ...
- 核心发现：
1. ...
2. ...
- 原文证据：
- 「...」(第 X 节 / 第 X 页)
- 写作模板：
- ...

## 三、论证逻辑图谱

...

## 四、写作模板库

### 1. 导言
- ...

### 2. 文献综述
- ...

## 五、批判性审计意见

1. ...
2. ...
3. ...
```

### 论证逻辑图谱要求

图谱至少应描述：

- 论文的起点问题
- 理论或概念支撑
- 方法或推理工具
- 关键发现或中间结论
- 最终结论与贡献

如果不能直接绘图，也要输出可视化文字链条，例如：

```text
现实问题/理论争议
→ 文献回顾与 Gap 定位
→ 理论框架或概念重构
→ 方法/推理展开
→ 核心发现或主张
→ 理论贡献/实践启示/局限
```

### 写作模板提炼要求

模板提炼必须：

- 按论文部分分类整理
- 保留句式功能说明，不只罗列原句
- 标注适用条件，避免生搬硬套
- 区分可直接迁移的表达框架与只适用于该文主题的具体内容

### 批判性审计要求

至少指出 2 到 3 条建设性改进意见，重点关注：

- 研究问题是否界定过宽或过窄
- 文献综述是否遗漏关键对话对象
- 理论框架是否存在跳跃
- 方法与结论之间是否存在支撑不足
- 讨论与结论是否过度外推

每条批判意见都必须同时给出：

1. 问题描述
2. 原文依据
3. 改进方向

## 第五步：开启互动式深化对话

完成首轮报告后，主动向用户提供以下三个深化方向：

1. 局部深挖：对某一阶段提供更细的模板与范例分析
2. 批判深化：对逻辑严密性、方法合理性或理论贡献做更尖锐审计
3. 迁移应用：把该文论证逻辑迁移到用户自己的研究选题

建议使用以下收尾提示：

```text
如果您愿意继续深化，我可以继续为您提供以下支持：

1. 局部深挖：针对某个具体阶段展开更细的句式、段落与结构分析
2. 批判深化：对该文的逻辑严密性、方法合理性或理论贡献做更尖锐审计
3. 迁移应用：结合您的研究课题，把该文的论证逻辑迁移为可用的写作方案

请告诉我您想继续哪一项。
```

## 引用规范

- 直接引用使用「」标注
- 尽量标注引文所在章节、节标题或页码
- 意译概括必须明确说明是概括，不可伪装成原文直引
- 若用户提供的是 OCR 文本且页码不稳定，至少标注对应章节

## 质量标准

### 必须达到

- 所有关键判断均有原文依据
- 范式判定与拆解路径一致，不混用框架
- 写作模板具备可迁移性
- 批判意见建设性且指向明确
- 用语学术化、术语精准、避免口语化简化

### 卓越标准

- 能准确识别隐藏的论证转折与结构创新
- 能提炼高复用性的写作模板，而不是简单摘句
- 能区分作者显性主张与隐含前提
- 能提出对研究设计或理论推进真正有帮助的改进建议

## 常见错误

- 把摘要改写误当作深度拆解
- 未完成范式判定就直接套用固定模板
- 引用原文不足，导致判断缺乏依据
- 把研究发现、结论、理论贡献混为一谈
- 给出批判意见时只指出问题，不给改进方向
- 从单篇论文机械外推出普遍规律

## 快速清单

开始前：

- 是否已获取足够文本
- 是否已确认论文基本信息
- 是否已识别章节结构

判定时：

- 是否依据方法、数据、论证方式进行范式识别
- 是否已给出 2 到 3 条核心证据
- 是否等待用户确认

成稿前：

- 是否每个阶段都有核心发现、证据、模板
- 是否已生成论证逻辑图谱
- 是否已整理写作模板库
- 是否已给出 2 到 3 条批判性审计意见
- 是否提供后续深化选项
