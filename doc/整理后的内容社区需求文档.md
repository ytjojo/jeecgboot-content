# 内容社区系统需求文档（整理版）

## 1. 文档说明

本文档基于原始需求内容进行整理与合并，统一术语、结构与表达方式，覆盖后台管理系统与内容服务能力，便于产品、设计、研发与测试协同推进。

### 1.1 目标
- 明确后台管理核心能力范围
- 规范内容系统业务模型与内容类型定义
- 提供非功能性要求作为交付标准

### 1.2 适用范围
- 管理后台（Admin）
- 内容服务（Content Service）
- 社区运营系统
- 审核与风控系统

---

# 2. 系统架构概览

## 2.1 技术栈（建议）
- 后端：Spring Boot + MyBatis-Plus
- 数据库：PostgreSQL（支持 JSONB）
- 前端：Vue 3
- 文件存储：对象存储（OSS / S3）
- 实时通信：WebSocket

## 2.2 设计原则
- 高内聚、低耦合
- 模块化扩展
- 数据一致性
- 权限隔离
- 安全合规
- 高可用与可观测性

---

# 3. 内容系统设计

## 3.1 内容类型
系统统一支持以下内容类型：

| 类型 | 说明 | 主要特征 |
|---|---|---|
| 文章 | 长内容输出 | 支持富文本 / Markdown |
| 笔记 | 图文记录 | 最多18张图片 |
| 动态 | 即时内容 | 图文 / 视频混排 |
| 视频 | 视频内容 | 单视频主内容 |
| 问题 | 问答提问 | 标题 + 描述 |
| 回答 | 问答回复 | 支持采纳最佳答案 |

## 3.2 编辑器能力
- 富文本编辑器（结构化 JSON 存储）
- Markdown 编辑器
- 图片、视频、音频、文档上传
- 草稿保存
- 自动恢复

## 3.3 内容附加元素
- @用户
- #话题
- 标签
- 外链
- 股票卡片
- 商品卡片
- 投票卡片
- 地理位置
- Emoji 表情
- 系统内内容引用卡片

## 3.4 媒体资源限制（建议）

| 类型 | 数量上限 | 描述 |
|---|---|---|
| 图片 | 18 | 支持说明文字 |
| 视频 | 18 | 支持封面 |
| 音频 | 18 | 支持标题 |
| 文档 | 18 | 支持附件下载 |

# 4. 后台管理系统功能需求

# 5. 非功能性需求

## 5.1 性能要求
- 页面响应时间 ≤ 3 秒
- 统计查询 ≤ 2 秒
- 支持 50 名管理员并发操作
- 支持大数据量分页

## 5.2 安全要求
- RBAC 权限控制
- 双因素认证（2FA）
- 敏感数据加密
- 防 SQL 注入 / XSS / CSRF
- 全量审计日志

## 5.3 可用性要求
- 响应式设计
- 操作简洁直观
- 误操作确认机制
- 帮助文档完善

## 5.4 可扩展性要求
- 支持水平扩容
- 插件化扩展
- 开放 API 接口
- 第三方系统集成

## 5.5 兼容性要求
- Chrome
- Firefox
- Safari
- Edge
- 多分辨率设备适配
- 多语言支持
