

# 内容社区统一禁言管理系统需求文档

## 1. 系统概述

### 1.1 背景
内容社区系统包含公共领域、频道(Channel)、社区(Community)等多个场景，未来还将扩展单聊和群聊功能。需要设计一套统一的禁言管理系统，支持不同粒度和范围的禁言控制。

### 1.2 目标
- 提供统一的禁言管理机制
- 支持多层级、多维度的禁言控制
- 确保禁言规则的一致性和可扩展性
- 提供完善的禁言审计和管理功能

## 2. 禁言范围分类

### 2.1 按作用域分类

#### 2.1.1 全系统禁言
- **范围**：整个平台所有功能
- **影响**：用户无法进行任何内容发布、评论、聊天操作
- **权限**：仅系统管理员可操作
- **使用场景**：严重违规用户、恶意用户

#### 2.1.2 功能模块禁言
- **内容发布禁言**：禁止发布文章、动态等内容
- **评论禁言**：禁止对内容进行评论和回复
- **聊天禁言**：禁止使用所有聊天功能
- **权限**：系统管理员、相关模块管理员

#### 2.1.3 区域范围禁言
- **公共领域禁言**：禁止在公共区域发言
- **频道禁言**：禁止在特定频道内发言
- **社区禁言**：禁止在特定社区内发言
- **群聊禁言**：禁止在特定群聊中发言
- **权限**：对应区域的管理员

### 2.2 按禁言类型分类

#### 2.2.1 完全禁言
- 禁止所有相关操作
- 包括发布、评论、聊天等

#### 2.2.2 部分禁言
- 仅禁止特定操作类型
- 如只禁评论，不禁发布

#### 2.2.3 临时禁言
- 设置禁言时长
- 到期自动解除

#### 2.2.4 永久禁言
- 需要手动解除
- 适用于严重违规

## 3. 禁言操作类型

### 3.1 内容相关操作
```
- CONTENT_CREATE: 发布内容
- CONTENT_EDIT: 编辑内容  
- CONTENT_DELETE: 删除内容
- COMMENT_CREATE: 发表评论
- COMMENT_REPLY: 回复评论
- LIKE_ACTION: 点赞操作
- SHARE_ACTION: 分享操作
```

### 3.2 聊天相关操作
```
- CHAT_SEND: 发送消息
- CHAT_VOICE: 发送语音
- CHAT_IMAGE: 发送图片
- CHAT_FILE: 发送文件
- GROUP_CREATE: 创建群聊
- GROUP_INVITE: 邀请入群
```

### 3.3 社交相关操作
```
- FOLLOW_ACTION: 关注操作

```

## 4. 数据库设计

### 4.1 禁言记录表 (ban_records)
```sql
CREATE TABLE ban_records (
    id VARCHAR(32) PRIMARY KEY COMMENT '主键ID',
    user_id VARCHAR(32) NOT NULL COMMENT '被禁言用户ID',
    ban_type VARCHAR(20) NOT NULL COMMENT '禁言类型：SYSTEM/MODULE/AREA',
    ban_scope VARCHAR(50) COMMENT '禁言范围：具体的频道ID、社区ID等',
    ban_actions TEXT COMMENT '禁言操作类型JSON数组',
    ban_reason VARCHAR(500) COMMENT '禁言原因',
    ban_level INTEGER DEFAULT 1 COMMENT '禁言级别：1-轻微 2-一般 3-严重',
    start_time DATETIME NOT NULL COMMENT '禁言开始时间',
    end_time DATETIME COMMENT '禁言结束时间，NULL表示永久',
    status INTEGER DEFAULT 1 COMMENT '状态：1-生效 2-已解除 3-已过期',
    operator_id VARCHAR(32) NOT NULL COMMENT '操作人ID',
    operator_type VARCHAR(20) DEFAULT 'ADMIN' COMMENT '操作人类型',
    create_by VARCHAR(32) COMMENT '创建人',
    create_time DATETIME DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    update_by VARCHAR(32) COMMENT '更新人',
    update_time DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
    del_flag INTEGER DEFAULT 0 COMMENT '删除标志',
    
    INDEX idx_user_id (user_id),
    INDEX idx_ban_type_scope (ban_type, ban_scope),
    INDEX idx_status_time (status, start_time, end_time),
    INDEX idx_operator (operator_id, create_time)
) COMMENT='禁言记录表';
```

### 4.2 禁言解除记录表 (ban_lift_records)
```sql
CREATE TABLE ban_lift_records (
    id VARCHAR(32) PRIMARY KEY COMMENT '主键ID',
    ban_record_id VARCHAR(32) NOT NULL COMMENT '禁言记录ID',
    lift_reason VARCHAR(500) COMMENT '解除原因',
    lift_type VARCHAR(20) DEFAULT 'MANUAL' COMMENT '解除类型：MANUAL-手动 AUTO-自动过期',
    operator_id VARCHAR(32) COMMENT '操作人ID',
    create_by VARCHAR(32) COMMENT '创建人',
    create_time DATETIME DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    
    INDEX idx_ban_record (ban_record_id),
    INDEX idx_operator_time (operator_id, create_time)
) COMMENT='禁言解除记录表';
```

### 4.3 禁言模板表 (ban_templates)
```sql
CREATE TABLE ban_templates (
    id VARCHAR(32) PRIMARY KEY COMMENT '主键ID',
    template_name VARCHAR(100) NOT NULL COMMENT '模板名称',
    ban_type VARCHAR(20) NOT NULL COMMENT '禁言类型',
    ban_actions TEXT COMMENT '禁言操作类型JSON数组',
    default_duration INTEGER COMMENT '默认禁言时长（分钟）',
    ban_reason VARCHAR(500) COMMENT '默认禁言原因',
    ban_level INTEGER DEFAULT 1 COMMENT '禁言级别',
    is_active INTEGER DEFAULT 1 COMMENT '是否启用',
    create_by VARCHAR(32) COMMENT '创建人',
    create_time DATETIME DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    update_by VARCHAR(32) COMMENT '更新人',
    update_time DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
    
    INDEX idx_type_active (ban_type, is_active)
) COMMENT='禁言模板表';
```

## 5. 核心业务逻辑

### 5.1 禁言检查流程
```
1. 用户执行操作 → 2. 权限拦截器 → 3. 禁言检查服务
                                    ↓
4. 返回检查结果 ← 5. 查询禁言记录 ← 6. 按优先级匹配规则
```

### 5.2 禁言优先级
```
1. 全系统禁言（最高优先级）
2. 功能模块禁言
3. 区域范围禁言
4. 特定操作禁言（最低优先级）
```

### 5.3 禁言继承规则
- 全系统禁言 > 所有其他禁言
- 社区禁言不影响其他社区
- 频道禁言不影响其他频道
- 群聊禁言仅在特定群内生效

## 6. 接口设计

### 6.1 禁言管理接口
```java
// 创建禁言
POST /api/ban/create
// 解除禁言  
POST /api/ban/lift/{banId}
// 查询用户禁言状态
GET /api/ban/check/{userId}
// 禁言记录列表
GET /api/ban/records
// 批量禁言
POST /api/ban/batch
```

### 6.2 权限检查接口
```java
// 检查操作权限
POST /api/ban/permission/check
// 获取禁言详情
GET /api/ban/detail/{userId}/{scope}
```
