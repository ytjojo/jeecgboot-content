# 草稿箱版本管理设计文档

## 1. 概述

本文档详细介绍了基于现有`content_versions`表实现草稿箱版本管理功能的设计方案。该方案采用差异存储策略，通过重用现有版本管理架构，为草稿箱提供完整的版本历史管理能力。

## 2. 设计原则

1. **重用现有架构**：充分利用现有的`content_versions`表结构和版本管理机制
2. **差异存储优化**：采用差异存储策略，节省存储空间
3. **版本链管理**：通过基础版本+增量版本的方式管理草稿版本链
4. **乐观锁机制**：防止并发编辑冲突
5. **自动保存支持**：支持自动保存和手动保存的版本管理
6. **发布转换**：支持草稿版本向正式内容版本的转换

## 3. 架构设计



**重用策略**：
- 通过`version_type = 'DRAFT'`区分草稿版本
- `content_id`字段关联到`content_drafts.id`
- 利用现有的版本号和差异存储机制

### 3.2 草稿版本管理架构

```
草稿箱版本管理架构：

ContentDraft (草稿表)
    ↓
ContentVersions (版本表)
    ├── version_type = 'DRAFT'
    ├── content_id = draft.id
    └── 差异存储机制
        ├── 基础版本 (version_number = 1)
        └── 增量版本 (version_number > 1)
```

### 3.3 版本存储策略

#### 3.3.1 基础版本存储
- **触发时机**：草稿首次更新时
- **存储内容**：完整的草稿内容
- **版本号**：1
- **存储格式**：JSON格式存储完整内容

#### 3.3.2 增量版本存储
- **首次更新**: 判断草稿首次更新,创建基础版本
- **触发时机**：草稿内容发生变更时（自动保存/手动保存）
- **存储内容**：与基础版本的差异数据
- **版本号**：递增（2, 3, 4...）
- **存储格式**：JSON Patch格式存储差异

### 3.4 版本管理流程

#### 3.4.1 草稿版本创建流程
```
1. 在草稿内容更新前,判断content_versions表是否有记录,如果无记录,则查询ContentDraft记录获取内容作为基础版本内容,
2. 创建ContentVersion记录（version_type='DRAFT', version_number=1）
3. 创建基础版本记录（写入content_patch ,rendered_patch）
```

#### 3.4.2 草稿保存和更新流程
```
1. 在草稿内容更新前,判断content_versions表是否有记录,如果无记录走草稿版本创建流程创建基础版本
2. 检查当前版本号（乐观锁）
1. 保存所有完整字段到content_drafts表
3. 计算与基础版本的差异
4. 创建新的增量版本记录
5. 更新content_versions表
```

#### 3.4.3 版本恢复流程
```
1. 获取目标版本记录
2. 如果是基础版本，直接返回内容
3. 如果是增量版本，应用差异恢复完整内容
4. 更新草稿表内容
```



## 9. 配置和优化

### 9.1 版本管理配置

```yaml
# 草稿版本管理配置
draft:
  version:
    # 最大保留版本数
    max-versions: 50
    # 自动清理间隔（小时）
    cleanup-interval: 24
    # 差异存储阈值（字符数）
    diff-threshold: 1000
    # 是否启用自动保存版本
    auto-save-enabled: true
    # 自动保存间隔（秒）
    auto-save-interval: 30
```

### 9.2 性能优化策略

1. **索引优化**：
   ```sql
   -- 草稿版本查询优化索引
   CREATE INDEX IF NOT EXISTS idx_versions_draft_content ON content_versions(content_id, version_type, version_number DESC) WHERE version_type = 'DRAFT';
   CREATE INDEX IF NOT EXISTS idx_versions_draft_time ON content_versions(create_time DESC) WHERE version_type = 'DRAFT';
   ```

## 10. 总结

本设计方案通过重用现有的`content_versions`表，为草稿箱提供了完整的版本管理功能：

2. **差异存储**：采用差异存储策略，优化存储空间
3. **版本链管理**：支持完整的版本历史追踪
4. **并发安全**：通过乐观锁机制保证数据一致性

该方案为草稿箱提供了企业级的版本管理能力，满足用户对草稿历史管理的需求。
