# JeecgBoot 后端技术规范 V1.0

## 1. 核心原则

1. **分层架构**：严格遵循 Controller -> Service -> Mapper 的分层架构
2. **职责分离**：各层级有明确职责，禁止跨层访问
3. **数据隔离**：Entity(数据层)、DTO(服务层)、VO(控制层)严格分离
4. **单一职责**：每个类都有明确用途，避免万能类
5. **语义明确**：类名、字段名、方法名清晰表达意图

## 3. 各层级详细规范

### 3.1 Entity层 (实体层)

**职责**：映射数据库表结构，承载核心业务数据

**继承关系**：
- Entity类直接实现 `Serializable` 接口
- 不继承任何基类（如BaseEntity）
- 使用MyBatis-Plus注解进行数据库映射

**规范**：
- 命名：`XxxEntity`
- 仅包含与数据库表字段对应的属性
- 使用MyBatis-Plus注解（如`@TableName`, `@TableId`, `@TableField`等）
- 禁止包含业务逻辑方法
- 禁止包含非数据库字段（除非使用`@TableField(exist = false)`注解）
- 必须实现`Serializable`接口

**示例**：
```java
@Data
@TableName(value = "sys_user", autoResultMap = true)
@Accessors(chain = true)
@EqualsAndHashCode(callSuper = false)
@Schema(description = "用户实体")
public class UserEntity implements Serializable {
    
    private static final long serialVersionUID = 1L;
    
    /**
     * 用户ID - 主键
     */
    @TableId(type = IdType.ASSIGN_ID)
    @Schema(description = "用户ID")
    private String id;
    
    /**
     * 用户名
     */
    @Excel(name = "用户名", width = 15)
    @Schema(description = "用户名")
    private String username;
    
    /**
     * 邮箱
     */
    @Excel(name = "邮箱", width = 20)
    @Schema(description = "邮箱")
    private String email;
    
    /**
     * 创建时间
     */
    @JsonFormat(timezone = "GMT+8", pattern = "yyyy-MM-dd HH:mm:ss")
    @DateTimeFormat(pattern = "yyyy-MM-dd HH:mm:ss")
    @Schema(description = "创建时间")
    private LocalDateTime createTime;
}
```

### 3.2 Mapper层 (数据访问层)

**职责**：负责与数据库交互，执行CRUD操作

**继承关系**：
- Mapper接口继承 `com.baomidou.mybatisplus.core.mapper.BaseMapper<Entity>`
- BaseMapper提供基础的CRUD操作方法
- 可以定义自定义查询方法

**规范**：
- 命名：`XxxMapper`
- 接口形式，继承`BaseMapper<XxxEntity>`
- 只包含数据库操作方法
- 禁止包含业务逻辑
- 使用MyBatis注解或XML配置SQL
- 自定义方法需要添加详细注释

**示例**：
```java
@Mapper
public interface UserMapper extends BaseMapper<UserEntity> {
    
    /**
     * 根据用户名查询用户
     * @param username 用户名
     * @return 用户实体
     */
    @Select("SELECT * FROM sys_user WHERE username = #{username} AND del_flag = 0")
    UserEntity findByUsername(@Param("username") String username);
    
    /**
     * 查询活跃用户列表
     * @param status 用户状态
     * @return 用户列表
     */
    @Select("SELECT * FROM sys_user WHERE status = #{status} AND del_flag = 0")
    List<UserEntity> selectActiveUsers(@Param("status") Integer status);
    
    /**
     * 根据部门ID查询用户数量
     * @param deptId 部门ID
     * @return 用户数量
     */
    @Select("SELECT COUNT(*) FROM sys_user WHERE dept_id = #{deptId} AND del_flag = 0")
    Integer countByDeptId(@Param("deptId") String deptId);
}
```

### 3.3 DTO层 (数据传输对象层)

**职责**：服务层内部数据传输，业务逻辑处理载体

**规范**：
- 命名：
  - 请求DTO：`XxxCreateDTO`, `XxxUpdateDTO`, `XxxQueryDTO`
  - 响应DTO：`XxxItemDTO`, `XxxDetailDTO`, `XxxSimpleDTO`
- 严格区分不同场景的DTO
- 禁止包含ORM注解
- 可包含业务相关的验证注解

**分类**：
1. **基础信息DTO**：包含最核心字段
2. **列表项DTO**：仅包含列表展示所需的最少字段
3. **详情DTO**：扩展详细信息
4. **复合DTO**：通过组合关联信息

**示例**：
```java
// 基础信息DTO
@Data
public class UserBaseDTO {
    protected Long id;
    protected String name;
    protected String avatarUrl;
}

// 列表项DTO
@Data
@EqualsAndHashCode(callSuper = true)
public class UserItemDTO extends UserBaseDTO {
    private String title;
    private String departmentName;
}

// 详情DTO
@Data
@EqualsAndHashCode(callSuper = true)
public class UserDetailDTO extends UserBaseDTO {
    private String email;
    private String phone;
    private String bio;
    private LocalDateTime createTime;
    private LocalDateTime lastLoginTime;
}

// 复合详情DTO
@Data
public class UserCompositeDetailDTO {
    private UserDetailDTO userInfo;
    private DepartmentItemDTO department;
    private List<RoleItemDTO> roles;
    private UserStatsDTO stats;
}
```

### 3.4 Service层 (业务逻辑层)

**职责**：处理业务逻辑，DTO与Entity转换

**继承关系**：
- Service接口继承 `com.baomidou.mybatisplus.extension.service.IService<Entity>`
- Service实现类继承 `com.baomidou.mybatisplus.extension.service.impl.ServiceImpl<Mapper, Entity>`
- IService提供基础的CRUD操作方法
- ServiceImpl提供基础CRUD操作的默认实现

**规范**：
- 接口命名：`IXxxService`，继承`IService<XxxEntity>`
- 实现类命名：`XxxServiceImpl`，继承`ServiceImpl<XxxMapper, XxxEntity>`
- 业务方法命名采用业务语义化命名（如`createUser`, `updateUser`, `getUserDetail`等）
- 负责 DTO ⇄ Entity 转换
- 禁止直接操作Entity（通过Mapper）
- 禁止直接返回VO
- 业务方法需要添加详细注释

**示例**：
```java
public interface IUserService extends IService<UserEntity> {
    UserDetailDTO createUser(UserCreateDTO createDTO);
    UserDetailDTO updateUser(UserUpdateDTO updateDTO);
    UserDetailDTO getUserDetail(String id);
    IPage<UserItemDTO> queryUserPage(UserQueryDTO queryDTO);
}

@Service
@Transactional(rollbackFor = Exception.class)
public class UserServiceImpl extends ServiceImpl<UserMapper, UserEntity> implements IUserService {
    @Autowired
    private UserMapper userMapper;
    
    @Override
    public UserDetailDTO createUser(UserCreateDTO createDTO) {
        // 参数校验
        if (createDTO == null || StringUtils.isBlank(createDTO.getUsername())) {
            throw new JeecgBootException("用户信息不能为空");
        }
        
        // DTO -> Entity
        UserEntity entity = UserDTOConverter.createDTOToEntity(createDTO);
        entity.setId(IdWorker.getIdStr());
        entity.setCreateTime(LocalDateTime.now());
        
        // 保存到数据库
        this.save(entity);
        
        // Entity -> DTO
        return UserDTOConverter.entityToDetailDTO(entity);
    }
    
    // ... 其他实现
}
```

### 3.5 VO层 (视图对象层)

**职责**：定义API输入输出契约，客户端交互协议

**规范**：
- 命名：
  - 请求VO：`XxxCreateReqVO`, `XxxUpdateReqVO`, `XxxQueryReqVO`
  - 响应VO：`XxxItemRespVO`, `XxxDetailRespVO`, `XxxSimpleRespVO`
- 字段只能是基本类型、String、其他VO或Enum
- 禁止包含DTO或Entity类型字段
- 字段扁平化，避免过深嵌套

**示例**：
```java
// 请求VO
@Data
public class UserCreateReqVO {
    @NotBlank
    private String name;
    
    @Email
    private String email;
    
    // VO -> DTO
    public UserCreateDTO toDTO() {
        UserCreateDTO dto = new UserCreateDTO();
        dto.setName(this.name);
        dto.setEmail(this.email);
        return dto;
    }
}

// 响应VO
@Data
public class UserItemRespVO {
    private Long id;
    private String name;
    private String title;
    private String departmentName;
    
    // DTO -> VO
    public static UserItemRespVO fromDTO(UserItemDTO dto) {
        UserItemRespVO vo = new UserItemRespVO();
        vo.setId(dto.getId());
        vo.setName(dto.getName());
        vo.setTitle(dto.getTitle());
        vo.setDepartmentName(dto.getDepartmentName());
        return vo;
    }
}
```

### 3.6 Controller层 (控制层)

**职责**：处理HTTP请求，VO与DTO转换，调用Service

**继承关系**：
- Controller继承 `org.jeecg.common.system.base.controller.JeecgController<Entity, IService>`
- JeecgController提供通用的CRUD接口实现
- 可以重写父类方法实现自定义逻辑
- 提供统一的异常处理和响应格式

**规范**：
- 命名：`XxxController`，继承`JeecgController<XxxEntity, IXxxService>`
- 负责 VO ⇄ DTO 转换
- 禁止直接操作Entity
- 禁止包含业务逻辑
- 使用统一的返回格式
- 接口方法需要添加详细注释和API文档注解

**示例**：
```java
@RestController
@RequestMapping("/sys/user")
@Api(tags = "用户管理")
@Slf4j
public class UserController extends JeecgController<UserEntity, IUserService> {
    
    @Autowired
    private IUserService userService;
    
    /**
     * 创建用户
     * @param reqVO 用户创建请求VO
     * @return 创建结果
     */
    @PostMapping("/add")
    @ApiOperation(value = "创建用户", notes = "创建新用户")
    public Result<UserDetailRespVO> createUser(@RequestBody @Valid UserCreateReqVO reqVO) {
        try {
            // VO -> DTO
            UserCreateDTO createDTO = reqVO.toDTO();
            // 调用Service
            UserDetailDTO detailDTO = userService.createUser(createDTO);
            // DTO -> VO
            UserDetailRespVO respVO = UserDetailRespVO.fromDTO(detailDTO);
            return Result.ok(respVO);
        } catch (Exception e) {
            log.error("创建用户失败", e);
            return Result.error("用户创建失败: " + e.getMessage());
        }
    }
    
    /**
     * 获取用户详情
     * @param id 用户ID
     * @return 用户详情
     */
    @GetMapping("/queryById")
    @ApiOperation(value = "获取用户详情", notes = "根据ID获取用户详情")
    public Result<UserDetailRespVO> getUserDetail(@RequestParam(name = "id") String id) {
        try {
            UserDetailDTO detailDTO = userService.getUserDetail(id);
            UserDetailRespVO respVO = UserDetailRespVO.fromDTO(detailDTO);
            return Result.ok(respVO);
        } catch (Exception e) {
            log.error("获取用户详情失败", e);
            return Result.error("获取用户详情失败: " + e.getMessage());
        }
    }
    
    /**
     * 分页查询用户列表
     * @param userQueryVO 查询条件
     * @param pageNo 页码
     * @param pageSize 页大小
     * @param req HttpServletRequest
     * @return 用户列表
     */
    @GetMapping("/list")
    @ApiOperation(value = "分页查询用户", notes = "分页查询用户列表")
    public Result<IPage<UserItemRespVO>> queryUserPage(
            UserQueryReqVO userQueryVO,
            @RequestParam(name = "pageNo", defaultValue = "1") Integer pageNo,
            @RequestParam(name = "pageSize", defaultValue = "10") Integer pageSize,
            HttpServletRequest req) {
        try {
            UserQueryDTO queryDTO = UserQueryReqVO.toDTO(userQueryVO);
            IPage<UserItemDTO> pageDTO = userService.queryUserPage(queryDTO);
            IPage<UserItemRespVO> pageVO = pageDTO.convert(UserItemRespVO::fromDTO);
            return Result.ok(pageVO);
        } catch (Exception e) {
            log.error("分页查询用户失败", e);
            return Result.error("查询失败: " + e.getMessage());
        }
    }
}
```

## 4. 数据流转规范

### 4.1 请求方向数据流转
```
Client Request 
    → Controller (接收 ReqVO) 
    → Converter (ReqVO → DTO) 
    → Service (处理 DTO) 
    → Mapper (DTO → Entity) 
    → Database
```

### 4.2 响应方向数据流转
```
Database 
    → Mapper (Entity → DTO) 
    → Service (处理 DTO) 
    → Converter (DTO → RespVO) 
    → Controller (返回 RespVO) 
    → Client Response
```

## 5. 转换规范

### 5.1 转换位置约定
- **Controller层**: 负责 `VO ⇄ DTO` 转换
- **Service层**: 负责 `DTO ⇄ Entity` 转换

### 5.2 转换方法规范
```java
// 在VO中定义转换方法
public class UserCreateReqVO {
    // VO -> DTO
    public UserCreateDTO toDTO() {
        // ...
    }
    
    // VO -> DTO (静态方法)
    public static UserCreateDTO toDTO(UserCreateReqVO vo) {
        // ...
    }
}

// 在VO中定义转换方法
public class UserItemRespVO {
    // DTO -> VO
    public static UserItemRespVO fromDTO(UserItemDTO dto) {
        // ...
    }
}

// 在DTO中定义转换方法
public class UserDTO {
    // Entity -> DTO
    public static UserDTO fromEntity(UserEntity entity) {
        // ...
    }
    
    // DTO -> Entity
    public UserEntity toEntity() {
        // ...
    }
}
```

## 6. 强制规则

1. **禁止** Controller直接操作Entity
2. **禁止** Service直接返回VO
3. **禁止** DTO包含ORM注解（如`@Entity`, `@Column`）
4. **禁止** VO用于服务层数据传输
5. **禁止** 在Entity中编写转换逻辑
6. **禁止** 创建包含所有字段的"上帝DTO"
7. **禁止** VO字段包含DTO或Entity类型
8. **禁止** 跨层访问（Controller不能直接访问Mapper）

## 7. 安全性规范

### 7.1 SQL注入防护
- 使用参数化查询，避免字符串拼接SQL
- 使用MyBatis-Plus条件构造器
- 对用户输入进行严格校验
- 使用@Param注解明确参数映射

### 7.2 数据权限控制
- 实现数据权限过滤
- 使用@DataScope注解控制数据范围
- 多租户数据隔离
- 敏感数据脱敏处理


## 8. 最佳实践建议

### 8.1 命名规范统一
- 创建类：`XxxCreateDTO`, `XxxCreateReqVO`
- 更新类：`XxxUpdateDTO`, `XxxUpdateReqVO`
- 查询类：`XxxQueryDTO`, `XxxQueryReqVO`
- 基础类：`BaseXxxDTO`
- 列表类：`XxxItemDTO`, `XxxItemRespVO`
- 详情类：`XxxDetailDTO`, `XxxDetailRespVO`

### 8.2 转换工具统一
```java
/**
 * 统一转换工具类
 * 负责DTO与Entity之间的数据转换
 */
@Component
public class UserDTOConverter {
    
    /**
     * 创建DTO转换为Entity
     */
    public static UserEntity createDTOToEntity(UserCreateDTO dto) {
        UserEntity entity = new UserEntity();
        BeanUtils.copyProperties(dto, entity);
        return entity;
    }
    
    /**
     * 更新DTO数据到Entity
     */
    public static void updateEntityFromDTO(UserEntity entity, UserUpdateDTO dto) {
        BeanUtils.copyProperties(dto, entity, getNullPropertyNames(dto));
    }
    
    /**
     * Entity转换为详情DTO
     */
    public static UserDetailDTO entityToDetailDTO(UserEntity entity) {
        UserDetailDTO dto = new UserDetailDTO();
        BeanUtils.copyProperties(dto, entity);
        return dto;
    }
}
```

### 8.3 验证规则统一
- 创建验证组：`Create.class`、`Update.class`
- 使用 `@Validated` 分组验证
- 统一错误消息格式

### 8.4 核心实践要点
1. 优先使用组合模式，即使对于简单场景
2. 保持DTO的纯洁性，不包含业务逻辑
3. 及时重构，当发现DTO被用于多个场景时立即拆分
4. 团队统一评审，确保设计符合规范
5. 显著减少代码冗余，提高代码的可维护性和一致性
