# 内容服务需求规格说明书 V2.0

## 1. 项目概述

### 1.1 文档目的
本文档旨在定义内容社区系统中内容服务的功能需求、技术架构和实现规范，为开发团队提供明确的开发指导。

### 1.2 核心功能
- **6种内容类型**：文章、笔记、动态、视频、问答问题、问答回答
- **11种特殊元素**：@用户、#话题、标签、股票、媒体文件、链接、投票卡片等
- **双编辑器支持**：Editor.js富文本编辑器 + Markdown编辑器
- **JSON格式存储**：Editor.js富文本json格式与扩展

### 1.3 技术架构
- **后端**：Spring Boot 3.4.5 + MyBatis-Plus + PostgreSQL
- **前端**：Vue3 + Editor.js编辑器
- **存储**：阿里云OSS + JSONB数据库存储
- **设计原则**：高内聚低耦合、可扩展性、数据一致性、安全合规

## 2. 内容类型定义

### 2.1 文章 (Article)
- **编辑器**：富文本编辑器（默认）+ Markdown编辑器
- **存储格式**：富文本内容以Delta JSON格式存储，Markdown以纯文本存储
- **封面图片**：支持设置单张封面图片
- **适用场景**：深度、结构化的内容分享

### 2.2 笔记 (Note)
- **图片轮播**：支持最多18张图片的轮播展示
- **编辑器**：富文本编辑器（默认）+ Markdown编辑器
- **适用场景**：图文并茂的生活记录、产品种草

### 2.3 动态 (Post)
- **布局**：文字内容在上方，媒体内容在下方
- **媒体支持**：图片/视频混合，支持九宫格布局
- **适用场景**：日常、即时性的图文或视频混合分享

### 2.4 视频 (Video)
- **视频限制**：限制上传1条视频
- **显示方式**：视频全屏显示
- **附加信息**：可附带独立的标题和简介

### 2.5 问答 (QA)
- **问题内容**：包含问题标题和问题描述
- **回答机制**：支持其他用户回答问题
- **采纳机制**：提问者可以采纳最佳答案

### 2.6 问答回答 (Answer)
- **回答内容**：包含回答内容
- **评论支持**：支持其他用户评论
- **采纳机制**：回答者可以采纳作为最佳答案

### 卡片与普通元素区别


1. 内容区别
   - **卡片元素**：是一种特殊的元素，它的内容是其他内容的引用，包含复杂数据结构,有可能包含图片 、视频、文字、 选项、按钮多种内容,
   - **普通元素**：支持添加文本、图片、视频、音频、文档、股票、链接、话题、标签、内容简单仅包含id、icon 、链接、显示文字信息,可在单行文字内显示,即使包含图片(比如头像)图片高度也基本和单行文字高度相近,方便在文字行内显示
2. 交互区别
   - **卡片元素**： 支持复杂交互功能，比如投票、播放视频、播放音频、点击跳转等
   - **普通元素**：如果元素含有连接可以点击跳转对应页面

3. 前端显示区别
   - **卡片元素**：在前端显示为一个独立的卡片，包含标题、内容、视频、图片、投票按钮等。
   - **普通元素**：在前端显示为普通的文本、或者文本+ icon 图标,文本可能会改变颜色或者加粗或者下划线等效果
4. 类似块元素(blocks),相对独立
   - **卡片元素**： 类似块(blocks/Node)元素,相对独立,多个卡片换行显示
   - **普通元素**： 类似行内元素(inline),可以在内容中插入多个普通元素

## 4. JSON格式规范

### 4.1 格式变更说明
为了与Editor.js官方格式保持兼容，本项目已将JSON格式从原有的`{"insert": {"element_type": {...}}}`结构调整为`{"type": "element_type", "data": {...}}`结构。

**变更对比：**
```json
// 旧格式
{"insert": {"tag": {"id": 789, "name": "前端开发"}}}

// 新格式
{"type": "tag", "data": {"id": 789, "name": "前端开发"}}
```

**兼容性说明：**
- 新格式与Editor.js官方格式完全兼容
- 转换函数已更新以支持新格式
- 保留向后兼容性，仍可处理旧格式数据

### 4.2 支持的内容类型
| 类型 | 标识符 | 描述 | 示例 |
|------|--------|------|------|
| paragraph | paragraph | 普通段落内容 | "Hello World"(参考Editor.js文档.md) |
| linkTool | linkTool | 链接 | https://example.com (参考Editor.js文档.md) |
| head | head | 标题块 | 这是一个标题 支持level 1-6 (参考Editor.js文档.md)|
| mention | mention | @用户提及 | @Alice |

    id VARCHAR(32) PRIMARY KEY,

    like_count INTEGER DEFAULT 0,
    comment_count INTEGER DEFAULT 0,
    share_count INTEGER DEFAULT 0,
    created_time TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_time TIMESTAMP DEFAULT CURRENT_TIMESTAMP

## 6. @用户功能实现

### 6.1 功能概述
在所有文本输入区域支持@用户功能，用户输入@符号后可以搜索并选择要提及的用户。

### 6.2 前端实现

#### 6.2.1 富文本编辑器插件
- 监听用户输入，检测@符号时触发用户搜索
- 在光标位置显示用户搜索下拉框
- 支持键盘上下键选择用户，回车键确认
- 插入格式：`<span class="mention" data-user-id="{userId}">@{nickname}</span>`

#### 6.2.2 用户搜索接口
- 实时调用搜索接口，支持用户名、昵称模糊搜索
- 优先显示用户关注的人
- 搜索结果限制在10个以内

#### 6.2.3 显示效果
- @用户在编辑器中以高亮样式显示
- 点击@用户可以跳转到用户主页
- 支持删除和重新编辑@用户

### 6.3 后端处理

#### 6.3.1 内容解析
- 发布时解析HTML内容，提取所有@用户信息
- 验证被@用户是否存在且有效
- 将@用户信息存储到content_mentions表

#### 6.3.2 权限控制
- 检查用户是否允许被@（隐私设置）
- 检查@用户是否在黑名单中
- 限制单条内容最多@10个用户

#### 6.3.3 通知机制
- 内容发布成功后，向所有被@用户发送通知
- 通知类型：MENTION
- 通知内容包含@用户的内容摘要和链接

#### 6.3.4 频率限制
- 单个用户每小时最多@50个用户
- 防止恶意@用户刷屏

### 6.4 用户名变更处理

#### 6.4.1 数据一致性策略
- content_mentions表存储用户ID而非用户名
- 内容HTML中的@用户标记使用data-user-id属性存储用户ID
- 显示时通过用户ID实时获取最新的用户名和昵称

#### 6.4.2 缓存更新机制
- 用户信息变更时，清除相关的@用户缓存
- 使用Redis发布订阅机制通知所有服务节点更新缓存
- 前端通过WebSocket接收用户信息变更通知

#### 6.4.3 显示逻辑
```javascript
// 前端渲染@用户时的处理逻辑
function renderMention(userId, fallbackName) {
    const userInfo = getUserFromCache(userId);
    if (userInfo && userInfo.status === 'ACTIVE') {
        return `@${userInfo.nickname || userInfo.username}`;
    } else if (userInfo && userInfo.status === 'DELETED') {
        return '@已注销用户';
    } else {
        return `@${fallbackName}`; // 降级显示原始名称
    }
}
```

## 8. 性能优化策略

### 8.1 数据库优化
- **JSONB索引**：使用JSONB类型存储JSON数据，创建GIN索引支持JSON查询
- **数据分离**：分离热数据和冷数据，提高查询效率
- **分页查询**：大数据量查询使用分页，避免一次性加载过多数据

### 8.2 缓存策略
- **用户信息缓存**：缓存用户基本信息、话题信息
- **内容缓存**：缓存渲染后的HTML内容
- **CDN加速**：使用CDN加速媒体文件访问

### 8.3 前端优化
