### MySQL 和 PostgreSQL 数据库表设计规范

#### 1. 命名规范
* **表名**：使用**小写复数**，下划线分隔（`snake_case`）。例如：`users`, `order_items`。
* **列名**：使用**小写单数**，下划线分隔。例如：`id`, `first_name`。
* **外键**：`referencing_table_id`，例如：`user_id`。
* **索引**：`idx_table_name_column_name`。
* **避免**使用缩写，除非是广为人知的。

---

#### 2. 数据类型
* 主键 id VARCHAR(32) NOT NULL PRIMARY KEY COMMENT '主键ID',
* **字符串**：使用 `VARCHAR(N)`，避免不必要的 `TEXT`。
* **日期/时间**：使用 `TIMESTAMP` 或 `DATETIME`。
* **布尔值**：
    * **MySQL**：`TINYINT(1)`。
    * **PostgreSQL**：`BOOLEAN`。
* **字符集**：MySQL 推荐 `utf8mb4`。

---

#### 3. 约束和索引
* **主键**：所有表都应有主键，通常为自增整数。
* **外键**：为外键列创建索引，并定义 `FOREIGN KEY` 约束。
* **索引**：
    * 为**`WHERE`**, **`ORDER BY`**, **`GROUP BY`** 中频繁使用的列创建索引。
    * 避免创建过多索引，以减少写入开销。
* **非空约束**：为所有必要的列添加 `NOT NULL` 约束。

---

#### 4. 特定平台
* **MySQL**：
    * **引擎**：使用 `InnoDB`。
* **PostgreSQL**：
    * **JSON**：使用 `JSONB` 存储 JSON 数据以提高性能。
    * **其他类型**：灵活使用 `uuid`, `array` 等高级数据类型。
