
  ## 任务：检查 OpenSpec Change 与代码实现一致性

  对 `openspec/changes/<change-name>/` 执行文档与实现双向对比，暴露漂移。

  ### 输入
  - change 目录路径
  - 后端代码根目录

  ### 检查清单

  #### 1. API 对比（规范文档 vs Controller）
  - 读取 `docs/backend-api/` 下相关 api 文档，列出所有已记录的端点
  - 读取 change 涉及的所有 Controller，列出实际端点
  - 逐条对比：路径、HTTP 方法、入参名/类型/必填、出参类型、出参字段
  - 输出差异表：`端点 | 文档 | 实际 | 是否一致 | 差距描述`

  #### 2. 入参/出参字段完整性（Spec vs VO/DTO）
  - 从 spec 文件提取每个 Requirement 声明的"系统 SHALL 展示/返回 XXX"字段
  - 从代码读取对应的 VO/DTO/Req 类，列出所有字段
  - 对比：spec 要求展示的字段，VO 是否都有？VO 有的字段，Service 是否都赋值？
  - 特别标注：字段已声明但从未赋值的"空字段"

  #### 3. 业务规则覆盖（Spec Scenario vs Service 代码分支）
  - 从 spec 提取所有 Scenario（WHEN/THEN/AND）
  - 在 Service 实现中查找对应分支（if/else/switch/return）
  - 标注：完全覆盖 / 部分覆盖（描述缺什么） / 完全缺失
  - 特别关注：边界条件（空值、超出范围、权限边界）

  #### 4. 设计决策遵循（design.md vs 实际代码）
  - 从 design.md Decisions 段提取每个决策点和选择理由
  - 验证实际代码是否按决策实现
  - 标注：遵循 / 偏离但更优（给出评价）/ 偏离需对齐

  #### 5. 数据一致性（Flyway vs Entity vs Spec 常量）
  - 读取 Flyway SQL 的表结构（字段名、类型、索引）
  - 读取 Java Entity 的字段映射
  - 对比 spec/design 描述的常量值（阈值、上限、枚举值）
  - 标注不一致处

  #### 6. 上下游引用检查
  - 搜索 change 涉及的 Controller/Service 是否被其他模块调用
  - 检查新 API 是否在 `docs/backend-api/` 的其他文档中有交叉引用
  - 标注引用链路和可能的文档遗漏点

  ### 输出格式
  对每项差异输出：

  ┌─────────────────────────────┬───────────────────────┬─────────────┬──────────┬──────────┬──────────┬──────────┐
  │          严重级别           │         类别          │ 源文件:行号 │ 规范描述 │ 实际实现 │   评价   │ 修复方向 │
  ├─────────────────────────────┼───────────────────────┼─────────────┼──────────┼──────────┼──────────┼──────────┤
  │ CRITICAL/WARNING/SUGGESTION │ API/VO/逻辑/设计/数据 │ 精确位置    │ 文档内容 │ 代码内容 │ 漂移评价 │ 同步方案 │
  └─────────────────────────────┴───────────────────────┴─────────────┴──────────┴──────────┴──────────┴──────────┘

  ### 严重级别定义
  - **CRITICAL**: spec 要求的功能完全未实现、API 路径不存在、数据表缺失
  - **WARNING**: 字段缺漏、边界条件未覆盖、文档与实现描述不一致
  - **SUGGESTION**: 旧代码未清理、命名不统一、多余但不影响功能的字段

  ### 最终输出
  1. 汇总表（总检查项 / CRITICAL / WARNING / SUGGESTION）
  2. 详细差异清单（按级别分组）
  3. 同步方案：哪些更新文档、哪些修复代码、哪些确认后决定

  ---
  使用时替换 <change-name> 即可。这个提示词可以作为 /opsx:verify 的增强版，或者放到 .claude/prompts/ 下做成可复用的 slash command。
