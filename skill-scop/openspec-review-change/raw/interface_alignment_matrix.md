## 状态说明

| 状态标识 | 含义 |
|----------|------|
| ✅ 已对齐 | 接口已在后端 SDD 中定义，且已实现，字段/行为与前端预期一致 |
| 📄 已设计未实现 | 接口在后端 SDD 中有定义，但代码尚未交付 |
| ❌ 完全缺失 | 接口在后端 SDD 和代码中均不存在 |
| ⚠️ 字段/行为不符 | 接口存在，但返回字段、数据类型、语义与前端预期不一致 |
| 🚫 性能不满足 | 接口存在且行为正确，但不满足前端调用频率或延迟要求 |
| 🔄 Mock 中 | GAP 存在，前端暂用 Mock 服务继续开发，等待后端交付 |
| 🗃️ 已废弃 | 前端已确认不再需要此接口 |

## 优先级说明

| 优先级 | 含义 |
|--------|------|
| BLOCKER | 阻塞前端开发，必须在进入开发前解决 |
| MAJOR | 不阻塞当前开发，但必须在里程碑前关闭 |
| MINOR | 影响较小，迭代内处理即可 |

## 附录 A：前端接口消费清单原始输出

> 此表由前端在 SDD 评审前独立输出，是接口对齐比对的输入文档。

| 页面 / 组件 | 依赖接口 | HTTP 方法 | 路径 | 调用时机 | 关键依赖字段 | 并发/顺序依赖 |
|------------|----------|-----------|------|----------|-------------|--------------|
| LoginPage | 用户登录 | POST | /v1/auth/login | 表单提交 | access_token, refresh_token, expires_in | 无 |
| App.tsx | 获取当前用户信息 | GET | /v1/auth/me | 应用初始化 | id, name, avatar, roles | 依赖登录成功后的 token |
| Header | 登出 | POST | /v1/auth/logout | 点击退出按钮 | — | 无 |
| AuthInterceptor | 刷新 Token | POST | /v1/auth/refresh | 响应拦截到 401 时 | access_token, expires_in | 触发后其他请求需排队等待 |
| ProductListPage | 商品列表（分页） | GET | /v1/products | 页面加载、翻页、筛选 | id, name, price, cover_image, stock_status | 无 |
| ProductDetailPage | 商品详情 | GET | /v1/products/:id | 页面加载 | id, name, price, description, images, specs | 无 |
| SearchBar | 商品搜索 | GET | /v1/products/search | 用户输入（debounce 500ms） | id, name, price, cover_image | 无 |
| CartPage | 批量获取商品库存 | POST | /v1/products/stock/batch | 购物车页加载 | product_id, available_stock, is_available | 无 |
| CheckoutPage | 创建订单 | POST | /v1/orders | 点击提交订单 | order_id, status, payment_url | 无 |
| OrderDetailPage | 查询订单状态 | GET | /v1/orders/:id/status | 页面加载后持续轮询 | status, updated_at | 依赖创建订单返回的 order_id |
