# 需求池（Backlog）

## 0. 使用说明

- 定位：V1 之外的待评审/待排期需求列表，用于持续维护与迭代计划输入。
- 约定：每条需求必须具备稳定 ID、状态、优先级、目标版本、Owner、验收标准与依赖。
- 实现状态：全部未实现。

## 1. 状态与字段定义

### 1.1 状态（Status）

- Proposed：已提出，未评审
- Grooming：评审中（范围/成本/依赖澄清）
- Planned：已排期（纳入某版本）
- In Progress：开发中
- Done：已完成
- Dropped：取消/不做

### 1.2 优先级（Priority）

- P0：阻塞主流程/必须做
- P1：重要但不阻塞
- P2：可延后
- P3：想法/探索

### 1.3 单条需求模板（维护规范）

每条需求至少包含：
- 元信息：ID / 模块 / 状态 / 优先级 / 目标版本 / Owner / 依赖
- 需求描述：一句话目标 + 核心范围（In/Out）
- 验收标准：Given-When-Then（至少 2 条）
- 数据与接口：新增/改动点（不确定则写 TBD）

**需求描述**
