# 社交关系扩展与互动增强

## 史诗概览

**史诗ID**: EPIC-07
**标题**: 提供互关标识、粉丝管理、邀请分享和社区角色标签等社交关系扩展能力
**背景**: 内容社区需要丰富的社交互动功能来增强用户粘性和社区氛围。当前缺少互关标识、粉丝数据分析、邀请机制和社区角色展示，导致社交关系不够清晰，用户难以优化内容创作方向
**用户**: 所有注册用户、创作者、版主/管理员
**成功标准**: 
- 互关率提升 35%
- 邀请转化率 >15%
- 粉丝管理功能使用率 >40%
- 社区角色识别准确率 >95%
**范围**: 
- ✅ 包含：互关标识、私密内容范围、粉丝列表与趋势、粉丝画像、邀请码/分享链接、社区角色标签、版主权限
- ❌ 不包含：独立 IM 即时通讯、企业组织通讯录
**预计持续时间**: 6 周
**优先级**: 中

---

## 特性列表

### 特性 1: 互关与私密内容

**描述**: 清晰标识互相关注关系，支持仅互关可见的私密内容范围
**关键交互**: 关注用户 → 对方回关 → 显示互关标识 / 发布私密内容 → 仅互关好友可见
**非功能需求**: 互关标识加载不影响性能，私密内容权限判定准确
**验收条件**: 用户能清晰识别互关关系，能发布仅互关可见的内容

#### 故事列表

##### 故事 7.1.1: 互关标识展示

**作为** 用户，  
**我希望** 互相关注有清晰标识（好友/互关），  
**以便** 识别关系强度。

**验收标准**:
1. **给定** 我与某用户互相关注，**当** 我查看该用户主页，**则** 我看到"互关"或"好友"标识
2. **给定** 我与某用户互相关注，**当** 我在评论区看到该用户的评论，**则** 我看到互关标识
3. **给定** 我只关注了某用户但对方未回关，**当** 我查看该用户主页，**则** 我仅看到"已关注"标识，无互关标识
4. **给定** 对方取消关注我，**当** 刷新页面，**则** 系统移除互关标识，仅显示"已关注"或"关注"按钮
5. **给定** 我想查看所有互关好友，**当** 我进入"互关好友"列表，**则** 我看到所有与我互关的用户

**附加说明**:
- **优先级**: 高
- **依赖项**: EPIC-04（关注体系）
- **技术说明**: 互关状态通过查询双向关注关系确定；前端组件复用标识

##### 故事 7.1.2: 仅互关可见的私密内容

**作为** 用户，  
**我希望** 能够设置"仅互关可见"的私密内容范围（若平台启用此可见性），  
**以便** 保护私密分享。

**验收标准**:
1. **给定** 我想发布私密内容，**当** 我选择可见性为"仅互关可见"并发布，**则** 系统保存内容并标记为私密
2. **给定** 我发布了仅互关可见的内容，**当** 我的互关好友浏览，**则** 他们能看到该内容
3. **给定** 我发布了仅互关可见的内容，**当** 非互关用户尝试查看，**则** 系统提示"该内容仅对互关好友可见"
4. **给定** 我取消了与某用户的互关，**当** 该用户刷新，**则** 他们不再看到我发布的仅互关可见内容
5. **给定** 我想将私密内容改为公开，**当** 我编辑可见性并保存，**则** 系统更新可见性并通知之前的互关好友（可选）

**附加说明**:
- **优先级**: 中
- **依赖项**: 故事 7.1.1
- **技术说明**: 可见性判定在内容查询层执行；私密内容不进入公共推荐流

### 特性 2: 粉丝管理与数据分析

**描述**: 提供粉丝列表查看、新增粉丝趋势和粉丝画像分析，帮助用户了解增长情况和优化创作方向
**关键交互**: 进入粉丝管理 → 查看列表/趋势/画像 → 分析数据 → 优化策略
**非功能需求**: 数据实时更新或 T+1 更新，隐私保护（不泄露粉丝个人信息）
**验收条件**: 用户能查看粉丝信息，能看到增长趋势和兴趣分布

#### 故事列表

##### 故事 7.2.1: 查看粉丝列表与新增粉丝趋势

**作为** 用户，  
**我希望** 能查看粉丝列表与新增粉丝趋势，  
**以便** 了解增长情况。

**验收标准**:
1. **给定** 我进入"粉丝"页面，**当** 页面加载，**则** 我看到粉丝总数和按时间倒序排列的粉丝列表
2. **给定** 我查看粉丝列表，**当** 我滚动页面，**则** 系统分页加载更多粉丝
3. **给定** 我想查看新增粉丝趋势，**当** 我切换到"趋势"标签，**则** 我看到按天/周/月统计的新增粉丝数图表
4. **给定** 我想了解某天的增长原因，**当** 我点击某天，**则** 系统显示当天新增粉丝列表和可能的原因（如发布了热门内容）
5. **给定** 我有大量粉丝，**当** 我搜索特定粉丝，**则** 系统通过昵称或用户名搜索并定位

**附加说明**:
- **优先级**: 中
- **依赖项**: EPIC-04（关注体系）
- **技术说明**: 粉丝列表需分页优化；趋势数据通过定时任务聚合

##### 故事 7.2.2: 查看粉丝画像

**作为** 用户，  
**我希望** 能查看粉丝画像（兴趣分布），  
**以便** 优化内容创作方向。

**验收标准**:
1. **给定** 我进入"粉丝画像"页面，**当** 页面加载，**则** 我看到粉丝的兴趣分布（如科技 30%、娱乐 25%、体育 20% 等）
2. **给定** 我想了解粉丝的地域分布，**当** 我查看地域图表，**则** 我看到粉丝的主要地区分布
3. **给定** 我想了解粉丝的活跃时段，**当** 我查看活跃时段图表，**则** 我看到粉丝最活跃的时间段
4. **给定** 我想导出数据，**当** 我点击"导出"，**则** 系统生成 CSV 文件供我下载（脱敏后）
5. **给定** 我的粉丝数较少（<100），**当** 我查看画像，**则** 系统提示"粉丝数量不足，暂无法生成画像"

**附加说明**:
- **优先级**: 低
- **依赖项**: 故事 7.2.1
- **技术说明**: 画像数据通过大数据分析生成，T+1 更新；需注意隐私合规

### 特性 3: 邀请与分享机制

**描述**: 提供个人邀请码/分享链接生成和追踪，绑定邀请收益与积分，促进社交传播
**关键交互**: 生成邀请码 → 分享给好友 → 好友注册 → 获得奖励 → 查看邀请记录
**非功能需求**: 邀请码唯一且可追踪，奖励发放准确
**验收条件**: 用户能成功邀请好友，能获得邀请奖励，能查看邀请效果

#### 故事列表

##### 故事 7.3.1: 生成邀请码与分享链接

**作为** 用户，  
**我希望** 生成个人邀请码/分享链接，并追踪邀请收益与积分绑定，  
**以便** 进行社交传播并获得奖励。

**验收标准**:
1. **给定** 我进入"邀请好友"页面，**当** 页面加载，**则** 我看到我的专属邀请码和分享链接
2. **给定** 我想分享链接，**当** 我点击"复制链接"或选择分享渠道（微信/QQ/微博），**则** 系统复制链接或调起分享
3. **给定** 好友通过我的链接注册，**当** 注册成功，**则** 系统记录邀请关系并发放奖励（如 50 积分）
4. **给定** 我想查看邀请记录，**当** 我进入"邀请记录"，**则** 我看到已邀请的好友列表、注册时间和获得的奖励
5. **给定** 我想查看邀请收益统计，**当** 我查看统计图表，**则** 我看到累计邀请人数、成功注册数和总奖励

**附加说明**:
- **优先级**: 中
- **依赖项**: EPIC-01（注册登录）和 EPIC-03（积分体系）
- **技术说明**: 邀请码通过唯一 ID 生成；邀请关系存储于邀请表；奖励自动发放

### 特性 4: 社区角色标签与管理权限

**描述**: 在评论区展示社区角色标签（普通用户/创作者/管理员/版主），并为版主/管理员提供部分管理权限
**关键交互**: 浏览评论 → 看到角色标签 / 版主登录 → 执行管理操作
**非功能需求**: 角色标签加载不影响性能，权限控制严格
**验收条件**: 用户能清晰识别社区角色，版主/管理员能执行管理操作

#### 故事列表

##### 故事 7.4.1: 展示社区角色标签

**作为** 平台，  
**我希望** 在评论区展示社区角色标签（普通用户/创作者/管理员/版主），  
**以便** 提升治理效率与信息透明。

**验收标准**:
1. **给定** 普通用户发表评论，**当** 查看评论，**则** 我看到该用户昵称旁无特殊标签或显示"用户"
2. **给定** 认证创作者发表评论，**当** 查看评论，**则** 我看到"创作者"标签及认证标识
3. **给定** 版主发表评论，**当** 查看评论，**则** 我看到"版主"标签
4. **给定** 管理员发表评论，**当** 查看评论，**则** 我看到"管理员"标签
5. **给定** 用户点击角色标签，**当** 点击查看，**则** 系统显示该角色的说明（如"版主负责维护社区秩序"）

**附加说明**:
- **优先级**: 中
- **依赖项**: EPIC-02（个人资料）
- **技术说明**: 角色信息从用户表或角色表读取；前端根据角色渲染不同标签样式

##### 故事 7.4.2: 版主/管理员管理权限

**作为** 版主/管理员，  
**我希望** 拥有部分管理权限（例如处理违规与审核），  
**以便** 维护社区秩序。

**验收标准**:
1. **给定** 我是版主，**当** 我查看评论区，**则** 我看到"删除评论"、"警告用户"等操作按钮
2. **给定** 我删除了违规评论，**当** 操作完成，**则** 系统记录操作日志并通知被处罚用户
3. **给定** 我是管理员，**当** 我查看用户管理页面，**则** 我能执行封禁、禁言、限制推荐等操作
4. **给定** 我执行了处罚操作，**当** 操作完成，**则** 系统生成审计日志（操作人、时间、原因、处罚类型）
5. **给定** 我想撤销处罚，**当** 我选择"撤销"并填写原因，**则** 系统恢复用户正常状态并记录撤销日志

**附加说明**:
- **优先级**: 中
- **依赖项**: EPIC-09（用户状态与治理）
- **技术说明**: 权限通过 RBAC 模型控制；所有管理操作必须留痕

## 风险与假设

- **风险**: 
  - 粉丝画像数据不准确导致误导
  - 邀请机制被滥用刷奖励
  - 版主权限过大可能导致滥用
- **假设**: 
  - 用户愿意邀请好友加入社区
  - 粉丝数据分析能帮助创作者优化内容
  - 版主/管理员能公正行使权限
