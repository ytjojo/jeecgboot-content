# 社交关系与内容订阅

## 史诗概览

**史诗ID**: EPIC-04
**标题**: 构建用户关注体系与多维度内容订阅能力，促进社交互动与内容发现
**背景**: 内容社区需要通过关注关系链和内容订阅机制，帮助用户建立社交连接并持续获取感兴趣的内容。当前缺少完善的关注管理、分组、特别关注和多源订阅能力，导致用户难以高效管理信息流
**用户**: 所有注册用户、内容创作者
**成功标准**: 
- 平均关注数提升 40%
- 关注流点击率 >30%
- 订阅转化率 >25%
- 用户留存率提升 20%
**范围**: 
- ✅ 包含：关注/取消关注、分组管理、特别关注、关注流、关注推荐、批量管理、订阅专题/话题/栏目、通知配置、订阅广场
- ❌ 不包含：独立支付结算、付费订阅（若平台未启用）
**预计持续时间**: 8 周
**优先级**: 高

---

## 特性列表

### 特性 1: 关注体系

**描述**: 支持用户关注其他用户，按分组管理，设置特别关注，并在关注流中查看动态
**关键交互**: 浏览用户 → 点击关注 → 管理分组 / 查看关注流 → 取消关注
**非功能需求**: 关注操作响应时间 <500ms，关注流加载时间 <2 秒
**验收条件**: 用户能成功关注/取消关注，能管理分组，能在关注流看到关注对象动态

#### 故事列表

##### 故事 4.1.1: 关注与取消关注

**作为** 用户，  
**我希望** 能够关注其他用户以订阅其内容动态，也能取消关注以调整关注列表，  
**以便** 控制我看到的内容来源。

**验收标准**:
1. **给定** 我浏览到其他用户的主页或内容，**当** 我点击"关注"按钮，**则** 系统建立关注关系并显示"已关注"
2. **给定** 我已关注某用户，**当** 我再次点击按钮，**则** 系统显示"取消关注"选项
3. **给定** 我点击"取消关注"并确认，**当** 确认后，**则** 系统解除关注关系并显示"关注"按钮
4. **给定** 我关注了某用户，**当** 该用户发布新内容，**则** 系统在我的关注流中展示该内容
5. **给定** 我取消关注某用户，**当** 刷新关注流，**则** 系统不再展示该用户的新内容

**附加说明**:
- **优先级**: 高
- **依赖项**: EPIC-01（用户需先注册登录）
- **技术说明**: 关注关系存储于关系表，建立索引优化查询；关注流通过事件驱动或定时聚合

##### 故事 4.1.2: 按分组管理关注对象

**作为** 用户，  
**我希望** 能按分组管理关注对象（默认分组与自定义标签），  
**以便** 更好地组织社交关系并分类查看内容。

**验收标准**:
1. **给定** 我关注了新用户，**当** 关注成功，**则** 系统默认将其加入"默认分组"
2. **给定** 我想创建自定义分组，**当** 我进入"分组管理"并创建新分组（如"朋友"、"创作者"），**则** 系统保存该分组
3. **给定** 我想将某用户移入分组，**当** 我在该用户主页选择分组并确认，**则** 系统将该用户加入指定分组
4. **给定** 我想查看某分组的内容，**当** 我选择分组，**则** 系统仅展示该分组内用户的动态
5. **给定** 我想将用户移出分组，**当** 我选择"移除"，**则** 系统将该用户从分组中移除（但不取消关注）

**附加说明**:
- **优先级**: 中
- **依赖项**: 故事 4.1.1
- **技术说明**: 分组关系存储于独立的分组表；支持一个用户在多个分组

##### 故事 4.1.3: 设置特别关注与强提醒

**作为** 用户，  
**我希望** 能够设置特别关注用户，并对其动态进行强提醒（推送/红点/优先展示），  
**以便** 不错过重要内容。

**验收标准**:
1. **给定** 我已关注某用户，**当** 我点击"设为特别关注"，**则** 系统标记该用户为特别关注
2. **给定** 特别关注用户发布新内容，**当** 内容发布，**则** 系统发送推送通知并显示红点提醒
3. **给定** 我查看关注流，**当** 加载内容，**则** 系统将特别关注用户的内容优先展示在顶部
4. **给定** 我想取消特别关注，**当** 我点击"取消特别关注"，**则** 系统移除特别关注标记，恢复正常提醒
5. **给定** 我有多个特别关注用户，**当** 他们发布内容，**则** 系统按发布时间倒序优先展示

**附加说明**:
- **优先级**: 中
- **依赖项**: 故事 4.1.1
- **技术说明**: 特别关注标记存储于关注表；推送通过消息队列异步发送

##### 故事 4.1.4: 关注流内容展示与配置

**作为** 用户，  
**我希望** 在首页"关注"信息流中查看关注对象发布/点赞/收藏的动态，并可配置展示维度，  
**以便** 更贴合我的兴趣。

**验收标准**:
1. **给定** 我进入"关注"标签页，**当** 页面加载，**则** 我看到按时间倒序排列的关注对象动态
2. **给定** 动态类型包括发布、点赞、收藏，**当** 我查看设置，**则** 我可以选择显示哪些类型的动态
3. **给定** 我只想看发布内容，**当** 我关闭"点赞"和"收藏"的显示开关，**则** 系统仅展示关注对象的发布内容
4. **给定** 关注流内容很多，**当** 我滚动页面，**则** 系统分页加载更多历史内容
5. **给定** 关注流无新内容，**当** 我下拉刷新，**则** 系统提示"暂无新内容"

**附加说明**:
- **优先级**: 高
- **依赖项**: 故事 4.1.1
- **技术说明**: 关注流可通过读扩散（实时查询）或写扩散（预聚合）实现；根据规模选择

##### 故事 4.1.5: 查看关注列表

**作为** 用户，  
**我希望** 能够查看我的关注列表，包括关注总数、分组筛选和搜索功能，  
**以便** 快速找到和管理我关注的用户。

**验收标准**:
1. **给定** 我进入"我的关注"页面，**当** 页面加载，**则** 我看到所有已关注的用户列表及关注总数
2. **给定** 我想按分组查看，**当** 我选择某个分组（如"朋友"、"创作者"），**则** 系统仅显示该分组内的关注用户
3. **给定** 我想搜索特定用户，**当** 我输入用户名或昵称并搜索，**则** 系统显示匹配的关注用户
4. **给定** 我查看某个关注用户，**当** 我点击进入其主页，**则** 我可以查看该用户的详细信息和动态
5. **给定** 关注列表很长，**当** 我滚动页面，**则** 系统分页加载更多关注用户
6. **给定** 我想快速操作，**当** 我在列表中点击用户卡片，**则** 我可以直接取消关注或调整分组

**附加说明**:
- **优先级**: 高
- **依赖项**: 故事 4.1.1 和 4.1.2
- **技术说明**: 关注列表支持分页查询；搜索使用索引优化；列表缓存提升加载速度

##### 故事 4.1.6: 查看特别关注列表

**作为** 用户，  
**我希望** 能够查看我的特别关注列表，包括特别关注总数和快速管理功能，  
**以便** 高效管理我特别关注的用户。

**验收标准**:
1. **给定** 我进入"特别关注"页面，**当** 页面加载，**则** 我看到所有设为特别关注的用户列表及特别关注总数
2. **给定** 我想查看某个特别关注用户的详情，**当** 我点击用户卡片，**则** 我可以跳转到该用户的主页
3. **给定** 我想取消特别关注，**当** 我在列表中点击"取消特别关注"按钮，**则** 系统移除该用户的特别关注标记
4. **给定** 特别关注用户发布新内容，**当** 我查看列表，**则** 我可以看到每个用户的最新动态提示（如"2小时前发布"）
5. **给定** 我没有特别关注用户，**当** 我进入页面，**则** 系统显示空状态并引导我设置特别关注
6. **给定** 我想批量管理，**当** 我进入批量操作模式，**则** 我可以批量取消特别关注或调整提醒方式

**附加说明**:
- **优先级**: 中
- **依赖项**: 故事 4.1.3
- **技术说明**: 特别关注列表通过查询关注表中特别关注标记获取；支持实时状态更新

##### 故事 4.1.7: 关注推荐与批量管理

**作为** 用户，  
**我希望** 系统基于兴趣与已有关注提供关注推荐，并附带推荐理由，也支持批量取消关注、移入/移出分组等批量管理能力，  
**以便** 提升关注转化并高效整理关注列表。

**验收标准**:
1. **给定** 我进入"发现"页面，**当** 我查看推荐列表，**则** 我看到基于我的兴趣和关注历史推荐的用户
2. **给定** 每个推荐用户，**当** 我查看，**则** 我看到推荐理由（如"与您关注的 XXX 相似"、"热门创作者"）
3. **给定** 我想批量取消关注，**当** 我进入"批量管理"并选择多个用户，**则** 系统一次性取消对这些用户的关注
4. **给定** 我想批量移动分组，**当** 我选择多个用户并选择目标分组，**则** 系统将这些用户移入指定分组
5. **给定** 批量操作完成，**当** 操作结束，**则** 系统显示操作结果（成功 X 个，失败 Y 个）

**附加说明**:
- **优先级**: 中
- **依赖项**: 故事 4.1.1 和 4.1.2
- **技术说明**: 推荐算法基于协同过滤或内容相似度；批量操作使用事务保证一致性

### 特性 2: 订阅体系

**描述**: 支持用户订阅专题/合集、话题/Tag、栏目/频道等内容源，配置通知方式，并在订阅广场发现新内容
**关键交互**: 浏览内容源 → 点击订阅 → 配置通知 → 接收更新 / 浏览订阅广场 → 发现新订阅
**非功能需求**: 订阅操作响应时间 <500ms，通知发送延迟 <5 分钟
**验收条件**: 用户能成功订阅各类内容源，能配置通知，能在统一管理界面查看所有订阅

#### 故事列表

##### 故事 4.2.1: 订阅专题/合集/话题/栏目

**作为** 用户，  
**我希望** 能够订阅专题/合集、话题/Tag、栏目/频道等内容源，  
**以便** 获取系列内容更新和特定主题下的新内容。

**验收标准**:
1. **给定** 我浏览到某个专题页面，**当** 我点击"订阅"按钮，**则** 系统建立订阅关系并显示"已订阅"
2. **给定** 我订阅了某个话题（如"#摄影"），**当** 该话题下有新内容，**则** 系统在我的订阅流中展示
3. **给定** 我订阅了官方栏目，**当** 栏目发布新内容，**则** 系统通知我有新内容
4. **给定** 我已订阅某内容源，**当** 我再次点击按钮，**则** 系统显示"取消订阅"选项
5. **给定** 我取消订阅，**当** 确认后，**则** 系统解除订阅关系并不再推送该内容源的更新

**附加说明**:
- **优先级**: 高
- **依赖项**: EPIC-01
- **技术说明**: 订阅关系存储于订阅表，支持多种订阅类型（专题、话题、栏目等）

##### 故事 4.2.2: 配置通知方式与频率

**作为** 用户，  
**我希望** 能选择通知方式（站内通知/推送/邮件摘要）并配置频率与渠道，  
**以便** 控制打扰程度。

**验收标准**:
1. **给定** 我订阅了某内容源，**当** 我进入"通知设置"，**则** 我可以看到通知方式选项（站内、推送、邮件）
2. **给定** 我想只接收站内通知，**当** 我关闭推送和邮件开关，**则** 系统仅通过站内消息通知我
3. **给定** 我想接收每日摘要邮件，**当** 我选择"每日摘要"，**则** 系统在每天固定时间发送汇总邮件
4. **给定** 我想实时接收推送，**当** 我选择"实时推送"，**则** 系统在有更新时立即发送推送
5. **给定** 我设置了免打扰时段，**当** 在免打扰时段内有更新，**则** 系统暂不发送通知，待免打扰结束后汇总发送

**附加说明**:
- **优先级**: 中
- **依赖项**: 故事 4.2.1
- **技术说明**: 通知配置存储于用户偏好表；邮件摘要通过定时任务生成

##### 故事 4.2.3: 订阅广场发现与搜索

**作为** 用户，  
**我希望** 在订阅广场发现热门订阅源，并按分类浏览与搜索，  
**以便** 探索新内容。

**验收标准**:
1. **给定** 我进入"订阅广场"，**当** 页面加载，**则** 我看到按热度排序的推荐订阅源
2. **给定** 我想按分类浏览，**当** 我选择分类（如科技、娱乐、体育），**则** 系统显示该分类下的订阅源
3. **给定** 我想搜索特定订阅源，**当** 我输入关键词并搜索，**则** 系统显示匹配的专题、话题、栏目
4. **给定** 我查看某个订阅源详情，**当** 我点击进入，**则** 我看到订阅源介绍、订阅人数和最近更新
5. **给定** 我想订阅某个订阅源，**当** 我点击"订阅"，**则** 系统建立订阅关系

**附加说明**:
- **优先级**: 中
- **依赖项**: 故事 4.2.1
- **技术说明**: 订阅广场数据通过定时任务聚合热门订阅源；搜索使用全文检索

##### 故事 4.2.4: 统一管理订阅项

**作为** 用户，  
**我希望** 在统一管理界面查看所有订阅项，并支持暂停/取消订阅，  
**以便** 便捷管理我的订阅列表。

**验收标准**:
1. **给定** 我进入"我的订阅"页面，**当** 页面加载，**则** 我看到所有已订阅的内容源列表
2. **给定** 每个订阅项，**当** 我查看，**则** 我看到订阅类型、订阅时间和最后更新时间
3. **给定** 我想暂时不接收某订阅源的更新，**当** 我点击"暂停订阅"，**则** 系统暂停推送但保留订阅关系
4. **给定** 我想恢复暂停的订阅，**当** 我点击"恢复订阅"，**则** 系统恢复推送
5. **给定** 我想取消订阅，**当** 我点击"取消订阅"并确认，**则** 系统解除订阅关系

**附加说明**:
- **优先级**: 中
- **依赖项**: 故事 4.2.1
- **技术说明**: 暂停状态存储于订阅表；支持批量操作

## 风险与假设

- **风险**: 
  - 关注流内容过多可能导致信息过载
  - 推荐算法不准确可能降低用户体验
  - 通知过于频繁可能导致用户关闭通知
- **假设**: 
  - 用户愿意主动关注感兴趣的内容创作者
  - 订阅机制能帮助用户发现优质内容
  - 分组管理能提升用户的信息组织效率
