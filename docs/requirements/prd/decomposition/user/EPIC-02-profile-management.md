# 个人资料与主页个性化

## 史诗概览

**史诗ID**: EPIC-02
**标题**: 提供灵活的个人资料管理与主页个性化定制能力
**背景**: 用户需要构建完整的社区身份，通过个人资料展示个性，同时控制隐私边界。当前缺少统一的资料管理、主页个性化和认证标识展示能力，导致用户体验割裂且隐私控制不足
**用户**: 所有注册用户、认证用户、管理员
**成功标准**: 
- 资料完善率 >80%
- 主页访问时长提升 30%
- 隐私设置使用率 >60%
- 认证用户信任度评分提升 25%
**范围**: 
- ✅ 包含：基础资料设置、主页个性化、认证标识展示、字段可见性控制、曾用昵称/头像历史记录
- ❌ 不包含：独立支付结算、企业组织通讯录
**预计持续时间**: 6 周
**优先级**: 高

---

## 特性列表

### 特性 1: 基础资料管理

**描述**: 支持用户设置和维护昵称、头像、简介等基础信息，构建完整的社区身份
**关键交互**: 进入个人资料页面 → 编辑信息 → 保存 → 审核（如需要）→ 生效
**非功能需求**: 图片上传支持格式 JPG/PNG/WebP，单张图片最大 5MB，资料修改频率限制为每天 5 次
**验收条件**: 用户能成功设置和更新资料，违规内容被拦截，修改频率受控

#### 故事列表

##### 故事 2.1.1: 设置基础个人信息

**作为** 用户，  
**我希望** 设置昵称、头像、简介、性别、生日、地区、职业、个人链接等基础信息，  
**以便** 构建我的社区身份并让其他人了解我。

**验收标准**:
1. **给定** 我是新用户或资料未完善，**当** 我进入"编辑资料"页面并填写必填项（昵称、头像），**则** 系统保存资料并显示"资料已更新"
2. **给定** 我设置的昵称包含敏感词或违规内容，**当** 我点击"保存"，**则** 系统提示"昵称包含不当内容，请修改"且不保存
3. **给定** 我上传的头像图片大于 5MB，**当** 我尝试保存，**则** 系统提示"图片大小必须小于 5MB"并阻止上传
4. **给定** 我上传的头像图片格式不支持（如 BMP），**当** 我尝试上传，**则** 系统提示"仅支持 JPG、PNG、WebP 格式"
5. **给定** 我在一天内修改资料超过 5 次，**当** 我第 6 次尝试修改，**则** 系统提示"今日修改次数已达上限，请明天再试"

**附加说明**:
- **优先级**: 高
- **依赖项**: EPIC-01（用户需先注册登录）
- **技术说明**: 头像图片存储于 CDN，自动压缩至最大分辨率 1000x1000px；昵称敏感词过滤使用关键词匹配 + AI 审核

##### 故事 2.1.2: 资料修改频率限制与内容审核

**作为** 平台系统，  
**我希望** 对昵称、头像、简介等资料修改实施频率限制和内容审核，  
**以便** 防止滥用和违规内容传播。

**验收标准**:
1. **给定** 用户修改昵称或头像，**当** 提交修改请求，**则** 系统检查是否在频率限制内（每天 5 次）
2. **给定** 用户提交的资料包含疑似违规内容，**当** 系统检测到，**则** 系统将资料标记为"待审核"状态，暂不公开显示
3. **给定** 资料审核通过，**当** 审核完成，**则** 系统公开显示新资料并通知用户
4. **给定** 资料审核不通过，**当** 审核完成，**则** 系统恢复原资料并提示用户"资料审核未通过，原因：XXX"
5. **给定** 用户在审核期间再次修改，**当** 提交修改，**则** 系统提示"您的资料正在审核中，请稍后"

**附加说明**:
- **优先级**: 高
- **依赖项**: 故事 2.1.1
- **技术说明**: 审核可使用 AI 自动审核 + 人工复审；审核队列按优先级排序；审核超时时间 24 小时

### 特性 2: 主页个性化定制

**描述**: 允许用户自定义个人主页的背景、主题色和模块布局，展现个性
**关键交互**: 进入主页设置 → 选择背景图/主题色 → 调整模块显隐和排序 → 预览 → 保存
**非功能需求**: 背景图加载时间 <2 秒，支持主流分辨率适配
**验收条件**: 用户能成功自定义主页外观，更改即时生效且在不同设备上正确显示

#### 故事列表

##### 故事 2.2.1: 设置个人主页背景图与主题色

**作为** 用户，  
**我希望** 设置个人主页背景图和主题色，  
**以便** 个性化展示我的主页风格。

**验收标准**:
1. **给定** 我想更换主页背景，**当** 我上传新背景图并保存，**则** 系统立即在我的主页显示新背景
2. **给定** 我上传的背景图尺寸不合适，**当** 系统处理时，**则** 系统自动裁剪或缩放以适配不同屏幕
3. **给定** 我选择主题色（如蓝色、红色、绿色），**当** 我保存，**则** 系统在我的主页应用该主题色（按钮、链接、边框等元素）
4. **给定** 我恢复默认设置，**当** 我点击"恢复默认"，**则** 系统清除自定义背景和主题色，使用平台默认样式

**附加说明**:
- **优先级**: 中
- **依赖项**: 故事 2.1.1
- **技术说明**: 背景图存储于 CDN，生成多种分辨率版本；主题色通过 CSS 变量动态切换

##### 故事 2.2.2: 自定义个人主页模块显隐与排序

**作为** 用户，  
**我希望** 自定义个人主页模块（动态、收藏、成就等）的显隐与排序，  
**以便** 突出我关心的内容并隐藏不想展示的部分。

**验收标准**:
1. **给定** 我进入"主页设置"页面，**当** 我查看可用模块列表，**则** 我看到所有可展示模块（动态、收藏、成就、关注、粉丝等）
2. **给定** 我想隐藏某个模块（如收藏），**当** 我关闭该模块的显示开关，**则** 系统在我的主页不再显示该模块
3. **给定** 我想调整模块顺序，**当** 我拖拽模块到新位置，**则** 系统保存新排序并在我的主页按新顺序展示
4. **给定** 我隐藏了所有模块，**当** 我尝试保存，**则** 系统提示"至少需要保留一个模块"
5. **给定** 其他用户访问我的主页，**当** 他们查看，**则** 他们只看到我设置为公开的模块

**附加说明**:
- **优先级**: 中
- **依赖项**: 故事 2.1.1
- **技术说明**: 模块配置存储于数据库，前端根据配置动态渲染；支持拖拽排序库

### 特性 3: 认证标识与信任展示

**描述**: 为认证用户、机构、达人提供清晰的认证标识展示，提升信任度和内容传播效率
**关键交互**: 申请认证 → 提交材料 → 审核 → 通过后展示标识
**非功能需求**: 认证标识加载不影响页面性能，标识清晰可辨
**验收条件**: 认证用户能在昵称附近展示可信的认证标识，不同类型认证有明确区分

#### 故事列表

##### 故事 2.3.1: 展示认证标识

**作为** 认证用户/机构/达人，  
**我希望** 在昵称附近展示清晰可信的认证标识，  
**以便** 获得身份识别并提升信任与内容传播效率。

**验收标准**:
1. **给定** 我通过了个人认证，**当** 我发布内容或评论，**则** 系统在昵称旁显示"个人认证"标识（蓝色对勾）
2. **给定** 我通过了企业认证，**当** 我发布内容，**则** 系统显示"企业认证"标识（金色徽章）及企业名称
3. **给定** 我是大V/达人，**当** 我展示主页，**则** 系统显示"大V认证"标识（紫色星标）及认证领域
4. **给定** 我是系统官方账号，**当** 我发布内容，**则** 系统显示"官方认证"标识（红色盾牌）
5. **给定** 用户点击认证标识，**当** 点击查看，**则** 系统显示认证详情（认证类型、认证时间、认证说明）

**附加说明**:
- **优先级**: 高
- **依赖项**: EPIC-01（用户需先注册）
- **技术说明**: 认证信息存储于独立的认证表，与用户资料解耦；标识通过组件复用

##### 故事 2.3.2: 手机号/邮箱绑定状态与实名认证标识展示

**作为** 用户，  
**我希望** 展示手机号/邮箱绑定状态和实名认证标识，  
**以便** 提升他人对我的信任度。

**验收标准**:
1. **给定** 我已绑定手机号，**当** 我查看个人资料页，**则** 我看到"手机号已验证"标识
2. **给定** 我已绑定邮箱，**当** 我查看个人资料页，**则** 我看到"邮箱已验证"标识
3. **给定** 我完成了实名认证，**当** 我展示主页，**则** 系统显示"实名认证"标识（灰色盾牌）
4. **给定** 其他用户查看我的资料，**当** 他们看到我的绑定状态，**则** 他们能看到我选择了公开的验证标识

**附加说明**:
- **优先级**: 中
- **依赖项**: 故事 2.1.1 和 EPIC-01
- **技术说明**: 验证状态从用户账户表读取；实名认证需对接第三方实名服务

### 特性 4: 隐私与可见性控制

**描述**: 允许用户对个人资料字段设置不同的可见性级别，精细控制隐私边界
**关键交互**: 进入隐私设置 → 选择字段 → 设置可见性 → 保存
**非功能需求**: 隐私设置调整后对新请求即时生效，缓存页面在 5 分钟内失效
**验收条件**: 用户能成功设置字段可见性，设置即时生效且不会泄露隐私

#### 故事列表

##### 故事 2.4.1: 设置个人资料字段可见性

**作为** 用户，  
**我希望** 对个人资料字段设置可见性（公开/仅关注者/仅自己/互关可见），  
**以便** 控制谁能看到我的个人信息。

**验收标准**:
1. **给定** 我想隐藏生日信息，**当** 我将生日字段设置为"仅自己"，**则** 系统保存设置且其他用户无法看到我的生日
2. **给定** 我想仅让关注者看到我的职业，**当** 我将职业字段设置为"仅关注者"，**则** 系统仅对关注我的人显示该信息
3. **给定** 我想让互关好友看到我的地区，**当** 我将地区字段设置为"互关可见"，**则** 系统仅对与我互关的用户显示该信息
4. **给定** 我将某字段设置为"公开"，**当** 任何用户查看我的资料，**则** 他们都能看到该信息
5. **给定** 我修改了可见性设置，**当** 其他用户刷新页面，**则** 他们在 5 分钟内看到更新后的可见性效果

**附加说明**:
- **优先级**: 高
- **依赖项**: 故事 2.1.1
- **技术说明**: 可见性规则在查询接口层统一判定；使用 Redis 缓存用户资料，可见性变更后 5 分钟内失效

##### 故事 2.4.2: 隐私设置即时生效与缓存失效

**作为** 平台系统，  
**我希望** 隐私设置调整对新请求即时生效，且缓存页面在可接受时间内失效，  
**以便** 保证隐私承诺不被破坏。

**验收标准**:
1. **给定** 用户将某字段从"公开"改为"仅自己"，**当** 修改保存成功，**则** 系统立即清除该字段的公共缓存
2. **给定** 其他用户在缓存失效前访问该用户资料，**当** 他们请求数据，**则** 系统实时判定可见性并返回正确结果
3. **给定** 用户频繁修改隐私设置，**当** 修改超过 10 次/小时，**则** 系统限制修改频率并提示"操作过于频繁"
4. **给定** 缓存失效策略执行，**当** 缓存过期，**则** 系统重新从数据库加载最新隐私设置

**附加说明**:
- **优先级**: 高
- **依赖项**: 故事 2.4.1
- **技术说明**: 使用 Cache-Aside 模式；隐私设置变更触发缓存 invalidation；关键接口不走缓存直接查库

### 特性 5: 曾用昵称/头像历史记录

**描述**: 保存用户的曾用昵称和头像历史记录，限时回溯展示，保留身份连续性与社交辨识
**关键交互**: 系统自动记录 → 用户查看历史 → 可选恢复
**非功能需求**: 历史记录保留 180 天，最多保存 20 条记录
**验收条件**: 系统能正确记录历史变更，用户能查看和恢复历史记录

#### 故事列表

##### 故事 2.5.1: 保存曾用昵称/头像历史记录

**作为** 平台系统，  
**我希望** 保存用户的曾用昵称和头像历史记录并限时回溯展示，  
**以便** 保留身份连续性与社交辨识度。

**验收标准**:
1. **给定** 用户修改了昵称，**当** 保存成功，**则** 系统将旧昵称存入历史记录（包含修改时间）
2. **给定** 用户修改了头像，**当** 保存成功，**则** 系统将旧头像 URL 存入历史记录（包含修改时间）
3. **给定** 历史记录超过 20 条，**当** 新增记录，**则** 系统删除最早的一条记录（FIFO）
4. **给定** 历史记录超过 180 天，**当** 系统执行清理任务，**则** 系统自动删除过期的历史记录
5. **给定** 用户查看"历史记录"页面，**当** 我点击进入，**则** 我看到按时间倒序排列的曾用昵称和头像列表

**附加说明**:
- **优先级**: 低
- **依赖项**: 故事 2.1.1
- **技术说明**: 历史记录存储于独立的 history 表；定期清理任务每天执行一次

##### 故事 2.5.2: 恢复曾用昵称/头像

**作为** 用户，  
**我希望** 从历史记录中恢复曾用的昵称或头像，  
**以便** 快速回到之前使用的名称或形象。

**验收标准**:
1. **给定** 我想恢复之前的昵称，**当** 我在历史记录中选择某个昵称并点击"恢复"，**则** 系统将该昵称设为当前昵称（计入修改次数限制）
2. **给定** 我想恢复之前的头像，**当** 我在历史记录中选择某个头像并点击"恢复"，**则** 系统将该头像设为当前头像
3. **给定** 我选择的昵称已被其他用户使用，**当** 我尝试恢复，**则** 系统提示"该昵称已被使用，请选择其他昵称"
4. **给定** 我今日修改次数已达上限，**当** 我尝试恢复，**则** 系统提示"今日修改次数已达上限，请明天再试"

**附加说明**:
- **优先级**: 低
- **依赖项**: 故事 2.5.1
- **技术说明**: 恢复操作等同于一次新的修改，需遵守频率限制和内容审核规则

## 风险与假设

- **风险**: 
  - 用户上传违规图片或昵称，审核压力大
  - 隐私设置复杂可能导致用户困惑
  - 缓存失效不及时可能导致隐私泄露
- **假设**: 
  - 用户愿意完善个人资料以提升社区体验
  - CDN 服务稳定可靠，图片加载速度快
  - AI 审核准确率 >90%，减少人工审核成本
