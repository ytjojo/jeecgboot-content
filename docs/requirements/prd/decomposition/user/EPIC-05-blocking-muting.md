# 降噪与内容保护

## 史诗概览

**史诗ID**: EPIC-05
**标题**: 提供拉黑（双向切断）和屏蔽（单向降噪）能力，保护用户个人空间并优化信息流体验
**背景**: 内容社区需要为用户提供有效的内容过滤和人际冲突处理机制。当前缺少清晰的拉黑和屏蔽功能边界，导致用户无法有效管理负面互动和信息噪音，影响社区体验
**用户**: 所有注册用户
**成功标准**: 
- 拉黑/屏蔽功能使用率 >30%
- 用户举报率降低 20%
- 信息流满意度提升 25%
- 用户留存率提升 15%
**范围**: 
- ✅ 包含：拉黑（双向切断）、屏蔽（单向降噪）、屏蔽词设置、不感兴趣反馈、黑名单/屏蔽列表管理、行为边界明确
- ❌ 不包含：内容审核系统、自动风控拦截
**预计持续时间**: 4 周
**优先级**: 高

---

## 特性列表

### 特性 1: 拉黑（双向切断）

**描述**: 允许用户拉黑指定用户，彻底切断双向互动与可见性，实现完全隔离
**关键交互**: 浏览用户/内容 → 选择拉黑 → 确认 → 生效 / 查看黑名单 → 解除拉黑
**非功能需求**: 拉黑操作响应时间 <500ms，拉黑后关系立即失效
**验收条件**: 拉黑后双方无法查看彼此内容、评论、动态，已有互关自动解除，@功能失效

#### 故事列表

##### 故事 5.1.1: 拉黑用户实现双向切断

**作为** 用户，  
**我希望** 能够拉黑指定用户，以便彻底切断双向互动与可见性，保护个人空间。

**验收标准**:
1. **给定** 我遇到骚扰或不想互动的用户，**当** 我在其主页或评论区点击"拉黑"并确认，**则** 系统将该用户加入我的黑名单
2. **给定** 我拉黑了某用户，**当** 我尝试访问该用户主页，**则** 系统提示"您已拉黑该用户，无法查看其内容"
3. **给定** 我拉黑了某用户，**当** 该用户尝试访问我的主页，**则** 系统对其显示"该用户不存在"或空白页面
4. **给定** 我拉黑了某用户，**当** 我浏览内容流，**则** 系统过滤掉该用户发布的所有内容和评论
5. **给定** 我拉黑了某用户，**当** 该用户浏览内容流，**则** 系统过滤掉我发布的所有内容和评论

**附加说明**:
- **优先级**: 高
- **依赖项**: EPIC-01（用户需先注册登录）
- **技术说明**: 拉黑关系存储于黑名单表；内容查询时统一过滤拉黑用户的内容

##### 故事 5.1.2: 拉黑后自动解除互关与禁止互动

**作为** 用户，  
**当我拉黑对方后，我希望双方无法查看彼此主页、内容、评论、动态，已有互关关系自动解除，对方无法评论/点赞/私信我，我也无法对其做同样操作，@功能对双方失效，以便实现双向隔离并避免冲突。

**验收标准**:
1. **给定** 我与某用户互相关注，**当** 我拉黑该用户，**则** 系统自动解除双方的关注关系
2. **给定** 我拉黑了某用户，**当** 该用户尝试评论我的内容，**则** 系统阻止评论并提示"操作失败"
3. **给定** 我拉黑了某用户，**当** 该用户尝试点赞我的内容，**则** 系统阻止点赞操作
4. **给定** 我拉黑了某用户，**当** 该用户尝试私信我，**则** 系统阻止发送并提示"无法发送消息"
5. **给定** 我拉黑了某用户，**当** 该用户在评论中 @我，**则** 系统不发送通知给我且不显示 @链接
6. **给定** 我拉黑了某用户，**当** 我尝试 @该用户，**则** 系统阻止 @操作并提示"无法 @已拉黑的用户"

**附加说明**:
- **优先级**: 高
- **依赖项**: 故事 5.1.1
- **技术说明**: 拉黑优先级高于所有互动功能；在权限校验层统一拦截

##### 故事 5.1.3: 查看与管理黑名单

**作为** 用户，  
**我希望** 在黑名单页面查看已拉黑用户与拉黑时间，并支持解除拉黑，  
**以便** 恢复关系。

**验收标准**:
1. **给定** 我进入"黑名单"页面，**当** 页面加载，**则** 我看到按拉黑时间倒序排列的已拉黑用户列表
2. **给定** 每个拉黑用户，**当** 我查看，**则** 我看到用户昵称、头像和拉黑时间
3. **给定** 我想解除拉黑，**当** 我点击某用户的"解除拉黑"按钮并确认，**则** 系统从黑名单中移除该用户
4. **给定** 我解除了拉黑，**当** 我刷新页面，**则** 该用户不再出现在黑名单中
5. **给定** 我解除了拉黑，**当** 我访问该用户主页，**则** 我可以正常查看其内容（但不会自动恢复关注关系）

**附加说明**:
- **优先级**: 中
- **依赖项**: 故事 5.1.1
- **技术说明**: 解除拉黑后需清除相关缓存；可选择是否恢复之前的关注关系（默认不恢复）

##### 故事 5.1.4: 拉黑操作不通知对方

**作为** 用户，  
**我希望** 拉黑操作不会通知对方，  
**以便** 保护隐私与减少对抗。

**验收标准**:
1. **给定** 我拉黑了某用户，**当** 操作完成，**则** 系统不向被拉黑用户发送任何通知
2. **给定** 被拉黑用户尝试与我互动，**当** 操作失败，**则** 系统仅提示"操作失败"而不告知具体原因（如被拉黑）
3. **给定** 被拉黑用户发现无法查看我的内容，**当** 他们疑惑，**则** 系统不透露是被拉黑还是其他原因
4. **给定** 我解除拉黑，**当** 操作完成，**则** 系统也不通知对方

**附加说明**:
- **优先级**: 高
- **依赖项**: 故事 5.1.1
- **技术说明**: 拉黑是静默操作；错误提示需模糊化以避免暴露拉黑状态

### 特性 2: 屏蔽（单向降噪）

**描述**: 允许用户屏蔽某个用户的动态或特定类型内容，使其不出现在推荐流与关注流中，但仍可访问其主页且保留关注关系
**关键交互**: 浏览内容 → 选择屏蔽/不感兴趣 → 配置屏蔽规则 → 生效 / 查看屏蔽列表 → 恢复
**非功能需求**: 屏蔽操作响应时间 <500ms，屏蔽后立即生效
**验收条件**: 屏蔽后内容不在信息流展示，但可访问主页，屏蔽可随时恢复

#### 故事列表

##### 故事 5.2.1: 屏蔽用户动态实现单向降噪

**作为** 用户，  
**我希望** 能够屏蔽某个用户的动态，使其内容不出现在推荐流与关注流中，但仍可访问其主页且保留关注关系，  
**以便** 减少噪音但不切断关系。

**验收标准**:
1. **给定** 我不想看到某用户的内容但不想拉黑，**当** 我在其内容上选择"屏蔽该用户"，**则** 系统将该用户加入我的屏蔽列表
2. **给定** 我屏蔽了某用户，**当** 我浏览推荐流或关注流，**则** 系统不再展示该用户的内容
3. **给定** 我屏蔽了某用户，**当** 我主动访问该用户主页，**则** 我可以正常查看其所有内容
4. **给定** 我屏蔽了某用户且之前关注了他，**当** 我查看关注列表，**则** 我仍然显示为关注状态
5. **给定** 我屏蔽了某用户，**当** 该用户查看我的主页，**则** 他可以正常查看我的内容（单向屏蔽）

**附加说明**:
- **优先级**: 高
- **依赖项**: EPIC-01
- **技术说明**: 屏蔽关系存储于屏蔽表；内容查询时过滤屏蔽用户的内容

##### 故事 5.2.2: 对内容选择"不感兴趣"并屏蔽类型或话题

**作为** 用户，  
**我希望** 对某条内容选择"不感兴趣"，并可选择屏蔽该类型内容或该话题，  
**以便** 快速优化信息流。

**验收标准**:
1. **给定** 我看到不感兴趣的内容，**当** 我点击"不感兴趣"，**则** 系统记录反馈并减少类似内容的推荐
2. **给定** 我点击"不感兴趣"后，**当** 系统弹出选项，**则** 我可以选择"屏蔽此类内容"或"屏蔽该话题"
3. **给定** 我选择屏蔽某类型内容（如视频），**当** 我浏览信息流，**则** 系统减少或不再展示该类型内容
4. **给定** 我选择屏蔽某话题（如"#八卦"），**当** 我浏览信息流，**则** 系统过滤包含该话题的内容
5. **给定** 我屏蔽了某话题，**当** 我查看屏蔽设置，**则** 我看到已屏蔽的话题列表并可取消屏蔽

**附加说明**:
- **优先级**: 中
- **依赖项**: 故事 5.2.1
- **技术说明**: "不感兴趣"反馈用于优化推荐算法；屏蔽规则存储于用户偏好表

##### 故事 5.2.3: 设置屏蔽词过滤不适内容

**作为** 用户，  
**我希望** 可以设置屏蔽词（关键词/正则规则），使包含屏蔽词的内容标题或正文被折叠/隐藏，  
**以便** 过滤不适内容。

**验收标准**:
1. **给定** 我想屏蔽包含特定词汇的内容，**当** 我进入"屏蔽词设置"并添加关键词，**则** 系统保存屏蔽词
2. **给定** 我添加了屏蔽词，**当** 我浏览信息流，**则** 系统检测内容标题和正文，若包含屏蔽词则折叠或隐藏该内容
3. **给定** 内容被屏蔽词过滤，**当** 我查看，**则** 我看到"该内容包含屏蔽词，已折叠"的提示，可选择展开
4. **给定** 我想使用正则表达式屏蔽，**当** 我添加正则规则，**则** 系统按正则匹配并过滤内容
5. **给定** 我想删除某个屏蔽词，**当** 我在列表中删除，**则** 系统不再使用该词过滤

**附加说明**:
- **优先级**: 低
- **依赖项**: 故事 5.2.1
- **技术说明**: 屏蔽词匹配可使用 Aho-Corasick 算法提高性能；支持批量导入导出

##### 故事 5.2.4: 临时屏蔽话题/Tag

**作为** 用户，  
**我希望** 可以设置在一段时间内屏蔽某话题/Tag，不再接收相关推荐，  
**以便** 阶段性降噪。

**验收标准**:
1. **给定** 我想暂时不看某话题，**当** 我选择"屏蔽此话题 7 天"，**则** 系统在该时间段内过滤该话题内容
2. **给定** 我设置了临时屏蔽，**当** 屏蔽期结束，**则** 系统自动取消屏蔽并恢复正常推荐
3. **给定** 我想提前取消临时屏蔽，**当** 我进入设置并取消，**则** 系统立即恢复该话题的推荐
4. **给定** 我查看临时屏蔽列表，**当** 我进入设置，**则** 我看到所有临时屏蔽的话题及剩余时间

**附加说明**:
- **优先级**: 低
- **依赖项**: 故事 5.2.2
- **技术说明**: 临时屏蔽通过 TTL 实现，到期自动删除

##### 故事 5.2.5: 独立管理屏蔽列表与黑名单

**作为** 用户，  
**我希望** 屏蔽列表与黑名单独立管理，且屏蔽可随时恢复，  
**以便** 保持可控与可逆。

**验收标准**:
1. **给定** 我进入"隐私设置"，**当** 我查看，**则** 我看到独立的"黑名单"和"屏蔽列表"入口
2. **给定** 我查看屏蔽列表，**当** 我点击进入，**则** 我看到所有被屏蔽的用户、话题、类型
3. **给定** 我想恢复某个屏蔽，**当** 我点击"取消屏蔽"，**则** 系统立即恢复该内容源的展示
4. **给定** 我想批量取消屏蔽，**当** 我选择多个项目并确认，**则** 系统一次性取消屏蔽
5. **给定** 我混淆了拉黑和屏蔽，**当** 我查看帮助文档，**则** 我看到清晰的说明："屏蔽是单向降噪、拉黑是双向切割"

**附加说明**:
- **优先级**: 中
- **依赖项**: 故事 5.1.3 和 5.2.1
- **技术说明**: 两个列表分别存储；前端清晰区分两种操作的文案和图标

### 特性 3: 行为边界明确与用户教育

**描述**: 明确"屏蔽是单向降噪、拉黑是双向切割"的行为边界，减少误用与纠纷
**关键交互**: 选择操作 → 查看说明 → 确认 → 生效
**非功能需求**: 提示信息清晰易懂，帮助用户做出正确选择
**验收条件**: 用户能理解拉黑和屏蔽的区别，能根据需求选择正确的操作

#### 故事列表

##### 故事 5.3.1: 明确拉黑与屏蔽的行为边界

**作为** 平台系统，  
**我希望** 明确"屏蔽是单向降噪、拉黑是双向切割"的行为边界，  
**以便** 减少误用与纠纷。

**验收标准**:
1. **给定** 用户准备拉黑某人，**当** 点击"拉黑"按钮，**则** 系统弹出确认框并说明"拉黑后双方将无法查看彼此内容，已有关注关系将解除"
2. **给定** 用户准备屏蔽某人，**当** 点击"屏蔽"按钮，**则** 系统弹出确认框并说明"屏蔽后您将不再看到该用户的内容，但仍可访问其主页，对方不受影响"
3. **给定** 用户在帮助中心，**当** 搜索"拉黑"或"屏蔽"，**则** 系统显示详细的对比说明和使用场景
4. **给定** 用户误用了拉黑，**当** 解除拉黑后，**则** 系统提示"拉黑已解除，但关注关系不会自动恢复，如需重新关注请手动操作"
5. **给定** 新用户首次使用，**当** 浏览到相关内容，**则** 系统显示引导提示介绍两种功能的区别

**附加说明**:
- **优先级**: 中
- **依赖项**: 故事 5.1.1 和 5.2.1
- **技术说明**: 帮助文档静态内容；引导提示可通过新手教程实现

## 风险与假设

- **风险**: 
  - 用户滥用拉黑功能导致社交孤立
  - 屏蔽规则过于复杂导致用户困惑
  - 拉黑后对方通过小号继续骚扰
- **假设**: 
  - 用户能理解拉黑和屏蔽的区别
  - 大部分用户会合理使用这些功能
  - 明确的边界说明能减少客服咨询
