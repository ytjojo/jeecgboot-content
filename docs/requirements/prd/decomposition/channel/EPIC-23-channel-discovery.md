# 频道分类、推荐与发现

## 执行摘要

EPIC-23 建立频道的分类、标签、推荐、榜单、精选、浏览和搜索能力，让用户能够在频道规模增长后仍然高效发现值得订阅和持续浏览的频道。它承接 EPIC-20 的频道基础信息、EPIC-21 的隐私与订阅关系、EPIC-22 的内容活跃信号，为后续 EPIC-24 的统计看板提供发现链路数据。

本史诗的产品愿景是：用户可以通过主动搜索、分类浏览、个性化推荐、排行榜和编辑精选等多种入口发现合适频道；平台运营可以用统一的分类体系和精选工具扶持优质频道；频道主可以通过准确分类和标签提升频道可发现性。发现能力必须优先保证公开可见边界、结果相关性和运营可解释性，避免私有频道误曝光、推荐理由不可理解或榜单口径不透明。

## 史诗概览

**史诗 ID**: EPIC-23
**标题**: 建立频道分类体系、推荐规则与多维度发现能力
**背景**: 随着频道数量增长，用户需要高效的发现机制来找到感兴趣的频道。需要建立完善的分类体系、个性化推荐、排行榜和搜索能力，帮助用户发现优质频道，同时为平台运营提供推荐引导工具。
**目标用户**: 内容消费者、平台运营、频道主、频道管理员
**优先级**: 中
**预计持续时间**: 4 周

**范围**:
- 包含：平台分类体系、频道内标签、个性化推荐、排行榜（热门/新晋/系统精选）、编辑精选、分类浏览、搜索（关键词/筛选/结果展示）。
- 不包含：频道创建（EPIC-20）、成员管理（EPIC-21）、内容发布（EPIC-22）、数据统计看板（EPIC-24）、底层推荐模型训练方案、搜索基础设施选型、平台级违规内容识别。

## 关键术语与业务规则

| 术语 | 定义 |
| --- | --- |
| 频道发现 | 用户通过推荐、榜单、编辑精选、分类浏览、关键词搜索等入口找到并进入频道的过程。 |
| 可发现频道 | 状态为 Active、隐私为公开、未被隐藏或限制公开曝光，且通过基础审核的频道。私有频道不得进入公开搜索、推荐、榜单或分类浏览结果。 |
| 平台分类 | 由平台运营维护的公共分类树，用于频道归属、浏览筛选和运营管理。分类最多支持 4 级，频道必须选择 1 个主分类，可选择 0 到 3 个副分类。 |
| 特殊分类 | 仅系统频道可使用的官方分类集合，包括“官方活动”“平台公告”“新手引导”“系统精选”。新增特殊分类需由平台运营配置并记录用途。 |
| 频道内标签 | 频道管理员在单个频道内维护的内容细分标签，仅用于该频道内容组织和频道内筛选，不替代平台分类。 |
| 个性化推荐 | 根据用户订阅、浏览、互动、屏蔽、不感兴趣反馈和频道可见性规则生成的频道候选列表。推荐结果需展示可理解理由，并遵守隐私和治理限制。 |
| 冷启动推荐 | 用户缺少足够行为数据时展示的推荐结果，由热门频道、编辑精选和系统频道组成。 |
| 不感兴趣反馈 | 用户对推荐频道表达负反馈的操作。反馈生效后，后续推荐中同一频道不再出现，相似主题频道的曝光需明显降低。 |
| 热门频道榜 | 反映频道当前整体受欢迎程度的榜单，支持日、周、月维度。 |
| 新晋频道榜 | 面向创建 30 天内频道的增长榜，反映新频道的订阅增长和内容活跃表现。 |
| 系统频道榜 | 展示平台官方系统频道的榜单，由平台运营控制展示资格和排序口径。 |
| 活跃度 | 用于排序和榜单的业务评分，范围 0 到 100。近 7 日活跃度 = 有效内容更新得分 40% + 有效互动得分 30% + 订阅增长得分 20% + 频道管理维护得分 10%。无效、违规或已删除内容不计入。 |
| 编辑精选 | 平台运营人工标记的优质频道集合，用于推荐页、频道发现页和冷启动场景展示。 |

## 用户旅程

### 旅程 1: 用户通过推荐发现并订阅频道

1. 用户进入频道发现页或首页推荐模块。
2. 系统展示推荐频道卡片，并标注频道类型、主分类、订阅数、简介和推荐理由。
3. 用户点击频道卡片进入频道主页，确认内容主题和活跃情况。
4. 用户选择订阅频道；若频道不可加入或需要申请，按 EPIC-21 的加入规则处理。
5. 用户对不感兴趣的推荐进行反馈，后续推荐减少同类频道曝光。

### 旅程 2: 用户通过分类浏览发现频道

1. 用户进入分类浏览页，查看平台分类树。
2. 用户选择一级分类或子分类，系统展示该分类及其子分类下的可发现频道。
3. 用户按订阅数、活跃度或创建时间排序，继续筛选个人、组织或系统频道。
4. 用户查看频道卡片，进入感兴趣频道主页。
5. 当分类下结果超过 20 个时，用户通过分页或滚动加载继续浏览。

### 旅程 3: 用户通过搜索快速找到目标频道

1. 用户在频道搜索入口输入关键词。
2. 系统返回覆盖频道名称、描述和标签的结果，并排除不可发现频道。
3. 用户按频道类型、平台分类、活跃度等条件筛选或排序。
4. 用户查看结果卡片的频道名称、图标、简介、订阅数、主分类和匹配原因。
5. 用户进入目标频道或在无结果时看到改写关键词、清除筛选和浏览分类入口。

### 旅程 4: 平台运营维护分类、榜单与精选

1. 平台运营进入频道发现配置后台。
2. 运营维护平台分类树、特殊分类和分类启停状态。
3. 运营查看推荐、榜单和搜索反馈数据，识别低质结果或冷启动不足。
4. 运营将优质频道加入编辑精选，配置展示位置和有效期。
5. 运营下线失效分类或取消精选标记，并确认对用户端发现入口的影响。

## UX/UI 与响应式要求

- 频道发现页需同时提供推荐频道、排行榜、编辑精选、分类入口和搜索入口，首屏优先展示对用户最可能有价值的频道入口。
- 推荐频道卡片需展示频道名称、图标、频道类型、主分类、简介、订阅数、推荐理由、订阅状态和“不感兴趣”操作。
- 分类浏览页需展示清晰的层级导航，用户可返回上级分类；当前分类、筛选条件和排序方式必须可见。
- 搜索结果页需展示关键词、筛选条件、结果数量、匹配原因和空状态引导。
- 排行榜页需清晰区分热门、新晋、系统频道榜，并展示榜单更新时间和榜单口径说明入口。
- 编辑精选模块需展示精选理由或运营推荐语，避免与系统自动推荐混淆。
- 分类管理后台需支持新增、编辑、停用、排序和查看分类影响范围；停用分类前需提示已关联频道数量。
- 标签管理入口需位于频道管理后台，标签删除时提示对历史内容筛选的影响。
- 空状态需覆盖无推荐、无分类结果、无搜索结果、无排行榜数据、无精选频道，并提供搜索、浏览分类或清除筛选入口。
- 错误态需说明具体原因，例如频道不可见、筛选条件无结果、推荐暂不可用或榜单尚未更新。
- 响应式要求：移动端频道卡片保留名称、图标、主分类、简介、订阅状态和主操作；次要指标折叠到详情或更多信息中。
- 响应式要求：窄屏下分类树采用分级进入方式，筛选条件收纳为筛选面板，避免横向溢出。
- 响应式要求：移动端搜索框、筛选、排序和结果列表必须可单手完成主要操作，榜单切换使用顶部标签或下拉选择。

## 特性列表

### 特性 1: 频道分类与标签体系

**描述**: 建立平台级多级树形分类体系和频道内自定义标签系统，为频道组织、搜索和推荐提供基础。
**关键交互**: 平台配置分类树 → 频道创建时选择分类 → 频道管理员创建标签 → 用户按分类或标签浏览。
**非功能需求**: 分类最多 4 级；分类和标签变更后 5 分钟内在用户端可见。
**验收条件**: 分类体系完整可用，频道内标签创建和使用正常，公开发现入口遵守频道可见性规则。

#### 故事列表

##### 故事 23.1.1: 平台分类体系管理

**作为** 平台运营，
**我希望** 管理平台的多级树形分类体系（如科技 → 人工智能 → 自然语言处理），
**以便** 为频道提供统一的分类标准。

**验收标准**:
1. **给定** 平台运营进入分类管理页，**当** 创建新分类并设置父级分类，**则** 新分类出现在分类树的对应位置，且分类层级不超过 4 级。
2. **给定** 频道创建时，**当** 用户选择分类，**则** 显示完整的分类树供选择，要求选择 1 个主分类和最多 3 个副分类。
3. **给定** 用户申请增设新分类，**当** 平台审批通过，**则** 新分类纳入公共分类体系，并可在频道创建和分类浏览中选择。
4. **给定** 系统频道，**当** 选择分类，**则** 可归属至特殊分类“官方活动”“平台公告”“新手引导”“系统精选”之一。
5. **给定** 平台运营停用某分类，**当** 该分类下仍有关联频道，**则** 系统展示影响范围并要求运营选择迁移分类或保留历史归属。

**附加说明**:
- **优先级**: 高
- **依赖项**: EPIC-20（频道创建）
- **产品规则**: 分类是平台公共发现口径，必须由平台运营维护；普通用户仅可申请新增分类，不可直接修改公共分类树。

##### 故事 23.1.2: 频道内标签管理

**作为** 频道管理员，
**我希望** 创建和管理频道内的自定义标签（如“教程”“讨论”“资源”），
**以便** 对频道内容进行细分组织。

**验收标准**:
1. **给定** 管理员进入频道标签管理页，**当** 创建新标签，**则** 标签可用于频道内内容分类。
2. **给定** 用户发布内容到频道，**当** 选择标签，**则** 显示该频道的自定义标签列表。
3. **给定** 用户浏览频道内容，**当** 按标签筛选，**则** 仅显示带有该标签的内容。
4. **给定** 管理员删除标签，**当** 操作完成，**则** 已使用该标签的内容保留历史标签展示，但该标签不再出现在新增内容选择和频道筛选入口中。
5. **给定** 管理员创建或编辑标签，**当** 标签名称为空、重复或超过 20 个字符，**则** 系统拒绝保存并提示具体原因。

**附加说明**:
- **优先级**: 中
- **依赖项**: 故事 23.1.1、EPIC-22（内容发布与治理）
- **产品规则**: 频道内标签仅在当前频道生效，不改变内容所属平台分类，也不自动影响其他频道的发现结果。

### 特性 2: 频道推荐与排行榜

**描述**: 基于用户兴趣、订阅关系、行为反馈和频道可见性规则提供个性化频道推荐，建立热门、新晋、系统频道榜单，并支持编辑精选。
**关键交互**: 用户进入发现入口 → 展示推荐或榜单 → 用户点击、订阅或反馈 → 平台运营调整精选和榜单展示。
**非功能需求**: 推荐反馈 5 分钟内反映主要变化；排行榜每日更新且延迟 <= 5 分钟。
**验收条件**: 推荐结果与用户兴趣相关，排行榜口径清晰且数据及时更新，精选频道可由运营维护。

#### 故事列表

##### 故事 23.2.1: 个性化频道推荐

**作为** 内容消费者，
**我希望** 看到根据我的兴趣和行为推荐的频道，
**以便** 发现我可能感兴趣的新频道。

**验收标准**:
1. **给定** 用户已订阅多个频道并有浏览历史，**当** 进入频道推荐页面，**则** 展示基于用户画像、行为和订阅历史推荐的相关频道。
2. **给定** 新用户无行为数据，**当** 进入推荐页面，**则** 展示热门频道、编辑精选和系统频道作为冷启动结果。
3. **给定** 用户对推荐频道点击“不感兴趣”，**当** 系统记录反馈，**则** 后续 30 天内不再推荐同一频道，并显著减少同主题频道曝光。
4. **给定** 推荐频道展示，**当** 用户查看，**则** 显示推荐理由（如“因为你订阅了 XX 频道”或“与你最近浏览的主题相关”）。
5. **给定** 推荐候选中存在私有、隐藏、冻结、限制公开曝光或未通过审核的频道，**当** 生成推荐结果，**则** 这些频道不得展示给无权访问的用户。

**附加说明**:
- **优先级**: 中
- **依赖项**: 故事 23.1.1、EPIC-21（订阅关系与可见性规则）
- **产品规则**: 推荐必须可解释、可反馈、可退出；推荐质量以点击率、订阅转化和不感兴趣反馈率综合评估。

##### 故事 23.2.2: 频道排行榜

**作为** 内容消费者，
**我希望** 查看热门频道榜、新晋频道榜等排行榜，
**以便** 了解当前社区最受欢迎和增长最快的频道。

**验收标准**:
1. **给定** 用户进入排行榜页面，**当** 查看热门频道榜，**则** 按订阅数、近 7 日活跃度等指标排序，分为日、周、月维度。
2. **给定** 用户查看新晋频道榜，**当** 加载榜单，**则** 展示创建 30 天内增长最快的频道。
3. **给定** 用户查看系统频道专属榜单，**当** 加载榜单，**则** 展示官方推荐的系统频道。
4. **给定** 排行榜数据，**当** 到达每日约定更新时间，**则** 用户可见榜单更新时间延迟 <= 5 分钟。
5. **给定** 用户查看榜单页面，**当** 榜单展示，**则** 显示榜单更新时间和榜单口径说明入口。

**附加说明**:
- **优先级**: 中
- **依赖项**: 故事 23.1.1、EPIC-22（内容活跃信号）
- **产品规则**: 榜单仅展示可发现频道；活跃度按本 PRD 术语定义计算，同分时按订阅数、创建时间倒序展示。

##### 故事 23.2.3: 编辑精选频道

**作为** 平台运营，
**我希望** 手动推荐优质频道为“编辑精选”，
**以便** 在独立模块中展示人工筛选的高质量频道。

**验收标准**:
1. **给定** 平台运营选择某频道，**当** 标记为“编辑精选”，**则** 该频道出现在编辑精选模块。
2. **给定** 用户浏览编辑精选模块，**当** 加载页面，**则** 展示所有当前有效的编辑精选频道。
3. **给定** 平台运营取消编辑精选标记，**当** 操作完成，**则** 频道从精选模块消失。
4. **给定** 平台运营配置精选频道，**当** 设置推荐语和有效期，**则** 用户端展示推荐语，并在有效期结束后不再展示该精选。
5. **给定** 频道变为私有、隐藏、冻结或限制公开曝光，**当** 该频道已被标记精选，**则** 用户端不再展示该频道，运营后台提示状态异常。

**附加说明**:
- **优先级**: 低
- **依赖项**: 故事 23.2.2
- **产品规则**: 编辑精选是人工运营入口，必须保留操作人、推荐语、有效期和上下线状态，便于运营复盘和用户解释。

### 特性 3: 频道分类浏览与搜索

**描述**: 支持按官方分类树浏览全部可发现频道，提供关键词搜索、多条件筛选和搜索结果展示。
**关键交互**: 用户选择分类或输入关键词 → 系统返回结果 → 用户筛选、排序、查看频道卡片 → 用户进入频道或订阅。
**非功能需求**: 搜索与分类浏览首屏结果响应 P99 <= 200ms；结果分页默认每页 20 条。
**验收条件**: 分类浏览和搜索功能完整，结果准确且响应快速，私有和不可公开曝光频道不进入公开结果。

#### 故事列表

##### 故事 23.3.1: 分类浏览

**作为** 内容消费者，
**我希望** 按官方分类树浏览全部公开频道，
**以便** 按兴趣领域发现频道。

**验收标准**:
1. **给定** 用户进入分类浏览页面，**当** 选择某一级分类，**则** 展示该分类及其子分类下的所有可发现频道。
2. **给定** 用户选择子分类，**当** 加载结果，**则** 展示该子分类下的频道，支持多条件排序（订阅数、活跃度、创建时间）。
3. **给定** 用户浏览分类结果，**当** 查看频道卡片，**则** 显示图标、名称、频道类型、主分类、订阅数和简介。
4. **给定** 分类下频道数量超过 20 个，**当** 用户滚动或翻页，**则** 支持按 20 条为一页加载更多结果。
5. **给定** 分类下没有可发现频道，**当** 页面加载完成，**则** 显示空状态，并提供返回上级分类、浏览热门频道或搜索入口。

**附加说明**:
- **优先级**: 中
- **依赖项**: 故事 23.1.1
- **产品规则**: 分类浏览必须遵守频道隐私、审核状态和治理状态；分类页展示结果需与频道详情页可见性一致。

##### 故事 23.3.2: 频道搜索

**作为** 内容消费者，
**我希望** 通过关键词搜索频道，按类型、分类、活跃度等条件筛选，
**以便** 快速找到目标频道。

**验收标准**:
1. **给定** 用户输入关键词，**当** 执行搜索，**则** 搜索覆盖频道名称、描述、标签字段。
2. **给定** 搜索结果已返回，**当** 用户按类型筛选（个人、组织、系统），**则** 仅显示对应类型的频道。
3. **给定** 搜索结果已返回，**当** 用户按分类筛选，**则** 仅显示该分类及其子分类下的频道。
4. **给定** 搜索结果已返回，**当** 用户按活跃度排序，**则** 按近 7 日活跃度从高到低排序。
5. **给定** 搜索结果展示，**当** 查看频道卡片，**则** 显示图标、名称、频道类型、主分类、订阅数、简介和匹配原因。
6. **给定** 搜索结果中存在私有、隐藏、冻结、限制公开曝光或未通过审核的频道，**当** 用户无权访问，**则** 这些频道不得出现在搜索结果中。
7. **给定** 搜索无结果，**当** 页面展示空状态，**则** 提供清除筛选、改写关键词和浏览分类入口。

**附加说明**:
- **优先级**: 高
- **依赖项**: 故事 23.1.1、EPIC-21（隐私与可见性规则）
- **产品规则**: 搜索结果相关性需结合关键词匹配、频道状态、分类一致性和活跃度排序；不得用搜索入口绕过频道隐私边界。

## 风险与假设

| 类型 | 内容 | 影响 | 应对 |
| --- | --- | --- | --- |
| 风险 | 推荐冷启动效果可能不佳 | 新用户发现效率低，推荐点击率不足 | 使用热门频道、编辑精选和系统频道兜底，并持续观察冷启动点击率 |
| 风险 | 搜索结果相关性不足 | 用户难以找到目标频道，满意度下降 | 提供结果反馈入口，按关键词、分类、活跃度和点击行为持续校准结果口径 |
| 风险 | 分类体系过深或过细 | 用户选择困难，频道创建和浏览成本升高 | 分类最多 4 级，新增分类需通过平台运营审批 |
| 风险 | 私有或受限频道误曝光 | 造成隐私和信任风险 | 将可见性边界作为搜索、推荐、榜单、分类浏览和精选的发布前验收重点 |
| 风险 | 榜单口径不透明 | 频道主和用户质疑榜单公平性 | 用户端展示更新时间和口径说明入口，运营后台保留配置与调整记录 |
| 假设 | 用户行为数据采集已具备基础能力 | 推荐、榜单和搜索质量评估可使用浏览、点击、订阅和反馈数据 | 若数据不足，先以热门、精选、分类浏览和明确反馈作为兜底 |
| 假设 | EPIC-21 的隐私与订阅规则可被发现入口调用 | 私有频道和限制公开曝光频道可被稳定排除 | 若依赖未就绪，EPIC-23 不发布公开搜索和推荐能力 |
