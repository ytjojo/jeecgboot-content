# 频道隐私、订阅与成员管理

## 执行摘要

EPIC-21 建立频道的访问控制、订阅关系、成员角色和频道级治理能力，让频道从“可创建的内容容器”升级为“可运营、可参与、可治理的社区单元”。本史诗承接 EPIC-20 的频道基础架构，重点解决用户是否能看、是否能加入、如何持续订阅，以及频道主如何管理成员和处理扰乱秩序的行为。

本史诗的产品愿景是：不同类型频道在隐私、加入、订阅和成员治理上保持一致、可解释、可验证的体验。内容消费者能清楚理解频道规则并顺畅订阅或申请加入；频道主和管理员能以明确权限管理成员；平台能通过审计和合规要求降低隐私泄露、骚扰和治理争议风险。

## 史诗概览

**史诗 ID**: EPIC-21

**标题**: 建立频道隐私控制、订阅机制与成员管理体系

**背景**: 频道创建后需要隐私设置、加入规则和成员管理能力，以保护内容质量、控制访问范围，并支持频道主维护成员秩序。系统频道、个人频道、组织频道在隐私和成员规则上存在差异，需要统一产品口径。

**目标用户**: 频道主、频道管理员、内容编辑、普通成员、内容消费者、组织管理员、平台运营

**范围**:
- 包含：频道隐私设置、加入方式配置、加入申请审核、订阅与取消订阅、订阅列表管理、成员角色体系、成员列表、移除、禁言、频道黑名单
- 不包含：频道创建与所有权管理（EPIC-20）、内容发布权限与内容治理（EPIC-22）、频道推荐发现算法（EPIC-23）、完整付费频道闭环

**预计持续时间**: 3 周

**优先级**: 高

## 关键术语与业务规则

| 术语 | 定义 |
| --- | --- |
| 公开频道 | 所有人可搜索、可进入频道主页并浏览公开内容的频道。 |
| 私有频道 | 仅成员可浏览受限内容，不进入公开搜索、公开推荐和游客可访问页面的频道。 |
| 自由加入 | 用户点击加入后，在满足基础限制条件时立即成为频道成员。 |
| 审核加入 | 用户提交申请后进入待审状态，由频道主或有权限管理员批准或拒绝。 |
| 邀请加入 | 用户凭频道主或管理员发出的邀请凭证加入频道。邀请凭证必须在有效期内，未超过可用次数，且未被撤销。 |
| 待审队列 | 频道内待处理加入申请列表，按提交时间默认倒序展示；同一用户对同一频道只能保留一个未处理申请。 |
| 默认关注策略 | 平台为新用户自动订阅指定系统频道的运营规则；启用范围、频道清单和退出能力需在平台运营后台配置并可审计。 |
| 按权限发布 | 普通成员仅能在频道发布模型允许时投稿或发布；不具备成员管理、频道设置、内容置顶、删除、精华等管理权限。 |
| 冷却期 | 成员被移除后 7 天内不可再次加入同一频道；频道主或平台运营可基于申诉结果提前解除。 |
| 禁言 | 成员在指定时长内不可发布内容、投稿或评论，但可继续浏览其有权限访问的内容。 |
| 频道黑名单 | 频道级阻断名单；被加入后不能申请加入、接受邀请或成为频道成员，直到管理员移出黑名单。 |
| 治理操作 | 移除成员、禁言、解除禁言、加入黑名单、移出黑名单、批量审核等影响成员资格或行为权限的操作。 |
| 组织授权 | 组织频道展示成员姓名、部门等组织信息前，必须确认组织已授权频道使用相关组织成员信息。 |

## 用户旅程

### 旅程 1: 内容消费者订阅公开频道

1. 用户通过搜索、推荐、排行榜或分类入口进入公开频道主页。
2. 用户查看频道名称、简介、规则、内容示例、订阅状态和加入方式。
3. 用户点击“订阅”。
4. 系统校验频道状态、用户状态和黑名单限制。
5. 订阅成功后，频道进入用户订阅列表，按钮变为“已订阅”，后续信息流可优先展示该频道内容。
6. 用户可在订阅列表中搜索、分组、关闭提醒或取消订阅。

### 旅程 2: 用户申请加入私有频道

1. 用户通过邀请、频道链接或可见入口进入私有频道访问页。
2. 页面展示频道基础信息、加入方式和可见范围，不展示受限内容。
3. 用户点击“申请加入”并填写申请理由。
4. 申请进入待审队列；用户看到待审核状态，不能重复提交。
5. 频道主或管理员批准后，用户成为成员并收到通知；拒绝后用户收到拒绝通知和可再次申请条件。

### 旅程 3: 频道主配置隐私与加入规则

1. 频道主进入频道设置页。
2. 选择公开或私有隐私状态，并配置自由加入、审核加入或邀请加入。
3. 系统根据频道类型展示可用选项；系统频道不可设为私有。
4. 频道主保存设置后，新访问和新加入请求按最新规则处理。
5. 若变更会影响搜索、推荐或现有成员体验，页面展示影响说明和确认提示。

### 旅程 4: 管理员治理违规成员

1. 管理员进入成员管理页，搜索或筛选目标成员。
2. 管理员查看成员角色、加入时间、贡献情况和历史治理状态。
3. 管理员选择移除、禁言或加入黑名单，并填写原因。
4. 系统校验管理员权限、目标成员角色和频道状态。
5. 操作完成后，被处理成员收到通知；审计记录可供频道主、平台运营和申诉流程追溯。

## UX/UI 与响应式要求

- 频道主页需清晰展示当前用户与频道的关系：未订阅、已订阅、非成员、待审核、已加入、被禁言、被黑名单命中。
- 公开频道展示“订阅”主操作；私有频道展示“申请加入”或“通过邀请加入”主操作；已订阅或已加入状态需提供明确的取消或退出入口。
- 频道设置页需将“隐私状态”和“加入方式”分组展示，并在系统频道、个人频道、组织频道下隐藏或禁用不可用选项。
- 成员管理页需支持按角色、状态、加入时间筛选，并提供搜索、批量审核、批量移除、禁言、黑名单入口。
- 治理操作必须有二次确认；确认文案需说明影响范围、冷却期或禁言到期时间。
- 空状态需覆盖无订阅、无成员、无待审申请、无黑名单用户，并提供下一步行动入口。
- 错误态需说明具体原因，例如权限不足、频道冻结、重复申请、邀请失效、黑名单命中。
- 响应式要求：移动端优先保证频道主页主操作、订阅状态、申请状态和成员治理动作可见；成员列表在窄屏下使用分段筛选和单行操作菜单，避免横向溢出。
- 管理端批量操作在移动端可降级为多选列表加底部固定操作栏，操作按钮需保持清晰可触达。

## 特性列表

### 特性 1: 频道隐私设置与加入规则

**描述**: 支持频道设置公开/私有隐私状态，配置自由加入、审核加入、邀请加入，系统频道强制公开。

**关键交互**: 频道主设置隐私 → 配置加入方式 → 用户根据规则订阅、加入或申请加入。

**产品规则**:
- 公开频道可被搜索、推荐和浏览；私有频道仅成员可浏览受限内容，不进入公开搜索和公开推荐。
- 系统频道必须公开，不允许设置为私有。
- 审核加入只允许保留一个未处理申请；邀请加入必须校验有效期、可用次数和撤销状态。

**验收条件**: 隐私设置在三类频道下正确生效，加入方式配置完整可用，私有内容不对非成员公开。

#### 故事列表

##### 故事 21.1.1: 频道隐私设置

**作为** 频道主，
**我希望** 设置频道的隐私状态（公开/私有），
**以便** 控制频道内容的可见范围。

**验收标准**:
1. **给定** 个人/组织频道主，**当** 将频道设为公开，**则** 所有人可搜索频道并浏览公开内容。
2. **给定** 个人/组织频道主，**当** 将频道设为私有，**则** 仅成员可浏览受限内容，频道不进入公开搜索和公开推荐。
3. **给定** 系统频道，**当** 操作者尝试设置为私有，**则** 系统拒绝并提示“系统频道必须公开”。
4. **给定** 频道隐私状态已修改，**当** 保存成功，**则** 新访问请求立即按新隐私规则处理，公开搜索和推荐入口在 5 分钟内完成可见性同步。

**附加说明**:
- **优先级**: 高
- **依赖项**: EPIC-20 故事 20.1.1
- **产品规则**: 隐私变更需提示对搜索、推荐、订阅和成员访问的影响。

##### 故事 21.1.2: 加入方式配置

**作为** 频道主，
**我希望** 为频道配置加入方式（自由加入/审核加入/邀请加入），
**以便** 控制新成员的加入流程和频道准入门槛。

**验收标准**:
1. **给定** 频道设置为自由加入，**当** 用户点击加入，**则** 用户在满足频道状态、账号状态和黑名单限制后立即成为成员。
2. **给定** 频道设置为审核加入，**当** 用户提交加入申请，**则** 申请进入待审状态，频道主或管理员可批准或拒绝。
3. **给定** 频道设置为邀请加入，**当** 用户使用邀请链接或邀请码，**则** 仅在邀请未过期、未超过可用次数、未撤销且用户未被黑名单命中时可加入。
4. **给定** 公开频道设置为自由加入，**当** 用户浏览频道，**则** 显示“加入”按钮并说明加入后可获得的权限。
5. **给定** 私有频道设置为审核加入，**当** 用户浏览频道，**则** 显示“申请加入”按钮、审核说明和当前申请状态。

**附加说明**:
- **优先级**: 高
- **依赖项**: 故事 21.1.1
- **产品规则**: 同一用户对同一频道只能存在一个未处理申请；已拒绝申请是否可再次提交遵循频道配置。

##### 故事 21.1.3: 加入申请审核

**作为** 频道管理员，
**我希望** 审核用户的加入申请，批准或拒绝申请者，
**以便** 维护频道成员质量。

**验收标准**:
1. **给定** 有新的加入申请，**当** 管理员进入审核页面，**则** 展示申请者昵称、申请时间、申请理由和当前处理状态。
2. **给定** 管理员批准申请，**当** 操作完成，**则** 申请者成为频道成员并收到通知。
3. **给定** 管理员拒绝申请，**当** 操作完成，**则** 申请者收到拒绝通知；若频道允许再次申请，通知需说明再次申请条件。
4. **给定** 管理员批量审核，**当** 选择多个申请并批量批准或拒绝，**则** 每个申请展示成功或失败结果，失败项保留原因。
5. **给定** 申请超过 48 小时未处理，**当** 管理员查看待审列表，**则** 超时申请被突出展示以便优先处理。

**附加说明**:
- **优先级**: 中
- **依赖项**: 故事 21.1.2
- **产品规则**: 审核处理需记录操作者、申请者、处理结果、时间和原因。

### 特性 2: 频道订阅机制

**描述**: 用户可订阅/取消订阅频道，订阅后频道内容在信息流中获得订阅加权展示，并支持订阅列表管理。

**关键交互**: 用户浏览频道 → 点击订阅 → 频道出现在订阅列表 → 信息流优先展示订阅频道内容。

**产品规则**:
- 取消订阅不影响历史互动数据和已发布内容。
- 私有频道需先成为成员，才能订阅和浏览受限内容。
- 系统频道的默认关注策略必须支持用户后续取消订阅，除非平台另有合规要求并展示说明。

**验收条件**: 订阅/取消订阅功能完整，订阅列表管理正常，订阅加权在信息流展示中可验证。

#### 故事列表

##### 故事 21.2.1: 订阅与取消订阅

**作为** 内容消费者，
**我希望** 订阅感兴趣的频道并在信息流中优先看到其内容，
**以便** 及时获取感兴趣的内容更新。

**验收标准**:
1. **给定** 用户浏览公开频道，**当** 点击“订阅”按钮，**则** 用户成功订阅该频道，按钮变为“已订阅”。
2. **给定** 用户已订阅频道，**当** 点击“取消订阅”，**则** 用户取消订阅，不影响已发布内容及历史互动数据。
3. **给定** 用户订阅频道后，**当** 刷新信息流，**则** 在同等内容质量和安全条件下，订阅频道内容优先于未订阅频道内容展示。
4. **给定** 私有频道，**当** 非成员用户尝试订阅，**则** 系统引导用户先申请加入或使用邀请加入。
5. **给定** 系统频道启用默认关注策略，**当** 新用户注册完成，**则** 自动订阅配置清单内的系统频道，并在订阅列表中允许用户查看来源和取消订阅。

**附加说明**:
- **优先级**: 高
- **依赖项**: 故事 21.1.1
- **产品规则**: 订阅成功、取消订阅、默认关注需给出明确反馈，并可追踪来源。

##### 故事 21.2.2: 订阅列表管理

**作为** 内容消费者，
**我希望** 管理我的频道订阅列表，支持分组和搜索，
**以便** 高效管理已订阅的频道。

**验收标准**:
1. **给定** 用户已订阅多个频道，**当** 进入订阅列表页面，**则** 展示频道图标、名称、订阅状态、最新内容摘要和提醒状态。
2. **给定** 用户在订阅列表中，**当** 按默认分组或自定义分组筛选，**则** 仅显示该分组下的频道。
3. **给定** 用户创建自定义分组，**当** 将频道移入分组，**则** 频道在新分组中显示，原分组关系按用户选择更新。
4. **给定** 用户在订阅列表中搜索，**当** 输入关键词，**则** 按频道名称模糊匹配筛选。
5. **给定** 用户关闭某频道消息提醒，**当** 该频道有更新，**则** 用户不再收到该频道的更新推送，但订阅关系保持不变。

**附加说明**:
- **优先级**: 中
- **依赖项**: 故事 21.2.1
- **产品规则**: 订阅分组仅影响用户个人管理视图，不影响频道成员关系和推荐资格。

### 特性 3: 频道成员角色与权限体系

**描述**: 定义频道主、管理员、内容编辑、普通成员四类频道内角色，不同角色拥有不同操作权限。

**关键交互**: 频道主任命管理员 → 管理员分配角色 → 各角色按权限操作。

**产品规则**:
- 频道主拥有频道最高管理权，可管理基础信息、成员角色和频道转让。
- 管理员可管理内容与成员，不可修改频道基础信息或转让频道。
- 内容编辑可编辑、删除、置顶、精华频道内容，不可管理成员。
- 普通成员可浏览其有权限访问的内容，并在频道发布模型允许时投稿或发布。

**验收条件**: 四种角色在系统、个人、组织频道下权限正确，角色变更对新操作立即生效。

#### 故事列表

##### 故事 21.3.1: 角色定义与权限分配

**作为** 频道主，
**我希望** 为频道成员分配不同角色（管理员/内容编辑/普通成员），
**以便** 实现频道的分层管理。

**验收标准**:
1. **给定** 频道主进入成员管理页，**当** 将某成员设置为管理员，**则** 该成员可管理内容与成员，但不可修改频道基础信息或转让频道。
2. **给定** 频道主将某成员设置为内容编辑，**当** 该成员操作频道，**则** 可执行内容编辑、删除、置顶、精华等内容管理操作，但不可管理成员。
3. **给定** 频道主将某成员设为普通成员，**当** 该成员操作频道，**则** 可浏览其有权访问的内容，并仅在频道发布模型允许时投稿或发布。
4. **给定** 组织频道，**当** 查看成员姓名、部门等组织信息，**则** 必须满足组织授权要求；未授权时仅展示平台公开身份信息。
5. **给定** 系统频道，**当** 查看角色体系，**则** 频道主为平台运营负责人，普通频道成员不可获得系统频道最高管理权。

**附加说明**:
- **优先级**: 高
- **依赖项**: EPIC-20 故事 20.3.1
- **产品规则**: 角色变更需提示影响范围，并记录变更前后角色、操作者和时间。

##### 故事 21.3.2: 成员列表展示

**作为** 频道管理员，
**我希望** 查看频道成员列表，了解成员的角色、加入时间和贡献情况，
**以便** 有效管理频道成员。

**验收标准**:
1. **给定** 管理员进入成员管理页，**当** 加载成员列表，**则** 展示昵称、头像、角色、加入时间、贡献数和治理状态。
2. **给定** 成员列表已加载，**当** 按角色筛选（频道主/管理员/内容编辑/普通成员），**则** 仅显示对应角色的成员。
3. **给定** 成员列表已加载，**当** 按加入时间排序，**则** 可按时间正序或倒序排列。
4. **给定** 成员列表已加载，**当** 搜索成员昵称，**则** 模糊匹配并显示结果。
5. **给定** 当前用户无成员管理权限，**当** 访问成员管理页，**则** 系统拒绝访问并提示权限不足。

**附加说明**:
- **优先级**: 中
- **依赖项**: 故事 21.3.1
- **产品规则**: 成员列表需支持空状态、加载状态和权限不足错误态。

### 特性 4: 成员治理操作

**描述**: 支持频道管理员对成员进行移除、禁言、加入黑名单等治理操作，维护频道秩序。

**关键交互**: 管理员选择成员 → 执行治理操作 → 成员收到通知 → 操作记录可追溯。

**产品规则**:
- 治理操作必须校验操作者权限、目标成员角色和频道状态。
- 被移除者在 7 天冷却期内不可再次加入同一频道，申诉通过后可提前解除。
- 黑名单优先级高于邀请和自由加入；黑名单命中时不得加入频道。
- 治理操作需提供原因填写、二次确认、结果反馈和申诉所需记录。

**验收条件**: 移除、禁言、黑名单功能在各频道类型下正确生效，关键操作可追溯。

#### 故事列表

##### 故事 21.4.1: 移除成员

**作为** 频道管理员，
**我希望** 将违规或不适合的成员从频道中移除，
**以便** 维护频道内容和互动质量。

**验收标准**:
1. **给定** 管理员选择某成员，**当** 点击“移除成员”并确认，**则** 该成员被移出频道，频道从其订阅列表消失。
2. **给定** 成员已被移除，**当** 该成员尝试重新加入，**则** 7 天冷却期内无法加入，并看到可再次申请时间。
3. **给定** 管理员批量移除成员，**当** 选择多个成员并确认，**则** 每个成员展示成功或失败结果，失败项保留原因。
4. **给定** 移除操作完成，**当** 查看治理记录，**则** 记录操作人、被移除者、操作时间、原因和冷却期结束时间。
5. **给定** 目标成员为频道主或权限高于操作者，**当** 管理员尝试移除，**则** 系统拒绝并提示权限不足。

**附加说明**:
- **优先级**: 高
- **依赖项**: 故事 21.3.1
- **产品规则**: 移除不删除成员历史互动记录，但会取消其频道成员资格和订阅关系。

##### 故事 21.4.2: 禁言成员

**作为** 频道管理员，
**我希望** 对违规成员设定禁言时长，限制其发布和评论能力，
**以便** 惩戒不当行为同时保留其浏览权限。

**验收标准**:
1. **给定** 管理员选择某成员，**当** 设置禁言时长（1 天/7 天/30 天/永久），**则** 该成员在禁言期间不可发布内容、投稿或评论。
2. **给定** 成员已被禁言，**当** 该成员尝试发布内容、投稿或评论，**则** 系统提示“您已被禁言，解封时间：YYYY-MM-DD”。
3. **给定** 禁言到期，**当** 成员再次尝试发布内容、投稿或评论，**则** 系统按其当前角色和频道发布规则处理。
4. **给定** 管理员手动解除禁言，**当** 操作完成，**则** 成员立即恢复其角色允许的发布、投稿或评论能力。
5. **给定** 禁言操作完成，**当** 查看治理记录，**则** 记录操作人、被禁言者、原因、开始时间、结束时间和解除方式。

**附加说明**:
- **优先级**: 高
- **依赖项**: 故事 21.3.1
- **产品规则**: 禁言不影响成员浏览其有权限访问的内容，也不自动取消订阅。

##### 故事 21.4.3: 黑名单管理

**作为** 频道管理员，
**我希望** 将严重违规或骚扰用户加入频道黑名单，
**以便** 阻止其继续申请加入或通过邀请进入频道。

**验收标准**:
1. **给定** 管理员将某用户加入黑名单，**当** 操作完成，**则** 该用户无法申请加入、接受邀请或成为该频道成员。
2. **给定** 黑名单中的用户尝试访问频道加入入口，**当** 系统检查，**则** 提示“您无法加入此频道”。
3. **给定** 管理员将用户从黑名单移除，**当** 操作完成，**则** 该用户可按频道当前加入规则重新申请或加入。
4. **给定** 管理员查看黑名单列表，**当** 加载页面，**则** 展示被拉黑用户昵称、拉黑时间、操作人、原因和当前状态。
5. **给定** 用户已在频道内，**当** 管理员将其加入黑名单，**则** 系统同时移除其成员资格，并记录影响范围。

**附加说明**:
- **优先级**: 中
- **依赖项**: 故事 21.4.1
- **产品规则**: 频道黑名单仅影响该频道；是否同步平台级黑名单由平台治理规则另行决定。

## 风险与假设

| 类型 | 内容 | 影响 | 缓解措施 |
| --- | --- | --- | --- |
| 风险 | 私有频道内容被搜索、推荐或分享入口误曝光 | 造成隐私事故和用户信任损失 | 将私有可见性作为发布前验收重点，覆盖搜索、推荐、直接访问和分享入口 |
| 风险 | 默认关注策略引发用户反感 | 影响新用户体验和订阅转化 | 默认关注需展示来源，并允许用户在订阅列表中取消 |
| 风险 | 加入申请和治理操作堆积 | 审核超时、成员管理效率下降 | 对超时申请突出展示，支持批量处理和失败结果反馈 |
| 风险 | 组织成员信息展示不符合授权要求 | 造成组织数据合规风险 | 未确认组织授权时仅展示平台公开身份信息 |
| 假设 | 通知系统已具备站内信和推送能力 | 审核结果、治理处罚和默认关注说明可及时触达用户 | 若通知能力未就绪，需先提供站内消息兜底 |
| 假设 | 推荐系统可接收订阅关系和频道可见性约束 | 订阅加权和私有频道排除可在 EPIC-23 中实现 | 本史诗仅定义产品规则和验收口径 |
