# 频道基础架构与创建

## 执行摘要

内容社区需要一个统一、可治理、可扩展的频道基础能力，让平台官方、个人创作者和认证组织都能用一致的产品规则创建和管理频道。EPIC-20 负责建立三类频道的基础类型、创建入口、审核状态、编辑、转让和删除流程，为后续 EPIC-21 隐私与成员、EPIC-22 内容发布、EPIC-23 推荐发现、EPIC-24 生命周期治理提供稳定前置。

本史诗的差异化定位是：先把频道作为可信的内容组织单元建立起来，而不是一次性实现完整的发现、订阅、发布和统计闭环。它保证频道创建后具备明确的类型、归属、审核状态、可展示基础信息和可追踪管理动作，使后续推荐、搜索、订阅和内容治理可以基于同一套频道规则扩展。

## 史诗概览

**史诗ID**: EPIC-20  
**标题**: 建立三类频道的基础架构与创建流程  
**背景**: 内容社区需要一个统一的频道体系来组织和发现内容。频道是内容聚合的核心单元，需要支持系统官方频道、个人频道、组织频道三种类型，各有不同的创建规则、权限模型和管理方式。  
**目标用户**: 平台管理员、普通用户、组织管理员、频道主  
**优先级**: 高  
**预计持续时间**: 3 周

**产品愿景**:
- 让平台可以快速建立官方内容入口，承载公告、活动、精选主题和运营分类。
- 让普通用户和组织可以创建可信频道，持续沉淀内容和订阅者。
- 让后续发现、发布、订阅、治理和统计能力共享同一套频道基础规则。

**范围**:
- 包含：频道类型定义、三类频道创建流程、频道信息编辑、频道转让与删除、基础审核状态、关键操作提示和审计要求。
- 不包含：频道隐私与加入规则（EPIC-21）、内容发布（EPIC-22）、推荐发现（EPIC-23）、数据统计（EPIC-24）。

## 关键术语与业务规则

| 术语 | 定义 |
| --- | --- |
| 内容安全审核 | 对频道名称、简介、图标、封面等用户可见信息进行平台安全检查，审核主体可以是内容安全服务或平台审核人员；结果至少包含通过、拒绝、退回修改、待人工复核。 |
| 审核状态 | 频道创建或关键字段修改后的处理状态。基础状态包含 Draft、PendingReview、Active、Rejected、DeleteCooling、Deleted；冻结、隐藏、归档等完整生命周期状态由 EPIC-24 扩展。 |
| 用户频道范围 | 个人频道和组织频道共同构成的名称唯一性范围，系统官方频道不参与该范围。处于 PendingReview、Active、DeleteCooling、ReadonlyFrozen、Hidden、Archived 的用户频道均占用名称；Deleted 后是否释放名称由平台策略确认。 |
| 基础账号要求 | 用户已登录，账号未被冻结，已完成手机号或邮箱验证，昵称可展示，且未处于禁止创建频道的风控状态。 |
| 组织最高管理员 | 组织认证系统中具备组织所有权或最高管理权限的账号，可代表组织确认组织频道删除等高风险操作。 |
| 未了结的付费订阅 | 与该频道直接关联且仍在有效期、退款期、争议处理期或待履约状态的付费订阅、订单或权益。若本期不实现付费频道，则该条件作为删除前置校验预留。 |
| 关键字段 | 影响频道识别、发现或安全判断的字段，至少包含频道名称、简介、图标、封面和分类；其中名称修改必须重新审核。 |
| 非关键字段 | 不改变频道身份且安全风险较低的展示字段，例如标签、部分封面裁剪信息等；具体字段清单由产品和审核策略维护。 |

## 用户旅程

### 旅程 1: 平台管理员创建系统官方频道

1. 平台管理员进入运营后台的频道管理入口。
2. 管理员选择创建系统官方频道，填写名称、简介、图标、封面、所属官方分类和置顶权重。
3. 系统校验管理员权限、必填字段和系统频道规则。
4. 创建成功后频道直接进入 Active 状态，默认公开、全站可见，并出现在后台频道列表。
5. 若权限不足或字段不完整，系统阻止创建并展示明确原因。

**关联故事**: 20.1.1、20.1.2、20.2.1、20.2.2

### 旅程 2: 普通用户创建个人频道并等待审核

1. 普通用户从用户端进入频道创建页。
2. 用户填写频道名称、简介、图标、隐私设置和归属分类。
3. 系统校验基础账号要求、个人频道数量上限、用户频道范围内名称唯一性和内容安全要求。
4. 校验通过后频道进入 PendingReview 状态，用户可看到审核进度。
5. 审核通过后频道进入 Active 状态并可公开展示；审核拒绝或退回修改时，用户收到原因和可操作下一步。

**关联故事**: 20.1.1、20.1.2、20.3.1、20.4.1

### 旅程 3: 组织管理员创建组织频道

1. 已认证组织的管理员进入频道创建入口并选择组织频道。
2. 系统展示创建表单并自动绑定所属组织。
3. 管理员填写频道信息后提交。
4. 系统校验组织认证状态、操作者组织权限、组织频道数量上限、名称唯一性和内容安全要求。
5. 校验通过后频道进入 PendingReview 状态；审核通过后展示组织信息且组织绑定不可更改。

**关联故事**: 20.1.1、20.1.2、20.3.2、20.4.1

### 旅程 4: 频道主维护频道信息和所有权

1. 频道主进入频道管理页。
2. 频道主编辑基础信息；非关键字段保存后生效，关键字段进入审核。
3. 个人频道主可以发起转让，目标用户确认后完成所有权变更。
4. 频道主申请删除时，系统先校验内容清理、订阅结清和权限要求，再进入二次确认。
5. 用户确认后频道进入 7 天冷静期，期间可撤销；冷静期结束后进入删除结果状态。

**关联故事**: 20.4.1、20.4.2、20.4.3

## UX/UI 与响应式要求

- 创建页必须清晰区分系统频道、个人频道、组织频道，并按频道类型展示对应字段、资格提示和不可用原因。
- 表单提交前必须展示必填项、名称唯一性、数量上限、组织认证和内容安全相关错误，错误文案需指向可操作修复方式。
- 审核状态必须在创建成功页、频道管理页和后台审核队列中可见，状态文案需区分待审核、通过、拒绝、退回修改和删除冷静期。
- 编辑、转让、删除等高风险操作必须提供二次确认；确认文案需说明影响范围、是否可撤销和下一步状态。
- 用户端创建页、审核结果页和频道管理页需适配移动端与桌面端；移动端表单字段分组展示，桌面端支持更高密度的信息扫描。
- 后台频道管理页面需支持按频道类型、审核状态、归属分类和创建时间筛选，便于平台管理员处理系统频道和审核任务。

## 非功能需求

| 分类 | 要求 | 度量方式 |
| --- | --- | --- |
| 性能 | 创建、编辑、转让、删除提交接口 P95 <= 500ms；审核状态查询 P95 <= 300ms | 接口监控按 channel_type、operation、result 维度统计 |
| 可用性 | 频道创建和审核状态查询可用性 >= 99.9% | 以核心接口成功响应和错误率监控计算 |
| 可靠性 | 创建、转让、删除等关键操作不得因重复提交产生重复频道、重复转让或重复删除结果 | 按 request_id 或业务单号审计重复提交处理结果 |
| 安全 | 系统频道创建、组织频道创建、转让、删除、审核处理必须校验操作者权限 | 权限拒绝日志、越权工单和安全测试结果 |

| 可观测性 | 创建失败、审核超时、名称冲突、删除冷静期到期等事件可追踪 | 日志和告警支持 channel_id、user_id、request_id、audit_id 检索 |

## 特性列表

### 特性 1: 频道类型定义与数据模型

**描述**: 定义系统官方频道、个人频道、组织频道三种类型的统一数据模型，为后续所有频道功能提供基础。  
**关键交互**: 系统初始化 → 建立频道类型规则 → 创建频道基础记录 → 配置类型差异化规则。  
**非功能需求**: 数据模型需支持扩展，频道类型字段创建后不可直接修改。  
**验收条件**: 三种频道类型在业务记录和展示层可明确区分，类型切换需走迁移流程。

#### 故事列表

##### 故事 20.1.1: 频道基础数据模型设计

**作为** 系统架构师，  
**我希望** 建立统一的频道数据模型，支持三种频道类型的基础字段和差异化字段，  
**以便** 为所有频道功能提供稳定的数据基础。

**验收标准**:
1. **给定** 频道数据模型，**当** 创建频道时，**则** 系统要求填写必填字段（名称、简介、类型、隐私设置、归属分类）
2. **给定** 频道已创建，**当** 查看频道类型字段，**则** 类型值为 system/personal/organization 之一且创建后不可直接修改
3. **给定** 组织频道，**当** 创建时，**则** 必须绑定组织 ID 且不可为空
4. **给定** 系统频道，**当** 创建时，**则** 可设置置顶权重且默认为 0

**附加说明**:
- **优先级**: 高
- **依赖项**: 无
- **产品规则**: 频道类型是频道身份的一部分，创建后不可直接修改；组织频道必须绑定已认证组织

##### 故事 20.1.2: 频道类型差异化规则配置

**作为** 系统，  
**我希望** 根据频道类型自动应用不同的创建规则和管理规则，  
**以便** 确保三类频道在创建、编辑、删除等操作上遵循各自的约束。

**验收标准**:
1. **给定** 系统频道，**当** 尝试设置为私有，**则** 系统拒绝并提示“系统频道必须公开”
2. **给定** 个人频道，**当** 尝试转让给其他符合条件的用户，**则** 系统允许进入转让流程
3. **给定** 组织频道，**当** 尝试转让给个人，**则** 系统拒绝并提示“组织频道仅可在组织管理员间转移”
4. **给定** 系统频道，**当** 用户尝试删除，**则** 系统拒绝并提示“系统频道仅平台可管理”

**附加说明**:
- **优先级**: 高
- **依赖项**: 故事 20.1.1
- **产品规则**: 三类频道必须遵循各自的隐私、归属、转让和删除约束，所有拒绝结果需返回可理解原因

### 特性 2: 系统官方频道创建

**描述**: 平台管理员在后台创建和管理系统频道，用于官方主题、活动、公告、精选推荐。  
**关键交互**: 管理员登录后台 → 填写频道信息 → 设置分类和权重 → 直接发布（无需普通创建审核）。  
**非功能需求**: 系统频道名称可使用保留关键词，不参与用户频道唯一性校验。  
**验收条件**: 管理员可成功创建系统频道，频道默认公开且全站可见。

#### 故事列表

##### 故事 20.2.1: 系统频道创建

**作为** 平台管理员，  
**我希望** 在后台创建系统官方频道，填写名称、简介、图标、封面、分类和置顶权重，  
**以便** 承载平台级的分类、活动与推荐内容。

**验收标准**:
1. **给定** 管理员已登录后台，**当** 进入频道管理页面点击“创建系统频道”，**则** 显示创建表单包含名称、简介、图标、封面、所属官方分类、置顶权重字段
2. **给定** 管理员填写完信息并提交，**当** 必填项已填写，**则** 系统直接创建频道（无需内容安全审核）并返回成功
3. **给定** 管理员创建系统频道，**当** 名称与已有用户频道重名，**则** 系统允许创建（系统频道名称不参与用户频道唯一性校验）
4. **给定** 系统频道已创建，**当** 查看频道状态，**则** 默认为公开、全站可见

**附加说明**:
- **优先级**: 高
- **依赖项**: 故事 20.1.1
- **产品规则**: 系统频道仅平台管理员可创建和维护，普通用户不可通过用户端创建系统频道

##### 故事 20.2.2: 系统频道管理限制

**作为** 系统，  
**我希望** 对系统频道施加特殊的管理限制，  
**以便** 确保系统频道的权威性和不可篡改性。

**验收标准**:
1. **给定** 系统频道已创建，**当** 普通用户尝试编辑频道信息，**则** 系统拒绝并提示无权限
2. **给定** 系统频道已创建，**当** 尝试将其设为私有，**则** 系统拒绝（系统频道强制公开）
3. **给定** 系统频道已创建，**当** 尝试删除频道，**则** 系统拒绝（仅平台可归档或关闭）
4. **给定** 系统频道已创建，**当** 尝试转让频道所有权，**则** 系统拒绝（不可转让）

**附加说明**:
- **优先级**: 高
- **依赖项**: 故事 20.2.1
- **产品规则**: 系统频道的编辑、删除、转让入口只对具备平台管理权限的操作者可见或可执行

### 特性 3: 个人与组织频道创建

**描述**: 普通用户创建个人频道，已认证组织管理员创建组织频道，支持名称唯一性校验和内容安全审核。  
**关键交互**: 用户进入创建页 → 填写必填信息 → 选择隐私和分类 → 提交审核 → 审核通过后公开。  
**非功能需求**: 创建后进入平台审核状态，审核通过后方可公开显示。  
**验收条件**: 个人/组织频道创建流程完整，名称唯一性校验准确，审核流程正常。

#### 故事列表

##### 故事 20.3.1: 个人频道创建

**作为** 普通用户，  
**我希望** 创建属于我的个人频道，设置名称、简介、图标、隐私和分类，  
**以便** 聚合我感兴趣或创作的内容并积累订阅者。

**验收标准**:
1. **给定** 用户已登录且满足基础账号要求，**当** 进入频道创建页面，**则** 显示创建表单包含名称、简介、图标、隐私设置（公开/私有）、归属分类字段
2. **给定** 用户填写完信息并提交，**当** 名称在用户频道范围内唯一且通过内容安全检测，**则** 系统创建频道并进入 PendingReview 状态
3. **给定** 用户提交的频道名称已存在（用户频道范围内），**当** 系统校验名称，**则** 提示“该频道名称已被使用，请更换”
4. **给定** 频道进入 PendingReview 状态，**当** 管理员审核通过，**则** 频道状态变更为 Active 并可公开显示
5. **给定** 用户已创建 20 个个人频道（达到上限），**当** 再次尝试创建，**则** 系统提示已达上限
6. **给定** 用户不满足基础账号要求，**当** 尝试创建个人频道，**则** 系统提示需完成账号验证或解除限制后再创建

**附加说明**:
- **优先级**: 高
- **依赖项**: 故事 20.1.1、用户认证系统、内容安全审核能力
- **产品规则**: 名称唯一性校验排除系统频道；审核结果需要通知用户并在创建结果页可见

##### 故事 20.3.2: 组织频道创建

**作为** 已认证组织的管理员，  
**我希望** 为我的组织创建专属频道，绑定组织信息并设置频道属性，  
**以便** 展示组织内容并管理成员。

**验收标准**:
1. **给定** 组织已完成认证且当前用户为组织管理员，**当** 进入频道创建页面，**则** 显示创建表单包含名称、简介、图标、隐私设置、归属分类，并自动绑定所属组织
2. **给定** 用户填写完信息并提交，**当** 名称在用户频道范围内唯一且通过内容安全检测，**则** 系统创建组织频道并进入 PendingReview 状态
3. **给定** 组织未完成认证，**当** 尝试创建组织频道，**则** 系统提示“请先完成组织认证”
4. **给定** 组织已创建 50 个频道（达到上限），**当** 再次尝试创建，**则** 系统提示已达上限
5. **给定** 组织频道已创建，**当** 查看频道详情，**则** 显示绑定的组织信息且不可更改
6. **给定** 当前用户不是组织管理员，**当** 尝试为该组织创建频道，**则** 系统拒绝并提示无组织管理权限

**附加说明**:
- **优先级**: 高
- **依赖项**: 故事 20.1.1、组织认证系统、内容安全审核能力
- **产品规则**: 组织频道创建资格由组织认证状态和用户组织角色共同决定，绑定组织后不可改绑

### 特性 4: 频道信息编辑与所有权管理

**描述**: 频道创建后支持信息编辑、所有权转让和删除操作，不同频道类型有差异化规则。  
**关键交互**: 频道主进入管理页 → 编辑信息/发起转让/申请删除 → 审核或确认后生效。  
**非功能需求**: 关键信息修改需平台审核，审核期间按原有信息展示。  
**验收条件**: 编辑、转让、删除功能在三类频道下行为正确，均有二次确认。

#### 故事列表

##### 故事 20.4.1: 频道信息编辑

**作为** 频道主/管理员，  
**我希望** 修改频道的名称、描述、图标、封面、分类、标签等信息，  
**以便** 保持频道信息的准确性和吸引力。

**验收标准**:
1. **给定** 频道主已登录，**当** 进入频道管理页编辑基本信息并保存，**则** 非关键字段立即生效
2. **给定** 频道主修改频道名称，**当** 提交保存，**则** 修改进入 PendingReview 状态，审核期间按原名称展示
3. **给定** 修改后的名称在用户频道范围内已存在，**当** 系统校验，**则** 提示名称冲突
4. **给定** 系统频道，**当** 管理员修改信息，**则** 无需普通创建审核直接生效
5. **给定** 频道关键字段修改被拒绝，**当** 频道主查看结果，**则** 原信息保持展示并显示拒绝原因

**附加说明**:
- **优先级**: 高
- **依赖项**: 故事 20.3.1 或 20.3.2
- **产品规则**: 名称等关键字段修改需重新审核；非关键字段保存后立即反馈结果并记录操作

##### 故事 20.4.2: 频道转让

**作为** 频道主，  
**我希望** 将频道所有权转让给其他符合条件的用户，  
**以便** 在我无法继续管理时频道仍能正常运营。

**验收标准**:
1. **给定** 个人频道的频道主，**当** 发起转让并选择目标用户，**则** 系统向目标用户发送确认请求
2. **给定** 目标用户确认接受转让，**当** 转让完成，**则** 原频道主降为管理员，目标用户成为新频道主
3. **给定** 组织频道，**当** 尝试转让给组织外的个人用户，**则** 系统拒绝并提示“组织频道仅可在组织管理员间转移管理权”
4. **给定** 系统频道，**当** 尝试转让，**则** 系统拒绝（系统频道不可转让）
5. **给定** 转让请求超时或目标用户拒绝，**当** 查看转让结果，**则** 频道所有权保持不变并显示失败原因

**附加说明**:
- **优先级**: 中
- **依赖项**: 故事 20.4.1
- **产品规则**: 转让需发起方和接收方确认；所有权变化前后需保留可追溯记录

##### 故事 20.4.3: 频道删除

**作为** 频道主/管理员，  
**我希望** 在满足条件时删除不再需要的频道，  
**以便** 清理无用资源。

**验收标准**:
1. **给定** 个人频道主申请删除频道，**当** 频道内所有内容已清理或转移且无未了结的付费订阅，**则** 系统进入二次确认流程
2. **给定** 用户确认删除，**当** 系统处理，**则** 频道进入 7 天冷静期，期间可撤销
3. **给定** 冷静期已过，**当** 系统自动处理，**则** 频道进入 Deleted 状态且用户不可再访问管理入口
4. **给定** 组织频道，**当** 组织最高管理员确认删除且满足条件，**则** 执行删除流程
5. **给定** 系统频道，**当** 尝试删除，**则** 系统拒绝（仅平台可归档或关闭）
6. **给定** 频道存在未清理内容、未完成转移或未了结的付费订阅，**当** 申请删除，**则** 系统拒绝进入删除确认并提示阻塞原因

**附加说明**:
- **优先级**: 中
- **依赖项**: 故事 20.4.1
- **产品规则**: 删除需要满足前置条件、二次确认和 7 天冷静期；冷静期内允许频道主撤销删除

- **Milestone 1**: Week 1 - 特性 1（数据模型）+ 特性 2（系统频道创建）。验收重点：频道类型、系统频道创建、系统频道限制可用。
- **Milestone 2**: Week 2 - 特性 3（个人/组织频道创建）。验收重点：基础账号要求、组织认证、名称唯一性、审核状态和通知可用。

## 风险与假设

| 类型 | 内容 | 影响 | 缓解措施 |
| --- | --- | --- | --- |
| 风险 | 组织认证系统未就绪 | 组织频道创建功能被阻塞 | 先完成系统频道和个人频道；组织频道以认证结果为前置发布条件 |
| 风险 | 内容安全审核服务响应时间或容量不足 | 个人/组织频道创建审核积压，影响用户体验 | 设置 24 小时 SLA、审核超时提醒和人工处理队列 |
| 风险 | 用户频道名称唯一性范围不清 | 创建、编辑和搜索结果可能冲突 | 按本文“用户频道范围”执行，并将策略作为发布前验收项 |
| 风险 | 删除条件涉及未来付费订阅能力 | 本期删除流程可能出现预留校验无法落地 | 本期若无付费订阅则返回“无未了结订阅”，但保留产品规则和审计字段 |
| 假设 | 用户认证系统可提供账号冻结、验证状态和风控状态 | 支撑基础账号要求校验 | 与用户域确认字段和错误码 |
| 假设 | 平台管理员后台已具备基础管理界面框架 | 支撑系统频道管理和审核队列展示 | 本 EPIC 只补充频道管理所需字段和交互要求 |
