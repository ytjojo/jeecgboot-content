# 频道数据统计与生命周期管理

## 执行摘要

EPIC-24 建立频道数据统计看板、数据导出、生命周期状态流转和合规治理能力，让频道从创建后的运营、增长、治理到归档或合并都具备可度量、可审计、可追踪的产品闭环。它承接 EPIC-20 的频道基础架构、EPIC-21 的成员与订阅关系、EPIC-22 的内容发布与治理事件、EPIC-23 的发现与活跃信号，是频道域管理能力的最终运营层。

本史诗的产品愿景是：频道主可以用可信数据判断内容和用户运营效果；平台运营可以用统一的状态流转和治理工具维护频道生态健康；系统可以按明确规则识别不活跃频道并触发提醒、归档或组织管理员处理。生命周期能力必须优先保证数据口径可解释、状态变更可审计、治理动作可申诉、敏感数据可控，避免统计误导、违规频道持续扩散或高风险操作缺少追溯。

## 史诗概览

**史诗 ID**: EPIC-24
**标题**: 建立频道数据统计看板与全生命周期管理能力
**背景**: 频道运营需要数据驱动决策，频道主和管理员需要了解频道核心指标、互动数据和用户分析。同时，频道从创建到归档、关闭、合并需要完整生命周期管理，包括审核、冻结、归档、合并、违规处置和不活跃治理。
**目标用户**: 频道主、频道管理员、组织最高管理员、平台运营、系统管理员
**优先级**: 中
**预计持续时间**: 3 周

**范围**:
- 包含：频道数据统计看板、互动与内容表现、用户分析、Excel/CSV 数据导出、频道审核、冻结与解冻、归档、合并、限制推荐、强制隐藏、永久关闭、不活跃提醒与处置、审计日志和申诉入口。
- 不包含：频道创建基础流程（EPIC-20）、成员和加入规则（EPIC-21）、内容发布与内容级审核（EPIC-22）、推荐发现与搜索排序（EPIC-23）、底层数据采集方案、审核系统建设方案、通知系统建设方案。

## 关键术语与业务规则

| 术语 | 定义 |
| --- | --- |
| 系统频道 | 由平台运营或系统管理员在后台创建和维护的官方频道，默认公开、全站可见，不走普通用户创建审核流程，仅平台可归档、关闭或删除。 |
| 个人频道 | 由普通用户创建并归属于个人账号的频道，频道主对频道运营负责，合并、归档、删除等高风险操作需频道主确认并按平台规则审核。 |
| 组织频道 | 绑定已认证组织的频道，由组织授权管理员管理。组织频道的合并、归档、关闭等高风险操作需组织最高管理员或具备同等权限的角色审批。 |
| 有效互动 | 来自非违规、非删除内容的点赞、评论、收藏、分享、订阅、有效访问等行为。重复刷量、被撤销、被判定违规或来自无权访问用户的行为不计入。 |
| 订阅增量 | 指定统计周期内新增的有效订阅关系数量。 |
| 订阅流失 | 指定统计周期内用户主动取消订阅、频道关闭导致订阅终止，或组织权限变化导致订阅关系失效的数量；长期未访问不等同于订阅流失。 |
| 公共推荐流 | 面向普通用户公开展示频道或频道内容的推荐入口，包括首页推荐、频道发现推荐、相关频道推荐、榜单和编辑精选等；不包括用户已订阅频道的私域列表。 |
| 不活跃频道 | 连续 6 个月无新增公开内容、无有效互动、无频道信息维护动作，且当前未处于 PendingReview、Deleted、Closed 或 Merged 状态的频道。 |
| 高风险生命周期操作 | 会影响频道可见性、发布能力、订阅关系、内容访问或历史数据的操作，包括冻结、解冻、隐藏、限制推荐、永久关闭、归档、合并、删除。 |
| 归档 | 频道停止公开运营但保留历史数据的生命周期状态。归档频道不再进入公共发现入口，历史内容按权限规则保留可访问或仅后台可查。 |
| 合并 | 将源频道的内容、订阅关系和访问指向迁移至目标频道的流程。合并完成后源频道进入 Merged 状态，并向访问者展示目标频道入口。 |
| 永久关闭 | 平台因严重违规或合规要求终止频道运营的治理结果。关闭后频道不可恢复，用户端不再展示，后台保留必要审计记录。 |

**生命周期状态规则**:
- PendingReview：个人或组织频道创建、关键字段修改、归档申请、合并申请等待审核。
- Active：频道正常运营，可按隐私和成员规则访问、发布、订阅和被发现。
- ReadonlyFrozen：频道只读，保留访问和历史数据，禁止发布新内容、投稿和关键信息修改。
- Hidden：频道对外不可见，仅平台运营和具备管理权限的角色可访问后台详情。
- Archived：频道停止公开运营，历史数据保留，不进入搜索、推荐、榜单或分类浏览。
- Merged：源频道已合并至目标频道，用户访问源频道时需看到目标频道跳转或说明。
- Closed：频道被平台永久关闭，不可恢复，不进入任何用户端入口。
- Deleted：频道已完成删除流程，用户端不可见，保留策略按平台数据保留规则执行。

## 用户旅程

### 旅程 1: 频道主查看数据看板并导出运营数据

1. 频道主进入频道管理后台的数据看板页。
2. 系统展示订阅数、内容数、PV、UV、互动量、趋势图和数据更新时间。
3. 频道主切换日、周、月或自定义时间范围，查看指标变化。
4. 频道主进入热门内容和用户分析模块，识别表现较好的内容、订阅增长和流失变化。
5. 频道主选择时间范围、字段和 Excel 或 CSV 格式导出数据。
6. 当导出范围超过 10,000 行时，系统展示导出处理中状态，并在生成完成后提供下载入口。

### 旅程 2: 平台运营审核频道创建或关键字段修改

1. 平台运营进入频道审核队列，按提交时间、频道类型、风险等级和状态筛选申请。
2. 运营查看频道名称、简介、图标、封面、分类、申请人和历史审核记录。
3. 运营选择通过、拒绝或退回修改，并填写原因。
4. 系统更新频道状态并通知申请人；审核期间用户端继续展示审核前的已生效信息。
5. 审核操作进入审计日志，后续可按频道、操作人和时间查询。

### 旅程 3: 平台运营处理违规频道并支持申诉

1. 平台运营从治理后台进入目标频道详情。
2. 运营查看频道类型、当前状态、近期内容、有效互动、历史处罚和相关举报。
3. 运营选择限制推荐、只读冻结、强制隐藏或永久关闭，并填写处理原因和影响范围。
4. 系统要求二次确认高风险操作，完成后通知频道主或组织管理员。
5. 频道主在处罚通知中进入申诉入口，提交说明和补充材料。
6. 平台运营处理申诉并记录复议结果，必要时恢复频道状态或维持原处理。

### 旅程 4: 系统识别不活跃频道并触发处置

1. 系统识别连续 6 个月无内容更新、无有效互动、无维护动作的频道。
2. 对个人频道，系统向频道主发送不活跃提醒，说明 1 个月内可通过发布内容或产生有效互动恢复活跃。
3. 提醒发出后 1 个月仍无改善时，个人频道进入自动归档流程。
4. 对组织频道，系统向组织最高管理员发送处置建议，由组织管理员决定恢复运营、手动归档或发起合并。
5. 频道归档后，用户端不再公开展示该频道，后台保留归档原因、提醒记录和处置结果。

### 旅程 5: 频道主发起频道合并

1. 频道主在频道管理后台选择合并频道。
2. 系统要求选择目标频道，并展示源频道内容、订阅者、历史链接和影响范围。
3. 频道主提交合并申请；组织频道需组织最高管理员审批。
4. 审核通过后，系统执行合并并通知相关订阅者和管理员。
5. 用户访问源频道时看到已合并说明和目标频道入口。

## UX/UI 与响应式要求

- 数据看板页需清晰展示数据更新时间、统计周期、筛选条件和权限范围，避免用户误解数据口径。
- 核心指标卡需展示订阅数、内容数、PV、UV；趋势图需支持日、周、月和自定义时间范围。
- 互动数据模块需展示点赞、评论、收藏、分享、有效访问等指标，并说明有效互动口径。
- 热门内容列表需展示内容标题、内容类型、发布时间、有效互动量和排名依据。
- 用户分析页需展示订阅增量、订阅流失、成员活跃度占比、贡献排行；组织频道可按部门或职务查看时必须展示权限提示。
- 数据导出入口需展示可导出字段、格式、时间范围、预计行数、处理状态、下载有效期和失败原因。
- 审核队列需支持按频道类型、提交时间、状态、申请类型和风险等级筛选；每条申请需展示可决策摘要和详情入口。
- 生命周期状态需在频道详情、后台列表和治理详情页统一展示，状态文案必须区分待审核、正常、只读冻结、强制隐藏、已归档、已合并、永久关闭和已删除。
- 冻结、隐藏、永久关闭、归档、合并、删除等高风险操作必须提供二次确认、原因填写、影响范围提示和操作后反馈。
- 违规处理结果需向频道主或组织管理员展示处理类型、原因、影响范围、申诉入口和申诉截止时间。
- 不活跃提醒需说明触发原因、恢复活跃方式、处置时限和逾期结果。
- 空状态需覆盖无统计数据、无热门内容、无用户分析数据、无审核申请、无审计日志、无可导出数据，并提供下一步入口。
- 错误态需说明具体原因，例如权限不足、频道状态不可操作、导出范围超限、审核申请已被他人处理或当前频道不可访问。
- 响应式要求：移动端数据看板优先展示核心指标、更新时间和主趋势图，次要维度折叠到分组详情。
- 响应式要求：移动端筛选条件使用收纳面板，导出和高风险治理操作需避免误触，并保留二次确认。
- 响应式要求：平台后台在桌面端支持高密度列表和批量筛选；窄屏下每条审核或治理记录必须保留状态、原因、主操作和详情入口。

## 特性列表

### 特性 1: 频道数据统计看板

**描述**: 为频道主和频道管理员提供频道运营数据看板，包含核心指标、互动数据、内容表现、用户分析和数据导出。
**关键交互**: 频道主进入数据看板 → 查看各维度数据 → 筛选时间范围 → 查看热门内容和用户分析 → 导出数据。
**非功能需求**: 看板核心数据延迟 P99 <= 5 分钟；常规查询 P95 <= 1 秒；导出遵守权限和字段白名单。
**验收条件**: 看板数据口径清晰、指标可见、筛选可用，Excel/CSV 导出内容完整且不越权。

#### 故事列表

##### 故事 24.1.1: 核心指标展示

**作为** 频道主，
**我希望** 在数据看板中查看频道的核心指标（订阅数、内容数、PV、UV），
**以便** 了解频道的整体运营状况。

**验收标准**:
1. **给定** 频道主进入数据看板，**当** 加载页面，**则** 展示当前订阅数、累计内容数、浏览量（PV）、访客数（UV）和数据更新时间。
2. **给定** 数据看板已加载，**当** 选择时间范围（日/周/月/自定义），**则** 指标数据按选定范围更新，并保留当前筛选条件。
3. **给定** 数据看板已加载，**当** 查看趋势图，**则** 展示选定时间范围内订阅数、内容数、PV、UV 的变化趋势。
4. **给定** 系统频道，**当** 平台运营查看数据，**则** 在平台运营后台展示与普通频道主视图一致的核心指标。
5. **给定** 频道核心统计事件产生，**当** 频道主查看看板，**则** 数据可见延迟 P99 <= 5 分钟。

**附加说明**:
- **优先级**: 高
- **依赖项**: EPIC-20（频道创建）、EPIC-22（内容发布事件）
- **产品规则**: 核心指标必须展示统计周期和更新时间；无权限用户不可查看频道看板。

##### 故事 24.1.2: 互动数据与内容表现

**作为** 频道主，
**我希望** 查看频道的互动数据（点赞、评论、收藏、分享）和热门内容排行，
**以便** 了解频道内容的受欢迎程度。

**验收标准**:
1. **给定** 频道主进入互动数据模块，**当** 加载页面，**则** 展示点赞、评论、收藏、分享、有效访问的总量及趋势。
2. **给定** 频道主查看热门内容，**当** 加载默认近 7 天热门内容 Top20，**则** 按有效互动量从高到低展示。
3. **给定** 频道主切换热门内容周期，**当** 选择近 7 天、近 30 天或近 90 天，**则** 热门内容排行按对应周期刷新。
4. **给定** 频道主查看新增内容统计，**当** 选择时间范围，**则** 展示该时间段内新增内容数量和类型分布。
5. **给定** 某内容已删除、违规下架或频道主无权查看，**当** 展示热门内容排行，**则** 该内容不进入用户可见排行。

**附加说明**:
- **优先级**: 中
- **依赖项**: 故事 24.1.1、EPIC-22（内容状态与互动事件）
- **产品规则**: 热门内容排序以有效互动为基础，必须排除违规、删除和不可访问内容。

##### 故事 24.1.3: 用户分析与数据导出

**作为** 频道主，
**我希望** 查看频道的用户分析数据（订阅增量/流失、成员活跃度、贡献排行），并支持数据导出，
**以便** 深入了解频道用户画像和运营效果。

**验收标准**:
1. **给定** 频道主进入用户分析模块，**当** 加载页面，**则** 展示订阅增量、订阅流失、成员活跃度占比和贡献排行。
2. **给定** 组织频道，**当** 操作者具备组织数据查看权限，**则** 额外提供按部门或职务统计贡献；无权限时隐藏该维度并提示权限不足。
3. **给定** 频道主点击“导出数据”，**当** 选择时间范围、字段和格式（Excel/CSV），**则** 系统生成可下载数据文件。
4. **给定** 导出范围预计超过 10,000 行，**当** 用户发起导出，**则** 页面展示导出处理中状态，并在生成完成后提供下载入口。
5. **给定** 导出完成，**当** 用户打开 Excel 或 CSV 文件，**则** 文件包含字段标题、时间范围、筛选条件、导出时间和符合权限范围的数据行。
6. **给定** 无权限用户尝试导出数据，**当** 提交导出请求，**则** 系统拒绝并提示权限不足。

**附加说明**:
- **优先级**: 中
- **依赖项**: 故事 24.1.1、EPIC-21（成员和权限）、EPIC-22（内容与互动数据）
- **产品规则**: 数据导出必须遵守字段白名单和操作者权限范围；导出记录需可追踪。

### 特性 2: 频道生命周期管理

**描述**: 管理频道从创建后审核、正常运营、冻结、归档、合并、关闭到删除的生命周期状态流转。
**关键交互**: 频道提交审核 → 平台运营审核 → 频道正常运营 → 触发冻结、归档、合并或关闭 → 通知相关责任人 → 记录审计日志。
**非功能需求**: 高风险状态变更需二次确认；生命周期操作审计记录覆盖率 = 100%。
**验收条件**: 各状态流转正确，非法流转被拦截，操作日志完整，相关方收到通知。

#### 故事列表

##### 故事 24.2.1: 频道审核流程

**作为** 平台运营，
**我希望** 审核用户创建的频道和频道信息修改，
**以便** 确保频道内容安全合规。

**验收标准**:
1. **给定** 用户创建个人或组织频道，**当** 提交创建申请，**则** 频道进入审核队列，平台运营可查看申请详情。
2. **给定** 平台运营审核通过，**当** 操作完成，**则** 频道状态变更为 Active，用户收到审核通过通知。
3. **给定** 平台运营审核拒绝，**当** 填写拒绝理由，**则** 用户收到拒绝通知、原因和可重新提交入口。
4. **给定** 频道主修改名称等关键字段，**当** 提交修改，**则** 进入审核流程，审核期间用户端按原已生效信息展示。
5. **给定** 系统频道，**当** 平台运营创建或修改，**则** 无需普通用户创建审核，但需记录后台操作日志。
6. **给定** 审核申请超过 24 小时未处理，**当** 平台运营查看审核队列，**则** 该申请标记为超时待处理。

**附加说明**:
- **优先级**: 高
- **依赖项**: EPIC-20（频道创建和关键字段定义）
- **产品规则**: 审核结果必须包含通过、拒绝或退回修改；拒绝和退回修改必须填写用户可理解原因。

##### 故事 24.2.2: 频道冻结与解冻

**作为** 平台运营，
**我希望** 对违规频道进行冻结（只读）处理，并在整改完成后解冻，
**以便** 惩戒违规行为同时保留频道数据。

**验收标准**:
1. **给定** 平台运营选择某频道，**当** 执行冻结操作并填写原因，**则** 频道进入 ReadonlyFrozen 状态，不可发布新内容、投稿或修改关键字段。
2. **给定** 频道已被冻结，**当** 频道成员尝试发布内容，**则** 系统提示“频道已被冻结，暂不可发布”。
3. **给定** 平台运营对已冻结频道执行解冻，**当** 填写解冻原因并确认，**则** 频道恢复到冻结前可用状态。
4. **给定** 冻结或解冻操作完成，**当** 查看审计日志，**则** 记录操作人、处理类型、处理原因、操作时间、前后状态和影响范围。
5. **给定** 频道被冻结，**当** 频道主查看处罚通知，**则** 可看到处理原因、影响范围和申诉入口。

**附加说明**:
- **优先级**: 高
- **依赖项**: 故事 24.2.1、EPIC-22（发布权限和内容状态）
- **产品规则**: 冻结是只读治理状态，不等同于隐藏或关闭；冻结期间历史内容是否可见需继续遵守原频道隐私和治理规则。

##### 故事 24.2.3: 频道归档

**作为** 平台运营或频道主，
**我希望** 将不再活跃的频道归档，
**以便** 清理无用资源同时保留历史数据。

**验收标准**:
1. **给定** 个人频道连续 6 个月无内容更新、无有效互动且无频道信息维护动作，**当** 系统识别为不活跃频道，**则** 通知频道主频道将进入归档风险期。
2. **给定** 通知发出后 1 个月仍无改善，**当** 系统处置个人频道，**则** 频道进入 Archived 状态。
3. **给定** 关联组织解散，**当** 系统识别到组织频道失去有效组织归属，**则** 组织频道进入归档流程并通知组织最高管理员或平台运营。
4. **给定** 平台运营手动归档某频道，**当** 填写原因并二次确认，**则** 频道进入 Archived 状态，用户端不再公开展示。
5. **给定** 频道主申请归档，**当** 提交申请，**则** 进入审核流程，审核通过后频道归档。
6. **给定** 频道已归档，**当** 用户通过搜索、推荐、榜单或分类浏览访问，**则** 不再展示该频道。

**附加说明**:
- **优先级**: 中
- **依赖项**: 故事 24.2.1、EPIC-23（发现入口可见性规则）
- **产品规则**: 归档保留历史数据和审计记录；归档频道是否允许恢复需由平台策略明确并进入 Open Questions。

##### 故事 24.2.4: 频道合并

**作为** 频道主，
**我希望** 将一个频道合并至另一个频道，
**以便** 整合分散的内容和订阅者。

**验收标准**:
1. **给定** 个人频道主发起合并申请，**当** 选择目标频道并提交，**则** 系统展示源频道内容、订阅者、历史链接和可见性影响范围，并进入审核流程。
2. **给定** 合并审核通过，**当** 系统执行合并，**则** 源频道的内容和订阅关系迁移至目标频道，并通知相关订阅者。
3. **给定** 合并完成，**当** 查看源频道，**则** 源频道状态变为 Merged，并展示目标频道入口。
4. **给定** 组织频道发起合并，**当** 提交申请，**则** 需组织最高管理员审批后才能进入平台审核。
5. **给定** 源频道或目标频道处于 Deleted、Closed、Hidden 或 PendingReview 状态，**当** 发起合并，**则** 系统拒绝并提示当前状态不可合并。
6. **给定** 合并完成，**当** 查看审计日志，**则** 记录源频道、目标频道、申请人、审批人、完成时间和影响范围。

**附加说明**:
- **优先级**: 低
- **依赖项**: 故事 24.2.1、EPIC-21（订阅关系）、EPIC-22（频道内容归属）
- **产品规则**: 合并前必须让申请人确认对内容、订阅者、历史链接和统计口径的影响；合并后源频道不可继续发布内容。

### 特性 3: 合规治理

**描述**: 平台对频道执行违规处理和不活跃处置，确保频道生态健康、公开发现边界可信、治理结果可申诉。
**关键交互**: 平台识别违规或不活跃 → 触发治理动作 → 通知相关方 → 提供申诉或复议入口 → 记录审计日志。
**非功能需求**: 治理操作需记录完整审计日志；冻结、隐藏、限制推荐和永久关闭需提供申诉入口。
**验收条件**: 违规处理和不活跃处置按规则执行，公共推荐流不展示被限制频道，申诉和审计可追踪。

#### 故事列表

##### 故事 24.3.1: 违规处理

**作为** 平台运营，
**我希望** 对违规频道进行限制推荐、强制隐藏或永久关闭等处理，
**以便** 维护平台内容生态健康。

**验收标准**:
1. **给定** 平台运营选择某违规频道，**当** 执行“限制推荐”操作，**则** 频道内容不再进入公共推荐流，但已订阅用户按原权限访问不受影响。
2. **给定** 平台运营执行“强制隐藏”操作，**当** 操作完成，**则** 频道对外不可见，仅平台运营和具备管理权限的角色可访问后台详情。
3. **给定** 平台运营执行“永久关闭”操作，**当** 二次确认后，**则** 频道进入 Closed 状态，不可恢复，用户端不再展示。
4. **给定** 违规处理完成，**当** 查看审计日志，**则** 记录操作人、处理类型、处理原因、操作时间、前后状态和影响范围。
5. **给定** 频道被限制推荐、隐藏或永久关闭，**当** 频道主查看通知，**则** 可看到处理原因、影响范围、申诉入口和申诉截止时间。
6. **给定** 频道处于 Hidden、Closed 或限制推荐状态，**当** EPIC-23 的搜索、推荐、榜单、精选或分类浏览展示频道，**则** 按治理状态排除或限制展示。

**附加说明**:
- **优先级**: 高
- **依赖项**: 故事 24.2.2、EPIC-23（发现入口可见性规则）
- **产品规则**: 违规处理需区分限制推荐、只读冻结、强制隐藏和永久关闭；不同处理对发布、访问、发现和恢复能力的影响必须在操作前展示。

##### 故事 24.3.2: 不活跃处置

**作为** 系统，
**我希望** 自动识别不活跃频道并触发处置流程，
**以便** 保持频道生态的活跃度和健康度。

**验收标准**:
1. **给定** 频道连续 6 个月无内容更新、无有效互动且无频道信息维护动作，**当** 系统识别为不活跃频道，**则** 在 24 小时内向频道主发送不活跃提醒通知。
2. **给定** 提醒发出后 1 个月仍无改善，**当** 系统处理个人频道，**则** 频道进入自动归档流程。
3. **给定** 提醒发出后 1 个月仍无改善，**当** 系统处理组织频道，**则** 向组织最高管理员发送处置建议，并在平台后台标记待跟进。
4. **给定** 频道主收到提醒后恢复运营，**当** 发布新内容、产生有效互动或完成频道信息维护，**则** 不活跃风险状态解除。
5. **给定** 频道已进入 Deleted、Closed、Merged 或 Archived 状态，**当** 进行不活跃识别，**则** 不重复发送不活跃提醒。
6. **给定** 不活跃处置完成，**当** 查看审计日志，**则** 记录触发原因、提醒时间、处置结果和责任人。

**附加说明**:
- **优先级**: 中
- **依赖项**: 故事 24.2.3、EPIC-22（内容更新与互动事件）、EPIC-21（组织管理员识别）
- **产品规则**: 不活跃处置应优先提醒和恢复运营，其次才进入归档或组织管理员处理；系统频道不适用自动归档。

## 风险与假设

| 类型 | 内容 | 影响 | 缓解方式 |
| --- | --- | --- | --- |
| 风险 | 统计口径不一致导致频道主误判运营效果 | 看板可信度下降，运营决策失真 | 在看板展示统计周期、更新时间和指标说明，导出文件保留筛选条件 |
| 风险 | 频道合并对内容、订阅和历史链接影响复杂 | 用户找不到原频道，订阅关系或统计口径产生争议 | 合并前展示影响范围，合并后源频道展示目标频道入口并通知订阅者 |
| 风险 | 高风险治理动作缺少二次确认或审计 | 误操作难以追责，频道主申诉困难 | 冻结、隐藏、关闭、归档、合并、删除必须二次确认并记录审计日志 |
| 风险 | 私有或受限频道在统计、导出或治理通知中越权暴露 | 数据安全和信任风险 | 看板、导出和通知按操作者权限过滤字段和对象 |
| 假设 | 用户行为数据采集已具备基础能力 | 看板可使用浏览、互动、订阅和内容事件计算指标 | 若数据不足，先发布核心指标和导出，再逐步开放细分分析 |
| 假设 | 通知系统可触达频道主、组织最高管理员和平台运营 | 审核结果、处罚、导出完成和不活跃提醒能闭环 | 若通知依赖未就绪，高风险生命周期流程不得发布为自动处置 |
