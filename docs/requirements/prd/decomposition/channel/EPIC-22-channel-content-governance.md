# 频道内容发布与治理

## 执行摘要

EPIC-22 建立频道内内容发布、审核、管理和公告能力，让频道从“可加入、可订阅的社区单元”进一步升级为“可持续产出、可治理、可沉淀优质内容的主题空间”。本史诗承接 EPIC-20 的频道基础架构和 EPIC-21 的隐私、成员与角色规则，重点解决内容如何进入频道、谁能发布、何时进入审核、发布后如何治理，以及频道如何通过公告传达规则。

本史诗的产品愿景是：创作者能够把合适内容投放到合适频道，频道主和管理员能够用清晰、可解释的规则维护内容质量，用户能够在频道中看到有秩序、有主题一致性的内容集合。平台需要在降低误发、刷屏、违规内容扩散风险的同时，保留内容恢复、修订追溯和作者知情等治理能力。

## 史诗概览

**史诗 ID**: EPIC-22
**标题**: 建立频道内容发布权限模型与内容治理能力
**背景**: 频道作为内容聚合的核心单元，需要完善的内容发布机制和内容治理能力，以确保频道内容质量和秩序。不同频道类型、成员角色和发布场景需要差异化的权限控制。
**目标用户**: 内容创作者、频道主、频道管理员、内容编辑、平台运营、普通成员、内容消费者
**优先级**: 高
**预计持续时间**: 3 周

**范围**:
- 包含：内容发布至频道、发布权限模型、先审后发、定时发布、发布限额、防灌水、多频道发布、内容置顶、精华、删除、回收站、移出频道、编辑协助、频道公告、已发布内容添加到频道。
- 不包含：频道创建与所有权管理（EPIC-20）、频道隐私与成员管理（EPIC-21）、推荐发现（EPIC-23）、频道数据统计（EPIC-24）、平台级内容安全审核策略、完整创作器能力。

## 关键术语与业务规则

| 术语 | 定义 |
| --- | --- |
| 内容类型 | 本史诗覆盖文章、图文帖子、视频、笔记、问答问题、问答答案。新增内容类型接入频道发布时，必须复用本史诗的频道选择、权限校验、审核和治理规则。 |
| 频道成员 | 已按 EPIC-21 加入频道且未被移除或加入频道黑名单的用户，包括频道主、管理员、内容编辑和普通成员。 |
| 发布权限模型 | 频道对内容进入方式的规则集合，包括仅管理员可发布、所有成员可发布、公开投稿和先审后发。 |
| 仅管理员可发布 | 频道主、管理员和具备内容管理权限的内容编辑可发布或添加内容；普通成员和非成员不可直接发布。 |
| 所有成员可发布 | 频道成员可直接发布；非成员不可直接发布。 |
| 公开投稿 | 非成员可向频道投稿，但内容需进入待审区；通过后才在频道展示。 |
| 先审后发 | 频道开启后，所有提交到频道的内容均先进入待审区；通过后展示，拒绝后不展示。 |
| 待审区 | 频道内待处理内容列表。审核人包括频道主、管理员和具备内容审核权限的内容编辑；审核结果为通过或拒绝，拒绝需填写原因并通知提交者。 |
| 发布限额 | 频道级防刷屏规则，包括每小时发布上限、每日发布上限和内容字数下限。每小时按滚动 60 分钟计算，每日按用户所在时区自然日计算。 |
| 定时发布 | 创作者选择未来时间发布内容；到达计划时间时系统重新校验频道状态、用户权限、禁言和限额，再决定发布、进入审核或失败。 |
| 频道回收站 | 保存从频道删除的内容关联，默认保留 30 天。保留期内可恢复到频道，超过保留期后不再支持频道级恢复。 |
| 移出频道 | 将内容从当前频道移除，或转入另一个目标频道。目标频道仍需按其发布权限模型校验。 |
| 编辑协助 | 频道管理者对内容标题、标签、摘要或明显错别字进行有限修订，保留原作者身份，并记录修订历史。 |
| 频道公告 | 频道顶部展示的规则、活动或运营信息，由频道主或管理员维护。 |

## 用户旅程

### 旅程 1: 创作者发布内容到频道

1. 创作者进入内容发布页面，完成内容主体编辑。
2. 创作者打开选择频道组件，查看自己可发布、可投稿或可申请审核发布的频道。
3. 创作者选择一个或多个频道，并看到每个频道的发布结果预期：直接发布、进入待审或不可发布。
4. 系统在提交时校验频道数量上限、发布权限、禁言状态、发布限额和内容字数要求。
5. 内容直接展示、进入待审区或返回明确失败原因。
6. 若为定时发布，到达计划时间时重新校验规则，并向创作者反馈发布、待审或失败结果。

### 旅程 2: 频道管理员审核投稿

1. 管理员进入频道管理后台的待审区。
2. 管理员查看投稿内容、提交者、目标频道、提交时间、来源场景和命中规则。
3. 管理员逐条或批量审核内容，选择通过或拒绝。
4. 通过后内容在频道展示；拒绝后内容不展示，并向提交者说明原因。
5. 管理员可通过筛选查看超时未处理、特定内容类型或特定提交者的待审内容。

### 旅程 3: 管理员治理已发布内容

1. 管理员进入频道内容管理页，通过搜索、筛选或排序找到目标内容。
2. 管理员对内容执行置顶、精华、取消置顶、取消精华、删除、恢复、移出频道或编辑协助。
3. 系统在关键操作前展示二次确认，说明影响范围和是否通知作者。
4. 操作完成后，频道内容列表按最新状态展示。
5. 治理记录可供频道主、平台运营和申诉流程追溯。

### 旅程 4: 频道主维护频道规则与公告

1. 频道主进入频道设置页，查看当前发布权限模型、先审后发状态、发布限额和公告内容。
2. 频道主调整发布权限或限额，并看到对普通成员、非成员和管理员的影响说明。
3. 频道主编辑公告内容，预览在频道主页的展示效果。
4. 保存后，新发布请求按最新规则处理，频道主页展示最新公告。

## UX/UI 与响应式要求

- 发布页需在选择频道时展示频道名称、频道类型、用户角色、发布结果预期和不可选原因。
- 多频道发布时，每个频道需有独立状态反馈，避免用户只能看到笼统失败。
- 待审区需支持按内容类型、提交者、提交时间、审核状态和超时状态筛选。
- 内容治理操作需放在内容管理页和内容详情管理入口，关键操作必须有二次确认。
- 删除、移出频道、编辑协助和公告删除需提供明确影响说明，避免误操作。
- 公告编辑需支持预览，保存后在频道主页顶部展示。
- 空状态需覆盖无可发布频道、无待审内容、无回收站内容、无公告、无治理记录，并提供下一步行动入口。
- 错误态需说明具体原因，例如权限不足、频道冻结、用户禁言、发布限额、内容字数不足、目标频道不可用。
- 响应式要求：移动端优先保证选择频道、提交结果、待审区审核动作和内容治理菜单可用；窄屏下批量操作收纳到操作栏或更多菜单，列表字段按优先级折叠，避免横向溢出。

## 特性列表

### 特性 1: 内容发布至频道

**描述**: 用户在发布内容时可选择一个或多个目标频道，系统根据频道发布权限模型判断内容直接展示、进入待审区或拒绝发布。
**关键交互**: 用户编辑内容 → 选择目标频道 → 查看发布结果预期 → 提交发布 → 系统返回直接展示、待审或失败结果。
**产品规则**:
- 选择频道组件仅展示用户可发布、可投稿或可管理的频道；不可发布频道可在搜索结果中展示为不可选并说明原因。
- 多频道发布按目标频道分别校验，提交结果需逐频道反馈。
- 私有频道、禁言成员、频道黑名单、成员角色和频道冻结状态需承接 EPIC-21 规则。
**验收条件**: 所有覆盖内容类型均可选择频道发布，发布权限和提交结果在核心场景下准确可解释。

#### 故事列表

##### 故事 22.1.1: 选择频道发布内容

**作为** 内容创作者，
**我希望** 在发布文章、图文帖子、视频、笔记、问答问题、问答答案时选择目标频道，
**以便** 将内容精准投放到目标受众。

**验收标准**:
1. **给定** 用户发布任一覆盖内容类型，**当** 进入发布页面，**则** 显示“选择频道”组件，列出用户有发布、投稿或管理权限的频道。
2. **给定** 用户选择一个或多个频道，**当** 提交发布，**则** 每个目标频道返回直接展示、进入待审区或失败原因。
3. **给定** 用户选择的频道数超过系统配置上限 N，**当** 尝试继续添加，**则** 阻止添加并提示“单篇内容最多同步至 N 个频道”。
4. **给定** 用户选择了需要审核的频道，**当** 提交发布，**则** 内容进入该频道待审区，并向用户展示待审核状态。
5. **给定** 用户对某频道无发布或投稿资格，**当** 尝试选择该频道，**则** 频道不可选并展示具体原因。

**附加说明**:
- **优先级**: 高
- **依赖项**: EPIC-20（频道创建）、EPIC-21（成员角色与禁言规则）
- **产品规则**: 同一内容不可在同一频道重复展示。

##### 故事 22.1.2: 发布权限模型配置

**作为** 频道主，
**我希望** 配置频道的发布权限模型（仅管理员可发布/所有成员可发布/公开投稿/先审后发），
**以便** 控制谁可以在频道中发布内容。

**验收标准**:
1. **给定** 频道设置为“仅管理员可发布”，**当** 普通成员或非成员尝试发布到该频道，**则** 系统拒绝并提示“仅管理员可发布”。
2. **给定** 频道设置为“所有成员可发布”，**当** 频道成员发布内容，**则** 内容直接在频道展示；非成员不可直接发布。
3. **给定** 频道设置为“公开投稿”，**当** 非成员用户投稿，**则** 内容进入待审区，审核通过后展示。
4. **给定** 频道开启“先审后发”，**当** 任一用户向该频道提交内容，**则** 内容均进入待审区，审核通过后展示。
5. **给定** 用户处于禁言状态或被频道黑名单命中，**当** 尝试发布或投稿，**则** 系统拒绝并展示对应原因。

**附加说明**:
- **优先级**: 高
- **依赖项**: 故事 22.1.1、EPIC-21
- **产品规则**: “先审后发”优先级高于直接发布规则；系统频道的发布规则由平台运营配置。

##### 故事 22.1.3: 定时发布

**作为** 内容创作者，
**我希望** 为内容设定未来发布时间，
**以便** 在最佳时间触达目标读者。

**验收标准**:
1. **给定** 用户在发布页面设置未来发布时间，**当** 提交发布，**则** 内容显示为“待发布”，并展示计划发布时间。
2. **给定** 到达计划发布时间，**当** 系统处理该内容，**则** 按目标频道当前规则发布、进入待审区或失败。
3. **给定** 定时发布前用户被移除发布权限、被禁言或目标频道不可用，**当** 到达计划发布时间，**则** 发布失败并通知用户具体原因。
4. **给定** 用户修改定时发布时间，**当** 保存修改，**则** 最新计划时间在内容状态中展示。
5. **给定** 计划发布时间早于当前时间或超过平台允许范围，**当** 用户保存，**则** 系统拒绝并提示可选择的时间范围。

**附加说明**:
- **优先级**: 中
- **依赖项**: 故事 22.1.1、故事 22.1.2
- **产品规则**: 定时发布到达时必须重新校验频道状态、成员状态、禁言状态、发布权限和限额。

##### 故事 22.1.4: 发布限额与防灌水

**作为** 频道主，
**我希望** 设置频道的发布限额（每日/每小时发布上限、内容字数下限），
**以便** 防止刷屏和低质量内容泛滥。

**验收标准**:
1. **给定** 频道设置每日发布上限为 5 篇，**当** 某成员当天已向该频道发布或投稿 5 篇后再次提交，**则** 系统拒绝并提示“今日发布次数已达上限”。
2. **给定** 频道设置内容字数下限为 100 字，**当** 用户发布内容字数不足，**则** 系统拒绝并提示“内容字数需不少于 100 字”。
3. **给定** 频道设置每小时发布上限为 M 篇，**当** 成员在滚动 60 分钟内已达到 M 篇后再次提交，**则** 系统拒绝并提示可再次发布的时间。
4. **给定** 用户同时选择多个频道，**当** 部分频道触发限额，**则** 仅对应频道失败，其他频道按各自规则处理并逐项展示结果。

**附加说明**:
- **优先级**: 低
- **依赖项**: 故事 22.1.1、故事 22.1.2
- **产品规则**: 限额命中应被视为业务拒绝，不计入发布成功率失败样本。

### 特性 2: 频道内容管理

**描述**: 频道管理员和内容编辑可对频道内容进行置顶、精华标记、编辑协助、删除、恢复、移出频道等治理操作。
**关键交互**: 管理员进入频道内容管理页 → 查找内容 → 执行治理操作 → 查看结果与治理记录。
**产品规则**:
- 内容治理权限承接 EPIC-21：频道主和管理员可管理内容；内容编辑可执行被授权的内容管理操作；普通成员不可治理内容。
- 关键治理操作需二次确认，并记录操作者、内容、动作、时间、原因和结果。
- 删除进入频道回收站，30 天内可恢复；平台级删除或内容作者删除不由频道回收站恢复。
**验收条件**: 所有内容治理操作功能完整，操作结果可见，治理记录完整。

#### 故事列表

##### 故事 22.2.1: 内容置顶与精华标记

**作为** 频道管理员，
**我希望** 将重要内容置顶或标记为精华，
**以便** 突出优质内容并引导用户关注。

**验收标准**:
1. **给定** 管理员选择某内容，**当** 点击“置顶”，**则** 该内容出现在频道顶部，并展示置顶状态。
2. **给定** 频道存在多条置顶内容，**当** 管理员调整置顶顺序，**则** 用户看到的置顶内容按最新顺序展示。
3. **给定** 管理员选择某内容，**当** 点击“标记精华”，**则** 该内容获得精华标识并聚合到精华列表。
4. **给定** 管理员取消置顶，**当** 操作完成，**则** 内容回到正常排序位置。
5. **给定** 管理员取消精华标记，**当** 操作完成，**则** 内容失去精华标识并从精华列表移除。

**附加说明**:
- **优先级**: 高
- **依赖项**: 故事 22.1.1
- **产品规则**: 置顶和精华不改变内容原作者和发布时间。

##### 故事 22.2.2: 内容删除与回收站

**作为** 频道管理员，
**我希望** 删除不符合主题或违规的内容，并能在回收站中恢复误删内容，
**以便** 维护频道内容质量。

**验收标准**:
1. **给定** 管理员选择某内容，**当** 点击“删除”并确认，**则** 内容从频道移除并进入频道回收站。
2. **给定** 管理员进入频道回收站，**当** 查看已删除内容，**则** 展示内容标题、内容类型、原作者、删除人、删除时间和删除原因。
3. **给定** 管理员在回收站中选择内容，**当** 点击“恢复”，**则** 内容恢复到频道中，并保留恢复记录。
4. **给定** 内容在回收站超过 30 天，**当** 管理员查看回收站，**则** 该内容不再支持频道级恢复。
5. **给定** 管理员批量删除内容，**当** 选择多条内容并确认，**则** 每条内容展示成功或失败结果，失败项保留原因。

**附加说明**:
- **优先级**: 高
- **依赖项**: 故事 22.1.1
- **产品规则**: 频道删除仅影响内容在该频道的展示；是否影响内容在其他频道或作者主页的展示由内容域规则决定。

##### 故事 22.2.3: 内容移出频道

**作为** 频道管理员，
**我希望** 将内容从当前频道移至其他频道，
**以便** 纠正内容分类错误或调整内容归属。

**验收标准**:
1. **给定** 管理员选择某内容，**当** 点击“移出频道”并选择目标频道，**则** 系统展示目标频道的发布权限要求和预期处理结果。
2. **给定** 目标频道设置为“仅管理员可发布”且操作者不是目标频道管理员，**当** 发起移入，**则** 目标频道管理员确认后内容才可进入目标频道。
3. **给定** 目标频道设置为“所有成员可发布”且作者或操作者满足目标频道规则，**当** 移出操作发起，**则** 内容进入目标频道。
4. **给定** 目标频道开启“先审后发”或采用“公开投稿”，**当** 移出操作发起，**则** 内容进入目标频道待审区。
5. **给定** 移出操作完成，**当** 查看原频道，**则** 内容不再出现在原频道中，并保留移出记录。

**附加说明**:
- **优先级**: 中
- **依赖项**: 故事 22.1.1、故事 22.1.2
- **产品规则**: 跨频道移动必须同时尊重原频道治理权限和目标频道发布权限。

##### 故事 22.2.4: 编辑协助

**作为** 频道管理员，
**我希望** 协助修正频道内容的标题、标签、摘要或明显错别字，
**以便** 提升内容质量同时保留作者信息。

**验收标准**:
1. **给定** 管理员选择某内容，**当** 点击“编辑协助”，**则** 进入编辑页面，可修改标题、标签、摘要和明显错别字。
2. **给定** 管理员修改内容并保存，**当** 操作完成，**则** 内容更新但保留原作者信息。
3. **给定** 管理员修改内容，**当** 查看修订历史，**则** 记录修改人、修改时间、修改字段、修改前后摘要和修改原因。
4. **给定** 管理员尝试修改正文核心观点、作者署名或内容归属，**当** 保存，**则** 系统拒绝或要求转入更高权限流程。

**附加说明**:
- **优先级**: 低
- **依赖项**: 故事 22.1.1
- **产品规则**: 编辑协助用于频道质量修正，不应改变作者表达的实质含义。

### 特性 3: 频道公告

**描述**: 频道管理员可发布顶部公告，说明频道规则、活动、运营安排或重要通知。
**关键交互**: 管理员进入频道设置 → 编辑公告内容 → 预览 → 发布公告 → 公告出现在频道顶部。
**产品规则**:
- 公告内容可以使用平台支持的富文本格式；展示前必须符合平台安全展示规则。
- 一个频道同一时间最多展示一条主公告；历史公告是否保留由后续公告管理能力决定。
**验收条件**: 公告发布、编辑、删除、预览和展示功能完整，公告在频道顶部正确展示。

#### 故事列表

##### 故事 22.3.1: 频道公告管理

**作为** 频道管理员，
**我希望** 发布和管理频道公告，说明频道规则、活动等重要信息，
**以便** 向所有频道成员传达关键信息。

**验收标准**:
1. **给定** 管理员进入频道公告管理页，**当** 编辑公告内容并保存，**则** 公告出现在频道页面顶部。
2. **给定** 公告已发布，**当** 用户进入频道页面，**则** 在顶部看到公告内容。
3. **给定** 管理员编辑公告，**当** 修改内容并保存，**则** 公告内容更新，并记录修改人和修改时间。
4. **给定** 管理员删除公告，**当** 确认删除，**则** 公告从频道顶部消失，并保留删除记录。
5. **给定** 公告内容包含富文本，**当** 系统预览或展示，**则** 仅展示平台允许的格式和链接。

**附加说明**:
- **优先级**: 中
- **依赖项**: 故事 22.1.1
- **产品规则**: 公告用于频道规则和活动沟通，不替代平台级系统通知。

### 特性 4: 已发布内容添加到频道

**描述**: 支持将已发布内容添加到频道，包括系统频道添加、个人/组织频道添加、频道主添加他人作品。
**关键交互**: 用户选择已发布内容 → 选择目标频道 → 查看权限结果 → 添加或进入待审。
**产品规则**:
- 添加已发布内容需遵循目标频道的发布权限模型、先审后发规则和禁言/黑名单限制。
- 添加他人作品需向原作者展示频道归属变化；是否允许作者申诉或撤回由内容域后续规则定义。
**验收条件**: 三种添加场景在对应权限模型下正确执行，并向操作者和作者提供清晰反馈。

#### 故事列表

##### 故事 22.4.1: 已发布内容添加到系统频道

**作为** 平台运营，
**我希望** 将已发布的优质内容添加到系统官方频道，
**以便** 在官方频道中推荐优质内容。

**验收标准**:
1. **给定** 平台运营选择已发布内容，**当** 选择目标系统频道并添加，**则** 内容出现在系统频道中。
2. **给定** 系统频道设置为“仅管理员可发布”，**当** 非管理员尝试添加，**则** 系统拒绝并提示权限不足。
3. **给定** 内容已添加到系统频道，**当** 查看内容详情，**则** 显示内容所在的频道列表。
4. **给定** 系统频道开启“先审后发”，**当** 平台运营添加内容，**则** 内容按系统频道当前规则进入待审或展示。

**附加说明**:
- **优先级**: 中
- **依赖项**: 故事 22.1.2
- **产品规则**: 系统频道添加行为需记录运营身份和添加原因。

##### 故事 22.4.2: 作者将已发布内容添加到个人/组织频道

**作为** 内容创作者，
**我希望** 将我已发布的内容添加到我的个人频道或所属组织频道，
**以便** 在频道中聚合我的内容。

**验收标准**:
1. **给定** 作者选择自己的已发布内容，**当** 选择目标个人/组织频道并添加，**则** 内容按目标频道规则展示或进入待审。
2. **给定** 目标频道设置为“仅管理员可发布”且作者非管理员，**当** 尝试添加，**则** 系统拒绝并提示权限不足。
3. **给定** 目标频道设置为“公开投稿”或开启“先审后发”，**当** 作者添加内容，**则** 内容进入待审区。
4. **给定** 作者不再是目标组织频道成员，**当** 尝试添加内容，**则** 系统拒绝并提示需先加入或获得授权。

**附加说明**:
- **优先级**: 中
- **依赖项**: 故事 22.1.2、EPIC-21
- **产品规则**: 组织频道需遵循组织授权与成员身份规则。

##### 故事 22.4.3: 频道主添加他人作品至频道

**作为** 频道主/管理员，
**我希望** 将他人的优质作品添加到我管理的频道，
**以便** 丰富频道内容。

**验收标准**:
1. **给定** 频道主选择他人已发布内容，**当** 选择目标频道并添加，**则** 内容按目标频道发布权限模型处理。
2. **给定** 目标频道设置为“仅管理员可发布”且操作者具备目标频道管理权限，**当** 添加内容，**则** 内容直接展示。
3. **给定** 目标频道开启“先审后发”，**当** 频道主或管理员添加他人作品，**则** 内容进入待审区或按平台运营规则处理。
4. **给定** 添加操作完成，**当** 原作者查看内容，**则** 显示内容已被添加到某频道。
5. **给定** 原内容已被作者删除、平台下架或不可见，**当** 频道主尝试添加，**则** 系统拒绝并提示内容不可添加。

**附加说明**:
- **优先级**: 中
- **依赖项**: 故事 22.1.2
- **产品规则**: 添加他人内容不得改变原作者署名和原始发布时间。

## 依赖关系

- **前置依赖**: EPIC-20（频道基础架构）、EPIC-21（成员角色、禁言、黑名单、隐私规则）。
- **内容域依赖**: 内容发布服务需提供内容创建、内容状态、作者身份、内容可见性和内容详情展示能力。
- **通知依赖**: 通知系统需支持发布失败、审核结果、内容被添加到频道、治理操作结果等通知。
- **后续依赖**: EPIC-23（推荐与发现，需要频道内容和可见性规则）、EPIC-24（数据统计，需要发布与治理事件）。

## 风险与假设

| 类型 | 内容 | 影响 | 应对 |
| --- | --- | --- | --- |
| 风险 | 多频道发布下部分频道成功、部分频道失败会增加用户理解成本 | 用户误以为全部成功或全部失败 | 提供逐频道结果反馈和可重试入口 |
| 风险 | 先审后发与公开投稿规则不清会导致审核负担和用户争议 | 频道管理成本上升 | 在发布前展示预期结果，并在待审区展示来源和命中规则 |
| 风险 | 编辑协助边界不清可能引发作者权益争议 | 作者信任下降 | 限定可修改字段，记录修订历史，保留作者身份 |
| 风险 | 回收站与平台级删除边界不清可能造成恢复失败争议 | 管理员误判可恢复范围 | 明确频道级删除仅影响频道展示，平台级删除不由频道回收站恢复 |
| 假设 | 内容发布系统已具备基础内容创建、展示和状态管理能力 | EPIC-22 不重复定义创作器能力 | 若内容域能力缺失，需前置补齐 |
| 假设 | EPIC-21 已提供成员角色、禁言和频道黑名单规则 | 发布权限可复用成员治理结果 | 若 EPIC-21 延迟，需先支持最小角色判断 |
