## 2. 背景与问题

### 2.1 当前现状

内容社区系统支持文章、图文帖子、视频、笔记、问答问题、问答答案等多类型内容发布。随着内容规模增长，平台需要一个稳定的主题组织单元，帮助用户发现内容、帮助创作者触达目标读者、帮助平台运营承载官方分类与活动。

频道是内容组织与发现的核心单元，需同时覆盖平台官方、个人创作者和组织机构三类运营场景。

### 2.2 用户痛点

- 内容创作者缺少稳定的主题归属，内容发布后难以持续沉淀目标受众。
- 内容消费者难以通过清晰的主题入口发现、订阅和持续浏览感兴趣内容。
- 频道主和组织管理员缺少成员准入、发布权限、内容治理和数据分析工具。
- 平台运营缺少官方频道、编辑精选、排行榜、生命周期治理等集中管理能力。

### 2.3 业务痛点

- 内容分发过度依赖推荐流和关注流，缺少可运营、可治理、可复用的频道资产。
- 频道规则若不明确，会导致权限冲突、内容刷屏、违规内容扩散和用户体验不一致。
- 系统官方频道、个人频道、组织频道的创建、审核、归属和删除规则不同，需要统一产品口径。

### 2.4 为什么现在要做

频道能力是内容发布、订阅、推荐、搜索、组织运营和后续商业化的基础。先建立频道域的标准模型和治理规则，可以降低后续内容分发、付费频道、活动频道、组织品牌频道等扩展成本。

### 2.5 数据或事实依据

- 来源需求已明确频道需要覆盖创建、编辑、隐私、加入、发布、治理、推荐、统计、生命周期等完整闭环。

## 3. 目标与成功指标

### 3.1 业务目标

- 建立覆盖系统官方、个人、组织三类场景的统一频道体系。
- 提升内容组织效率和频道订阅转化，形成可持续运营的内容资产。
- 为平台推荐、搜索、分类浏览、组织品牌展示和未来付费频道提供基础能力。

### 3.2 产品目标

- 用户可创建、编辑、订阅、加入、发布和浏览频道内容。
- 频道主/管理员可配置频道规则、管理成员、治理内容、查看数据。
- 平台运营可管理系统频道、分类体系、推荐精选、审核与违规治理。
- 三类频道在权限、生命周期、可见性和归属规则上行为一致且可验证。

### 3.3 成功指标

- 指标 1：频道创建成功率 >= 99%，创建失败原因可追踪。
- 指标 2：频道搜索响应时间 P99 <= 200ms。
- 指标 3：频道相关核心接口可用性 >= 99.9%。
- 指标 4：频道统计数据延迟 <= 5 分钟。
- 指标 5：定时发布准时率 >= 99.9%。
- 指标 6：频道审核处理 SLA：普通创建/修改审核 24 小时内完成比例 >= `[待确认]%`。
- 指标 7：上线 3 个月内，频道订阅转化率、频道内容消费占比达到 `[待确认目标]`。

## 4. 范围

### 4.1 In Scope

- 频道类型定义：系统官方频道、个人频道、组织频道。
- 频道创建、编辑、转让、删除、归档、合并、冻结等生命周期能力。
- 频道隐私设置、加入方式、订阅、订阅分组、成员管理与角色权限。
- 内容发布至频道、多频道发布、先审后发、定时发布、发布限额、防灌水。
- 频道内容治理：置顶、精华、编辑协助、删除、回收站、移出频道、频道公告、排序筛选。
- 平台分类体系、频道内标签、个性化推荐、排行榜、编辑精选、分类浏览、搜索。
- 频道数据看板、用户分析、数据导出。
- 平台审核、内容安全、操作日志和合规治理。

### 4.2 Out of Scope

- 独立支付结算、创作者收益提现。
- 频道内直播、课程、电商、广告投放。
- 频道独立 App、小程序或站外分发系统。
- 组织认证流程本身，仅消费组织认证结果和组织权限。
- 推荐算法底层模型训练方案，仅定义产品侧输入、输出和兜底策略。

### 4.3 非目标

- 本期不实现完整付费频道闭环；付费频道仅作为扩展能力预留规则。
- 本期不将系统频道、个人频道、组织频道互相转换；如需转换，走新建频道、内容迁移、订阅迁移提示流程。
- 本期不解决所有内容安全审核系统建设问题，仅定义频道域接入审核的触发点和结果处理。

## 5. 用户与场景

### 5.1 目标用户

| 角色 | 描述 | 核心需求 |
| --- | --- | --- |
| 游客 | 未登录用户 | 浏览公开频道与公开内容，了解频道价值 |
| 注册用户 / 内容消费者 | 已登录普通用户 | 订阅频道、加入频道、浏览频道内容、管理订阅 |
| 内容创作者 | 发布内容的用户 | 将内容发布到一个或多个目标频道 |
| 频道主 | 个人频道创建者或组织频道指定负责人 | 设置频道信息、规则、成员、内容治理 |
| 频道管理员 / 内容编辑 | 被授权的频道协作者 | 审核成员、治理内容、维护频道秩序 |
| 组织管理员 | 已认证组织的授权管理者 | 创建组织频道、管理组织频道成员和组织展示信息 |
| 平台运营 / 系统管理员 | 平台后台角色 | 创建系统频道、管理分类、审核、推荐、冻结、归档和审计 |

### 5.2 用户故事

- 作为一个 `内容创作者`，我希望将内容发布到特定频道，以便更好地组织和分类我的内容。
- 作为一个 `频道主/组织管理员`，我希望设定频道的加入与发布规则，以便保护频道内容和互动质量。
- 作为一个 `平台运营`，我希望创建和管理系统官方频道，并通过推荐引导用户发现优质内容。
- 作为一个 `内容消费者`，我希望通过清晰的信息和规则了解一个频道，从而决定是否订阅或参与。
- 作为一个 `普通用户`，我希望创建属于自己的个人频道，以便聚合我感兴趣或创作的内容并积累订阅者。
- 作为一个 `组织负责人`，我希望为组织创建专属频道，以便展示组织内容并管理成员。
- 作为一个 `系统管理员`，我希望审计频道关键操作，以便满足社区治理与合规追溯要求。

### 5.3 核心场景

#### 场景 1：创建个人/组织频道

- 前置条件：用户已登录；创建组织频道时，组织已认证且用户具备组织管理权限。
- 主流程：
  1. 用户进入频道创建入口。
  2. 选择频道类型，填写名称、简介、图标、封面、隐私设置和归属分类。
  3. 系统校验必填项、频道数量上限、名称唯一性、组织认证状态。
  4. 系统进行内容安全检测，频道进入审核状态。
  5. 审核通过后频道公开显示，创建者获得频道主权限。
- 备选流程：系统频道由平台管理员在后台创建，默认公开且无需普通审核。
- 异常流程：名称重复、超出创建上限、组织未认证、内容安全不通过时，提示明确原因。
- 结束状态：频道状态为正常或待审核；操作日志可查询。

#### 场景 2：订阅或加入频道

- 前置条件：频道存在且未关闭；用户具备浏览或申请权限。
- 主流程：
  1. 用户通过搜索、推荐、排行榜或分类浏览进入频道主页。
  2. 用户查看频道信息、规则和内容可见范围。
  3. 用户点击订阅、加入或申请加入。
  4. 系统根据公开/私有、自由加入/审核加入/邀请加入规则处理。
  5. 成功后频道进入用户订阅列表，并影响信息流展示权重。
- 备选流程：邀请加入时校验邀请码或邀请链接有效期与使用次数。
- 异常流程：用户被频道黑名单命中、申请重复提交、频道冻结或归档时，阻断操作并提示。
- 结束状态：用户成为订阅者/成员，或申请进入待审状态。

#### 场景 3：发布内容到频道

- 前置条件：用户已登录；内容类型受平台支持；用户对目标频道具备发布、投稿或审核后发布权限。
- 主流程：
  1. 用户在发布文章、图文、视频、笔记、问答等内容时选择目标频道。
  2. 系统展示用户有权限发布或投稿的频道。
  3. 用户选择一个或多个频道并提交。
  4. 系统校验频道数量上限、发布权限、禁言状态、发布频率和内容合规。
  5. 内容直接发布或进入频道待审区。
- 备选流程：定时发布到达执行时间时重新校验发布权限。
- 异常流程：权限变化、频道冻结、内容审核失败、重复提交或队列超时时，发送失败通知并保留可追踪原因。
- 结束状态：内容在目标频道展示、待审或发布失败。

#### 场景 4：频道内容治理

- 前置条件：操作者为频道主、管理员、内容编辑或平台运营，具备对应治理权限。
- 主流程：
  1. 管理员进入频道内容管理页。
  2. 对内容执行置顶、精华、编辑协助、删除、批量删除、移出频道或恢复。
  3. 系统校验权限和目标频道规则。
  4. 操作生效并记录审计日志。
- 备选流程：跨频道移入需目标频道同意或满足平台规则。
- 异常流程：权限不足、内容已删除、目标频道不可用、并发修改冲突时，提示并阻止错误状态写入。
- 结束状态：内容状态更新，用户侧展示同步刷新。

## 6. 需求层级（Epic / Feature / Story）

### EPIC-20 频道基础架构与创建

- 业务目标：建立三类频道的基础模型、创建流程和所有权管理能力。
- Source / Reference：`docs/requirements/频道需求说明.md` 3、4.1、4.2；`EPIC-20-channel-infrastructure.md`
- Owner：[待确认]
- 范围说明：包含频道类型、数据模型、系统频道创建、个人/组织频道创建、编辑、转让、删除。

#### FEATURE-20.1 频道类型定义与基础模型

- Feature 目标：统一定义系统官方频道、个人频道、组织频道的数据模型和不可变类型规则。
- 优先级：Must
- 参与角色：系统管理员、平台运营、普通用户、组织管理员
- 业务规则：
  - 每个频道必须且仅能属于一种类型。
  - 频道类型创建后不可直接修改。
  - 组织频道创建时必须绑定组织 ID，绑定后不可更改。
  - 系统频道默认公开、全站可见，不可设为私有。
- 核心 Stories：
  - STORY-20.1.1：频道基础数据模型设计。
  - STORY-20.1.2：频道类型差异化规则配置。

##### STORY-20.1.1 频道基础数据模型设计

- 用户 / 角色：系统管理员、平台运营、频道创建者
- 用户价值：通过统一频道模型支撑三类频道创建、展示、权限和生命周期管理。
- 前置条件：频道类型枚举、组织认证结果、分类体系基础数据可用。
- 主流程：
  1. 用户或平台管理员提交频道创建请求。
  2. 系统写入频道基础字段、类型字段、归属字段和状态字段。
  3. 系统根据频道类型初始化默认配置。
- 验收标准：
  - Given 创建任一频道
    When 查询频道详情
    Then 频道类型必须为 system、personal、organization 之一且创建后不可直接修改。
- 依赖：组织认证结果、平台分类体系。
- 边界与异常：
  - 空状态：无频道时创建入口展示默认引导。
  - 权限不足：无组织权限时不得创建组织频道。
  - 重复操作：同一 request_id 重复提交不得创建重复频道。

##### STORY-20.1.2 频道类型差异化规则配置

- 用户 / 角色：系统、平台运营、频道主
- 用户价值：确保系统频道、个人频道、组织频道在隐私、归属、转让和删除规则上行为一致。
- 前置条件：频道基础模型已定义。
- 主流程：
  1. 系统读取频道类型。
  2. 按类型应用创建、编辑、隐私、转让、删除规则。
  3. 对不允许的操作返回明确原因。
- 验收标准：
  - Given 系统频道已创建
    When 操作者尝试将其设为私有
    Then 系统拒绝并提示系统频道必须公开。
- 依赖：频道基础模型、权限服务。
- 边界与异常：
  - 权限不足：非平台管理员不得管理系统频道。
  - 并发修改：同一频道配置并发修改时需使用版本校验。

#### FEATURE-20.2 三类频道创建

- Feature 目标：支持平台管理员创建系统频道，普通用户创建个人频道，组织管理员创建组织频道。
- 优先级：Must
- 业务规则：
  - 系统频道仅平台后台可创建，可使用平台保留关键词。
  - 个人/组织频道名称在用户频道范围内唯一，系统频道名称不参与唯一性校验。
  - 个人/组织频道创建后进入审核状态，审核通过后公开显示。
  - 单用户/组织创建数量受平台配置限制，默认个人 20 个、组织 50 个。
- 核心 Stories：
  - STORY-20.2.1：系统频道创建。
  - STORY-20.3.1：个人频道创建。
  - STORY-20.3.2：组织频道创建。

#### FEATURE-20.3 编辑与所有权管理

- Feature 目标：支持频道信息编辑、个人频道转让、频道删除申请和冷静期。
- 优先级：Should
- 业务规则：
  - 名称修改需重新审核，审核期间按原信息展示。
  - 个人频道可转让给符合条件的用户。
  - 组织频道不可转让给个人，仅可调整组织内管理权限。
  - 频道删除需满足内容清理/转移、付费订阅结清等条件，并二次确认。
- 核心 Stories：
  - STORY-20.4.1：频道信息编辑。
  - STORY-20.4.2：频道转让。
  - STORY-20.4.3：频道删除。

### EPIC-21 频道隐私、订阅与成员管理

- 业务目标：建立频道访问控制、订阅关系、成员角色与频道级治理能力。
- Source / Reference：`docs/requirements/频道需求说明.md` 4.3、4.6；`EPIC-21-channel-privacy-membership.md`
- Owner：[待确认]
- 范围说明：包含隐私设置、加入方式、订阅、订阅分组、角色权限、成员列表、移除、禁言、黑名单。

#### FEATURE-21.1 隐私与加入规则

- Feature 目标：支持公开/私有频道和自由加入、审核加入、邀请加入。
- 优先级：Must
- 业务规则：
  - 公开频道可搜索、可浏览；私有频道仅成员可浏览且不可公开搜索。
  - 系统频道强制公开。
  - 审核加入需进入待审队列；邀请加入需校验邀请凭证。
- 核心 Stories：
  - STORY-21.1.1：频道隐私设置。
  - STORY-21.1.2：加入方式配置。
  - STORY-21.1.3：加入申请审核。

#### FEATURE-21.2 订阅与订阅管理

- Feature 目标：用户可订阅、取消订阅和分组管理频道。
- 优先级：Must
- 业务规则：
  - 取消订阅不影响历史互动数据和已发布内容。
  - 私有频道需先成为成员后再订阅或浏览受限内容。
  - 订阅后频道内容在信息流中加权展示。
- 核心 Stories：
  - STORY-21.2.1：订阅与取消订阅。
  - STORY-21.2.2：订阅列表管理。

#### FEATURE-21.3 成员角色与治理

- Feature 目标：支持频道主、管理员、内容编辑、普通成员的权限分层和成员治理。
- 优先级：Must
- 业务规则：
  - 管理员可管理内容与成员，不可修改基础信息或转让频道。
  - 内容编辑可编辑、删除、置顶内容，不可管理成员。
  - 被移除用户在冷却期内不可再次加入。
  - 被黑名单用户无法申请加入。
- 核心 Stories：
  - STORY-21.3.1：角色定义与权限分配。
  - STORY-21.3.2：成员列表展示。
  - STORY-21.4.1：移除成员。
  - STORY-21.4.2：禁言成员。
  - STORY-21.4.3：频道黑名单。

### EPIC-22 频道内容发布与治理

- 业务目标：让内容能按频道规则发布、审核、展示和治理，保障频道内容质量。
- Source / Reference：`docs/requirements/频道需求说明.md` 4.5、4.7；`EPIC-22-channel-content-governance.md`
- Owner：[待确认]
- 范围说明：包含发布至频道、发布权限模型、先审后发、定时发布、限额、防灌水、置顶、精华、删除、回收站、公告。

#### FEATURE-22.1 内容发布至频道

- Feature 目标：用户发布所有平台支持内容类型时，可选择一个或多个目标频道。
- 优先级：Must
- 业务规则：
  - 发布前需校验目标频道发布权限。
  - 单篇内容最多同步至 N 个频道，N 由平台配置。
  - 频道可设置仅管理员、所有成员、公开投稿三类发布模型。
  - 频道可开启先审后发，所有内容进入待审区。
- 核心 Stories：
  - STORY-22.1.1：选择频道发布内容。
  - STORY-22.1.2：发布权限模型配置。
  - STORY-22.1.3：定时发布。
  - STORY-22.1.4：发布限额与防灌水。

#### FEATURE-22.2 已发布内容加入频道

- Feature 目标：支持作者或频道管理员将已发布内容添加到频道，并遵循频道权限模型。
- 优先级：Should
- 业务规则：
  - 作者可将自己的已发布内容加入个人/组织频道，需满足目标频道发布权限。
  - 频道主/管理员可把他人作品添加至频道，需满足授权、投稿审核或平台规则。
  - 添加到系统频道需遵循系统频道发布权限和平台运营审核。
- 核心 Stories：
  - STORY-22.2.1：作者添加已发布内容至频道。
  - STORY-22.2.2：管理员收录他人作品至频道。
  - STORY-22.2.3：系统频道内容收录审核。

#### FEATURE-22.3 频道内容治理

- Feature 目标：频道管理员可维护频道内容秩序和展示质量。
- 优先级：Must
- 业务规则：
  - 置顶支持多条并可排序。
  - 删除内容进入频道回收站，30 天内可恢复。
  - 编辑协助需保留原作者信息和修订历史。
  - 频道公告支持说明规则、活动和运营信息。
- 核心 Stories：
  - STORY-22.3.1：内容置顶与精华标记。
  - STORY-22.3.2：内容删除与回收站。
  - STORY-22.3.3：内容移出频道。
  - STORY-22.3.4：编辑协助。
  - STORY-22.3.5：频道公告与内容排序筛选。

### EPIC-24 频道数据统计与生命周期管理

- 业务目标：让频道运营可量化、生命周期可治理、关键操作可审计。

#### FEATURE-24.2 生命周期状态管理

- Feature 目标：管理频道从创建、审核、正常、冻结、归档、合并到删除的状态流转。
- 优先级：Must
- 业务规则：
  - 个人/组织频道创建和关键修改需审核。
  - 系统频道仅平台可归档、关闭或删除。
  - 个人频道连续 6 个月无内容更新且无有效互动，通知频道主；再经 1 个月无改善可自动归档。

  - 合并需审核并保证内容、订阅者和跳转关系一致。
- 核心 Stories：

  - STORY-24.2.2：频道冻结与解冻。
  - STORY-24.2.3：频道归档。
  - STORY-24.2.4：频道合并。

- Feature 目标：平台可对违规频道执行限制推荐、强制隐藏、只读冻结或永久关闭。
- 优先级：Must
- 业务规则：
  - 所有治理操作需记录操作人、对象、原因、时间、影响范围和恢复条件。
  - 用户或频道主应可查看必要的处罚原因和申诉入口。
  - 限制推荐不影响已订阅用户浏览，但影响公共推荐流曝光。
- 核心 Stories：
  - STORY-24.3.1：违规处理。
  - STORY-24.3.2：不活跃处置。
  - STORY-24.3.3：治理申诉与审计。

## 7. 业务流程 / 状态流转

### 7.1 频道状态列表

| 状态 | 说明 | 典型触发 |
| --- | --- | --- |
| Draft | 创建草稿，尚未提交 | 用户保存草稿 |
| PendingReview | 待审核 | 个人/组织频道创建或关键字段修改 |
| Active | 正常运营 | 审核通过或系统频道创建成功 |
| ReadonlyFrozen | 只读冻结 | 违规、风控或平台人工处理 |
| Hidden | 强制隐藏 | 严重违规或平台治理 |
| RestrictedRecommendation | 限制推荐 | 低质、风险或处罚 |
| Archived | 已归档 | 不活跃、组织解散、人工归档 |
| Merged | 已合并 | 合并至目标频道 |
| DeleteCooling | 删除冷静期 | 频道主确认删除 |
| Deleted | 已删除 | 冷静期结束或平台永久删除 |

### 7.2 主流程说明

1. 用户或平台管理员创建频道。
2. 系统根据频道类型决定是否进入审核。
3. 审核通过后频道进入正常状态。
4. 频道可被订阅、加入、发布内容和运营治理。
5. 若发生违规、不活跃、合并或删除请求，进入对应生命周期流程。
6. 所有关键状态变更写入审计日志并通知相关方。

### 7.3 超时 / 重试 / 回滚 / 补偿策略

- 审核超时：超过 SLA 未处理时进入运营待办提醒。
- 定时发布重试：执行失败时按配置重试，仍失败则通知作者并记录失败原因。
- 统计任务重试：聚合失败时保留上一版本数据，并在下一周期补偿计算。
- 合并回滚：合并执行前生成迁移批次号，失败时按批次回滚关联关系。
- 删除冷静期：冷静期内允许撤销；冷静期结束后进入不可逆删除或平台保留审计数据。

## 11. 非功能需求

### 11.1 性能

- 频道列表、分类浏览、搜索响应时间 P99 <= 200ms。
- 频道创建、编辑、订阅、加入、发布权限校验接口 P95 <= 500ms。
- 数据看板常规查询 P95 <= 1s；大数据量导出走异步任务。
- 搜索和推荐结果分页加载，单页默认不超过 20 条。

### 11.2 可用性 / 可靠性

- 频道核心能力可用性 >= 99.9%。
- 定时发布准时率 >= 99.9%，失败需通知作者并可追踪。
- 统计聚合失败时保留上一版本数据，不展示空白误导用户。
- 合并、删除、转让等关键流程需具备幂等和并发保护。

### 11.3 安全 / 合规

- 频道名称、简介、图标、封面和公告等 UGC 信息必须接入内容安全审核。
- 系统频道、组织频道、成员角色、审核、冻结、删除等敏感操作必须鉴权。
- 关键操作审计日志至少保留 `[待确认：建议 180 天或更长]`。
- 组织频道展示成员姓名、部门等组织信息前需校验组织授权。
- 私有频道内容不得出现在公开搜索、推荐或游客可访问页面。

### 11.4 兼容性

- 支持平台现有内容类型：文章、图文帖子、视频、笔记、问答问题、问答答案、转发帖子。
- 与用户域订阅、通知、黑名单、屏蔽、禁言等能力保持规则一致。
- 频道类型字段需支持未来扩展，但现有三类类型不可直接转换。

### 11.5 可观测性

- 记录频道创建、审核、订阅、加入、发布、治理、统计导出、生命周期状态变更日志。
- 对审核队列积压、定时发布失败、搜索超时、统计延迟、导出失败设置告警。
- 支持按 channel_id、user_id、request_id、audit_id 追踪问题链路。

### 11.6 扩展性

- 预留付费频道、频道内直播、课程、活动专题等扩展字段和状态。
- 分类体系支持多级树，频道标签支持频道内隔离和批量管理。
- 推荐、搜索、统计均通过事件或异步任务解耦，避免阻塞发布主链路。

## 12. 验收标准

- AC-001：Given 任一频道已创建，When 查询频道详情，Then 频道类型必须为 system、personal、organization 之一且创建后不可直接修改。
- AC-002：Given 创建组织频道，When 组织未认证或操作者无组织管理权限，Then 系统拒绝创建并提示原因。
- AC-003：Given 平台管理员创建系统频道，When 提交合法信息，Then 频道直接创建为公开、全站可见且无需普通审核。
- AC-004：Given 普通用户或组织管理员创建用户频道，When 名称唯一且内容安全通过，Then 频道进入审核状态并在审核通过后公开显示。
- AC-005：Given 频道主删除个人频道，When 内容已清理或转移且无未了结付费订阅，Then 系统二次确认并进入 7 天冷静期。
- AC-006：Given 私有频道设置为审核加入，When 用户提交加入申请，Then 申请进入待审队列且管理员可批准或拒绝。
- AC-007：Given 用户订阅公开频道，When 刷新信息流，Then 该频道内容获得订阅加权展示；取消订阅不影响历史互动。
- AC-008：Given 成员被禁言，When 其尝试发布或评论，Then 系统阻止操作并展示解禁时间。
- AC-009：Given 用户选择多个目标频道发布内容，When 频道数量超过平台配置 N，Then 系统阻止继续添加并提示上限。
- AC-010：Given 作者将已发布内容添加到个人/组织频道，When 目标频道开启先审后发，Then 内容进入待审区而非直接展示。
- AC-011：Given 管理员删除频道内容，When 操作确认，Then 内容进入频道回收站并可在 30 天内恢复。
- AC-012：Given 创建频道，When 选择分类，Then 必须选择 1 个主分类且最多 3 个副分类。
- AC-013：Given 新用户无行为数据，When 进入频道推荐页，Then 展示热门频道和编辑精选作为冷启动结果。
- AC-014：Given 用户搜索频道关键词，When 搜索完成，Then 结果覆盖名称、描述、标签，并支持类型、分类、活跃度筛选。
- AC-015：Given 频道主查看数据看板，When 选择时间范围，Then 展示订阅、内容、PV、UV、互动和趋势数据，延迟 <= 5 分钟。
- AC-016：Given 个人频道连续 6 个月无内容更新且无有效互动，When 系统检测到不活跃，Then 通知频道主；再经 1 个月无改善可归档。
- AC-017：Given 平台运营对频道执行冻结、隐藏、限制推荐或关闭，When 操作完成，Then 审计日志记录操作人、原因、时间和影响范围。

## 15. 词汇表 (Glossary)

| 术语 | 定义 |
| --- | --- |
| 频道 | 内容社区中组织、聚合和发现内容的主题单元 |
| 系统官方频道 | 平台管理员创建和维护的官方频道，默认公开且具备更高公信力 |
| 个人频道 | 普通用户创建的频道，用于个人创作或兴趣聚合 |
| 组织频道 | 已认证组织创建的频道，归属于组织实体 |
| 频道主 | 频道最高管理者；个人频道通常为创建者，组织频道为指定组织管理员 |
| 内容编辑 | 频道内可协助管理内容但不可管理成员的角色 |
| 公开频道 | 所有人可搜索、浏览和订阅的频道 |
| 私有频道 | 仅成员可浏览内容且不可被公开搜索的频道 |
| 自由加入 | 用户点击后立即成为成员的加入方式 |
| 审核加入 | 用户提交申请后由管理员审批的加入方式 |
| 邀请加入 | 用户通过邀请链接或邀请码加入的方式 |
| 先审后发 | 内容提交后先进入频道待审区，通过后展示 |
| 频道回收站 | 保存被频道删除内容的恢复区域，默认保留 30 天 |
| 限制推荐 | 频道仍可被直接访问或订阅浏览，但降低或取消公共推荐流曝光 |
| 归档 | 频道停止公开运营但保留历史数据的生命周期状态 |
| 合并 | 将源频道内容、订阅和访问指向迁移至目标频道的流程 |
