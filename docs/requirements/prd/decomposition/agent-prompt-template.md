# 前端 PRD 生成 Agent Prompt 模板

## 你的任务

根据后端 PRD 和 openspec change 文档，生成前端 PRD 文档。

## 上下文（由调度器填充）

- **Change 目录**: `{change_name}`
- **PRD 文档**: `{prd_path}`
- **Epic 编号**: `{epic_number}`
- **域**: `{domain}`

## 输入文件（你必须读取）

1. `{prd_dir}/frontend-standards.md` - **前端规范与组件清单**（必读，用于组件选型）
2. `{changes_dir}/{change_name}/design.md` - 技术设计文档
3. `{changes_dir}/{change_name}/proposal.md` - 变更提案
4. `{changes_dir}/{change_name}/specs/` - 规格说明目录（读取所有 .md 文件）
5. `{prd_dir}/{prd_path}` - 后端 PRD 文档

## 输出要求

生成前端 PRD 文档，保存到：`{output_dir}/{epic_number}-frontend-prd.md`

## 前端 PRD 文档结构

```markdown
# {Epic 编号} - {功能名称} 前端 PRD

## 1. 概述
- 功能目标
- 用户价值
- 与后端 API 的关系

## 2. 用户故事
- 作为 [角色]，我想要 [功能]，以便 [价值]
- 验收标准

## 3. 页面设计

### 3.1 页面列表
| 页面名称 | 路由 | 描述 |
|---------|------|------|

### 3.2 页面详情
对每个页面：
- 页面目的
- UI 布局描述
- 交互流程
- 状态管理

## 4. 组件设计

### 4.1 通用组件
| 组件名称 | 用途 | Props |
|---------|------|-------|

### 4.2 业务组件
| 组件名称 | 用途 | 依赖 API |
|---------|------|----------|

## 5. API 对接

### 5.1 API 列表
| API 端点 | 方法 | 前端用途 | 请求/响应格式 |
|---------|------|---------|--------------|

### 5.2 数据流
- 请求发起时机
- 数据缓存策略
- 错误处理

## 6. 状态管理

### 6.1 全局状态
- 需要跨页面共享的数据

### 6.2 页面状态
- 页面级状态说明

## 7. 组件选型（必须基于 frontend-standards.md）

### 7.1 组件映射
| 功能需求 | 选用组件 | 组件路径 | 理由 |
|---------|---------|---------|------|
| 示例: 数据列表 | JVxeTable | src/components/jeecg/JVxeTable/ | 支持行编辑 |

### 7.2 复用说明
- 列出需要复用的现有组件
- 说明是否需要扩展现有组件
- 如需新组件，说明为何现有组件不满足

## 8. 交互设计

### 8.1 用户操作流程
- 操作步骤
- 反馈机制
- 异常处理

### 8.2 动画与过渡
- 页面切换动画
- 操作反馈动画

## 9. 响应式设计
- 移动端适配方案
- 断点说明

## 9. 性能要求
- 首屏加载时间
- 接口响应时间
- 优化策略

## 10. 测试要点
- 功能测试用例
- 边界条件
- 兼容性要求
```

## 执行步骤

1. 读取 `{changes_dir}/{change_name}/design.md`
2. 读取 `{changes_dir}/{change_name}/proposal.md`
3. 读取 `{changes_dir}/{change_name}/specs/` 下的所有文件
4. 读取 `{prd_dir}/{prd_path}`
5. 分析后端功能和 API 设计
6. 转换为前端视角的需求文档
7. 写入到 `{output_dir}/{epic_number}-frontend-prd.md`

## 质量检查

- [ ] 所有后端 API 都有对应的前端对接说明
- [ ] 用户故事完整且可验收
- [ ] 页面和组件设计覆盖所有功能
- [ ] 状态管理方案清晰
- [ ] 响应式设计考虑移动端
