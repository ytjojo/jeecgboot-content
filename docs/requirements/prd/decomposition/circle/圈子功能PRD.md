## 2. 背景与问题

### 2.1 当前现状

内容社区系统支持用户发布多种类型内容（文章、图文、视频、笔记、问答），但缺乏基于兴趣聚合的垂直社区能力。用户只能通过关注、话题标签等方式发现内容，缺少围绕特定主题进行深度交流和持续互动的空间。

### 2.2 用户痛点

- 用户希望围绕共同兴趣主题建立持续交流的社区，但现有系统缺乏聚合能力
- 内容创作者希望有专属的内容分发渠道，积累垂直领域粉丝
- 普通用户难以发现和参与特定主题的深度讨论

### 2.3 业务痛点

- 用户粘性不足，缺乏社区归属感
- 内容消费场景单一，主要依赖推荐流和关注流
- 无法满足用户对垂直领域深度交流的需求

### 2.4 为什么现在要做

圈子功能是社区系统的核心社交能力之一，能有效提升用户活跃度、留存率和内容消费深度。通过圈子，用户可以建立自己的兴趣社区，形成自组织的内容生态。

## 5. 用户与场景

### 5.1 目标用户

| 角色 | 描述 | 核心需求 |
|------|------|----------|
| 圈子创建者 | 创建和管理圈子的用户 | 创建圈子、管理成员、运营内容、查看数据 |
| 圈子版主/主持人 | 协助管理圈子的用户 | 管理内容、处理举报、协助运营 |
| 普通成员 | 加入圈子参与互动的用户 | 浏览内容、发布内容、参与讨论 |
| 游客 | 未加入圈子的用户 | 发现圈子、浏览公开内容、申请加入 |
| 系统管理员 | 平台运营人员 | 审核圈子、处理违规、查看全局数据 |

### 5.2 用户故事

- 作为一个 `内容创作者`，我希望创建自己的兴趣圈子，以便聚合垂直领域粉丝并建立专属社区
- 作为一个 `普通用户`，我希望加入感兴趣的圈子，以便与志同道合的用户交流特定主题内容
- 作为一个 `圈子版主`，我希望协助管理圈子内容和成员，以便维护圈子秩序和内容质量
- 作为一个 `系统管理员`，我希望审核和管理所有圈子，以便确保平台内容合规和社区健康

### 5.3 核心场景

#### 场景 1：创建圈子

- 前置条件：用户已登录，完成基础账号要求
- 主流程：
  1. 用户点击"创建圈子"按钮
  2. 填写圈子基础信息（名称、简介、图标、分类标签）
  3. 设置圈子属性（隐私类型、加入方式、加入条件、是否收费）
  4. 提交创建申请
  5. 系统校验名称唯一性、敏感词审核
  6. 创建成功，用户成为圈子创建者
- 备选流程：名称不合法时提示修改；敏感词触发时进入人工审核
- 异常流程：系统校验失败时提示具体原因
- 结束状态：圈子创建成功，进入圈子主页

#### 场景 2：加入圈子

- 前置条件：用户已登录
- 主流程：
  1. 用户通过搜索或推荐发现圈子
  2. 查看圈子信息和规则
  3. 点击"加入"按钮
  4. 若为公开圈子：直接加入成功
  5. 若为私有圈子：提交加入申请，等待审核
  6. 若需密码：输入密码验证后加入
- 备选流程：申请被拒绝时提示原因
- 异常流程：圈子已满员时提示无法加入
- 结束状态：用户成功加入圈子

#### 场景 3：圈子内发布内容

- 前置条件：用户已加入圈子，具备发帖权限
- 主流程：
  1. 用户在圈子内点击"发布"按钮
  2. 选择内容类型（文章、笔记、动态、视频等）
  3. 编写内容，添加话题标签
  4. 提交发布
  5. 系统根据圈子规则进行内容审核（如需）
  6. 内容发布成功，展示在圈子内容列表
- 备选流程：内容触发敏感词时进入审核队列
- 异常流程：用户被禁言时提示无法发布
- 结束状态：内容发布成功

#### 场景 4：管理圈子成员

- 前置条件：用户为圈子创建者或管理员
- 主流程：
  1. 进入圈子管理页面，查看成员列表
  2. 对成员进行操作（设置角色、禁言、移除）
  3. 处理加入申请（批准/拒绝）
  4. 查看成员活跃度数据
- 备选流程：批量操作成员
- 异常流程：权限不足时提示
- 结束状态：成员管理操作完成

- Owner：[待确认]
- 范围说明：包含圈子创建、成员管理、内容管理、访问控制、审核机制、数据统计、搜索推荐、成长激励等核心能力

#### FEATURE-001 圈子创建与设置

- Feature 目标：支持用户创建圈子并进行基础设置
- 优先级：Must
- 参与角色：圈子创建者、系统管理员
- 触发条件：用户点击"创建圈子"按钮
- 前置条件：用户已登录，完成基础账号要求
- 输入：圈子名称、简介、图标、分类标签、隐私类型、加入方式等
- 输出：创建成功的圈子
- 主逻辑：
  1. 用户填写圈子基础信息
  2. 系统校验名称唯一性（同类型内唯一或全局唯一，按平台策略配置）
  3. 系统进行敏感词审核
  4. 设置圈子属性（隐私、加入方式、条件等）
  5. 创建成功，用户成为圈子创建者
- 业务规则：
  - 圈子名称在同类型内唯一（或全局唯一，按平台策略配置）
  - 圈子图标和封面图需符合平台规范
  - 敏感词审核通过后方可创建
  - 创建后不可直接改类型（如需转换需走迁移流程）
- 权限与数据范围：
  - 创建者拥有圈子最高管理权限
  - 系统管理员可审核和管理所有圈子
- 状态变化：无 → 已创建
- 异常处理：
  - 名称重复：提示用户修改
  - 敏感词触发：进入人工审核队列
  - 校验失败：提示具体失败原因
- 边界条件：
  - 空状态：首次创建时的引导
  - 极限输入：名称长度限制、简介长度限制
  - 权限不足：未满足创建条件时的提示
- 后置条件：圈子创建成功，创建者成为圈子主

##### STORY-001 填写圈子基础信息

- 用户 / 角色：圈子创建者
- 用户价值：能够创建属于自己的兴趣圈子
- 前置条件：用户已登录
- 主流程：
  1. 用户进入创建圈子页面
  2. 填写圈子名称、简介、图标、封面图
  3. 选择分类标签
  4. 提交创建
- 验收标准：
  - Given 用户已登录
    When 用户填写完整的圈子基础信息并提交
    Then 系统创建圈子并返回成功提示
  - Given 用户输入的圈子名称已存在
    When 用户提交创建
    Then 系统提示"该圈子名称已存在，请修改"
  - Given 用户输入包含敏感词
    When 用户提交创建
    Then 系统提示"内容包含敏感词，需人工审核"
- 依赖：用户账号系统
- 边界与异常：
  - 空状态：创建表单的初始状态
  - 权限不足：未满足创建条件时的提示
  - 重复操作：重复提交时的防重处理

##### STORY-002 设置圈子隐私与访问控制

- 用户 / 角色：圈子创建者
- 用户价值：能够控制圈子的可见性和加入方式
- 前置条件：圈子已创建
- 主流程：
  1. 进入圈子设置页面
  2. 设置隐私类型（公开、私有）
  3. 设置加入方式（直接加入、申请加入、邀请加入）
  4. 设置加入条件（需要审核、无需审核）
  5. 设置搜索可见性（公开、私有）
  6. 设置密码保护（可选）
- 验收标准：
  - Given 圈子创建者进入设置页面
    When 设置圈子为"私有"类型
    Then 普通用户无法直接加入，需申请审核
  - Given 圈子设置为"密码保护"
    When 用户输入正确密码
    Then 用户可加入圈子
- 依赖：圈子创建
- 边界与异常：
  - 空状态：默认设置的展示
  - 重复操作：重复修改时的处理

#### FEATURE-002 圈子成员管理

- Feature 目标：提供完善的圈子成员管理体系，支持多角色和权限分配
- 优先级：Must
- 参与角色：圈子创建者、圈子版主、普通成员
- 触发条件：用户加入/退出圈子，管理员管理成员
- 前置条件：圈子已创建
- 输入：成员操作请求（加入、退出、角色变更、禁言、移除）
- 输出：成员状态变更结果
- 主逻辑：
  1. 成员加入：根据圈子设置进行审核或直接加入
  2. 角色管理：创建者可任命版主、设置角色权限
  3. 成员治理：禁言、移除、黑名单管理
  4. 成员退出：成员主动退出圈子
- 业务规则：
  - 角色体系：创建者 > 版主 > 普通成员
  - 创建者不可被移除或降级
  - 版主由创建者任命，可协助管理内容和成员
  - 禁言按时长设置，到期自动解除
  - 黑名单成员无法申请加入
- 权限与数据范围：
  - 创建者：所有管理权限
  - 版主：内容管理、成员管理（除角色变更）
  - 普通成员：浏览、发帖、互动
- 状态变化：未加入 → 已加入 → 版主/管理员 → 已退出/已移除
- 异常处理：
  - 圈子已满员：提示无法加入
  - 申请被拒绝：提示原因
  - 权限不足：提示无权操作
- 边界条件：
  - 空状态：无成员时的默认状态
  - 极限输入：成员数量上限
  - 超时：加入申请审核超时
  - 重复提交：重复加入申请
  - 权限不足：非管理员尝试管理操作
- 后置条件：成员状态变更完成

##### STORY-003 加入圈子

- 用户 / 角色：普通用户
- 用户价值：能够加入感兴趣的圈子参与讨论
- 前置条件：用户已登录
- 主流程：
  1. 用户通过搜索或推荐发现圈子
  2. 查看圈子信息
  3. 点击"加入"按钮
  4. 系统根据圈子设置处理加入请求
- 验收标准：
  - Given 圈子为公开类型，无需审核
    When 用户点击"加入"
    Then 用户直接加入成功，可浏览和发布内容
  - Given 圈子为私有类型，需要审核
    When 用户提交加入申请
    Then 系统提示"申请已提交，请等待审核"
  - Given 圈子设置密码保护
    When 用户输入正确密码
    Then 用户加入成功
- 依赖：圈子创建、访问控制设置
- 边界与异常：
  - 空状态：圈子无内容时的展示
  - 权限不足：被黑名单用户无法加入
  - 重复操作：已加入时的提示

##### STORY-004 管理成员角色与权限

- 用户 / 角色：圈子创建者
- 用户价值：能够管理圈子成员角色，分配管理权限
- 前置条件：用户为圈子创建者
- 主流程：
  1. 进入成员管理页面
  2. 选择目标成员
  3. 设置角色（版主、普通成员）
  4. 确认操作
- 验收标准：
  - Given 用户为圈子创建者
    When 将某成员设置为版主
    Then 该成员获得版主权限，可协助管理内容和成员
  - Given 用户尝试将创建者降级
    When 执行操作
    Then 系统提示"创建者角色不可变更"
- 依赖：成员加入
- 边界与异常：
  - 空状态：无成员可管理时的提示
  - 权限不足：非创建者尝试角色变更
  - 重复操作：重复设置同一角色

##### STORY-005 成员禁言与移除

- 用户 / 角色：圈子创建者、圈子版主
- 用户价值：能够处理违规成员，维护圈子秩序
- 前置条件：用户为圈子创建者或版主
- 主流程：
  1. 进入成员管理页面
  2. 选择目标成员
  3. 执行禁言或移除操作
  4. 确认操作
- 验收标准：
  - Given 用户为圈子创建者
    When 对某成员执行禁言操作，设置时长为 24 小时
    Then 该成员在 24 小时内无法发布内容和评论
  - Given 用户为圈子版主
    When 尝试移除其他版主
    Then 系统提示"权限不足，仅创建者可管理版主"
- 依赖：成员角色管理
- 边界与异常：
  - 空状态：无可操作成员时的提示
  - 权限不足：版主尝试管理同级或上级
  - 超时：禁言到期自动解除

#### FEATURE-003 圈子内容管理

- Feature 目标：支持圈子内内容的发布、分类、排序和管理
- 优先级：Must
- 参与角色：圈子成员、圈子版主、圈子创建者
- 触发条件：成员发布内容，管理员管理内容
- 前置条件：用户已加入圈子
- 输入：内容发布请求、内容管理操作
- 输出：内容发布结果、管理操作结果
- 主逻辑：
  1. 内容发布：成员在圈子内发布各类内容
  2. 内容分类：支持帖子分类管理
  3. 内容排序：按时间、热度等维度排序
  4. 内容管理：置顶、精华、下架、删除、移动
- 业务规则：
  - 内容类型支持：文章、笔记、动态、视频、问答
  - 置顶内容优先展示
  - 精华内容有特殊标识
  - 内容可移动到其他频道（需目标频道权限通过）
- 权限与数据范围：
  - 普通成员：发布内容、浏览内容
  - 版主：置顶、精华、下架、删除内容
  - 创建者：所有内容管理权限
- 状态变化：草稿 → 已发布 → 已置顶/已精华 → 已下架/已删除
- 异常处理：
  - 权限不足：非管理员尝试管理操作
  - 内容审核：触发敏感词时进入审核
  - 重复发布：防重处理
- 边界条件：
  - 空状态：圈子无内容时的展示
  - 极限输入：内容数量上限
  - 超时：内容审核超时
  - 权限不足：非成员尝试发布
- 后置条件：内容状态变更完成

##### STORY-006 在圈子内发布内容

- 用户 / 角色：圈子成员
- 用户价值：能够在圈子内发布内容参与讨论
- 前置条件：用户已加入圈子，具备发帖权限
- 主流程：
  1. 在圈子内点击"发布"按钮
  2. 选择内容类型
  3. 编写内容
  4. 添加话题标签
  5. 提交发布
- 验收标准：
  - Given 用户已加入圈子，未被禁言
    When 用户发布内容
    Then 内容发布成功，展示在圈子内容列表
  - Given 用户被禁言
    When 用户尝试发布内容
    Then 系统提示"您已被禁言，无法发布内容"
- 依赖：成员加入、内容发布系统
- 边界与异常：
  - 空状态：首次发布时的引导
  - 权限不足：被禁言用户的提示
  - 重复操作：重复发布时的防重

##### STORY-007 内容置顶与精华

## 12. 验收标准

### 12.1 基础功能验收

- Given 用户已登录
  When 用户填写完整的圈子基础信息并提交创建
  Then 系统创建圈子并返回成功提示

- Given 用户输入的圈子名称已存在
  When 用户提交创建
  Then 系统提示"该圈子名称已存在，请修改"

- Given 用户输入包含敏感词
  When 用户提交创建
  Then 系统提示"内容包含敏感词，需人工审核"

- Given 圈子设置为"私有"类型
  When 用户尝试加入
  Then 系统提示需申请审核

### 12.2 成员管理验收

- Given 圈子创建者将某成员设置为版主
  When 操作完成
  Then 该成员获得版主权限

- Given 圈子创建者尝试将创建者降级
  When 执行操作
  Then 系统提示"创建者角色不可变更"

- Given 用户被禁言
  When 用户尝试发布内容
  Then 系统提示"您已被禁言，无法发布内容"

### 12.3 内容管理验收

- Given 用户已加入圈子，未被禁言
  When 用户发布内容
  Then 内容发布成功，展示在圈子内容列表

- Given 圈子版主对某内容执行置顶操作
  When 操作完成
  Then 该内容在圈子内优先展示

### 12.4 审核机制验收

- Given 圈子设置为需要审核
  When 管理员批准某用户的加入申请
  Then 该用户成功加入圈子

- Given 某内容被举报
  When 管理员删除该内容
  Then 内容被移除，举报者收到处理结果通知

### 12.5 异常场景验收

- Given 圈子已满员
  When 用户尝试加入
  Then 系统提示"圈子已满员，无法加入"

- Given 用户被黑名单
  When 用户尝试加入
  Then 系统提示"无法加入该圈子"

- Given 系统异常
  When 用户执行操作
  Then 系统提示"操作失败，请稍后重试"
