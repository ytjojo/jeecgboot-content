## 1. 概述

### 产品目标

为内容社区建立统一的频道基础架构前端页面，覆盖三类频道（系统/个人/组织）的创建、信息编辑、审核管理、转让和删除完整流程。

- **一句话概述**: 前端需实现频道创建、管理、审核、转让和删除的完整页面与交互
- **要解决的问题**: 内容社区缺少统一的频道管理前端入口，平台管理员、普通用户和组织管理员无法以一致的界面规则创建和管理频道
- **期望达成的结果**: 用户端和后台管理端的频道相关页面全部可用，覆盖 EPIC-20 定义的全部用户旅程

### 目标用户与使用场景

| 角色 | 使用场景 | 入口 |
|------|----------|------|
| 平台管理员 | 后台创建/管理系统频道、审核频道、管理所有频道 | 后台管理菜单 > 内容管理 > 频道管理 |
| 普通用户 | 用户端创建/管理个人频道、查看审核状态 | 用户端菜单 > 我的频道 / 创建频道 |
| 组织管理员 | 用户端创建/管理组织频道 | 用户端菜单 > 组织频道 / 创建频道 |
| 频道主 | 编辑频道信息、发起转让、申请删除 | 频道管理页 > 设置 |

### 范围定义

#### 本期范围

- 用户端：频道创建页（三类频道）、我的频道列表页、频道管理页（编辑/转让/删除）
- 后台端：频道管理页（列表/筛选/详情）、系统频道创建、审核队列页

#### 非本期范围

- 频道隐私与加入规则（EPIC-21）
- 频道内容发布与治理（EPIC-22）
- 频道推荐与发现（EPIC-23）
- 频道数据统计与生命周期管理（EPIC-24）
- 付费订阅能力

## 3. 功能详细说明

### 3.1 系统频道创建（后台端）

#### 核心流程

1. 管理员进入后台 > 内容管理 > 频道管理
2. 点击"创建系统频道"按钮
3. 填写创建表单（名称、简介、图标、封面、所属官方分类、置顶权重）
4. 提交后系统直接创建频道，状态为 Active，默认公开、全站可见

#### 交互要求

- **入口**: 频道管理页面顶部操作区，按钮文案"创建系统频道"
- **表单字段**:
  - 频道名称: 必填，文本输入，最大 50 字符，实时校验格式
  - 频道简介: 必填，多行文本，最大 200 字符
  - 频道图标: 必填，图片上传，支持裁剪，建议尺寸 200x200px
  - 频道封面: 选填，图片上传，建议尺寸 1200x400px
  - 所属官方分类: 必填，下拉选择（复用平台分类体系）
  - 置顶权重: 选填，数字输入，默认 0，范围 0-9999
- **提交行为**: 点击"创建"后按钮显示 loading 并禁用，成功后跳转到频道详情页，失败时保留表单内容并显示错误信息
- **校验规则**: 名称不可为空；系统频道名称不参与用户频道唯一性校验，无需实时查重

#### 布局与结构

- 使用 Modal（弹窗）承载创建表单，宽度 640px
- 表单采用单列布局，字段间距统一
- 图标和封面上传区域水平排列
- 底部固定"取消"和"创建"按钮

#### 状态与边界

| 状态 | 表现 |
|------|------|
| 非管理员访问 | 页面显示无权限提示，隐藏创建按钮 |
| 提交中 | 按钮 loading，表单不可编辑 |
| 提交成功 | 关闭弹窗，列表刷新，顶部消息提示"系统频道创建成功" |
| 提交失败 | 保留表单内容，顶部显示错误原因 |

### 3.2 个人频道创建（用户端）

#### 核心流程

1. 用户进入"我的频道"页面
2. 点击"创建频道"按钮
3. 选择频道类型为"个人频道"
4. 填写表单（名称、简介、图标、隐私设置、归属分类）
5. 提交后系统校验基础账号要求、数量上限、名称唯一性
6. 校验通过后频道进入 PendingReview 状态，展示审核等待页

#### 交互要求

- **入口**: "我的频道"页面顶部，按钮文案"创建频道"
- **频道类型选择**: 使用分步向导（Steps 组件）实现，共两步：
  - **Step 1 — 选择类型**: 页面展示两个大卡片供用户选择，每个卡片包含图标、类型名称和简短说明：
    - 个人频道卡片：图标 + "个人频道" + "适合个人创作者，独立管理内容社区"
    - 组织频道卡片：图标 + "组织频道" + "需要组织认证，团队协作管理"
  - 用户点击卡片后高亮选中，底部"下一步"按钮激活
  - **Step 2 — 填写信息**: 根据所选类型动态渲染对应的表单字段
  - Steps 组件显示在页面顶部，标记当前进度（Step 1 选择类型 → Step 2 填写信息）
  - Step 2 页面顶部提供"返回上一步"链接，允许用户修改频道类型
- **表单字段**:
  - 频道名称: 必填，文本输入，最大 50 字符
    - 失焦时触发用户频道范围内的唯一性校验
    - 重名时输入框下方红色提示"该频道名称已被使用，请更换"
    - 校验中时输入框右侧显示 loading 图标
  - 频道简介: 必填，多行文本，最大 200 字符
  - 频道图标: 必填，图片上传，支持裁剪
  - 隐私设置: 必填，单选（公开/私有），默认"公开"
  - 归属分类: 必填，下拉选择
- **提交行为**: 点击"提交审核"后按钮显示 loading，成功后跳转到审核等待页
- **审核等待页**: 独立页面，包含以下内容：
  - 频道名称和当前状态（PendingReview）展示
  - 审核进度时间线组件（已提交 → 审核中 → 完成），当前停留在"已提交"步骤
  - 预计审核时间："预计 24 小时内完成"，并显示"已等待 X 小时"的动态计时
  - 引导提示："在等待期间，您可以先准备频道内容"
  - 帮助入口区域：FAQ 链接列表，包含"审核需要多长时间？"、"审核不通过怎么办？"
  - "返回我的频道"按钮（主操作）

#### 布局与结构

- 独立页面，非弹窗
- 页面顶部显示"创建频道"标题和面包屑导航
- 表单居中，最大宽度 720px
- 隐私设置使用 Radio.Group 水平排列
- 底部固定操作栏："取消"和"提交审核"按钮

#### 状态与边界

| 状态 | 表现 |
|------|------|
| 用户未完成账号验证 | 创建按钮禁用，hover 提示"请先完成账号验证" |
| 用户账号被冻结 | 创建按钮禁用，hover 提示"账号已被冻结，无法创建频道" |
| 已达个人频道上限（20个） | 创建按钮禁用，提示"已达个人频道创建上限" |
| 名称重名 | 输入框下方红色提示，提交按钮保持可用但点击后阻止提交 |
| 提交成功 | 跳转审核等待页，显示"频道已提交审核，预计 24 小时内完成" |
| 提交失败 | 保留表单内容，顶部显示具体错误原因 |

### 3.3 组织频道创建（用户端）

#### 核心流程

1. 组织管理员进入"创建频道"页面
2. 选择频道类型为"组织频道"
3. 系统自动展示绑定的组织信息（不可编辑）
4. 填写表单（名称、简介、图标、隐私设置、归属分类）
5. 提交后校验组织认证状态、名称唯一性
6. 校验通过后进入 PendingReview 状态

#### 交互要求

- 与个人频道创建共用同一页面，通过频道类型切换动态展示
- **组织信息展示区**: 页面顶部展示绑定的组织名称和认证状态，灰色背景不可编辑
- **表单字段**: 与个人频道一致，但不含隐私设置选项（组织频道默认公开）
- **差异校验**: 组织频道数量上限为 50 个

#### 状态与边界

| 状态 | 表现 |
|------|------|
| 组织未完成认证 | 选择"组织频道"后显示提示"请先完成组织认证"，表单不可填写 |
| 当前用户非组织管理员 | 选择"组织频道"后显示提示"您不是组织管理员，无法创建组织频道" |
| 已达组织频道上限（50个） | 提交时提示"已达组织频道创建上限" |

### 3.4 我的频道列表（用户端）

#### 核心流程

1. 用户进入"我的频道"页面
2. 查看自己创建的所有频道列表
3. 点击频道进入频道管理页

#### 交互要求

- **列表展示**: 使用 JVxeTable 表格，列包含：频道图标、频道名称、频道类型（Tag 标签）、审核状态（Tag 标签）、创建时间（支持排序）、操作
- **排序**: 创建时间列默认倒序排列，支持点击切换排序；审核状态列支持排序（PendingReview 和 Rejected 状态置顶，因为这些需要用户关注）
- **筛选区**: 频道类型下拉筛选、审核状态下拉筛选
- **操作列**:
  - Active 状态: "管理"按钮
  - PendingReview 状态: "查看详情"按钮
  - Rejected 状态: "重新提交"按钮
  - DeleteCooling 状态: "撤销删除"按钮
- **空状态**: 无频道时显示空状态插图和"创建你的第一个频道"引导按钮

#### 布局与结构

- 使用 Page 容器
- 顶部筛选区 + 数据表格（典型列表页模式）
- 表格支持分页，默认每页 10 条

### 3.5 频道信息编辑（用户端）

#### 频道管理页信息架构

频道管理页（`/content/channel/manage/:id`）是频道主最常访问的页面，采用 Tab 结构组织内容：

| Tab | 名称 | 内容 |
|-----|------|------|
| Tab 1 | 概览 | 频道基本信息摘要（名称、图标、类型、状态）+ 关键数据摘要（内容数、成员数等，本期可为占位） |
| Tab 2 | 编辑信息 | 频道信息编辑表单（即本节 3.5 的编辑功能） |
| Tab 3 | 设置 | 转让频道、删除频道等危险操作（放在深层级，避免误操作） |

- 页面顶部始终显示当前频道状态标签（Active / PendingReview / Rejected / DeleteCooling），使用 ChannelStatusTag 组件
- 当频道处于 DeleteCooling 状态时，页面顶部显示醒目通知条"频道正在删除冷静期中，剩余 X 天。[撤销删除]"
- 默认选中 Tab 1（概览）

#### 核心流程

1. 频道主进入频道管理页
2. 点击"编辑信息"
3. 修改字段后保存
4. 非关键字段立即生效；关键字段（名称、简介、图标、封面、分类）进入审核

#### 交互要求

- **编辑入口**: 频道管理页顶部操作区，按钮文案"编辑信息"
- **表单承载**: 使用 Drawer（抽屉）承载编辑表单，从右侧滑出，宽度 480px
- **关键字段标识**: 关键字段区域使用浅黄色/浅橙色背景色进行视觉区分，每个关键字段右侧始终可见橙色 Tag 标签"修改需审核"（不仅依赖 Tooltip）；用户首次修改关键字段时，弹出一次性提示："您正在修改需审核的字段，保存后将进入平台审核流程"
- **保存行为**:
  - 仅修改非关键字段: 保存后立即生效，顶部消息提示"保存成功"
  - 包含关键字段修改: 保存后进入审核，顶部消息提示"关键信息修改已提交审核，审核期间按原信息展示"
- **名称唯一性**: 修改名称时实时校验，排除当前频道自身
- **系统频道编辑**: 管理员编辑系统频道时无"修改需审核"标识，修改直接生效

#### 布局与结构

- Drawer 内部单列布局
- 关键字段区域用分割线和标题"以下信息修改需平台审核"与非关键字段区分
- 底部固定"取消"和"保存"按钮

#### 状态与边界

| 状态 | 表现 |
|------|------|
| 审核期间再次编辑 | 弹窗提示"当前有审核中的修改，请审核完成后再编辑" |
| 关键字段修改被拒绝 | 频道管理页顶部显示通知条"您的频道信息修改被拒绝，原因：xxx"，可点击"重新编辑" |
| 名称冲突 | 输入框下方红色提示 |

### 3.6 频道转让（用户端）

#### 核心流程

1. 频道主进入频道管理页 > 设置
2. 点击"转让频道"
3. 选择目标用户（搜索用户）
4. 确认转让信息
5. 目标用户收到转让请求
6. 目标用户确认后转让完成

#### 交互要求

- **入口**: 频道管理页 > 设置区域，按钮文案"转让频道"
- **转让弹窗**: 使用 Modal 承载，宽度 520px
  - 第一步: 搜索并选择目标用户（带用户头像和昵称的下拉搜索），搜索交互规格如下：
    - 输入 2 个字符后触发搜索，300ms 防抖
    - 搜索结果下拉列表显示用户头像和昵称
    - 无结果时显示"未找到匹配的用户"
    - 排除当前用户自身（不出现在搜索结果中）
    - 冻结用户在搜索结果中显示但置灰，hover 提示"该用户账号已被冻结"，不可选中
    - 组织频道场景下，搜索框 placeholder 改为"搜索组织内管理员"，搜索范围限定为同组织管理员
  - 第二步: 确认转让信息（显示当前频道名、目标用户名、转让后原频道主降为管理员）
- **二次确认**: 最终确认弹窗，文案"确认将频道 [频道名] 转让给 [用户名]？转让后您将降为管理员，此操作不可撤销。"
- **目标用户视角**: 目标用户在"消息通知"中收到转让请求，可"接受"或"拒绝"
- **转让记录**: 频道管理页设置区域展示转让历史（发起时间、目标用户、状态、结果）

#### 状态与边界

| 状态 | 表现 |
|------|------|
| 系统频道 | "转让频道"按钮不显示 |
| 组织频道 | 目标用户选择范围限定为同组织管理员，提示"组织频道仅可在组织管理员间转移" |
| 转让请求已存在 | 按钮禁用，提示"已有进行中的转让请求" |
| 目标用户拒绝 | 消息通知显示"转让被拒绝"，频道所有权不变 |
| 转让超时（7天） | 消息通知显示"转让请求已过期"，频道所有权不变 |

### 3.7 频道删除（用户端）

#### 核心流程

1. 频道主进入频道管理页 > 设置
2. 点击"删除频道"
3. 系统校验前置条件（内容清理、订阅结清）
4. 条件满足后进入二次确认
5. 确认后频道进入 7 天冷静期
6. 冷静期内可撤销

#### 交互要求

- **入口**: 频道管理页 > 设置区域，红色危险按钮"删除频道"
- **前置条件校验**: 点击"删除频道"按钮后，按钮立即显示 loading 状态并禁用（防止重复点击），同时调用接口校验前置条件。接口返回后：
  - 满足条件 → 按钮恢复，弹出二次确认弹窗
  - 不满足条件 → 按钮恢复，弹窗显示阻塞原因列表（如"频道内仍有 3 篇未清理内容"、"存在未了结的付费订阅"），无"继续删除"按钮
  - 接口失败 → 按钮恢复，顶部消息提示"校验失败，请重试"
- **二次确认弹窗**: Modal，宽度 480px
  - 标题: "确认删除频道"
  - 正文: "删除后频道将进入 7 天冷静期，期间可撤销。冷静期结束后频道将永久删除，不可恢复。"
  - 确认操作: 需输入频道名称进行确认（输入框 + 频道名对照）
  - 按钮: "取消"（默认）和"确认删除"（红色危险按钮）
- **冷静期展示**: 删除后频道管理页顶部显示醒目通知条"频道正在删除冷静期中，剩余 X 天。[撤销删除]"
- **撤销删除**: 点击"撤销删除"后弹窗确认，确认后频道恢复为 Active 状态

#### 状态与边界

| 状态 | 表现 |
|------|------|
| 系统频道 | "删除频道"按钮不显示 |
| 前置条件不满足 | 弹窗显示阻塞原因列表，无"继续删除"按钮 |
| 组织频道 | 确认弹窗提示"需要组织最高管理员确认"，发起后等待最高管理员审批 |
| 冷静期到期 | 定时任务处理，用户端不再展示该频道的管理入口 |

### 3.8 后台频道管理（后台端）

#### 核心流程

1. 管理员进入后台 > 内容管理 > 频道管理
2. 查看所有频道列表，支持多维度筛选
3. 点击频道查看详情或执行管理操作

#### 交互要求

- **列表展示**: 使用 JVxeTable 表格
  - 列: 频道图标、频道名称、频道类型（Tag）、审核状态（Tag）、归属（用户/组织名）、分类、置顶权重（仅系统频道）、创建时间、操作
- **筛选区**: 频道类型、审核状态、归属分类、创建时间范围
- **搜索**: 频道名称模糊搜索
- **操作列**:
  - 所有状态: "查看详情"
  - 系统频道: "编辑"按钮
  - DeleteCooling 状态: "强制删除"按钮（需二次确认）
- **批量操作**: 支持批量审核（选中多条 PendingReview 记录后批量通过/拒绝）

#### 布局与结构

- 典型列表页模式：Page > 筛选区 + 操作按钮 + 数据表格
- 筛选区默认展开，支持收起
- 表格支持列排序（创建时间、置顶权重）
- 分页默认每页 20 条

### 3.9 审核队列（后台端）

#### 核心流程

1. 管理员进入后台 > 内容管理 > 审核队列
2. 查看所有待审核频道
3. 点击频道进入审核详情
4. 执行审核操作（通过/拒绝/退回修改）

#### 交互要求

- **列表展示**: 使用 Table 组件，仅显示 PendingReview 状态的频道
  - 列: 频道名称、频道类型、提交人、提交时间、等待时长（超过 24h 标红）、操作
- **审核详情**: 使用 Drawer 承载，宽度 560px
  - 展示频道完整信息（名称、简介、图标、封面、分类、隐私设置）
  - 如为编辑触发的审核，展示修改前后对比（diff 视图），展示规则如下：
    - **文本字段**（名称、简介等）：左右并排对比，删除内容用红色删除线标注，新增内容用绿色背景高亮
    - **图片字段**（图标、封面）：左右并排展示，左侧标注"当前"（旧图），右侧标注"待审核"（新图）
    - **未修改的字段**：折叠显示或灰显，让审核员聚焦于变更内容
    - 对比区域顶部显示"共修改 N 个字段"的摘要信息
  - 底部操作区: "通过"（绿色）、"拒绝"（红色）、"退回修改"（橙色）
- **审核操作**:
  - 通过: 无需填写原因，点击后直接通过
  - 拒绝: 必须填写拒绝原因（多行文本，必填），点击后确认
  - 退回修改: 必须填写修改建议（多行文本，必填），点击后确认
- **审核结果反馈**: 操作成功后关闭 Drawer，列表刷新，顶部消息提示"审核完成"

#### 布局与结构

- 典型列表页模式
- 顶部显示待审核数量统计（如"待审核频道：12 个"）
- 超过 24 小时未处理的记录行背景色高亮

#### 状态与边界

| 状态 | 表现 |
|------|------|
| 无待审核频道 | 显示空状态"暂无待审核频道" |
| 审核超时（>24h） | 记录行标红，等待时长列显示"已超时" |
| 并发审核 | 两人同时打开同一审核详情时，后操作者提交时提示"该频道已被审核" |

## 9. 响应式设计

### 桌面端（>= 1024px）

- 列表页：筛选区 + 表格，左右留白
- 创建/编辑：弹窗或抽屉，表单单列
- 审核详情：右侧抽屉，信息分栏展示

### 移动端（< 768px）

#### 触摸交互规格

- **触摸目标尺寸**: 所有可点击元素最小 44x44px，按钮间距不小于 8px
- **卡片列表操作**: 频道列表卡片操作使用底部 ActionSheet（从底部滑出的操作菜单），而非内嵌按钮，节省卡片空间
- **图片上传**: 支持相册选择 + 拍照上传，裁剪使用移动端友好的全屏裁剪组件
- **列表交互**: 优先使用下拉刷新 + 分页（而非无限滚动，因为频道列表有明确的分页需求，上限 20 个）

#### 页面适配

| 页面 | 移动端适配 |
|------|-----------|
| 频道列表 | 表格转为卡片列表，每条卡片显示图标、名称、状态 Tag，点击卡片弹出 ActionSheet 显示操作项 |
| 创建频道 | 弹窗转为全屏页面，表单字段垂直堆叠 |
| 编辑频道 | 抽屉转为全屏页面 |
| 审核队列 | 表格转为卡片列表，审核操作按钮固定在底部 |
| 筛选区 | 折叠为"筛选"按钮，点击展开筛选面板 |
| 导航 | 左侧菜单收起为汉堡菜单 |

### 平板端（768px - 1023px）

- 列表保持表格形式，减少列数，次要列隐藏
- 弹窗宽度自适应 90vw
- 抽屉宽度自适应 80vw

## 10. 性能要求

| 指标 | 目标 | 说明 |
|------|------|------|

- 路由懒加载：频道相关页面使用动态 import
- 图片压缩：上传前客户端压缩，减少上传时间
- 接口缓存：频道类型、分类等静态数据缓存到 store
- 防抖处理：名称校验接口 300ms 防抖
- 骨架屏：列表页和详情页使用骨架屏提升感知速度
