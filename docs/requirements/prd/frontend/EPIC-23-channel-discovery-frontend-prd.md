# EPIC-23 频道分类、推荐与发现 - 前端 PRD

> 变更 ID: channel-discovery | EPIC-23 | 优先级: 中 | 预计周期: 4 周

## 1. 概述

### 1.1 需求目标

建立频道发现的完整前端能力，让用户通过推荐、排行榜、编辑精选、分类浏览和搜索等多种入口高效发现频道，同时为平台运营提供分类管理、精选配置等后台工具。

**要解决的问题**: 频道数量增长后，用户缺少高效发现机制，只能通过直接链接或订阅关系找到频道；新用户冷启动体验差，优质频道缺乏曝光渠道。

**期望达成的结果**: 用户可从多个入口发现频道；搜索 P99 响应 <= 200ms；推荐频道点击率 > 15%；搜索满意度 >= 85%。

### 1.2 目标用户与使用场景

| 角色 | 使用场景 | 入口 |
|------|---------|------|
| 内容消费者 | 发现感兴趣的频道并订阅 | 频道发现页、首页推荐模块、搜索入口 |
| 频道管理员 | 管理频道内标签，提升内容组织效率 | 频道管理后台 |
| 平台运营 | 维护分类体系、配置精选频道、监控推荐质量 | 运营后台 |

### 1.3 范围定义

**本期范围**:
- 频道发现聚合页（推荐 + 排行榜 + 精选 + 分类入口 + 搜索入口）
- 分类浏览页
- 频道搜索结果页
- 排行榜页
- 运营后台：分类管理页
- 运营后台：编辑精选管理页
- 频道管理后台：标签管理模块
- 频道创建/编辑表单中的分类选择组件

**非本期范围**:
- 推荐模型训练与算法迭代
- 搜索基础设施迁移到 Elasticsearch
- 数据统计看板（EPIC-24）
- 频道创建流程（EPIC-20）
- 成员管理（EPIC-21）

## 1A. 术语定义

| 术语 | 英文 | 定义 | 示例值 |
|------|------|------|--------|
| 频道类型 | channelType | 频道的归属主体类型 | `personal`（个人）、`organization`（组织）、`system`（系统） |
| 频道状态 | channelStatus | 频道在编辑精选中的有效状态 | `active`（生效中）、`expired`（已过期）、`abnormal`（状态异常） |
| 频道可见性 | channelVisibility | 频道对外可见程度，由后端 `ChannelVisibilityService` 管控 | 公开、私有、隐藏、冻结、限制公开曝光、未通过审核 |
| 分类 | category | 平台官方维护的多级分类体系，频道归属分类以提升可发现性 | 技术/前端/Vue |
| 标签 | tag | 频道管理员自定义的内容标签，用于频道内内容细分 | TypeScript、入门教程 |
| 精选 | editorialPick | 运营手动标记的优质频道，附带推荐语和有效期 | - |
| 推荐理由 | reason | 推荐系统返回的推荐依据说明 | "与您订阅的频道相似" |
| 匹配原因 | matchReason | 搜索结果的匹配依据 | 名称匹配、标签匹配、描述匹配 |

> **注意**: 文档中"频道类型"特指 channelType（个人/组织/系统），与"频道状态"（active/expired/abnormal）和"频道可见性"（公开/私有/隐藏等）是三个独立维度。筛选条件中如需同时使用，应作为独立筛选项呈现。

## 3. 页面设计

### 3.1 频道发现页（首页入口）

**用途**: 频道发现的聚合入口，集中展示推荐、排行榜、精选和搜索入口。

**页面结构** (从上到下):

| 区块 | 内容 | 展示规则 |
|------|------|---------|
| 搜索入口区 | 搜索框 + 快捷筛选标签（热门分类） | 始终可见，置于页面顶部 |
| 推荐频道区 | 推荐频道卡片列表（横向滚动或网格） | 登录用户展示个性化推荐；未登录展示冷启动结果 |
| 排行榜入口区 | 热门榜/新晋榜/系统榜 Tab 切换，每榜展示 Top 5 | 始终可见 |
| 编辑精选区 | 精选频道卡片 + 运营推荐语 | 有精选数据时展示 |
| 分类入口区 | 一级分类卡片网格，点击进入分类浏览页 | 始终可见 |

**入口与跳转方式**:

| 入口位置 | 跳转方式 | 用户流说明 |
|---------|---------|-----------|
| 频道模块顶部导航 | 路由跳转至 `/channel/discovery` | 点击"发现"导航项，完整加载频道发现页 |
| 首页频道推荐区域 | 路由跳转至 `/channel/discovery` | 首页展示推荐频道摘要（Top 3-6），点击"查看更多"或"频道发现"跳转至完整发现页 |
| 底部 Tab 切换 | 路由跳转至 `/channel/discovery` | 移动端底部 Tab 栏的"发现"Tab，点击切换到发现页 |

> **注意**: 首页频道推荐区域为嵌入式摘要模块，仅展示推荐频道卡片（不含搜索、排行榜、精选、分类入口），点击后跳转至完整发现页。发现页为独立路由页面，非首页内嵌。

**频道推荐卡片信息层级**:
- 第一层: 频道图标 + 频道名称 + 频道类型标签
- 第二层: 主分类 + 简介（限 2 行）
- 第三层: 订阅数 + 推荐理由
- 操作区: 订阅/已订阅按钮 + "不感兴趣"按钮（... 菜单内）

### 3.2 分类浏览页

**用途**: 按官方分类树浏览全部可发现频道。

**页面结构**:

| 区块 | 内容 |
|------|------|
| 面包屑导航 | 显示当前分类层级路径，可点击返回上级 |
| 分类导航区 | 左侧分类树（PC）/ 顶部分类 Tab（移动端） |
| 筛选排序区 | 频道类型筛选（个人/组织/系统）+ 排序方式（订阅数/活跃度/创建时间） |
| 频道列表区 | 频道卡片网格，每页 20 条，滚动加载更多 |
| 空状态区 | 无结果时展示返回上级分类、浏览热门频道或搜索入口 |

**交互流程**:
1. 用户进入分类浏览页，默认展示所有一级分类
2. 点击一级分类 -> 展示该分类及其子分类下的频道
3. 点击子分类 -> 展示该子分类下的频道
4. 支持排序和筛选，结果实时刷新
5. 超过 20 条结果时，滚动触发加载更多

### 3.3 搜索结果页

**用途**: 关键词搜索频道，支持多条件筛选。

**页面结构**:

| 区块 | 内容 |
|------|------|
| 搜索框区 | 搜索输入框（带清除按钮）+ 搜索按钮 |
| 筛选条件区 | 频道类型筛选 + 分类筛选 + 活跃度排序切换 |
| 结果统计区 | "共找到 N 个频道" + 当前筛选条件标签 |
| 结果列表区 | 搜索结果卡片列表，每页 20 条 |
| 空状态区 | 无结果时展示清除筛选、改写关键词和浏览分类入口 |
| 反馈入口 | "结果有帮助"反馈按钮（用于满意度统计） |

**搜索结果卡片信息层级**:
- 第一层: 频道图标 + 频道名称（关键词高亮）+ 频道类型标签
- 第二层: 主分类 + 简介（限 2 行）
- 第三层: 订阅数 + 匹配原因（如"名称匹配""标签匹配"）
- 操作区: 订阅/已订阅按钮

### 3.4 排行榜页

**用途**: 展示热门、新晋、系统频道排行榜。

**页面结构**:

| 区块 | 内容 |
|------|------|
| 榜单切换区 | Tab 切换：热门频道榜 / 新晋频道榜 / 系统频道榜 |
| 维度切换区 | 热门榜支持日/周/月维度切换（下拉或 Tab）；新晋榜固定为"近 30 天"维度，不展示维度切换；系统榜不展示维度切换（全量展示） |
| 榜单列表区 | 带排名序号的频道列表，每页 20 条 |
| 榜单信息区 | 榜单更新时间 + 口径说明入口 |

**热门榜列表项信息**:
- 排名序号（1-3 带特殊样式）
- 频道图标 + 频道名称 + 频道类型
- 主分类 + 订阅数
- 活跃度评分

### 3.5 运营后台 - 分类管理页

**用途**: 平台运营维护多级树形分类体系。

**页面结构**:

| 区块 | 内容 |
|------|------|
| 操作栏 | 新增一级分类按钮 + 搜索分类名称 |
| 分类树区 | 左侧可展开的分类树，展示层级关系 |
| 分类详情区 | 右侧展示选中分类的名称、层级、排序、状态、关联频道数 |
| 编辑操作区 | 编辑名称 / 新增子分类 / 启用停用 / 调整排序 |

**核心交互**:
- 点击分类树节点 -> 右侧展示详情
- 新增分类: 弹窗表单，含名称、父级选择、排序号
- 编辑分类: 弹窗表单，含名称、排序号
- 停用分类: 二次确认弹窗，若有关联频道则展示影响范围并要求选择"保留历史归属"或"迁移到新分类"
- 删除分类: 仅无关联频道时允许删除，二次确认

### 3.6 运营后台 - 编辑精选管理页

**用途**: 平台运营标记和管理编辑精选频道。

**页面结构**:

| 区块 | 内容 |
|------|------|
| 操作栏 | 添加精选频道按钮 |
| 精选列表区 | 当前有效的精选频道列表（表格或卡片） |
| 状态筛选 | 全部 / 生效中 / 已过期 / 状态异常 |

**列表项信息**: 频道名称、频道图标、推荐语、有效期、状态、操作人、操作时间

**核心交互**:
- 添加精选: 弹窗表单，含频道搜索选择、推荐语输入（限 200 字）、有效期选择
- 编辑精选: 弹窗表单，修改推荐语和有效期
- 取消精选: 二次确认弹窗
- 状态异常提示: 频道变为私有/隐藏/冻结/限制公开曝光/未通过审核时，列表中高亮提示（与 11.2 节可见性测试用例一致）

### 3.7 频道管理后台 - 标签管理模块

**用途**: 频道管理员管理频道内的自定义标签。

**页面结构**:

| 区块 | 内容 |
|------|------|
| 操作区 | 新增标签输入框 + 添加按钮 |
| 标签列表区 | 标签名称 + 使用该标签的内容数量 + 编辑/删除操作 |

**核心交互**:
- 新增标签: 内联输入框，输入后点击添加；校验空名称、重复名称、超长（>20 字符）
- 编辑标签: 点击标签名称进入编辑态，回车或失焦保存
- 删除标签: 二次确认弹窗，提示"该标签已被 N 条内容使用，删除后不再出现在新增选择中"

### 3.8 频道创建/编辑 - 分类选择组件

**用途**: 频道创建和编辑时选择主分类和副分类。

**组件结构**:
- 主分类: 必填，TreeSelect 组件，展示完整分类树
- 副分类: 可选，最多 3 个，MultiSelect 或 Tag 输入，不能与主分类相同
- 系统频道: 可选择特殊分类（官方活动/平台公告/新手引导/系统精选）

## 4. 组件设计

### 4.1 ChannelCard（频道卡片）

**用途**: 在推荐、搜索、分类浏览、排行榜中统一展示频道信息。

**Props**:
| 属性 | 类型 | 说明 |
|------|------|------|
| channel | ChannelInfo | 频道数据对象 |
| mode | 'recommend' \| 'search' \| 'browse' \| 'ranking' | 展示模式，控制信息层级 |
| showReason | boolean | 是否展示推荐理由或匹配原因 |
| showNotInterested | boolean | 是否展示"不感兴趣"操作 |
| showRank | boolean | 是否展示排名序号 |
| rank | number | 排名序号 |

**展示规则**:
- `recommend` 模式: 展示完整信息（图标、名称、类型、分类、简介、订阅数、推荐理由、订阅状态、不感兴趣按钮）
- `search` 模式: 展示图标、名称（高亮关键词）、类型、分类、简介、订阅数、匹配原因
- `browse` 模式: 展示图标、名称、类型、分类、订阅数、简介
- `ranking` 模式: 展示排名序号、图标、名称、类型、分类、订阅数、活跃度

### 4.2 CategoryTreeNav（分类树导航）

**用途**: 在分类浏览页和分类选择中展示分类树。

**Props**:
| 属性 | 类型 | 说明 |
|------|------|------|
| categories | CategoryTree[] | 分类树数据 |
| selectedKey | string | 当前选中分类 ID |
| mode | 'browse' \| 'select' \| 'manage' | 浏览/选择/管理模式 |
| maxLevel | number | 最大展示层级，默认 4 |

**交互规则**:
- 浏览模式: 点击分类展开子分类并加载对应频道列表
- 选择模式: 单选或父子联动选择
- 管理模式: 支持拖拽排序、右键菜单（新增子分类/编辑/停用）

### 4.3 SearchBar（搜索栏）

**用途**: 频道搜索输入组件。

**Props**:
| 属性 | 类型 | 说明 |
|------|------|------|
| value | string | 搜索关键词 |
| placeholder | string | 占位文案 |
| hotCategories | Category[] | 热门分类快捷标签 |

**交互规则**:
- 输入防抖 300ms 后触发搜索
- 回车键或点击搜索按钮执行搜索
- 展示搜索历史（最近 10 条），可清除
- 空输入时展示热门搜索词

### 4.4 FilterPanel（筛选面板）

**用途**: 搜索和分类浏览的筛选条件组件。

**Props**:
| 属性 | 类型 | 说明 |
|------|------|------|
| filters | FilterConfig[] | 筛选条件配置 |
| values | Record<string, any> | 当前筛选值 |
| collapsible | boolean | 是否可折叠（移动端） |

**筛选条件配置**:
- 频道类型: 多选（个人/组织/系统）
- 分类: 树形选择
- 排序: 单选（相关性/活跃度/订阅数/创建时间）

### 4.5 RankingList（排行榜列表）

**用途**: 排行榜页的带排名列表。

**Props**:
| 属性 | 类型 | 说明 |
|------|------|------|
| data | RankingItem[] | 排行数据 |
| dimension | 'daily' \| 'weekly' \| 'monthly' | 时间维度 |
| loading | boolean | 加载状态 |

**排名样式**:
- 第 1 名: 金色背景/图标
- 第 2 名: 银色背景/图标
- 第 3 名: 铜色背景/图标
- 第 4-10 名: 普通样式但突出显示
- 第 11+ 名: 普通列表样式

### 4.6 CategoryManageTree（分类管理树）

**用途**: 运营后台分类管理的可编辑分类树。

**Props**:
| 属性 | 类型 | 说明 |
|------|------|------|
| categories | CategoryTree[] | 分类树数据 |
| onSelect | (category) => void | 选中回调 |
| onAdd | (parentId) => void | 新增子分类回调 |
| onEdit | (category) => void | 编辑回调 |
| onToggleStatus | (category) => void | 启停回调 |

```typescript
// useChannelDiscoveryStore
interface ChannelDiscoveryState {
  recommendations: ChannelRecommendationVO[];   // 推荐频道列表

interface ChannelSearchState {

  page: number;                                 // 当前页码
  loading: boolean;
  searchHistory: string[];                      // 搜索历史
}
```

### 6.2.1 缓存失效规则

| Store | 缓存数据 | 缓存时长 | 失效条件 | 刷新方式 |
|-------|---------|---------|---------|---------|
| `useChannelDiscoveryStore` | 推荐频道列表 | 5 分钟 | 1) 缓存过期；2) 用户反馈"不感兴趣"；3) 用户订阅/取消订阅 | 过期后下次访问自动刷新；操作后立即刷新对应数据 |
| `useChannelDiscoveryStore` | 排行榜数据 | 5 分钟 | 缓存过期 | 过期后下次访问自动刷新 |
| `useChannelDiscoveryStore` | 编辑精选列表 | 5 分钟 | 缓存过期 | 过期后下次访问自动刷新 |
| `useChannelCategoryStore` | 分类树 | 会话级 | 1) 运营后台新增/编辑/停用/删除分类；2) 用户刷新页面 | 写操作后立即重新拉取完整分类树；页面刷新时重新加载 |
| `useChannelSearchStore` | 搜索结果 | 不缓存 | 每次搜索重新请求 | 关键词或筛选条件变更时重新搜索 |
| `useChannelSearchStore` | 搜索历史 | 持久化（localStorage） | 用户手动清除 | 写入 localStorage，页面加载时恢复 |

> **注意**: 缓存过期判断基于 `lastFetchTime` 字段，计算方式为 `Date.now() - lastFetchTime > CACHE_TTL`。`CACHE_TTL` 常量定义为 `5 * 60 * 1000`（5 分钟）。

### 10.2 前端优化策略

| 策略 | 实施方式 |
|------|---------|
| 数据缓存 | 推荐数据 5 分钟缓存（lastFetchTime 判断）；分类树缓存到 Store，变更后刷新 |

| 图片懒加载 | 频道图标使用懒加载 |
| 虚拟滚动 | 排行榜列表超过 100 条时启用虚拟滚动 |
| 接口合并 | 发现页使用聚合 API `/content/channel/discovery/home` 一次性获取推荐+榜单+精选 |

## 12. 待确认问题 / 默认假设

### 12.1 待确认问题

| 问题 | 影响范围 | 建议处理 |
|------|---------|---------|
| 推荐模块是否需要区分首页推荐、频道发现页推荐和频道详情页相关推荐的不同口径？ | 推荐卡片组件复用性 | 先统一口径，通过 `mode` prop 区分展示样式 |
| 编辑精选是否需要按地区、用户身份或频道类型差异化展示？ | 精选管理页筛选条件 | 先全站统一，差异化进入后续版本 |
| 搜索建议（输入时的下拉建议）是否需要实现？ | 搜索框组件复杂度 | 本期先做关键词搜索，搜索建议作为 P2 延伸 |
| 排行榜是否需要支持无限滚动或仅分页？ | 排行榜列表组件 | 建议分页，每页 20 条 |
| 频道卡片的"不感兴趣"操作放在 ... 菜单中还是直接展示为图标按钮？ | 卡片信息密度 | PC 端放 ... 菜单，移动端长按卡片弹出操作菜单 |

### 12.2 默认假设

| 假设 | 说明 |
|------|------|
| 频道发现页作为独立页面 | 存在独立的频道发现页路由，而非嵌入在首页中 |
| 分类树数据量有限 | 平台分类最多 4 级（产品决策，与后端 `CategoryTreeVO.level` 约束一致），预估总分类数 < 500，前端可一次性加载完整分类树。前端新增分类时需校验层级不超过 4 级，若后端返回 level > 4 的数据应视为异常并提示用户 |
| 搜索使用数据库能力 | 本期搜索基于 MySQL FULLTEXT，前端无需处理搜索基础设施差异 |
| 推荐结果服务端已过滤可见性 | 前端展示推荐结果时无需额外过滤，后端 `ChannelVisibilityService` 已完成过滤 |
| 运营后台与用户端共用路由体系 | 运营后台页面在现有菜单和权限体系下配置，不需要独立的路由模块 |
| 标签管理嵌入频道管理后台 | 标签管理作为频道管理的一个 Tab 或子页面，不需要独立路由 |
| 移动端使用同一套代码 | 通过响应式适配而非独立移动端页面实现 |
