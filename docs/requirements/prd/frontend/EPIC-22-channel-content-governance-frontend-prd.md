# 频道内容发布与治理 - 前端 PRD

> **史诗ID**: EPIC-22
> **域**: channel（频道域）
> **变更ID**: channel-22-content-governance-frontend
> **版本**: 1.0
> **前置依赖**: EPIC-20
> **日期**: 2026-06-02

## 1. 概述

### 需求目标

- **需求名称**: 频道内容发布与治理（EPIC-22）
- **一句话概述**: 建立频道内内容发布、审核、治理和公告的完整前端能力
- **要解决的问题**: 频道缺少发布权限分层、审核流程、内容治理策略和公告机制，无法控制内容质量
- **期望达成的结果**: 频道从"可加入的社区单元"升级为"可持续产出、可治理的主题空间"

### 目标用户与使用场景

| 角色 | 说明 | 主要场景 |
|------|------|----------|
| 内容创作者 | 发布内容到频道的用户 | 选择频道发布、定时发布、查看发布结果 |
| 频道主 | 频道所有者 | 配置发布权限、管理公告、治理内容 |
| 频道管理员 | 被授权的频道管理者 | 审核投稿、治理内容、编辑协助 |
| 内容编辑 | 具备内容管理权限的成员 | 审核投稿、编辑协助 |
| 平台运营 | 系统频道管理者 | 添加内容到系统频道 |
| 普通成员 | 频道订阅者 | 发布内容（受限权限）、查看公告 |

- **使用场景**: 桌面端 + 移动端跨端
- **入口路径**: 内容发布页 → 频道选择；频道管理后台 → 待审区/内容管理/公告管理/回收站

### 范围定义

#### 本期范围

1. 频道选择组件（发布时选择目标频道）
2. 发布权限模型配置页（四种模式切换）
3. 待审区管理页（审核列表、逐条/批量审核）
4. 定时发布设置（时间选择器、状态展示）
5. 发布限额配置（每小时/每日上限、字数下限）
6. 内容治理页（置顶、精华、删除、移出、编辑协助）
7. 频道回收站页（已删除内容列表、恢复操作）
8. 治理日志页（操作记录查询）
9. 频道公告管理（发布、编辑、删除、预览）
10. 已发布内容添加到频道（三种场景入口）

#### 非本期范围

- 频道创建与所有权管理（EPIC-20）
- 频道隐私与成员管理（EPIC-21）
- 推荐发现（EPIC-23）
- 频道数据统计（EPIC-24）
- 平台级内容安全审核策略
- 完整创作器能力

## 2. 用户故事

### 旅程 1: 创作者发布内容到频道

**故事 22.1.1 - 选择频道发布内容**

作为内容创作者，我希望在发布文章、图文帖子、视频、笔记、问答问题、问答答案时选择目标频道，以便将内容精准投放到目标受众。

> **说明**: 覆盖后端 EPIC-22 定义的 6 种内容类型。其中"问答答案"的频道选择跟随其所属问答问题所在的频道，无需独立选择（详见 3.1 节交互规则补充）。

**故事 22.1.2 - 发布权限模型配置**

作为频道主，我希望配置频道的发布权限模型（仅管理员可发布/所有成员可发布/公开投稿/先审后发），以便控制谁可以在频道中发布内容。

**故事 22.1.3 - 定时发布**

作为内容创作者，我希望为内容设定未来发布时间，以便在最佳时间触达目标读者。

**定时发布业务边界规则**:
1. **定时发布期间用户被禁言**: 到达发布时间时若用户已被禁言，定时发布任务自动取消，前端通过站内消息通知用户"定时发布已取消：您在频道 {channelName} 的定时发布因禁言被取消"
2. **定时发布期间频道权限变更**: 若频道发布权限从"所有成员可发布"变更为"仅管理员可发布"，普通成员的定时发布任务在到达发布时间时由后端校验权限，若不满足则取消并通知用户
3. **定时发布编辑/取消**: 定时发布支持编辑时间和取消操作。前端在"我的定时发布"列表中每条任务展示"编辑时间"和"取消发布"按钮，取消需二次确认
4. **定时发布任务管理**: 前端增加"我的定时发布"入口（内容发布页右上角或个人中心），展示当前用户所有待执行的定时发布任务列表（标题、目标频道、计划发布时间、状态），支持编辑时间和取消操作

**故事 22.1.4 - 发布限额与防灌水**

作为频道主，我希望设置频道的发布限额（每日/每小时发布上限、内容字数下限），以便防止刷屏和低质量内容泛滥。

### 旅程 2: 频道管理员审核投稿

**故事 22.2.1 - 内容置顶与精华标记**

作为频道管理员，我希望将重要内容置顶或标记为精华，以便突出优质内容并引导用户关注。

**故事 22.2.2 - 内容删除与回收站**

作为频道管理员，我希望删除不符合主题或违规的内容，并能在回收站中恢复误删内容，以便维护频道内容质量。

**故事 22.2.3 - 内容移出频道**

作为频道管理员，我希望将内容从当前频道移至其他频道，以便纠正内容分类错误或调整内容归属。

**故事 22.2.4 - 编辑协助**

作为频道管理员，我希望协助修正频道内容的标题、标签、摘要或明显错别字，以便提升内容质量同时保留作者信息。

### 旅程 3: 管理员治理已发布内容

**故事 22.3.1 - 频道公告管理**

作为频道管理员，我希望发布和管理频道公告，说明频道规则、活动等重要信息，以便向所有频道成员传达关键信息。

### 旅程 4: 已发布内容添加到频道

**故事 22.4.1 - 已发布内容添加到系统频道**

作为平台运营，我希望将已发布的优质内容添加到系统官方频道，以便在官方频道中推荐优质内容。

**故事 22.4.2 - 作者将已发布内容添加到个人/组织频道**

作为内容创作者，我希望将已发布的内容添加到我的个人频道或所属组织频道，以便在频道中聚合我的内容。

**故事 22.4.3 - 频道主添加他人作品至频道**

作为频道主/管理员，我希望将他人的优质作品添加到我管理的频道，以便丰富频道内容。

## 3. 页面设计

### 3.1 频道选择组件（ChannelSelector）

**入口**: 内容发布页面中的"选择频道"按钮/区域

**页面结构**:

```
+--------------------------------------------------+
| 搜索频道 [输入框]                    [已选 N/M]  |
+--------------------------------------------------+
| 已选频道标签区（可移除）                          |
+--------------------------------------------------+
| 推荐频道                                         |
|   [频道卡片] [频道卡片] [频道卡片]               |
+--------------------------------------------------+
| 我管理的频道                                      |
|   [频道卡片] [频道卡片]                           |
+--------------------------------------------------+
| 我加入的频道                                      |
|   [频道卡片] [频道卡片] [频道卡片]               |
+--------------------------------------------------+
| 搜索结果（搜索时展示）                            |
|   [频道卡片 - 可选] [频道卡片 - 不可选+原因]     |
+--------------------------------------------------+
```

**频道卡片信息**:
- 频道名称
- 频道类型标签（系统/个人/组织）
- 用户角色标签（频道主/管理员/成员）
- 发布结果预期（直接发布/进入待审/不可发布+原因）
- 选中状态（已选/可选/不可选置灰）

**交互规则**:
- 已选频道数达到上限 N 时，不可选频道展示"已达上限"提示，阻止继续添加
- 不可发布频道在搜索结果中展示为置灰不可选，hover 展示具体原因
- 已选频道支持标签形式快速移除
- 搜索支持频道名称模糊匹配
- **重复发布检测**: 若内容已在某频道发布，该频道卡片展示"已在此频道"标识并置灰不可选，防止同一内容在同一频道重复展示
- **频道数上限来源**: 上限值 N 由后端接口 `/api/v1/content/channel/publish/available` 返回（系统配置或用户等级决定），前端从接口响应中读取 `maxChannelCount` 字段
- **重试失败项规则**: 重试时仅允许重试原来的失败频道，不可更换目标频道；若需更换频道，需重新发起发布流程

**状态**:
- 空状态: 无可发布频道时展示引导文案"暂无可发布的频道，去加入或创建频道"
- 加载状态: 骨架屏
- 搜索无结果: "未找到匹配的频道"

### 3.3 发布权限配置页（PublishPermissionConfig）

**入口**: 频道设置页 → 发布权限

**页面结构**:

```
+--------------------------------------------------+
| 发布权限模型                                      |
+--------------------------------------------------+
| ( ) 仅管理员可发布                                |
|     说明：频道主、管理员和内容编辑可发布           |
| ( ) 所有成员可发布                                |

| 发布限额设置                                      |
```

**交互规则**:

### 3.4 待审区管理页（ReviewQueue）

**入口**: 频道管理后台 → 待审区

**页面结构**:

```
+--------------------------------------------------+
| 待审区                           [批量操作 ▼]     |
+--------------------------------------------------+
| 筛选栏: [内容类型▼] [提交者] [提交时间]          |
|         [审核状态▼] [超时状态▼]    [搜索]        |
+--------------------------------------------------+
| [ ] 标题          | 类型 | 提交者 | 提交时间     |
|   来源场景        | 命中规则             | [通过][拒绝] |
|--------------------------------------------------|
| [ ] 如何提升内容质量 | 文章 | 张三 | 10分钟前   |
|   公开投稿         | 先审后发规则         | [通过][拒绝] |
|--------------------------------------------------|
| [!] 超时未处理内容  | 视频 | 李四 | 3小时前    |
|   管理员添加       | 仅管理员可发布       | [通过][拒绝] |
+--------------------------------------------------+
| [全选]  [批量通过] [批量拒绝]                     |
+--------------------------------------------------+
| 分页: < 1 2 3 ... >                              |
+--------------------------------------------------+
```

**交互规则**:
- 单条审核: 点击"通过"直接通过；点击"拒绝"弹出拒绝原因填写弹窗（必填）
- 批量审核: 勾选多条 → 点击"批量通过"/"批量拒绝" → 展示确认弹窗（含影响说明）
- 超时内容高亮标识（红色边框或背景）
- 拒绝原因弹窗: 文本域（必填，最少 10 字）+ 预设原因快捷选择
- 审核完成后从列表移除，展示"审核完成"toast 提示
- **批量操作确认弹窗文案**:
  - 批量通过: "确认通过选中的 {N} 条内容？通过后内容将在频道中展示。"
  - 批量拒绝: "确认拒绝选中的 {N} 条内容？将向 {N} 位提交者发送拒绝通知。"
- **超时机制业务规则**:
  - 超时阈值: 24 小时（从提交时间起算），超时后前端仅做高亮标识，不触发自动通过或自动拒绝
  - 超时提醒对象: 所有有审核权限的管理员（频道主 + 频道管理员 + 内容编辑），通过站内消息通知
  - 超时提醒频率: 首次超时时提醒一次，之后每 12 小时重复提醒，直到内容被处理
  - 待审区 badge 统计: `超时数` = 待审列表中提交时间距当前超过 24 小时的内容数量，由 `/api/v1/content/channel/review/stats` 接口返回，前端每 60 秒刷新一次

**状态**:
- 空状态: "暂无待审核内容" + 引导返回频道管理页
- 加载状态: 表格骨架屏
- 批量操作结果: 逐条展示成功/失败，失败项保留原因

**交互规则**:
- 编辑器使用 Tinymce 富文本编辑器
- 预览: 点击"预览"在下方预览区展示渲染效果，过滤不安全内容
- 发布公告: 二次确认弹窗"确认发布此公告？发布后将展示在频道顶部"
- 删除公告: 二次确认弹窗"确认删除公告？删除后频道顶部不再展示"，删除后公告进入公告历史（保留最近 3 个版本），支持从历史中恢复
- 一个频道同一时间最多一条主公告
- 保存后记录修改人和修改时间
- **公告版本保留**: 保留最近 3 个历史版本，超过的自动清除。公告管理页底部增加"公告历史"区域，展示历史版本列表（版本号、修改人、修改时间），支持"恢复此版本"操作
- **并发编辑冲突**: 保存时后端校验版本号（乐观锁），若检测到冲突，前端提示"公告已被其他人修改，请刷新后重试"，展示最新版本内容供用户合并
- **发布生效延迟**: 公告发布后前端立即更新频道顶部展示，同时调用后端接口触发缓存刷新。若后端返回缓存刷新失败，提示"公告已发布，频道顶部展示可能有短暂延迟"

## 8. 交互设计

### 8.1 通用交互规则

| 场景 | 交互行为 |
|------|---------|
| 加载状态 | 数据请求期间显示骨架屏（JVxeTable/Table 内置 loading） |
| 空状态 | 无数据时展示空状态插图 + 引导文案 + 下一步操作入口 |
| 错误反馈 | 操作失败时 toast 提示具体原因，保留用户已输入内容 |
| 表单校验 | 必填字段实时校验，提交时触发全量校验，错误提示在字段下方 |
| 防重复提交 | 提交按钮在请求期间显示 loading 并 disabled |
| 成功反馈 | 操作成功后 useMessage.success 提示，关键操作后刷新列表 |

### 8.2 关键操作交互

**审核拒绝**:
1. 点击"拒绝" → 弹出 Modal
2. Modal 内容: 预设原因快捷标签 + 自定义原因文本域（必填，最少 10 字）
3. 点击"确认拒绝" → 调用 API → 成功后从列表移除 → toast "已拒绝并通知提交者"
4. 失败时保留输入内容，展示错误原因

**删除内容**:
1. 点击"删除" → 弹出确认 Modal
2. Modal 内容: "确认将《{标题}》从频道删除？内容将进入回收站，30天内可恢复。"
3. 可选: 删除原因输入框
4. 勾选"通知作者"（默认勾选）
5. 确认后调用 API → 成功后从列表移除 → toast "已删除并移入回收站"

**移出频道**:
1. 点击"移出频道" → 弹出 Modal
2. Modal 内容: 目标频道选择器（复用 ChannelSelector 简化版）
3. 选择目标频道后展示预期结果: "将进入目标频道待审区" / "将直接展示"
4. 确认后调用 API → 成功后从当前列表移除 → toast "已移出"

**编辑协助**:
1. 点击"编辑协助" → 打开 Drawer（右侧滑出）
2. Drawer 顶部: 原作者信息、内容标题（只读）
3. Drawer 中部: 可编辑字段表单（标题、标签、摘要）
4. Drawer 底部: 修订历史列表（修改人、时间、字段、前后摘要）
5. 保存时必填"修改原因"
6. 保存成功后关闭 Drawer，刷新列表
7. **作者通知**: 保存后自动向原作者发送站内消息通知，通知内容包含：修改人、修改字段、修改前后对比（diff 格式）、修改原因
8. **作者申诉入口**: 通知消息中附带"申诉"链接，点击后跳转到申诉表单页（复用平台通用申诉组件），申诉后由频道主/管理员处理
9. **修订历史可见范围**: 修订历史对原作者可见（在内容详情页展示"修订记录"入口），管理员和内容编辑可在编辑协助 Drawer 中查看完整修订历史

**置顶/精华**:
1. 点击"置顶" → 直接调用 API（无需确认）→ 列表刷新，内容移至顶部
2. 点击"标记精华" → 直接调用 API → 内容添加精华标识
3. 取消操作同理，状态即时切换

### 8.3 发布流程交互

1. 用户在发布页编辑内容主体（已有能力）
2. 点击"选择频道" → 展开 ChannelSelector（弹窗或页内展开）
3. 选择频道后，每个频道卡片展示发布结果预期
4. 可选: 开启定时发布 → 展示日期时间选择器
5. 点击"发布" → 前端预校验（频道数上限、字数下限）→ 调用发布 API
6. 展示 PublishResult 组件，逐频道反馈结果
7. 失败项支持"重试"操作

## 9. 响应式设计

### 9.1 断点定义

遵循项目 Ant Design Vue 4 响应式断点：
- `xs`: < 576px（手机竖屏）
- `sm`: >= 576px（手机横屏）
- `md`: >= 768px（平板）
- `lg`: >= 992px（桌面）
- `xl`: >= 1200px（大桌面）

### 9.2 各页面响应式策略

**ChannelSelector（频道选择组件）**:
- PC: 弹窗内分栏展示（推荐/我管理的/我加入的）
- 移动端: 全屏弹窗，频道列表纵向堆叠，已选频道标签可横向滚动

**待审区（ReviewQueueTable）**:
- PC: 完整表格展示所有列
- 移动端 (< md): 表格转为卡片列表，每张卡片展示标题、类型、提交者、时间，审核操作按钮固定在卡片底部；批量操作收纳到顶部"更多"菜单

**内容治理（ContentGovernance）**:
- PC: 完整表格 + 操作列
- 移动端: 卡片列表，治理操作收纳到 Dropdown "更多"菜单；筛选区折叠为"筛选"按钮触发的抽屉

**回收站（RecycleBinTable）**:
- PC: 完整表格
- 移动端: 卡片列表，恢复按钮在卡片底部

**治理日志（GovernanceLogTable）**:
- PC: 完整表格
- 移动端: 列表形式，详细信息可展开查看

**公告管理（AnnouncementEditor）**:
- PC: 编辑器和预览左右分栏
- 移动端: 编辑器全宽，预览切换为 Tab 页

**发布权限配置（PublishPermissionConfig）**:
- PC: 表单纵向排列
- 移动端: 同样纵向排列，底部按钮固定

**发布结果（PublishResult）**:
- PC: 列表形式逐频道展示
- 移动端: 卡片形式，每个频道一张结果卡片

## 10. 性能要求

| 指标 | 目标 | 实现方式 |
|------|------|---------|
| 频道选择列表加载 | P95 <= 500ms | 虚拟滚动（频道数 > 50 时）、搜索防抖 300ms |
| 发布提交响应 | P95 <= 500ms | 乐观 UI、提交后异步轮询结果 |
| 待审区列表加载 | P95 <= 500ms | 分页加载（默认 20 条/页）、筛选防抖 |
| 内容治理列表加载 | P95 <= 500ms | 分页加载、虚拟滚动（大数据量） |
| 定时发布状态更新 | 实时 | WebSocket 或轮询（30s 间隔） |
| 富文本编辑器加载 | <= 2s | Tinymce 按需加载、懒加载 |
| 批量操作反馈 | 即时 | 乐观 UI 更新 + 后台异步处理 |

### 优化策略

- **虚拟滚动**: 频道选择器列表超过 50 项时启用虚拟滚动
- **搜索防抖**: 所有搜索输入框 300ms 防抖
- **分页加载**: 所有列表默认分页，避免一次性加载大量数据
- **懒加载**: 公告编辑器（Tinymce）按需加载
- **缓存**: 频道列表、权限配置等低频变更数据使用 Pinia store 缓存
- **骨架屏**: 所有列表页和表单页使用骨架屏提升感知性能

## 13. 跨 EPIC 依赖矩阵

本 EPIC 依赖 EPIC-20（频道基础架构）和 EPIC-21（隐私与成员管理），以下明确各依赖点的数据流和前端响应策略：

### 13.1 依赖 EPIC-20（频道基础架构）

| 依赖点 | 数据流向 | 状态变更时前端响应 | 兜底策略 |
|--------|---------|-------------------|---------|
| 频道状态查询 | EPIC-20 提供 → EPIC-22 消费 | 频道被冻结时：ChannelSelector 中该频道展示"频道已冻结"置灰状态；已设定的定时发布任务自动取消并通知用户 | 若接口超时，ChannelSelector 展示加载失败，提供"重试"按钮；不展示可能已过期的缓存状态 |
| 频道删除通知 | EPIC-20 推送 → EPIC-22 监听 | 频道被删除时：待审区中该频道的待审内容自动移除；回收站中该频道的内容标记为"频道已删除"；定时发布任务自动取消 | 若 WebSocket 断连，前端每 5 分钟轮询频道状态，发现删除后同步清理 |
| 频道信息更新 | EPIC-20 提供 → EPIC-22 消费 | 频道名称/头像变更时：ChannelSelector 和已选频道标签同步更新 | 使用 Pinia store 缓存频道信息，设置 5 分钟 TTL，过期后重新拉取 |

### 13.2 依赖 EPIC-21（隐私与成员管理）

| 依赖点 | 数据流向 | 状态变更时前端响应 | 兜底策略 |
|--------|---------|-------------------|---------|
| 用户角色变更 | EPIC-21 推送 → EPIC-22 监听 | 用户从管理员降为普通成员时：该用户立即失去审核权限，待审区操作按钮对其隐藏；已分配的内容编辑权限自动回收 | 若推送丢失，用户执行操作时后端返回 403，前端提示"权限已变更，请刷新页面"并自动跳转到无权限状态 |
| 禁言状态 | EPIC-21 提供 → EPIC-22 消费 | 禁言用户发布时：前端在发布提交前校验禁言状态，展示"您在频道 {channelName} 已被禁言，无法发布。原因：{reason}"，发布按钮禁用 | 若校验接口超时，允许提交但后端会拒绝，前端捕获错误后展示禁言提示 |
| 黑名单状态 | EPIC-21 提供 → EPIC-22 消费 | 黑名单用户发布时：展示"您已被移出频道 {channelName}，无法发布"，发布按钮禁用。黑名单用户无法在 ChannelSelector 中看到该频道 | 同禁言状态兜底策略 |
| 成员列表查询 | EPIC-21 提供 → EPIC-22 消费 | 待审区筛选"提交者"时使用成员列表 | 接口超时时展示输入框自由输入，不提供下拉建议 |

### 13.3 通知依赖

| 通知场景 | 通知渠道 | 通知内容 | 触发时机 |
|---------|---------|---------|---------|
| 审核结果通知 | 站内消息 | "您提交到频道 {channelName} 的内容《{title}》已{通过/被拒绝}。{拒绝原因}" | 审核操作完成后 |
| 编辑协助通知 | 站内消息 | "管理员 {operator} 修改了您在频道 {channelName} 的内容《{title}》的{字段列表}。[查看详情] [申诉]" | 编辑协助保存后 |
| 超时提醒 | 站内消息 | "频道 {channelName} 有 {N} 条待审内容超过 24 小时未处理，请及时审核。[前往待审区]" | 超时后 + 每 12 小时 |
| 定时发布取消 | 站内消息 | "您在频道 {channelName} 的定时发布《{title}》已取消。原因：{reason}" | 取消触发时 |
| 定时发布失败 | 站内消息 | "您在频道 {channelName} 的定时发布《{title}》发布失败。原因：{reason}。[重新发布]" | 定时发布时间到达后 |

> **说明**: 通知能力复用 EPIC-06（隐私与通知）的通知基础设施，前端通过消息中心统一展示。

## 14. 待确认问题 / 默认假设

### 待确认问题

| 问题 | 影响范围 | 建议负责人 |
|------|---------|-----------|
| 单篇内容最多可同步到多少个频道 N，是否按内容类型或用户等级区分？ | ChannelSelector 上限逻辑 | 产品/运营 |
| 待审区内容超过多长时间未处理需要提醒管理员？ | 超时标识阈值 | 产品/社区治理 |
| 编辑协助是否需要作者确认后才生效？ | EditAssistDrawer 流程 | 产品/法务 |
| 作者是否可以拒绝他人将自己的作品添加到频道？ | AddContentDialog 流程 | 产品/内容域 |
| 定时发布使用哪个调度框架？ | 定时发布状态轮询方式 | 后端架构 |
| 频道选择器是否需要展示频道头像/封面？ | ChannelSelector 视觉设计 | 设计 |

### 默认假设

| 假设 | 理由 | 影响 |
|------|------|------|
| 单篇内容最多同步到 5 个频道 | 常见社区产品默认值，可配置 | ChannelSelector 上限校验 |
| 待审区超时阈值为 24 小时 | 平衡审核及时性和管理员负担 | 超时标识展示，超时后仅标识不自动处理，提醒所有有审核权限的管理员 |
| 编辑协助修改即时生效，不需作者确认 | 简化流程，修订历史可追溯 | EditAssistDrawer 保存逻辑；保存后自动通知作者，作者可通过通知申诉 |
| 频道列表按用户角色分组展示（我管理的/我加入的/推荐） | 提升选择效率 | ChannelSelector 分组逻辑 |
| 公告保留最近 3 个历史版本 | 防止误删无法恢复，与回收站"可恢复"理念一致 | 公告管理逻辑，支持从历史版本恢复 |
| 回收站保留 30 天，治理日志保留 180 天 | EPIC-22 明确要求 | 数据展示和过期处理 |
| 定时发布时间范围为当前时间到 30 天内 | 合理业务范围 | 时间选择器范围限制 |
