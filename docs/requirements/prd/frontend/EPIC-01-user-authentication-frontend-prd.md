## 1. 需求说明

### 需求目标

- **需求名称**: 用户认证与账号安全
- **一句话概述**: 为内容社区提供统一的注册、登录、会话管理、账号绑定、密码找回、风控拦截和账号注销前端能力
- **要解决的问题**: 当前内容社区缺少独立的认证前端体系，注册登录依赖平台层，无设备管理、账号安全设置、风控交互和注销流程的前端入口
- **期望达成的结果**: 注册转化率提升 25%，登录成功率 >99.5%，账号被盗率 <0.01%，异常登录检测准确率 >95%

### 数据指标与埋点方案

#### 基线定义

| 指标 | 定义 | 当前基线（预估） | 目标值 | 计算公式 |
|------|------|------------------|--------|----------|
| 注册转化率 | 访问注册页 → 完成注册 | 待测量（上线前采集 2 周） | 提升 25% | 完成注册用户数 / 注册页 PV |
| 注册漏斗转化率 | 点击注册按钮 → 完成注册 | 待测量 | >60% | 完成注册用户数 / 点击注册按钮用户数 |
| 登录成功率 | 登录请求成功 / 登录请求总数 | 待测量 | >99.5% | 登录成功次数 / 登录请求总次数 |
| 异常登录检测准确率 | 正确拦截 / 总拦截次数 | 待测量 | >95% | 用户确认"非本人"次数 / 异常通知弹出总次数 |

### 目标用户与使用场景

| 角色 | 说明 | 使用场景 |
|------|------|----------|
| 游客（未注册用户） | 访问内容社区的未注册访客 | 手机号注册、邮箱注册、第三方登录 |
| 注册用户 | 已完成注册的社区用户 | 登录、设备管理、账号绑定、密码找回、注销 |
| 管理员 | 平台运营管理人员 | 查看风控日志、处理误判申诉 |

- **使用场景**: 桌面端 + 移动端跨端适配
- **入口路径**:
  - 未登录状态 → 登录页/注册页
  - 个人中心 → 账号安全（绑定/解绑、设备管理、密码修改）
  - 个人中心 → 账号安全 → 账号注销
  - 登录页 → 忘记密码 → 密码找回
  - 风控拦截 → 图形验证码 / 账号锁定提示
  - 内容浏览 → 点赞/评论/收藏 → 弹出登录引导弹窗 → 登录后回到原位置
  - 内容浏览 → Token 过期 → 无感刷新（refresh_token 有效时）→ 刷新失败时跳转登录页

## 3. 功能详细说明

### 3.1 F1 手机号注册

#### 页面: 注册页（手机号 Tab）

**入口**: 登录页 → "注册" 链接 → 切换到手机号注册 Tab

**页面结构**:
```
注册表单卡片
├── Tab 切换（手机号注册 | 邮箱注册）
├── 手机号输入框（必填，带国际区号选择，默认 +86）
├── 图形验证码输入框（必填，带图形刷新）
├── 短信验证码输入框（必填，带"获取验证码"按钮和 60s 倒计时）
├── 勾选协议（"我已阅读并同意《用户协议》和《隐私政策》"，复选框 + 链接）
├── "注册" 按钮（主操作，全宽）
└── 底部链接（"已有账号？去登录"）
```

**交互要求**:
- 手机号输入框失去焦点时校验格式（11 位数字），不通过时下方显示红色提示"请输入有效的手机号码"
- 图形验证码点击图片刷新，输入框支持自动填充
- 点击"获取验证码"前校验手机号格式和图形验证码是否填写，不通过则不发送
- 点击"获取验证码"后按钮进入 60 秒倒计时状态，显示"重新获取(XXs)"，倒计时期间按钮禁用
- 同一手机号 1 分钟内只能请求 1 次，返回"请稍后再试"时在按钮上方显示提示
- 验证码输入框支持 6 位数字自动提交（输入满 6 位自动触发注册）
- 未勾选协议时"注册"按钮禁用，点击时提示"请先阅读并同意用户协议"
- 注册按钮点击后进入 loading 状态，防止重复提交
- 注册成功后自动登录，跳转到首页（或引导完善资料页）
- **注册后个性化推荐引导**（适用于手机号注册和邮箱注册验证通过后）:
  - 注册成功后首次进入首页时，弹出"发现你感兴趣的内容"引导弹窗（全屏或大面积覆盖）
  - 弹窗内容：频道/话题标签选择网格（如：科技、生活、游戏、音乐、影视、设计、职场、教育等，由后端配置返回），至少展示 10 个选项
  - 用户点选标签（至少选 3 个），选中态高亮显示
  - 点击"开始探索"按钮，提交选择到后端（`POST /api/v1/content/user/preferences/topics`），关闭弹窗，首页内容流根据选择优先展示相关内容
  - 提供"稍后设置"链接，点击后跳过引导直接进入首页
  - 跳过后在个人中心"兴趣设置"入口保留入口，用户可随时设置
  - 若用户既未选择也未跳过，下次登录时再次弹出引导弹窗
  - 首页内容流顶部显示"个性化推荐"标签，告知用户推荐内容基于其兴趣选择
- 手机号已注册时提示"该手机号已注册，是否直接登录？"，提供"去登录"链接
- 验证码错误超过 3 次，要求重新获取验证码并冷却 60 秒
- 验证码过期（超过 5 分钟）提示"验证码已过期，请重新获取"

**状态与边界情况**:
- **加载中**: 图形验证码加载时显示占位图
- **网络错误**: 提示"网络异常，请重试"，保留已输入内容
- **短信发送失败**: 提示"验证码发送失败，请稍后重试"
- **风控拦截**: 详见 F11 风控拦截交互

### 3.2 F2 邮箱注册

#### 页面: 注册页（邮箱 Tab）

**入口**: 登录页 → "注册" 链接 → 切换到邮箱注册 Tab

**页面结构**:
```
注册表单卡片
├── Tab 切换（手机号注册 | 邮箱注册）
├── 邮箱输入框（必填）
├── 密码输入框（必填，带 StrengthMeter 强度指示器）
├── 确认密码输入框（必填）
├── 勾选协议（同 F1）
├── "注册" 按钮（主操作，全宽）
└── 底部链接（"已有账号？去登录"）
```

**交互要求**:
- 邮箱输入框失去焦点时校验格式，不通过时显示"请输入有效的邮箱地址"
- 密码输入框实时显示强度指示器（StrengthMeter 组件）：弱/中/强
- 密码强度规则：至少 8 位，必须包含至少一个数字；含字母+数字为中，含大小写+数字+特殊字符为强
- 密码不符合要求时，输入框下方显示具体规则提示"密码长度至少为 8 位且必须包含至少一个数字"
- 确认密码与密码不一致时显示"两次输入的密码不一致"
- 注册成功后显示"验证邮件已发送至 xxx@xxx.com，请在 24 小时内完成验证"
- 邮箱已注册时提示"该邮箱已注册，是否直接登录？"
- 提供"重新发送验证邮件"链接，60 秒内只能发送 1 次

**未验证状态限制策略**:

邮箱注册成功但未完成验证的用户，处于"未验证"状态，功能受限如下：

| 功能 | 未验证状态 | 已验证状态 |
|------|-----------|-----------|
| 浏览内容 | 正常 | 正常 |
| 发布内容（帖子/评论） | 禁止，提示"请先验证邮箱后再发布" | 正常 |
| 互动功能（点赞/收藏/关注） | 禁止，提示"请先验证邮箱后再操作" | 正常 |
| 编辑个人资料 | 正常 | 正常 |
| 接收系统通知 | 仅接收验证提醒 | 正常 |

**前端视觉提示**:
- 未验证用户登录后，在页面顶部显示常驻警告横幅（Alert 组件，warning 类型）："您的邮箱尚未验证，部分功能受限。验证邮件已发送至 xxx@xxx.com"，附带"重新发送"按钮和"前往验证"链接
- 个人中心头像旁显示"未验证"角标
- 发布内容和互动按钮处于禁用态（灰色），点击时弹出提示弹窗，提供"去验证"和"重新发送验证邮件"两个操作
- 验证成功后自动解锁所有功能，移除警告横幅

**状态与边界情况**:
- **验证邮件未收到**: 显示"未收到邮件？请检查垃圾邮件箱或重新发送"
- **验证链接已过期**: 点击过期链接后显示"链接已过期，请重新注册或申请新的验证邮件"
- **网络错误**: 同 F1

### 3.3 F3 第三方登录注册

#### 组件: 第三方登录按钮组

**入口**: 登录页底部"其他登录方式"区域

**布局**:
```
登录表单底部
├── 分割线（"其他登录方式"）
├── 第三方登录图标组
│   ├── 微信图标按钮
│   ├── Apple 图标按钮
│   └── Google 图标按钮
└── （首次登录时）引导完善资料弹窗
```

**交互要求**:
- 点击第三方图标按钮后，PC 端打开新窗口跳转到第三方授权页，移动端跳转到第三方 App 或 WebView
- 授权成功后回调到前端，前端携带授权码调用后端登录接口
- 首次使用第三方登录时，系统自动创建账户并显示"欢迎加入！请完善基础资料"引导弹窗
- 引导弹窗包含：昵称（必填）、头像（可选，默认使用第三方头像）
- 用户可跳过引导，后续在个人中心完善资料
- **资料完善渐进式引导策略**:
  - 跳过引导后，个人中心头像区域显示红色圆点提示，直至用户完善昵称和头像
  - 用户首次发布内容（帖子/评论）时，弹出非阻塞式提示条："完善资料可让更多人认识你，去完善 >"，点击跳转个人资料编辑页
  - 用户首次关注其他用户时，弹出类似提示
  - 个人资料编辑页顶部显示资料完善度进度条（头像 +25%、昵称 +25%、简介 +25%、兴趣标签 +25%），低于 75% 时显示"建议完善"提示
  - 资料完善度达到 75% 后，所有引导提示自动消失
- 已绑定第三方账号的用户，授权后直接登录跳转首页
- 用户在第三方授权页取消授权，返回后显示"登录已取消"，停留在登录页
- 第三方 API 返回错误时显示"第三方登录失败，请稍后重试"，并提供其他登录方式入口

**状态与边界情况**:
- **授权超时**: 10 秒内未完成授权，提示"授权超时，请重试"
- **第三方服务不可用**: 对应图标按钮置灰，hover 提示"该登录方式暂时不可用"
- **第三方账号已被其他账户绑定**: 提示"该第三方账号已被其他账户绑定，请使用其他登录方式"

**页面结构**:
```
登录表单卡片
├── Tab 切换（密码登录 | 验证码登录）
├── 手机号输入框（必填）
```

- 确认后账号恢复正常，显示"注销已取消，账号恢复正常"
- 冷静期结束后，后端定时任务执行注销，用户再次登录时提示"该账号已注销"

**状态与边界情况**:
- **检查接口失败**: 显示"检查失败，请重试"，"申请注销"按钮禁用
- **已处于冷静期**: 直接显示冷静期状态页，不显示检查项
- **注销申请被拒绝**: 提示拒绝原因
- **网络错误**: 同 F1

### 6.3 缓存策略

- 异常登录通知数量缓存在 `useUserStore` 中，定期刷新（进入首页时）

## 7. 交互设计

### 7.1 核心交互流程

#### 流程 1: 新用户注册（手机号）
```
进入登录页 → 点击"注册" → 切换到手机号 Tab
→ 输入手机号 → 获取图形验证码 → 输入图形验证码
→ 点击"获取验证码" → 输入短信验证码
→ 勾选用户协议 → 点击"注册"
→ 成功：自动登录，跳转首页（或引导完善资料）
→ 失败（手机号已注册）：提示"已注册，去登录"
→ 失败（验证码错误）：提示错误，允许重试
→ 失败（风控）：弹出图形验证码验证
```

#### 流程 2: 用户登录（密码）
```
进入登录页（携带 redirect 参数或直接访问）→ 默认密码登录 Tab
→ 输入手机号/邮箱 → 输入密码 → 点击"登录"
→ 成功：跳转来源页（有 redirect 参数时）或首页（无 redirect 时）
→ 失败（密码错误）：提示"账号或密码错误"
→ 失败（连续 5 次）：锁定 15 分钟，显示倒计时
→ 失败（风控 10 次）：弹出图形验证码
→ 失败（风控 20 次）：锁定 30 分钟
```

#### 流程 3: 设备管理
```
进入账号安全页 → 点击"设备管理" → 加载设备列表
→ 查看设备信息 → 点击某设备"下线"
→ 确认弹窗 → 确认 → 设备下线成功
→ 被下线设备下次操作时需重新登录
```

#### 流程 4: 账号绑定
```
进入账号安全页 → 查看绑定状态
→ 点击"绑定手机号" → 弹出绑定弹窗
→ 输入手机号 → 获取验证码 → 输入验证码 → 确认
→ 成功：绑定成功，列表更新
→ 失败（已被绑定）：提示"该手机号已被其他账户绑定"
```

#### 流程 5: 密码找回
```
登录页 → "忘记密码" → 进入找回密码页
→ 选择找回方式（手机号/邮箱）→ 输入手机号/邮箱
→ 输入验证码 → 点击"下一步"
→ 输入新密码 + 确认密码 → 点击"确认重置"
→ 成功：显示"密码重置成功" → 点击"去登录"
```

#### 流程 6: 账号注销
```
进入账号安全页 → 点击"账号注销"
→ 系统检查前置条件 → 全部通过
→ 点击"申请注销" → 确认弹窗（输入"确认注销"）
→ 确认 → 进入冷静期，显示倒计时
→ 冷静期内登录 → 显示注销中提示 → 可点击"取消注销"
→ 冷静期结束 → 账号注销完成 → 下次登录提示"已注销"
```

| 数据加载中 | 内容区骨架屏 | Skeleton |
| 按钮操作中 | 按钮内 loading 图标 | Button loading |

| 注销冷静期 | 页面横幅 + 倒计时 | Alert + 倒计时 |

## 9. 性能要求

| 指标 | 目标值 | 说明 |
|------|--------|------|
| 登录页首屏加载 | < 2s | 包含背景图和表单渲染 |
| 注册页首屏加载 | < 2s | 同登录页 |
| 登录/注册接口响应 | < 1.5s | 后端接口响应时间 |
| 图形验证码加载 | < 500ms | 图片加载和渲染 |
| 短信验证码发送 | < 2s | 后端发送 + 前端反馈 |
| 设备列表加载 | < 1s | 接口响应 + 列表渲染 |
| 账号安全页加载 | < 1s | 绑定状态查询 + 页面渲染 |
| 密码找回流程 | < 5s | 整个流程（验证身份 + 设置密码） |
| 账号注销检查 | < 1s | 前置条件检查接口响应 |

**优化策略**:
- 登录/注册页使用懒加载，非首屏内容（如第三方登录图标）延迟加载
- 图形验证码使用 base64 内联，减少额外 HTTP 请求
- 设备列表使用分页加载（如果设备数较多）
- 表单校验使用防抖（300ms）避免频繁触发
- 登录成功后预加载用户信息和权限数据，减少首页白屏时间
- 使用骨架屏提升加载感知体验

## 11. 待确认问题 / 默认假设

### 待确认问题

1. **登录页独立性**: 内容社区登录页是独立页面（`/content/login`），还是复用现有平台登录页（`/sys/login`）？当前假设为独立页面，因为内容社区有独立认证体系。
2. **第三方登录配置**: 微信/Apple/Google 的 App ID 和回调地址是内容社区独立配置，还是复用平台配置？当前假设为独立配置。
3. **图形验证码组件**: 是否复用平台层现有的图形验证码（`LoginController.randomImage()`），还是内容社区独立实现？当前假设复用平台层。
4. ~~**滑块验证组件**~~（已确认）: 本期仅使用图形验证码（`GET /api/v1/content/auth/captcha/image` + `POST /api/v1/content/auth/captcha/verify`），不实现滑块验证。
5. **账号注销数据处理**: 匿化优先还是物理删除优先？设计文档标注为"匿名化优先"，前端假设只展示状态，不涉及数据处理细节。
6. **异常登录检测时机**: 异常登录通知是在用户登录时实时弹出，还是仅在消息中心展示？当前假设为登录时实时弹出 + 消息中心历史记录。
7. **密码找回中的高风险验证**: 高风险找回场景的"安全问题或身份证明"产品形态尚未确定，前端本期预留接口调用入口，暂不实现具体 UI。
8. **设备信息采集**: 设备类型、操作系统、设备指纹等信息如何采集？是否使用现有库或自研？当前假设使用 `navigator.userAgent` 解析 + 后端 IP 地理位置解析。
9. ~~**登录后跳转**~~（已确认）: 登录成功后跳转到来源页（如有 redirect 参数）或首页，作为默认行为写入功能规格（F4/F5）。
10. **注册后引导完善资料**: 注册成功后是否强制跳转到完善资料页？当前假设为可选引导弹窗，用户可跳过。

### 默认假设

1. **登录页风格**: 沿用现有平台登录页的深色背景 + 白色卡片风格，保持视觉一致性。
2. **验证码有效期**: 短信验证码 5 分钟，邮箱验证链接 24 小时，由后端控制，前端只做倒计时展示。
3. **密码强度规则**: 至少 8 位，包含字母和数字；StrengthMeter 组件显示弱/中/强三级。
4. **设备列表排序**: 按最后登录时间倒序，当前设备置顶。
5. **最大在线设备数**: 5 台，超出自动挤出最早设备。
6. **密码锁定规则**: 5 次错误锁定 15 分钟，10 次触发图形验证码，20 次锁定 30 分钟。
7. **注销冷静期**: 默认 7 天，可配置 7-30 天，前端从接口获取配置值。
8. **第三方登录按钮**: 微信、Apple、Google 三个，具体显示哪些由后端配置决定，前端根据接口返回动态渲染。
9. **国际化**: 本期仅支持中文，不做多语言适配。
10. **协议链接**: 用户协议和隐私政策链接由后端配置返回，前端渲染为可点击链接。
