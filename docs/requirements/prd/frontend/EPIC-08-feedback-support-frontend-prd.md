# EPIC-08 反馈、举报与客服支持 -- 前端 PRD

> **史诗ID**: EPIC-08
> **变更名称**: feedback-support-system
> **所属域**: user
> **优先级**: 高
> **预计周期**: 6 周（分三期交付）
> **技术栈**: Vue 3 + TypeScript + Vite 6 + Ant Design Vue 4 + Pinia

---

## 1. 概述

### 1.1 需求目标

内容社区缺少举报入口、申诉流程和客服支持体系。本变更补齐以下能力：

- 用户可对违规用户/内容/评论发起举报，上传证据，查看处理进度
- 用户可对处罚决定发起申诉，最多 3 次，支持审核结果通知
- 帮助中心提供常见问题分类与全文搜索，降低客服压力
- 客服通道支持智能客服机器人 + 人工客服，高等级用户（LV.15+）优先

### 1.2 目标用户与使用场景

| 角色 | 使用场景 | 入口 |
|------|---------|------|
| 注册用户 | 举报违规内容、发起申诉、查看帮助、联系客服 | 内容详情页"举报"按钮、个人中心菜单、帮助中心入口 |
| 高等级用户（LV.15+） | 享受人工客服优先排队 | 客服入口自动识别等级 |
| 被处罚用户 | 收到处罚通知后发起申诉 | 通知消息中的"申诉"链接 |

### 1.3 范围定义

#### 本期范围
- 举报系统：举报表单弹窗、举报列表页、举报详情、防重复校验
- 申诉系统：申诉提交页、申诉列表页、申诉详情、次数限制提示
- 帮助中心：分类浏览、全文搜索、文章详情、有用/无用反馈
- 更新日志：版本列表、版本详情、搜索
- 客服通道：智能客服对话界面、转人工排队、实时对话、服务评分、会话历史

#### 非本期范围
- 独立工单系统
- 电话客服
- 智能客服 NLP 引擎训练（使用现有引擎）
- 帮助中心 CMS 完整后端（仅前端搜索接口对接）

## 2. 用户故事

### US-01 举报违规用户/内容/评论
**作为** 用户，**我希望** 对违规用户/内容/评论发起举报，选择举报类型并上传证据，**以便** 参与社区治理。

**验收标准**:
1. 点击"举报"按钮后弹出举报表单
2. 举报类型包括：色情、暴力、诈骗、骚扰、其他，每个类型带说明文字
3. 支持上传图片/视频证据，单文件最大 10MB，超出时提示"文件大小不能超过 10MB"
4. 提交成功后显示"举报已提交，我们将尽快处理"并返回举报编号
5. 同一用户对同一对象重复举报时提示"您已举报过该内容，请勿重复举报"

### US-02 查看举报处理进度
**作为** 用户，**我希望** 跟踪举报的处理进度，**以便** 了解举报是否被处理。

**验收标准**:
1. "我的举报"页面展示所有举报记录列表
2. 每条记录显示：举报对象、举报类型、提交时间、当前状态（待处理/处理中/已处理）
3. 已处理的举报可查看处理结果（已删除内容/已警告用户/举报不成立）
4. 处理完成后系统发送通知
5. 对处理结果不满意时可点击"申诉"进入申诉流程

### US-03 对处罚发起申诉
**作为** 用户，**我希望** 对被删除内容、被封禁账户、被扣除积分/勋章等处罚发起申诉，**以便** 获得复核机会。

**验收标准**:
1. 处罚通知中包含"申诉"入口
2. 填写申诉理由后提交，生成申诉编号
3. "我的申诉"页面展示申诉列表和处理状态
4. 审核中状态显示"审核中"及预计处理时间
5. 审核完成后收到通知（维持原判/撤销处罚）

### US-04 申诉审核与恢复
**作为** 平台系统，**我希望** 支持申诉恢复，恢复后保留历史记录用于审计，**以便** 平衡治理与公平。

**验收标准**:
1. 申诉成功后系统通知"申诉成功，处罚已撤销"
2. 申诉驳回后系统通知"申诉未通过，原因：XXX"
3. 同一事件最多申诉 3 次，第 4 次提示"您已申诉多次，请耐心等待最终结果"
4. 所有申诉操作生成不可篡改的审计日志（前端只展示，不生成）

### US-05 访问帮助中心与搜索问题
**作为** 用户，**我希望** 能够访问帮助中心，**以便** 自助解决问题。

**验收标准**:
1. 帮助中心页面展示常见问题分类（账号、隐私、举报、积分等）
2. 输入关键词搜索后返回相关帮助文章，响应时间 <500ms
3. 搜索无结果时显示"未找到相关文章"并提示联系客服
4. 文章详情页展示步骤说明和截图
5. 文章底部提供"有用/无用"反馈按钮；点击"无用"后提示"是否联系客服？"

### US-06 查看功能更新日志
**作为** 用户，**我希望** 查看功能更新日志，**以便** 了解平台最新变化。

**验收标准**:
1. 更新日志页面按时间倒序展示版本记录
2. 每个版本显示：版本号、更新日期、新增功能、优化内容、修复问题
3. 支持搜索功能名称
4. 有新版本发布时，首次登录弹出"查看最新版本更新"提示

### US-07 智能客服与人工客服入口
**作为** 用户，**我希望** 有客服通道且高等级用户可优先，**以便** 提升支持体验。

**验收标准**:
1. 点击"客服"入口后首先启动智能客服机器人对话界面
2. 点击"转人工"后加入人工客服队列
3. 高等级用户（LV.15+）排在优先位置，预计等待 <2 分钟；普通用户 <5 分钟
4. 人工客服接入后可实时对话
5. 会话结束后邀请用户评分

### US-08 客服会话管理与历史记录
**作为** 用户，**我希望** 查看客服会话历史记录，**以便** 回顾之前的咨询内容。

**验收标准**:
1. "客服记录"页面按时间倒序展示历史会话列表
2. 点击会话可查看完整对话记录
3. 支持"继续咨询"创建新会话并关联历史记录
4. 超过 30 天的会话提示"历史记录仅保留 30 天"

## 3. 页面设计

### 3.1 页面清单

| 页面 | 路由建议 | 类型 | 入口 |
|------|---------|------|------|
| 我的举报列表 | `/user/reports` | 列表页 | 个人中心菜单 |
| 举报表单弹窗 | - | 弹窗（Modal） | 内容详情页"举报"按钮 |
| 我的申诉列表 | `/user/appeals` | 列表页 | 个人中心菜单 / 处罚通知 |
| 申诉提交页 | `/user/appeals/create` | 表单页 | 处罚通知"申诉"链接 |
| 帮助中心 | `/help` | 分类+搜索页 | 顶部导航 / 页脚 |
| 帮助文章详情 | `/help/article/:id` | 详情页 | 帮助中心文章列表 |
| 更新日志 | `/changelog` | 列表页 | 页脚 / 帮助中心侧边栏 |
| 客服对话 | `/customer-service` | 对话页 | 悬浮客服按钮 |
| 客服历史记录 | `/user/service-history` | 列表页 | 个人中心菜单 |

### 3.2 页面结构

#### 3.2.1 举报表单弹窗（Modal + Form）

```
Modal（宽度 520px）
  ├── 标题：举报
  ├── 被举报对象摘要区域
  │     └── 展示被举报内容摘要（如"举报内容：{文章标题前20字}..."或"举报用户：{用户名}"）
  ├── 举报类型（Radio.Group，竖排）
  │     ├── 色情内容 - 包含色情、裸露等不当内容
  │     ├── 暴力内容 - 包含血腥、恐怖等暴力内容
  │     ├── 诈骗信息 - 涉及欺诈、钓鱼等行为
  │     ├── 骚扰行为 - 包含辱骂、威胁、骚扰等
  │     └── 其他 - 其他违规行为
  ├── 举报说明（TextArea，选填，最大 500 字）
  ├── 证据上传（Upload，支持图片/视频，最大 10MB）
  │     └── 已上传文件列表（可预览/删除/重试）
  ├── 防重复提示区域（仅重复举报时显示）
  └── 底部按钮：取消 | 提交举报
```

**交互规则**:
- 弹窗顶部展示被举报对象摘要（根据 `targetType` 显示对应内容：文章显示标题前 20 字 + "...", 评论显示评论内容前 30 字 + "...", 用户显示用户名 + 头像），帮助用户确认操作对象正确
- 选择举报类型后，下方显示该类型说明文字
- 证据上传状态机：空闲 → 上传中（进度条 + 禁用提交按钮） → 成功 / 失败（失败标识 + 重试按钮）
- 上传过程中提交按钮处于禁用状态，直到所有文件上传完成或失败
- 部分文件上传失败时：允许带已成功的文件提交，失败文件需重试或删除后方可提交；未处理的失败文件阻止提交
- 网络断开时上传队列暂停，恢复网络后自动继续上传剩余文件
- 文件超过 10MB 时立即提示，不发起上传
- 提交按钮在请求期间显示 loading 并禁用
- 提交成功后弹窗关闭，页面顶部显示成功消息"举报已提交，我们将尽快处理"

#### 3.2.2 我的举报列表页（Page + Table）

```
Page
  ├── 查询表单区（Form，inline 模式）
  │     ├── 举报状态（Select：全部/待处理/处理中/已处理/已撤回）
  │     ├── 举报类型（Select：全部/色情/暴力/诈骗/骚扰/其他）
  │     └── 查询 | 重置 按钮
  └── 数据表格（Table）
        ├── 列：举报编号 | 举报对象 | 举报类型 | 提交时间 | 状态（Tag） | 操作
        ├── 状态 Tag 颜色：待处理=橙色+时钟图标、处理中=蓝色+旋转图标、已处理=绿色+勾选图标、已撤回=灰色+撤回图标
        └── 操作列：查看详情 | 撤回（仅待处理状态显示） | 申诉（仅已处理且结果不满意时显示）
```

**交互规则**:
- 点击"查看详情"打开 Drawer 展示举报详情（举报对象、类型、说明、证据预览、处理结果）
- 点击"撤回"弹出二次确认弹窗"确认撤回该举报？撤回后不可恢复"，确认后调用撤回 API，成功后刷新列表
- 举报详情 Drawer 中增加"对处理结果不满意？发起申诉"醒目入口，点击跳转申诉提交页
- 处理完成的通知消息中直接包含"申诉"链接
- 点击"申诉"跳转至申诉提交页，预填举报编号
- 空状态显示引导文案"暂无举报记录"和"遇到违规内容？点击举报"按钮
- 表格默认按提交时间倒序排列

#### 3.2.3 申诉提交页（Page + Form）

```
Page
  ├── 面包屑：个人中心 > 我的申诉 > 提交申诉
  ├── 表单区（Form）
  │     ├── 申诉类型（Select：内容删除/账号封禁/积分扣除/勋章扣除）
  │     ├── 关联记录（只读，从处罚通知或举报详情带入）
  │     ├── 申诉理由（TextArea，必填，最大 1000 字）
  │     └── 附件上传（Upload，选填，支持图片/视频）
  ├── 申诉次数提示
  │     └── "本次为第 X/3 次申诉"
  └── 底部按钮：取消 | 提交申诉
```

**交互规则**:
- 申诉理由为必填项，为空时提交按钮禁用
- 第 3 次申诉时提交前弹出确认框"这是最后一次申诉机会，确认提交？"
- 超过 3 次后页面显示提示"您已申诉多次，请耐心等待最终结果"，提交按钮隐藏
- 提交成功后跳转至"我的申诉"列表页，顶部显示成功消息

#### 3.2.4 我的申诉列表页（Page + Table）

```
Page
  ├── 查询表单区（Form，inline 模式）
  │     ├── 申诉状态（Select：全部/审核中/已通过/已驳回/已撤回）
  │     └── 查询 | 重置 按钮
  └── 数据表格（Table）
        ├── 列：申诉编号 | 申诉类型 | 关联处罚 | 提交时间 | 状态（Tag） | 操作
        ├── 状态 Tag 颜色：审核中=蓝色+旋转图标、已通过=绿色+勾选图标、已驳回=红色+叉号图标、已撤回=灰色+撤回图标
        └── 操作列：查看详情 | 撤回（仅审核中状态显示）
```

**交互规则**:
- 点击"查看详情"打开 Drawer 展示：申诉理由、附件、审核结果、审核时间
- 点击"撤回"弹出二次确认弹窗"确认撤回该申诉？撤回后不可恢复"，确认后调用撤回 API，成功后刷新列表
- 审核中状态额外显示预计处理时间
- 空状态显示"暂无申诉记录"

#### 3.2.5 帮助中心页（Page）

```
Page
  ├── 顶部搜索区
  │     ├── 搜索输入框（大尺寸，居中，带搜索图标）
  │     └── 搜索按钮
  ├── 分类导航区（Card 分组）
  │     ├── 账号安全（图标 + 名称 + 文章数）
  │     ├── 隐私设置
  │     ├── 举报申诉
  │     ├── 积分规则
  │     └── 其他问题
  ├── 热门问题列表（可选，默认展示 Top 10）
  └── 最近更新文章（可选）
```

**交互规则**:
- 搜索框支持回车触发搜索
- 搜索结果页展示文章标题列表，关键词高亮
- 搜索无结果时显示空状态插图 + "未找到相关文章" + "联系客服"按钮
- 点击分类后展示该分类下的文章列表
- 分类导航区在移动端转为 2 列网格布局

#### 3.2.6 帮助文章详情页（Page）

```
Page
  ├── 面包屑：帮助中心 > 分类名称 > 文章标题
  ├── 文章标题
  ├── 文章正文（Markdown 渲染，支持图片/截图）
  ├── 文章底部
  │     ├── "这篇文章有帮助吗？" + 有用 / 无用 按钮
  │     └── 点击"无用"后展开："是否联系客服？" + 联系客服按钮
  └── 相关文章推荐（可选）
```

**交互规则**:
- 点击"有用"/"无用"后按钮变为已选中状态，不可重复点击
- 点击"无用"后在下方展开"是否联系客服？"提示，点击跳转客服对话页
- 文章内容支持 Markdown 渲染，图片可点击放大预览

#### 3.2.7 更新日志页（Page）

```
Page
  ├── 顶部搜索区（可选，搜索功能名称）
  └── 时间线列表（Timeline）
        └── 每个版本节点
              ├── 版本号 + 更新日期
              ├── 新增功能（绿色标签 + 列表）
              ├── 优化内容（蓝色标签 + 列表）
              └── 修复问题（橙色标签 + 列表）
```

**交互规则**:
- 默认按时间倒序展示
- 搜索时实时过滤，匹配的版本节点高亮关键词
- 首次登录有新版本时弹出 Modal 提示"查看最新版本更新"，点击跳转至更新日志页
- 移动端时间线节点改为卡片堆叠布局

#### 3.2.8 客服对话页（Page）

```
Page（全屏或 Drawer）
  ├── 顶部栏
  │     ├── 客服名称/状态
  │     ├── 排队等待提示（排队时显示）
  │     └── 关闭按钮
  ├── 对话区域（消息列表）
  │     ├── 智能客服消息（左侧气泡）
  │     ├── 用户消息（右侧气泡）
  │     ├── 系统消息（居中，如"人工客服已接入"、"会话已结束"）
  │     └── 消息类型支持：文本、图片、链接
  ├── 快捷回复区（智能客服阶段）
  │     └── 常见问题快捷按钮
  ├── 底部输入区
  │     ├── 输入框 + 发送按钮
  │     └── "转人工"按钮（智能客服阶段显示）
  └── 评分弹窗（会话结束后弹出）
        └── 星级评分（1-5 星） + 评价输入框 + 提交
```

**交互规则**:
- 进入客服页面后自动启动智能客服机器人
- 智能客服阶段底部显示快捷回复按钮（常见问题分类）
- 点击"转人工"后显示排队等待提示和预计等待时间
- 高等级用户（LV.15+）显示"优先排队中"标识
- 人工客服接入后"转人工"按钮隐藏
- 新消息自动滚动到底部
- 会话结束后弹出评分 Modal，评分后不可再次评分
- 移动端全屏展示

**消息发送失败与断连处理**:
- 消息发送失败时：在气泡旁显示红色感叹号图标 + "重试"文字按钮，点击重新发送
- 消息发送中状态：气泡旁显示 loading 转圈图标，气泡半透明
- 消息状态流转：发送中（灰色气泡 + loading） → 已发送（正常气泡） → 发送失败（红色感叹号 + 重试）
- WebSocket 断连时：顶部显示"连接已断开，正在重连..."黄色横幅，输入框禁用
- 重连成功后：横幅自动消失，自动重新发送所有失败消息，输入框恢复可用
- 长时间无法重连（超过 30 秒）：横幅变为红色，提示"连接失败，请刷新页面重试"，附带"刷新页面"按钮
- 排队期间断连：自动重新加入队列并恢复排队位置，顶部提示"连接已恢复，您仍在排队中"

**排队状态持久化与离开提醒**:
- 排队状态持久化到 sessionStorage，刷新页面后自动恢复排队状态并重新连接 WebSocket
- 客服入口悬浮按钮在用户排队期间显示排队状态徽标（如"排队中 #3"），点击可进入客服对话页
- 排队期间用户点击页面内导航（非关闭浏览器）时，弹出确认弹窗"您正在排队中，离开将取消排队，确认离开？"，确认后取消排队
- 用户关闭浏览器或标签页后排队自动取消，下次进入需重新排队
- 排队期间客服入口按钮显示倒计时或排队位置变化动画，增强用户等待信心

#### 3.2.9 客服历史记录页（Page + Table）

```
Page
  └── 数据表格（Table）
        ├── 列：会话时间 | 客服类型（智能/人工） | 问题摘要 | 状态 | 操作
        ├── 状态：进行中/已结束
        └── 操作列：查看详情 | 继续咨询
```

**交互规则**:
- 点击"查看详情"打开 Drawer 展示完整对话记录
- 点击"继续咨询"创建新会话并关联历史记录
- 超过 30 天的会话显示"历史记录仅保留 30 天"提示，查看详情时弹出该提示
- 空状态显示"暂无客服记录"

## 8. 交互设计

### 8.1 通用交互规则

以下交互行为适用于所有页面：

| 场景 | 反馈方式 |
|------|---------|
| 数据加载中 | 骨架屏（Skeleton）或 Spin |
| 操作成功 | 顶部全局消息提示（绿色），3 秒自动关闭 |
| 操作失败 | 顶部全局消息提示（红色），保留用户已输入内容 |
| 空状态 | 空状态插图 + 引导文案 + 可选操作按钮 |
| 表单校验失败 | 字段下方红色提示文字，提交按钮不禁用但触发全量校验 |
| 防重复提交 | 提交按钮在请求期间显示 loading 并禁用 |
| 危险操作 | 二次确认弹窗（如删除证据、取消申诉） |
| 文件上传失败 | 文件列表中显示失败标识 + 重试按钮 |

### 8.2 举报流程交互

```
点击"举报" → 检查是否重复 → [重复] 显示提示 → 结束
                            → [不重复] 弹出举报表单
                                → 顶部展示被举报对象摘要（确认上下文）
                                → 选择类型 → 显示类型说明
                                → 上传证据 → 显示进度（上传中禁用提交） → 成功/失败（可重试）
                                → 填写说明（选填）
                                → 点击提交 → loading → 成功 → 关闭弹窗 + 消息提示

举报列表 → 待处理状态行 → 点击"撤回" → 二次确认弹窗 → 确认 → 调用撤回 API → 刷新列表
         → 已处理状态行 → 点击"查看详情" → Drawer 展示详情 + "对处理结果不满意？发起申诉"入口
```

### 8.3 申诉流程交互

```
点击"申诉"入口 → 跳转申诉提交页
    → 填写申诉理由（必填）
    → 上传附件（选填）
    → 显示"第 X/3 次申诉"
    → [第3次] 提交前弹出确认框
    → 点击提交 → loading → 成功 → 跳转列表页 + 消息提示
    → [超过3次] 显示提示，提交按钮隐藏

申诉列表 → 审核中状态行 → 点击"撤回" → 二次确认弹窗 → 确认 → 调用撤回 API → 刷新列表
```

### 8.4 客服对话交互

```
进入客服页 → 启动智能客服
    → 显示欢迎消息 + 快捷回复按钮
    → 用户发送消息 / 点击快捷回复
    → 智能客服回复 → 继续对话
    → 点击"转人工" → 显示排队提示 + 预计等待时间（排队状态持久化到 sessionStorage）
    → [高等级用户] 显示"优先排队中"
    → [排队中离开页面] 弹出确认弹窗"您正在排队中，离开将取消排队"
    → 人工客服接入 → 系统消息"人工客服已接入"
    → 实时对话
    → [消息发送失败] 气泡旁显示红色感叹号 + "重试"按钮
    → [WebSocket 断连] 顶部黄色横幅"连接已断开，正在重连..."
    → [重连成功] 自动重发失败消息
    → [超过30秒未重连] 红色横幅"连接失败，请刷新页面重试"
    → 客服结束会话 → 系统消息"会话已结束"
    → 弹出评分 Modal → 用户评分 → 提交 → 关闭

移动端 → 点击"返回" → 最小化会话（不结束）→ 客服入口显示绿色圆点 → 点击重新进入
```

### 8.5 搜索交互

```
输入关键词 → 按回车或点击搜索
    → 显示加载状态
    → [有结果] 展示文章列表，关键词高亮
    → [无结果] 显示空状态 + "联系客服"按钮
    → 点击文章标题 → 进入文章详情
    → 点击"有用" → 按钮变为已选中
    → 点击"无用" → 按钮变为已选中 + 展开"联系客服"提示
```

## 10. 性能要求

| 指标 | 目标值 | 说明 |
|------|-------|------|
| 举报提交响应时间 | < 1 秒 | 前端提交到后端返回 |
| 申诉提交响应时间 | < 1 秒 | 前端提交到后端返回 |
| 帮助中心搜索响应时间 | < 500ms | 从输入到结果展示 |
| 客服消息发送延迟 | < 500ms | WebSocket 优先 |
| 页面首屏加载 | < 2 秒 | 路由懒加载，按需引入 |
| 列表页分页切换 | < 300ms | 本地缓存 + 防抖 |
| 文件上传 | 进度条实时更新 | 支持断点续传（如后端支持） |

### 10.1 优化策略

- **路由懒加载**: 所有页面组件使用动态 import
- **组件按需引入**: 仅引入使用的 Ant Design Vue 组件
- **列表分页**: 默认每页 20 条，避免一次性加载过多数据
- **搜索防抖**: 帮助中心搜索输入 300ms 防抖
- **WebSocket 复用**: 客服对话使用单个 WebSocket 连接
- **图片懒加载**: 帮助文章中的图片使用懒加载
- **缓存策略**: 帮助分类列表和更新日志首次加载后缓存，减少重复请求

## 12. 无障碍设计规范

### 12.1 ARIA 标签与语义标记

| 元素 | ARIA 要求 |
|------|----------|
| 举报类型 Radio.Group | 每个 Radio 添加 `aria-label`（如 `aria-label="色情内容"`），Group 添加 `aria-labelledby` 指向标题 |
| 表单控件 | 所有 Form.Item 使用 `<label>` 关联输入控件，或通过 `aria-labelledby` / `aria-label` 标注 |
| 状态 Tag | 不仅依赖颜色，同时使用图标辅助区分：待处理=橙色+时钟图标、处理中=蓝色+旋转图标、已处理=绿色+勾选图标；Tag 文字内容包含完整状态名称（如"待处理"而非仅颜色块） |
| 客服消息气泡 | 使用 `role="log"` 和 `aria-live="polite"` 包裹消息区域；用户消息标注 `aria-label="我发送的消息"`，客服消息标注 `aria-label="客服消息"`，系统消息标注 `aria-label="系统通知"` |
| 弹窗（Modal） | 使用 `role="dialog"` + `aria-modal="true"` + `aria-labelledby` 指向弹窗标题 |
| 抽屉（Drawer） | 使用 `role="dialog"` + `aria-modal="true"` + `aria-labelledby` 指向抽屉标题 |
| 操作按钮 | 所有图标按钮添加 `aria-label`（如"查看详情"、"撤回举报"、"重试发送"） |

### 12.2 焦点管理

- **弹窗/Drawer 打开时**: 焦点自动移入弹窗/Drawer 内第一个可交互元素
- **弹窗/Drawer 关闭时**: 焦点回到触发打开的元素
- **焦点陷阱**: 弹窗/Drawer 打开期间，Tab 键循环限定在弹窗/Drawer 内部，不跳到背景元素
- **评分弹窗**: 打开时焦点移至第一颗星，关闭后焦点回到触发元素

### 12.3 键盘导航

- **Tab 顺序**: 遵循页面视觉布局的从左到右、从上到下的自然顺序
- **表单提交**: Enter 键提交表单（单行输入时）
- **列表操作**: 支持上下方向键在列表行间导航，Enter 键触发查看详情
- **消息列表**: 支持上下方向键浏览消息历史

### 12.4 色觉辅助

- 所有状态信息同时使用颜色 + 图标 + 文字三重标识，不依赖纯颜色区分
- 上传状态：成功=绿色+勾选图标+"上传成功"、失败=红色+叉号图标+"上传失败"、上传中=蓝色+旋转图标+"上传中"
- 消息发送状态：发送中=灰色+loading、已发送=正常+勾号、失败=红色+感叹号+"发送失败"
