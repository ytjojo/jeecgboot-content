# EPIC-12 - 圈子数据与推荐 前端 PRD

## 1. 概述

### 1.1 背景

圈子功能已具备基础能力（EPIC-10）和内容互动能力（EPIC-11），但管理员缺乏数据支撑来优化运营决策，用户发现圈子的方式单一（仅搜索）。本 PRD 定义圈子数据统计和推荐发现两大特性的前端实现需求。

### 1.2 目标

- 提供圈子运营数据统计面板，支持成员增长、内容发布、活跃度等核心指标的可视化展示
- 提供基于用户兴趣的圈子推荐能力，帮助用户发现感兴趣的圈子
- 提供热门圈子榜单和新增圈子榜单，覆盖冷启动场景
- 支持推荐来源追踪，用于统计点击率和加入转化
- 支持 PC 端和移动端响应式布局

### 1.3 技术栈约束

- 基于现有 JeecgBoot Vue3 前端框架
- 使用 Vue 3 + TypeScript + Ant Design Vue
- 遵循项目现有的组件、路由、状态管理规范

### 1.4 范围

| 包含 | 不包含 |
|------|--------|
| 圈子数据统计面板页面 | 复杂机器学习推荐算法 |
| 时间范围筛选与数据导出 | 成长激励体系（EPIC-13） |
| 推荐圈子列表展示 | 付费功能 |
| 热门圈子榜单与新增圈子榜单 | 推荐算法权重配置后台 |
| 推荐来源追踪（前端埋点） | 实时数据流计算 |
| 响应式布局适配 | - |

### 1.5 里程碑

| 阶段 | 时间 | 交付内容 |
|------|------|----------|
| Milestone 1 | 第 1-2 周 | 圈子数据统计面板页面 |
| Milestone 2 | 第 2-3 周 | 推荐列表、榜单展示、全量联调与测试 |

## 3. 页面设计

### 3.1 页面列表

### 3.2 页面详情

#### 3.2.1 圈子数据统计页 `/circle/:id/analytics`

**功能描述**: 展示圈子运营数据，支持时间筛选和数据导出。仅圈子创建者和版主可访问。

**入口**:
- 圈子详情页 `/circle/:id` 中，创建者/版主的操作按钮区增加"数据统计"入口按钮
- 圈子设置页侧边导航增加"数据统计"入口

**页面元素**:

**顶部操作区**:
- 页面标题："圈子数据统计"
- 返回按钮（返回圈子详情页）
- 时间范围选择器：
  - 快捷选项：近 7 天 / 近 30 天
  - 自定义范围：日期区间选择器（DatePicker.RangePicker）
  - 默认选中"近 7 天"
- 导出按钮：点击后下载 CSV 文件，文件内容与当前筛选时间范围一致

**核心指标卡片区**（一行 4 个卡片）:
- 成员总数（数字 + 与上期对比变化百分比）
- 新增成员数（数字 + 与上期对比变化百分比）
- 发帖总数（数字 + 与上期对比变化百分比）
- 活跃用户数（数字 + 与上期对比变化百分比）
- 变化百分比：正数绿色上箭头，负数红色下箭头，零灰色横线

**趋势图表区**:
- 成员增长趋势折线图（X 轴日期，Y 轴人数）
- 内容发布趋势折线图（X 轴日期，Y 轴发帖数）
- 活跃度趋势折线图（X 轴日期，Y 轴活跃用户数）
- 图表支持 Hover 查看具体数值（Tooltip）

**交互逻辑**:
- 切换时间范围后自动刷新所有数据和图表
- 导出按钮在数据加载完成前禁用
- 图表数据为空时展示空状态插图 + "暂无数据"文案，保留时间筛选入口
- 数据加载失败时展示错误提示 + "重试"按钮
- 普通成员访问此页面时展示"权限不足"提示，提供返回按钮

#### 3.2.2 圈子列表页增强 `/circle`

**功能描述**: 在现有圈子列表页中增加推荐区域和榜单 Tab，帮助用户发现圈子。

**页面元素**:

**Tab 切换区**（页面顶部）:
- "推荐" Tab（默认选中，已登录用户）
- "热门榜" Tab
- "新锐榜" Tab
- 游客默认展示"热门榜" Tab

**推荐 Tab 内容**:
- 推荐圈子卡片列表（每行 3 个，PC 端）
- 每张卡片展示：圈子图标、名称、简介摘要（2 行截断）、成员数、分类标签、公开/私有标识、加入按钮
- 私有圈子卡片：仅展示基础信息，不展示私有内容预览，加入按钮文案改为"申请加入"
- 推荐为空时：自动切换展示热门榜单作为兜底内容，页面顶部显示提示"暂无推荐，为您展示热门圈子"

**热门榜 Tab 内容**:
- 按成员数和活跃度排名的圈子列表，展示 Top 20
- 每行展示：排名序号（1-3 用金银铜色标识）、圈子图标、名称、成员数、分类标签
- 候选不足 20 个时展示全部符合条件的公开圈子
- 无符合条件的圈子时展示空状态

**新锐榜 Tab 内容**:
- 按创建时间倒序展示的圈子列表
- 展示字段与热门榜一致（无排名序号，改为创建时间）
- 空状态处理同热门榜

**交互逻辑**:
- 点击推荐/榜单中的圈子卡片跳转至圈子详情页
- 点击"加入"按钮触发加入流程（复用 EPIC-10 的 JoinModal 逻辑）
- 推荐来源标识：通过推荐 Tab 点击进入详情页时，URL 携带 `source=recommend` 参数
- 通过热门榜点击进入详情页时，URL 携带 `source=hot_rank` 参数
- 通过新锐榜点击进入详情页时，URL 携带 `source=new_rank` 参数
- Tab 切换时加载对应数据，使用骨架屏占位

## 4. 组件设计

### 4.1 通用组件

#### StatCard 统计指标卡片

**用途**: 展示单个核心指标（成员数、发帖数等），用于数据统计页顶部。

**Props**:
| 属性 | 类型 | 说明 |
|------|------|------|
| title | string | 指标名称 |
| value | number | 当前值 |
| change | number | 与上期对比变化百分比（正数/负数/零） |
| prefix | string | 数值前缀（如"¥"），可选 |
| suffix | string | 数值后缀（如"人"），可选 |
| loading | boolean | 加载状态 |

**展示规则**:
- 数值使用 `CountTo` 组件实现数字动画
- 变化百分比：正数绿色 + 上箭头，负数红色 + 下箭头，零灰色横线
- 加载状态时展示骨架块
- **"上期"计算规则**: 与当前选择等长的前一时间段。例如当前选择"近 7 天"（6/1-6/7），上期为前 7 天（5/25-5/31）；当前选择"自定义 10 天"，上期为前 10 天
- **基数为零的边界处理**: 当上期值为 0 且本期值大于 0 时，不展示百分比，改为展示"新增"文案；当上期值和本期值均为 0 时，展示"--"（无变化）

#### TrendChart 趋势折线图

**用途**: 展示指标随时间变化的趋势，用于数据统计页图表区。

**Props**:
| 属性 | 类型 | 说明 |
|------|------|------|
| title | string | 图表标题 |
| data | Array<{date: string, value: number}> | 数据点 |
| xField | string | X 轴字段名，默认 'date' |
| yField | string | Y 轴字段名，默认 'value' |
| color | string | 折线颜色 |
| loading | boolean | 加载状态 |

**交互**:
- Hover 数据点展示 Tooltip（日期 + 具体数值）
- 数据为空时展示空状态插图

#### CircleCard 圈子卡片（复用 EPIC-10）

**用途**: 展示圈子摘要信息，复用 EPIC-10 的 CircleCard 组件。

**核心 Props**（保持精简，避免膨胀）:
| 属性 | 类型 | 说明 |
|------|------|------|
| source | string | 推荐来源标识，点击时携带到详情页 URL |

**Slots**（替代额外 Props，遵循组合优于继承）:
| Slot 名称 | 说明 |
|-----------|------|
| prefix | 卡片前置区域，用于榜单排名标识（金银铜色序号、创建时间等） |
| extra | 卡片附加信息区域，用于展示额外标签或操作 |

**使用方式**:
- 推荐列表：直接使用 CircleCard，不传 prefix slot
- 榜单列表：组合使用 `RankCircleCard`（封装 CircleCard + 排名标识），或通过 prefix slot 注入排名

#### RankCircleCard 榜单圈子卡片（组合模式）

**用途**: 在 CircleCard 基础上叠加排名标识，遵循组合优于继承原则。

**Props**:
| 属性 | 类型 | 说明 |
|------|------|------|
| rank | number | 排名序号 |
| rankType | 'hot' \| 'new' | 榜单类型，决定排名展示样式 |
| source | string | 透传给 CircleCard |

**实现**: 内部渲染 CircleCard，通过 prefix slot 插入排名序号（热门榜 1-3 金银铜色，新锐榜展示创建时间）。

#### RankList 榜单列表

**用途**: 展示热门榜或新锐榜的圈子列表。

**Props**:
| 属性 | 类型 | 说明 |
|------|------|------|
| type | 'hot' \| 'new' | 榜单类型 |
| data | CircleRankVO[] | 榜单数据 |
| loading | boolean | 加载状态 |

**展示规则**:
- 热门榜：排名 1-3 使用金银铜色序号标识
- 新锐榜：展示创建时间替代排名
- 列表项可点击跳转圈子详情

### 4.2 业务组件

#### DataAnalyticsPanel 数据统计面板

**用途**: 圈子数据统计页核心面板，集成时间筛选、指标卡片、趋势图表和导出功能。

**功能**:
- 时间范围选择器（快捷选项 + 自定义区间）
- 核心指标卡片组（一行 4 个）
- 趋势图表组（3 个折线图）
- 导出按钮
- 空状态 / 错误状态 / 权限不足状态处理

**内部状态**:
- 时间范围（startDate, endDate）
- 统计数据（指标 + 图表数据）
- 加载状态、错误状态
- 导出中状态

#### RecommendationPanel 推荐面板

**用途**: 圈子列表页"推荐"Tab 的核心面板，展示推荐圈子卡片列表。

**功能**:
- 推荐圈子卡片网格布局
- 推荐为空时自动降级为热门榜单
- 推荐来源埋点（点击时上报 source）

**内部状态**:
- 推荐列表数据
- 加载状态、空状态
- 是否处于降级模式（展示兜底内容）

**降级状态转换规则**:
- **触发条件**: 推荐接口返回空数组**或**接口请求失败（网络错误、500 等），均触发降级
- **降级行为**: 自动加载热门榜单数据作为兜底内容，页面顶部显示提示"暂无推荐，为您展示热门圈子"
- **Tab 切换行为**: 降级状态下，用户手动切回"推荐"Tab 时，再次请求推荐接口（不保持降级状态），若仍为空/失败则再次降级
- **状态持久化**: `fallbackMode` 状态**不**持久化到 URL，刷新页面后重新请求推荐接口，重新判定是否降级
- **恢复条件**: 推荐接口返回非空数组时，自动退出降级模式，展示正常推荐内容，隐藏顶部提示

#### TimeRangeSelector 时间范围选择器

**用途**: 数据统计页的时间范围筛选组件。

**Props**:
| 属性 | 类型 | 说明 |
|------|------|------|
| value | { startDate: string, endDate: string } | 当前选中范围 |
| presets | Array<{label: string, value: [string, string]}> | 快捷选项，默认近 7 天/近 30 天 |
| maxRangeDays | number | 最大可选天数，默认 90。超出时 DatePicker 禁用对应日期 |

**Events**:
| 事件 | 参数 | 说明 |
|------|------|------|
| change | { startDate: string, endDate: string } | 范围变更 |

**交互**:
- 快捷选项点击后立即生效
- 自定义范围使用 DatePicker.RangePicker
- DatePicker.RangePicker 使用 `disabledDate` 函数禁用超出 `maxRangeDays` 的日期：当用户选择起始日期后，结束日期超过起始日期 + maxRangeDays 的日期变为不可选状态（灰色），前端校验优先，不依赖提交后才提示错误
- 切换范围后触发 change 事件，父组件刷新数据

### 5.1.1 API 契约规范

#### 请求参数规范

| 接口 | 参数 | 类型 | 格式 | 说明 |
|------|------|------|------|------|
| GET /analytics | startDate | string | `YYYY-MM-DD` | 开始日期，不含时区 |
| GET /analytics | endDate | string | `YYYY-MM-DD` | 结束日期，不含时区 |
| GET /analytics/export | startDate | string | `YYYY-MM-DD` | 同上 |
| GET /analytics/export | endDate | string | `YYYY-MM-DD` | 同上 |
| GET /recommend | limit | number | 整数，默认 20，最大 50 | 控制返回数量，预留扩展能力 |
| GET /ranking/hot | limit | number | 整数，默认 20 | 返回条数 |
| GET /ranking/new | limit | number | 整数，默认 20 | 返回条数 |

#### 错误响应格式

所有接口统一错误响应体结构：

```typescript
interface ApiErrorResponse {
  code: number;       // 业务错误码
  message: string;    // 用户可读的错误描述
  success: boolean;   // 固定为 false
}
```

#### 错误码定义

| HTTP 状态码 | 业务错误码 | 场景 | 前端处理 |
|-------------|-----------|------|----------|
| 401 | 401001 | 未登录 | 跳转登录页 |
| 403 | 403001 | 非圈子创建者/版主访问数据统计页 | 展示 403 页面"权限不足" |
| 404 | 404001 | 圈子不存在 | 展示 404 页面 |
| 404 | 404002 | 统计数据不存在（圈子刚创建） | 展示空状态 |
| 400 | 400001 | 时间范围超过 90 天 | Toast 提示"时间范围不能超过 90 天" |
| 400 | 400002 | startDate 晚于 endDate | Toast 提示"起始日期不能晚于结束日期" |
| 429 | 429001 | 请求频率过高 | Toast 提示"操作过于频繁，请稍后重试" |
| 500 | 500001 | 服务端内部错误 | 页面错误状态 + 重试按钮 |

#### 导出接口响应说明

- **响应类型**: 后端返回文件流（`Content-Type: text/csv`）
- **前端处理**: 使用 `fetch` + `Blob` + `<a download>` 触发下载
- **文件名格式**: `{圈子名称}_{startDate}_{endDate}.csv`（中文名需 URL 编码）
- **列头**: 中文，与页面展示的指标名称一致
- **日期格式**: `YYYY-MM-DD`
- **数值精度**: 百分比保留 1 位小数，成员数取整
- **下载方式**: 后端直接返回文件流，前端使用 `fetch` + `Blob` + `URL.createObjectURL` + `<a download>` 触发下载，无需额外文件下载库，浏览器原生下载进度即可，无需自定义进度条

#### CSV 字段映射表

**核心指标导出**:

| CSV 列头 | 数据来源 | 格式 |
|---------|---------|------|
| 成员总数 | memberTotal | 整数 |
| 新增成员数 | memberNew | 整数 |
| 成员变化百分比 | memberChange | 百分比，保留 1 位小数，如 "+5.2%" |
| 发帖总数 | postTotal | 整数 |
| 新增发帖数 | postNew | 整数 |
| 发帖变化百分比 | postChange | 百分比，保留 1 位小数 |
| 活跃用户数 | activeUserCount | 整数 |
| 活跃用户变化百分比 | activeUserChange | 百分比，保留 1 位小数 |

**趋势数据导出**（附加在核心指标下方，空行分隔）:

| CSV 列头 | 数据来源 | 格式 |
|---------|---------|------|
| 日期 | date | YYYY-MM-DD |
| 成员增长 | memberTrend[].value | 整数 |
| 内容发布 | postTrend[].value | 整数 |
| 活跃用户 | activeTrend[].value | 整数 |

### 5.2 数据流

#### 数据统计查询流程

```
用户进入数据统计页
  → 权限校验（前端路由守卫 + 后端接口校验）
  → 默认加载近 7 天数据（GET /analytics?circleId=xxx&startDate=xxx&endDate=xxx）
  → 渲染指标卡片 + 趋势图表
  → 用户切换时间范围 → 重新请求数据 → 刷新视图
  → 用户点击导出 → GET /analytics/export → 浏览器触发文件下载
```

#### 推荐浏览流程

```
用户进入圈子列表页
  → 判断是否已登录
  → 已登录：加载推荐列表（GET /recommend?limit=20）
  → 未登录：默认展示热门榜 Tab
  → 推荐列表返回 → IntersectionObserver 检测卡片可见 → 加入待上报集合
  → 延迟 500ms 批量 flush → POST /recommend/exposure（携带圈子 ID 列表）
  → 推荐接口返回空数组或请求失败 → 降级展示热门榜单 + 顶部提示
  → 用户点击推荐圈子 → POST /recommend/click（携带圈子 ID + source）
  → 跳转圈子详情页（URL 携带 source=recommend）
  → 用户离开页面 → visibilitychange 事件触发 → navigator.sendBeacon 保底上报剩余数据
```

#### 榜单浏览流程

```
用户切换到热门榜/新锐榜 Tab
  → 加载对应榜单数据（GET /ranking/hot 或 /ranking/new）
  → 渲染榜单列表
  → 用户点击圈子 → 跳转详情页（URL 携带 source=hot_rank 或 source=new_rank）
```

### 6.2 页面级状态

各页面内部使用组件 `ref` / `reactive` 管理局部状态：

- **数据统计页**: 时间选择器临时值、图表 Hover 状态、导出进度
- **圈子列表页（增强）**: 当前 Tab 选中态、卡片 Hover 态

### 6.3 缓存策略

- 统计数据：页面级缓存，切换时间范围时重新请求，离开页面时清除
- 推荐列表：会话级缓存，同一会话内切换 Tab 不重新请求（除非手动刷新）
- 榜单数据：会话级缓存，榜单每小时更新，前端不做额外缓存失效
- 曝光上报：批量上报，页面可见时延迟 500ms 合并上报

### 6.4 曝光上报实现规范

#### 触发时机

使用 `IntersectionObserver` 检测推荐卡片进入视口，而非列表渲染完成时一次性上报。阈值设为 `threshold: 0.5`（卡片 50% 面积可见时触发）。

#### 去重策略

- 维护一个 `Set<string>` 记录本次会话已上报的圈子 ID
- 同一圈子在同一页面生命周期内仅上报一次曝光
- 用户反复滚动同一区域不会重复上报

#### 批量合并上报

- 卡片进入视口后加入待上报集合
- 延迟 500ms 后批量 flush，调用 `POST /recommend/exposure`
- 若 500ms 内有新卡片进入，重置计时器（防抖模式）

#### 页面离开保底

- 监听 `document.visibilitychange` 事件，当 `visibilityState === 'hidden'` 时立即 flush
- 使用 `navigator.sendBeacon` 发送剩余未上报数据，保证页面关闭/跳转时不丢失
- 监听 `beforeunload` 事件作为备用保底

#### 上报失败处理

- 上报失败静默处理，不重试，不阻塞用户操作
- 仅在开发环境控制台打印日志，生产环境不展示

#### 虚拟滚动兼容

- 若后续引入虚拟滚动，`IntersectionObserver` 的 root 需设为滚动容器而非 `null`（默认 viewport）
- 虚拟滚动回收节点时，已进入上报集合的 ID 不受影响（去重集合独立于 DOM 生命周期）

## 10. 性能要求

### 10.1 前端性能指标

| 指标 | 目标值 |
|------|--------|
| 首屏加载时间（FCP） | < 2s（WiFi）/ < 3s（4G） |
| 最大内容绘制（LCP） | < 2.5s |
| 累积布局偏移（CLS） | < 0.1 |
| 交互响应延迟（INP） | < 200ms |

### 10.2 接口响应要求

| 接口 | P95 响应时间 |
|------|-------------|
| 数据统计查询 | < 1s |
| 数据导出 | < 5s（大数据量时分片下载） |
| 推荐列表 | < 1s |
| 榜单列表 | < 1s |
| 曝光/点击上报 | 异步，不阻塞用户操作 |

### 10.3 前端优化策略

- **路由懒加载**: 数据统计页和推荐模块按页面级别拆分路由，按需加载
- **图表按需加载**: ECharts 按需引入，仅加载折线图模块（`echarts/charts` + `echarts/components`），使用 tree-shaking 减小 bundle 体积
- **骨架屏**: 数据统计页、推荐列表、榜单列表使用骨架屏占位
- **批量曝光上报**: 延迟 500ms 合并上报推荐曝光事件，减少请求数
- **推荐列表缓存**: 同一会话内 Tab 切换不重新请求
- **图表数据点聚合**: 时间范围超过 30 天时，图表数据按周聚合展示
- **防抖**: 时间范围切换防抖 300ms，避免频繁请求
- **CSV 导出**: 大数据量时使用流式下载，展示下载进度

## 12. 待确认问题 / 默认假设

### 待确认问题

1. **图表库选型**: **[已确认]** 优先使用 ECharts。开发前需确认项目是否已有 ECharts 依赖：若有，直接使用并移除 AntV fallback 假设；若无，需在开发启动前完成引入。无论选型结果如何，TrendChart 组件必须封装图表库细节（通过 `useChart` composable 处理实例创建、resize 监听、销毁等生命周期），对外暴露统一 Props 接口，降低后续切换成本。移动端使用 ECharts 时需关注性能，建议关闭动画、减少数据点、使用 canvas 渲染器而非 svg
2. **数据统计的历史数据保留策略**: 后端统计数据保留多长时间？影响自定义时间范围的最大跨度
3. **推荐算法权重**: 前端是否需要展示推荐理由（如"因为你加入了 XX 圈子"）？
4. **导出文件大小限制**: 单次导出的 CSV 文件是否有大小限制？
5. **推荐列表分页**: 推荐列表是否需要分页或无限滚动？还是固定返回一批？

### 默认假设

1. 图表库锁定使用 ECharts，移除 AntV G2 fallback 假设。开发前需确认项目已有 ECharts 依赖，若无则在 Milestone 1 启动前完成引入
2. 自定义时间范围最大跨度假设为 90 天
3. 推荐列表假设一次性返回，不分页（初期推荐量可控）
4. 榜单列表假设一次性返回 Top 20，不分页
5. 推荐曝光上报假设批量延迟 500ms 合并上报
6. 数据统计页的图表数据假设按天聚合（非按小时）
7. 推荐来源追踪仅在前端通过 URL 参数传递，不使用独立的推荐会话存储
