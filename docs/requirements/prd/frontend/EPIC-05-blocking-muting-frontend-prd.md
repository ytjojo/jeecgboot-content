## 1. 概述

### 产品目标
- 为内容社区用户提供**拉黑**（双向切断）和**屏蔽**（单向降噪）能力，保护个人空间并优化信息流体验。
- 核心边界：**屏蔽是单向降噪，拉黑是双向切断**。前端必须在所有操作入口和确认流程中清晰传达这一区别。

### 目标用户与使用场景
- **主要用户**：所有已注册登录的内容社区用户
- **使用场景**：
  - 浏览用户主页或评论区时触发拉黑/屏蔽操作
  - 在信息流中对内容选择"不感兴趣"
  - 在隐私设置中管理黑名单、屏蔽列表、屏蔽词和屏蔽规则
- **入口来源**：用户主页、内容卡片操作菜单、评论区、隐私设置页

### 本期范围

| 包含 | 不包含 |
|------|--------|
| 拉黑操作（含确认弹窗）及黑名单管理页 | 内容审核系统、自动风控拦截 |
| 屏蔽用户操作及屏蔽列表管理 | 反小号骚扰识别 |
| 不感兴趣反馈及内容类型/话题屏蔽 | 推荐算法完整重构 |
| 屏蔽词（关键词/正则）设置 | 管理后台批量治理 |
| 临时话题屏蔽（含倒计时展示） | 私信模块底层实现（仅对接已有接口） |
| 操作确认与行为边界说明文案 | — |
| 隐私设置页聚合入口 | — |

## 3. 页面设计

### 3.1 隐私设置聚合页

**路径**：设置 > 隐私设置

**页面结构**：

```
Page (隐私设置)
  ├── 区块：关系管理
  │   ├── 入口卡片「黑名单」→ 跳转黑名单管理页
  │   └── 入口卡片「屏蔽列表」→ 跳转屏蔽列表管理页（含 Tab：屏蔽用户 / 屏蔽话题 / 屏蔽内容类型 / 临时屏蔽）
  ├── 区块：内容过滤
  │   └── 入口卡片「屏蔽词设置」→ 跳转屏蔽词管理页
  └── 区块：帮助说明
      └── 折叠面板「拉黑与屏蔽的区别」
```

**交互规则**：
- 每个入口卡片显示对应列表当前数量（如"黑名单 (3)"、"屏蔽列表 (5)"）
- 「屏蔽列表」卡片的角标数为所有 Tab 项的合计数（屏蔽用户 + 屏蔽话题 + 屏蔽内容类型 + 临时屏蔽）
- 数量为 0 时不显示角标
- 点击卡片进入对应子管理页
- 帮助说明折叠面板默认收起，展开后显示对比表格

**布局**：
- PC 端：两列卡片网格，每列 2-3 个卡片
- 移动端：单列堆叠，卡片全宽

### 3.2 黑名单管理页

**路径**：设置 > 隐私设置 > 黑名单

**页面结构**：

```
Page (黑名单管理)
  ├── 顶部：标题 + 返回按钮
  ├── 搜索区：用户昵称搜索框（可选）
  ├── 列表区：已拉黑用户列表
  │   └── 列表项（头像 | 昵称 | 拉黑时间 | 解除拉黑按钮）
  └── 底部：分页（PC 端）/ 加载更多（移动端）
```

**列表项字段**：

| 字段 | 展示方式 | 说明 |
|------|---------|------|
| 用户头像 | 圆形图片 40x40 | 点击跳转用户主页（拉黑中仍可跳转，显示拉黑提示页） |
| 用户昵称 | 文本，单行截断 | — |
| 拉黑时间 | 灰色辅助文本，格式"YYYY-MM-DD" | — |
| 操作 | 「解除拉黑」按钮 | 点击弹出确认弹窗 |

**空状态**：显示空状态插图 + 文案"暂无拉黑用户"

**交互规则**：
- 列表按拉黑时间倒序排列
- 解除拉黑前弹出确认弹窗，文案："解除拉黑后，您可以正常查看该用户内容，但之前的关注关系不会自动恢复。确定解除拉黑？"
- 确认后调用接口，成功则从列表移除并显示全局消息"已解除拉黑"
- 解除拉黑不通知对方

### 3.3 屏蔽列表管理页

**路径**：设置 > 隐私设置 > 屏蔽列表

**页面结构**：

```
Page (屏蔽列表管理)
  ├── 顶部：标题 + 返回按钮
  ├── Tab 切换：屏蔽用户 | 屏蔽话题 | 屏蔽内容类型 | 临时屏蔽
  ├── 列表区：按 Tab 分类展示
  │   ├── 屏蔽用户 Tab：用户头像 | 昵称 | 屏蔽时间 | 取消屏蔽
  │   ├── 屏蔽话题 Tab：话题名称 | 屏蔽时间 | 取消屏蔽
  │   ├── 屏蔽内容类型 Tab：类型名称 | 屏蔽时间 | 取消屏蔽
  │   └── 临时屏蔽 Tab：话题名称 | 剩余时间（倒计时） | 提前取消
  └── 批量操作栏：全选 | 批量取消屏蔽
```

**交互规则**：
- Tab 默认选中"屏蔽用户"
- 取消屏蔽单条：点击后立即生效，无需确认弹窗（屏蔽是可逆的低风险操作）
- 批量取消：勾选多条后点击"批量取消屏蔽"，弹出确认弹窗"确定取消选中的 N 项屏蔽？"
- 临时屏蔽 Tab 显示剩余天数/小时，格式如"剩余 5 天"或"剩余 3 小时"
- 临时屏蔽到期后自动从列表消失

**空状态**：每个 Tab 无数据时显示对应空状态文案，如"暂无屏蔽用户"、"暂无屏蔽话题"

### 3.4 屏蔽词设置页

**路径**：设置 > 隐私设置 > 屏蔽词设置

**页面结构**：

```
Page (屏蔽词设置)
  ├── 顶部：标题 + 返回按钮
  ├── 输入区：文本输入框 + 添加按钮
  │   └── 提示文字：支持关键词和正则表达式（正则以 / 开头和结尾）
  ├── 已添加列表：
  │   └── 列表项（屏蔽词文本 | 类型标签[关键词/正则] | 删除按钮）
  └── 说明区：屏蔽词匹配范围说明（标题和正文）
```

**交互规则**：
- 输入框支持回车快捷添加
- 添加关键词时，若已存在相同关键词，提示"该屏蔽词已存在"
- 正则表达式格式：以 `/` 开头和结尾，如 `/广告.*推广/`
- 无效正则表达式提示"正则表达式格式错误，请检查"
- 屏蔽词数量上限 100 个，达到上限提示"屏蔽词数量已达上限"
- 删除时无需确认弹窗，点击即删，支持撤销（3 秒内显示撤销提示）

**空状态**：显示文案"暂无屏蔽词，添加后包含该词的内容将被折叠"

### 3.5 拉黑/屏蔽操作入口（嵌入式组件）

以下入口嵌入现有页面，不是独立页面：

#### 3.5.1 用户主页入口

**触发位置**：用户主页右上角「更多操作」下拉菜单

**菜单项**：
- 「拉黑该用户」— 图标：禁止符号，文字颜色：红色
- 「屏蔽该用户」— 图标：眼睛划线

> **术语统一说明**：所有入口的屏蔽操作统一使用"屏蔽该用户"文案。屏蔽的实际效果是**信息流降噪**（该用户内容不再出现在信息流中），不影响主页访问、评论互动等。此边界说明在确认弹窗中以正文形式告知用户，而非在菜单文案中体现。

#### 3.5.2 内容卡片入口

**触发位置**：内容卡片右上角「···」更多操作菜单

**菜单项**：
- 「不感兴趣」
- 「屏蔽该用户」
- 「拉黑该用户」（红色文字）

#### 3.5.3 评论区入口

**触发位置**：评论项的「···」操作菜单

**菜单项**：
- 「屏蔽该用户」
- 「拉黑该用户」（红色文字）

### 3.7 被拉黑状态页面

**场景**：当用户访问已被拉黑的用户主页时

**页面展示（拉黑发起方访问被拉黑用户主页时）**：
- 显示模糊占位图 + 文案"无法查看该用户"
- 副文案："您已拉黑该用户，如需查看可在黑名单中解除拉黑"
- 底部按钮："前往黑名单管理"

**页面展示（被拉黑方访问拉黑发起方主页时）**：
- 显示标准 404 风格占位页，文案"该用户不存在"
- 不展示任何拉黑相关提示，不暴露拉黑原因
- 页面布局与真实不存在的用户主页一致，避免被拉黑方通过页面差异推断出被拉黑状态

### 3.8 屏蔽词命中内容展示

**场景**：信息流中内容被屏蔽词命中

**展示方式**：
- 内容卡片默认折叠，显示提示条："该内容包含屏蔽词，已折叠"
- 提示条右侧「展开」按钮
- 点击展开后正常显示完整内容，提示条变为「收起」
- 折叠状态下仅显示内容标题（标题本身也被屏蔽词命中时显示"***"）

```typescript
// 状态结构
interface BlockMuteState {
  // 当前用户与某目标用户的关系状态缓存
  relationCache: Record<string, {
    isBlocked: boolean;      // 我是否拉黑了对方
    isMuted: boolean;        // 我是否屏蔽了对方
    isBlockedBy: boolean;    // 对方是否拉黑了我
  }>;
  // 黑名单数量（用于隐私设置页角标）
  blacklistCount: number;
  // 屏蔽列表总数量
  muteListCount: number;
}
```

### 6.2 Store 方法

| 方法 | 说明 |
|------|------|
| `checkRelation(targetUserId)` | 查询并缓存与目标用户的关系状态 |
| `blockUser(targetUserId)` | 拉黑用户，更新缓存 |
| `unblockUser(targetUserId)` | 解除拉黑，更新缓存 |
| `muteUser(targetUserId)` | 屏蔽用户，更新缓存 |
| `unmuteUser(targetUserId)` | 取消屏蔽，更新缓存 |
| `clearRelationCache(targetUserId?)` | 清除指定或全部关系缓存 |
| `refreshCounts()` | 刷新黑名单和屏蔽列表数量 |

### 6.3 缓存策略

- 关系状态查询结果缓存在内存中，页面切换时保留
- 拉黑/屏蔽/解除操作成功后立即更新本地缓存，不等待下次查询
- 用户登出时清空缓存

## 7. 交互设计

### 7.1 操作流程

#### 拉黑流程

```
用户点击「拉黑该用户」
  → 弹出确认弹窗（说明拉黑后果）
  → 用户点击「确认拉黑」
  → 按钮显示 loading + 禁用
  → 调用拉黑接口
  → 成功：关闭弹窗 + 全局消息"已拉黑" + 刷新当前页面内容（移除该用户内容）
  → 失败：关闭弹窗 + 全局错误消息"操作失败，请重试"
```

#### 屏蔽流程

```
用户点击「屏蔽该用户」
  → 弹出确认弹窗（说明屏蔽后果）
  → 用户点击「确认屏蔽」
  → 按钮显示 loading + 禁用
  → 调用屏蔽接口
  → 成功：关闭弹窗 + 全局消息"已屏蔽，该用户内容将不再出现在信息流中"
  → 失败：关闭弹窗 + 全局错误消息
```

#### 不感兴趣流程

```
用户点击「不感兴趣」
  → 立即从信息流移除该内容卡片（乐观更新，无需等待接口响应）
  → 同时调用不感兴趣接口（静默，无确认弹窗）
  → 接口成功后，在内容卡片原位置显示气泡选项
  → 气泡选项根据当前内容数据动态生成（见下方数据来源说明）
  → 选择"屏蔽此类内容" → 调用添加屏蔽规则接口(ruleType=category) → 全局消息"已屏蔽该类型内容"
  → 选择"屏蔽该话题" → 调用添加屏蔽规则接口(ruleType=topic) → 全局消息"已屏蔽该话题"
  → 选择"知道了" → 关闭气泡
  → 接口失败：静默处理（内容已从视图移除，不影响用户操作）
```

**气泡选项数据来源**：

| 气泡选项 | 数据来源 | 说明 |
|---------|---------|------|
| 「屏蔽此类内容」 | 内容卡片数据的 `category` 字段（如"视频"、"图文"） | 仅当 `category` 字段存在时显示此选项 |
| 「屏蔽该话题」 | 内容卡片数据的 `topics` 数组字段（如["科技", "数码"]） | 仅当 `topics` 数组非空时显示此选项 |
| 「知道了」 | 固定选项 | 始终显示 |

- 气泡中仅展示当前内容实际拥有的分类/话题选项，无对应数据时隐藏对应选项
- 若 `category` 和 `topics` 均为空，气泡仅显示"知道了"
- 气泡展示位置：默认在内容卡片原位置底部向上弹出（见 11 节问题 6 默认假设）

#### 解除拉黑流程

```
黑名单列表点击「解除拉黑」
  → 弹出确认弹窗（说明关注关系不恢复）
  → 用户点击「确认解除」
  → 按钮显示 loading + 禁用
  → 调用解除拉黑接口
  → 成功：从列表移除 + 全局消息"已解除拉黑"
  → 失败：全局错误消息
```

### 7.2 交互反馈规则

| 场景 | 反馈方式 |
|------|---------|
| 操作成功 | 全局消息提示（useMessage.success），2 秒后自动消失 |
| 操作失败 | 全局错误提示（useMessage.error），3 秒后自动消失，保留用户操作上下文 |
| 接口请求中 | 按钮显示 loading 状态并禁用，防止重复提交 |
| 列表加载中 | 表格区域显示 Spin 加载指示 |
| 列表为空 | 显示 Empty 组件 + 对应引导文案 |
| 删除屏蔽词 | 点击即删 + 3 秒内显示撤销提示条 |
| 正则格式错误 | 输入框下方显示红色错误提示 |

### 7.3 操作确认策略

本节将所有操作按风险等级分为三类，每类对应不同的确认机制：

**第一类：强制确认弹窗（高风险、不可逆或影响重大）**
- **拉黑用户** — 需用户明确确认，弹窗标题"确认拉黑"，按钮为红色 danger 样式
- **解除拉黑** — 需用户明确确认，提示关注关系不自动恢复

**第二类：说明性弹窗（中风险、可逆但需告知边界）**
- **屏蔽用户** — 弹窗目的不是"确认操作"而是"说明屏蔽的实际范围和边界"，弹窗不可跳过，标题为"确认屏蔽"，按钮为默认样式
- **批量取消屏蔽** — 需确认批量操作的范围，弹窗显示选中数量

**第三类：无需弹窗（低风险、可逆或可撤销）**
- **取消单条屏蔽** — 点击即生效，无需确认
- **删除屏蔽词** — 点击即删，支持 3 秒内撤销
- **不感兴趣反馈** — 静默记录，无确认弹窗

## 9. 性能要求

| 指标 | 目标值 | 说明 |
|------|--------|------|
| 拉黑/屏蔽操作响应 | < 500ms | 用户点击确认到收到成功反馈 |
| 黑名单/屏蔽列表加载 | < 1s | 首屏列表数据加载完成 |
| 信息流过滤延迟 | 用户无感知 | 屏蔽/拉黑后刷新信息流，不出现已屏蔽内容 |
| 关系状态查询 | < 200ms | 进入用户主页时判断关系状态 |
| 屏蔽词匹配 | 后端处理 | 前端仅展示折叠/展开状态 |

### 前端优化策略

- 关系状态查询结果缓存在 Pinia Store 中，避免重复请求
- 黑名单/屏蔽列表采用分页加载，每页 20 条
- 拉黑/屏蔽操作成功后直接更新本地缓存和页面状态，不重新请求全量数据
- 信息流刷新时复用已有过滤参数，减少不必要的重新渲染
