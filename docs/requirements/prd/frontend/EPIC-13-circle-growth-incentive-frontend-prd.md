## 1. 概述

### 1.1 产品目标

建立圈子等级、成员经验值、贡献值、成就徽章和排行榜体系，形成可感知、可追踪、可持续的正向激励循环。

**成功指标（上线后 30 天）**:
- 参与成长体系的活跃成员日均发帖量提升 20%
- 参与成长体系的活跃成员 7 日留存率提升 15%
- 至少 30% 活跃成员获得 1 枚及以上成就徽章

**基线数据与测量方案**:
- **基线采集**: 上线前 30 天为基线期，采集各圈子日均发帖量、7 日留存率、活跃成员数
- **测量方式**: 通过埋点事件记录用户行为，对比基线期与上线后 30 天数据
- **埋点事件清单**:
  - `growth_exp_gain`: 经验值获得（参数：circleId, expAmount, actionType）
  - `growth_badge_earned`: 徽章获得（参数：circleId, badgeId）
  - `growth_level_up`: 等级提升（参数：circleId, oldLevel, newLevel）
  - `growth_streak_update`: 连续参与更新（参数：circleId, streakDays）
  - `growth_leaderboard_view`: 排行榜查看（参数：circleId, dimension, period）
- **灰度发布**: 建议按圈子灰度，选取 10% 圈子先行上线，隔离变量
- **排除条件**: 排除测试账号、机器人账号、异常数据（单日发帖 > 50 的账号）

| 圈子创建者 | 查看圈子等级、成长分、升级条件；通过等级权益运营圈子 |

## 2. 用户故事

### US-13.1.1 圈子等级计算与展示

**作为** 圈子成员，**我希望** 看到圈子的等级和成长进度，**以便** 了解社区发展状态并形成归属感。

**验收标准**:
1. 圈子成长分达到下一等级门槛时，圈子等级提升并展示在圈子主页、圈子详情和圈子列表卡片
2. 圈子等级提升时，圈子创建者和近 7 天活跃成员收到站内通知
3. 圈子详情页展示当前等级、成长分、下一等级所需条件、进度百分比和已解锁权益
4. 未达到下一等级时，展示距离下一等级还需补足的条件

### US-13.2.1 成员经验值系统

**作为** 圈子成员，**我希望** 通过在圈子内的活跃行为获得经验值，**以便** 展示我的参与度和成长进度。

**验收标准**:
1. 发布内容成功后获得 10 点经验值和 10 点贡献值
2. 发表评论成功后获得 3 点经验值和 3 点贡献值
3. 内容被标记为精华后获得额外 30 点经验值和 50 点贡献值
4. 个人资料页展示各圈子的经验值、贡献值、当前等级、下一等级进度和圈内排名
5. 每日经验值达到 100 点上限后，后续行为不再增加当日经验值

**每日经验值上限设计依据**:
- **防刷机制**: 上限值设定为 100 点，按当前规则（发帖 10 + 评论 3）计算，需发 10 帖或评论 33 次才能达到上限，有效防止刷经验行为
- **活跃度平衡**: 100 点上限对应每日 10 篇帖子或 33 条评论，覆盖绝大多数活跃用户的日常行为量
- **成功指标影响**: 上限值不影响"日均发帖量提升 20%"目标，因为目标针对的是活跃成员群体，而非单日峰值
- **动态调整**: 上限值固定为 100 点，暂不根据用户等级动态调整，保持规则简单透明

**经验值扣除规则**:
1. 内容删除：删除已发布内容后，扣减该内容对应的经验值（发帖 -10、精华额外 -30）
2. 内容撤回：撤回内容后，扣减该内容对应的经验值（同删除规则）
3. 内容违规：内容被管理员标记为违规后，扣减该内容对应的经验值（同删除规则）
4. 等级下降：经验值扣减导致低于当前等级门槛时，等级下降一级
5. 排行榜影响：经验值扣减后，排行榜数据实时更新，用户排名相应调整

**徽章撤销规则**:
1. 内容删除/违规导致徽章获得条件不再满足时，徽章标记为「已撤销」
2. 退出圈子后，该圈子内获得的所有徽章标记为「已撤销」
3. 重新加入圈子后，已撤销徽章需重新满足条件才能再次获得
4. 撤销徽章在徽章墙中展示为灰色，标注「已撤销」状态，不再计入已获得徽章总数

### US-13.2.2 连续参与进度

**作为** 圈子成员，**我希望** 看到自己近期参与圈子的连续进度，**以便** 有明确目标驱动我持续回访和互动。

**验收标准**:
1. 近 7 天内至少 3 天完成有效参与行为时，展示连续参与进度和已完成天数
2. 完成当日有效参与行为后，当日参与状态标记为已完成
3. 连续参与进度达到 3 天、7 天或 14 天时，展示对应成就进度或已获得徽章
4. 近 7 天没有任何有效参与行为时，展示空状态和可参与的圈内行为入口

**有效参与行为定义**:
- **发帖**: 在圈子内发布可见内容（图文、视频、动态）
- **评论**: 对圈子内内容发表评论
- **点赞**: 对圈子内内容进行点赞
- **不计入**: 浏览、分享、收藏等行为不计入有效参与（激励门槛过低，效果弱）

### US-13.3.1 成就徽章系统

**作为** 圈子成员，**我希望** 通过达成特定成就获得徽章，**以便** 展示我的特殊贡献和荣誉。

**验收标准**:
1. 累计发布 10 篇可见内容后自动获得"持续创作者"徽章
2. 累计获得 5 篇精华内容后自动获得"优质贡献者"徽章
3. 近 7 天至少 3 天完成有效参与行为后自动获得"活跃参与者"徽章
4. 获得新徽章后发送站内通知提醒
5. 个人资料页展示已获得徽章、未获得徽章、每个徽章的达成条件和当前进度

**徽章种类定义（6 种）**:

| 徽章名称 | 类型 | 达成条件 | 激励目标 |
|---------|------|----------|----------|
| 持续创作者 | 内容型 | 累计发布 10 篇可见内容 | 鼓励内容产出 |
| 优质贡献者 | 质量型 | 累计获得 5 篇精华内容 | 鼓励高质量内容 |
| 活跃参与者 | 活跃型 | 近 7 天至少 3 天完成有效参与行为 | 鼓励持续参与 |
| 圈内新星 | 活跃型 | 近 7 天经验值排名前 10 | 激励短期冲刺 |
| 内容里程碑 | 里程碑型 | 累计发布 100 篇可见内容 | 鼓励长期贡献 |
| 社交达人 | 社交型 | 成功邀请 5 人加入圈子 | 鼓励圈子扩张 |

**徽章类型说明**:
- **内容型**: 鼓励基础内容产出，降低参与门槛
- **质量型**: 引导高质量内容，提升社区价值
- **活跃型**: 保持用户活跃度，促进留存
- **里程碑型**: 长期成就激励，增强用户粘性
- **社交型**: 促进圈子增长，扩大用户规模

### US-13.3.2 圈子内排行榜

**作为** 圈子成员，**我希望** 查看圈子内的成员排行榜，**以便** 了解活跃成员并获得参与目标。

**验收标准**:
1. 排行榜展示经验值榜、贡献值榜和发帖数榜 3 个维度
2. 每个榜单展示 Top 50 成员；不足 50 人时展示全部
3. 当前成员未进入 Top 50 时，在榜单底部展示当前成员排名、数值和距离上一名的差距
4. 圈子没有符合条件的排行成员时，展示空状态和参与入口
5. 支持本周、本月、累计 3 个周期切换

**反作弊与公平性规则**:
1. **质量加权**: 精华帖在经验值榜和贡献值榜中权重更高（精华帖经验值 ×2）
2. **新用户机会**: "本周"和"本月"排行周期性重置数据，给新用户提供排名机会
3. **并列排名处理**: 多人数值相同时，按以下顺序排序：
   - 优先按经验值排序（经验值相同再按贡献值）
   - 贡献值相同再按发帖数
   - 所有数值相同则按注册时间排序（先注册者排名靠前）
4. **异常数据过滤**: 排除测试账号、机器人账号、单日发帖 > 50 的异常账号

├─────────────────────────────────────────────┤

- 等级标识使用颜色区分：L1 灰色、L2 绿色、L3 蓝色、L4 橙色、L5 金色
- 进度条可点击，点击后展开详细的成员规模、内容贡献、活跃互动三类指标分项
- 已解锁权益用勾选图标，未解锁权益用锁定图标并置灰
- 最高等级（L5）时进度条显示为满格，文案变为「已达最高等级」
- 私有圈子：未加入成员不展示成长区块，显示「加入圈子后查看」

- 当前成员在 Top 50 内时，该行使用主题色背景高亮
- 当前成员未进入 Top 50 时，榜单底部固定展示「我的排名」区域，与主列表用分隔线隔开
- 排名前三名使用金银铜色序号标识
- 列表使用虚拟滚动（成员数量可控在 50 以内，暂不强制，但预留接口）
- 空状态：圈子无符合条件的排行成员时，展示空状态插图和「去发帖」「去评论」操作入口
- 列表项点击头像或用户名可跳转该成员的个人资料页

import { defHttp } from '/@/utils/http/axios';

enum Api {
  CircleLevel = '/api/v1/content/user/growth/level/info',

}

export function getMemberGrowth(circleId: string, userId: string) {
  return defHttp.get({ url: Api.MemberGrowth, params: { circleId, userId } });
}

export function getCircleBadges(circleId: string, userId: string) {
  return defHttp.get({ url: Api.Badges, params: { circleId, userId } });
}

export function getLeaderboard(circleId: string, dimension: string, period: string, currentUserId: string) {
  return defHttp.get({
    url: Api.Leaderboard,
    params: { circleId, dimension, period, currentUserId },
  });

### 6.2 缓存策略

| 数据 | 缓存粒度 | 失效策略 |
|------|---------|----------|
| 圈子等级信息 | circleId | 页面切换时重新请求；等级进度 30 分钟更新一次，无需轮询 |
| 成员成长信息 | circleId | 进入页面时请求；成长行为完成后通过 WebSocket 或轮询刷新 |
| 徽章列表 | circleId | 进入页面时请求；获得新徽章后刷新 |
| 排行榜 | circleId + dimension + period | 切换维度/周期时请求；每小时更新一次，页面停留时可选轮询 |

### 6.3 通知处理

等级提升和徽章获得的通知通过已有站内通知系统（WebSocket）推送。前端在收到通知后：
- 触发对应圈子的等级信息和成长信息刷新
- 弹出轻提示（Toast）展示通知内容
- 通知中心列表自动更新

## 10. 性能要求

| 指标 | 目标 | 说明 |
|------|------|------|
| 个人成长页加载时间 | P95 < 1 秒 | 含 3 个接口并行请求 |
| 排行榜页加载时间 | P95 < 1 秒 | 单接口请求，数据量可控（Top 50） |
| 徽章墙加载时间 | P95 < 1 秒 | 单接口请求，数据量可控（4 种徽章） |
| 经验值/贡献值展示延迟 | P95 < 10 秒 | 成长行为完成后数值更新 |
| 等级进度数据延迟 | <= 30 分钟 | 后端定时任务刷新，前端无需特殊处理 |

**前端优化措施**:
- 接口并行请求：个人成长页的等级信息、成长信息、徽章摘要使用 `Promise.all` 并行加载
- 骨架屏：所有页面首屏使用 Skeleton 组件，减少白屏感知
- 路由懒加载：成长相关页面使用动态 import 按需加载
- 图片懒加载：徽章图标使用 `loading="lazy"`
- 排行榜列表：Top 50 数据量小，暂不做虚拟滚动，预留接口
