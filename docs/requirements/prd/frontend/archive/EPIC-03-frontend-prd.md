# EPIC-03 - 徽章积分成长 前端 PRD

## 1. 概述

### 1.1 功能目标

构建内容社区的勋章、积分和成长等级激励体系前端，为用户提供完整的勋章获取与佩戴展示、积分获取与消耗、等级成长与权益展示、经验衰减与降级保护的交互体验。

### 1.2 用户价值

- **成就感**：通过勋章分类、进度展示和佩戴展示，让用户获得可视化的成就反馈
- **长期激励**：积分获取/消耗 + 等级成长形成闭环，驱动用户持续活跃
- **身份标识**：等级徽章、评论特效、功能特权增强用户身份感
- **公平透明**：积分明细可查、衰减规则公示、降级保护保障用户体验

### 1.3 与后端 API 的关系

前端完全依赖后端 RESTful API 驱动，不持有本地状态持久化。所有数据变更（积分增减、勋章佩戴、等级升降）通过 API 调用后端服务，前端负责展示和交互反馈。后端统一奖励编排服务保证幂等性、每日上限和事务原子性。

### 1.4 技术栈假设

- Vue 3 + TypeScript（与 JeecgBoot 前端体系一致）
- Pinia 状态管理
- Ant Design Vue 组件库
- HTTP 请求使用项目已有 Axios 封装

### 3.2 页面详情

#### P-01 我的勋章页

**路由**: `/user/badges`
**入口**: 个人主页 > 我的勋章 Tab 或个人中心导航

**页面结构**:
1. **页面头部**: "我的勋章"标题 + 用户当前佩戴勋章概览（最多 5 个图标 + "去佩戴"按钮）
2. **分类筛选栏**: 全部 / 成就类 / 身份类 / 活动类 / 关系类 / 已过期（Tab 切换）
3. **勋章卡片网格**: 响应式网格布局，每个勋章卡片包含：
   - 勋章图标（支持动态特效勋章的 Lottie/WebP 动画）
   - 勋章名称
   - 获得状态标签：未获得 / 已获得 / 已过期 / 已回收
   - 进度条（未获得时显示，如 "7/10"）
4. **勋章详情弹窗**: 点击勋章卡片弹出，展示：
   - 勋章大图标 + 名称 + 分类标签
   - 获取条件描述
   - 当前进度（进度条 + 数字）
   - 获得时间（已获得时）
   - 有效期（有期限的勋章显示到期时间）
   - 回收原因（已回收时显示）

**交互逻辑**:
- 进入页面时调用 `GET /api/v1/content/user/growth/badge/catalog` 获取勋章分类列表
- 切换分类 Tab 时可前端过滤或重新请求
- 点击勋章卡片调用 `GET /api/v1/content/user/growth/badge/{badgeCode}/detail` 获取详情
- 已过期分类调用 `GET /api/v1/content/user/growth/badge/catalog?status=expired`

#### P-02 勋章佩戴设置页

**路由**: `/user/badges/wear`
**入口**: 我的勋章页 > "去佩戴"按钮 或 个人主页 > 佩戴设置

**页面结构**:
1. **已佩戴区域**: 顶部展示当前佩戴的勋章（最多 5 个），每个勋章卡片右上角有移除按钮
2. **可选勋章列表**: 下方展示用户已获得但未佩戴的活跃勋章
3. **操作栏**: "保存佩戴设置"按钮 + 已选数量提示（如 "已选 3/5"）

**交互逻辑**:
- 进入页面时调用 `GET /api/v1/content/user/growth/badge/worn` 获取当前佩戴列表，调用 `GET /api/v1/content/user/growth/badge/owned` 获取已拥有勋章
- 用户点击可选勋章添加到已佩戴区域（最多 5 个）
- 点击移除按钮从已佩戴区域移除
- 保存时调用 `PUT /api/v1/content/user/growth/badge/wear` 提交勋章 ID 列表
- 校验：不可重复选择、不可选择已过期/已回收勋章、不超过 5 个
- 保存成功后返回我的勋章页

#### P-03 积分明细页

**路由**: `/user/points`
**入口**: 个人主页 > 积分明细 或 个人中心导航

**页面结构**:
1. **积分概览卡片**: 当前积分余额（大数字）+ 今日获得积分 + 今日剩余上限
2. **筛选栏**: 类型筛选（全部/获取/消耗）+ 时间范围筛选（日期范围选择器）+ 重置按钮
3. **明细列表**: 按时间倒序，无限滚动或分页加载，每条记录展示：
   - 变更时间
   - 变更类型图标（绿色向上箭头=获取，红色向下箭头=消耗）
   - 变更数量（+10 或 -50）
   - 来源说明（如 "登录奖励"、"兑换虚拟装扮"）
   - 变更后余额
4. **空状态**: 无记录时展示空状态插画和提示文字

**交互逻辑**:
- 进入页面时调用 `GET /api/v1/content/user/growth/point/ledger?page=1&size=20`
- 筛选时调用 `GET /api/v1/content/user/growth/point/ledger?type=earn&startDate=xxx&endDate=xxx&page=1&size=20`
- 滚动加载更多，触底时 page+1
- 积分余额调用 `GET /api/v1/content/user/growth/point/balance`

#### P-04 兑换商城页

**路由**: `/points/exchange`
**入口**: 积分明细页 > "去兑换" 或 个人中心 > 兑换商城

**页面结构**:
1. **积分余额横幅**: 顶部展示当前积分余额
2. **商品分类**: Tab 或标签切换（全部/虚拟装扮/曝光券/其他权益）
3. **商品卡片网格**: 每个商品卡片包含：
   - 商品图标/图片
   - 商品名称
   - 所需积分
   - 库存状态（有库存/库存不足/已售罄）
   - "立即兑换"按钮
4. **兑换确认弹窗**: 展示商品信息、所需积分、当前余额，确认/取消按钮
5. **兑换成功弹窗**: 展示兑换结果，可跳转查看积分明细

**交互逻辑**:
- 进入页面时调用 `GET /api/v1/content/user/growth/point/exchange/goods` 获取商品列表
- 点击"立即兑换"弹出确认弹窗，检查余额是否充足
- 确认兑换调用 `POST /api/v1/content/user/growth/point/exchange/order` 创建兑换订单
- 积分不足时按钮置灰并显示"积分不足"
- 库存为 0 时按钮置灰并显示"已售罄"

#### P-05 功能解锁与礼物页

**路由**: `/points/features`
**入口**: 个人中心 > 功能解锁 或 帖子/评论区 > 赠送礼物入口

**页面结构**:
1. **功能解锁区域**: 展示可解锁功能列表，每个功能卡片包含：
   - 功能图标 + 名称 + 描述
   - 解锁所需积分
   - 解锁状态（未解锁/已解锁/已过期）
   - "解锁"按钮
2. **虚拟礼物区域**: 展示可赠送的礼物列表，每个礼物卡片包含：
   - 礼物动画/图标 + 名称
   - 所需积分
   - "赠送"按钮
3. **赠送弹窗**: 选择接收人 + 附言输入 + 确认赠送

**交互逻辑**:
- 功能解锁调用 `POST /api/v1/content/user/growth/point/feature/unlock` 提交解锁请求
- 已解锁功能再次使用不再扣积分，通过 `GET /api/v1/content/user/growth/point/feature/unlock/status` 查询状态
- 赠送礼物调用 `POST /api/v1/content/user/growth/point/gift/send` 提交赠送请求
- 接收人不能是自己

#### P-06 等级与成长页

**路由**: `/user/growth`
**入口**: 个人主页 > 等级 Tab 或个人中心导航

**页面结构**:
1. **等级展示卡片**:
   - 当前等级大图标（LV.X）+ 等级名称
   - 经验值进度条（当前经验值 / 升级阈值）
   - 距离下一级还需 N 经验值
2. **积分与成长值对比区**:
   - 积分余额（可消耗）
   - 成长值（只增不减，驱动等级）
   - 分别展示两者的获取来源说明
3. **等级权益展示**: 当前等级享有的权益列表：
   - 等级徽章标识
   - 评论框特效（如有）
   - 文件上传额度
   - 视频清晰度上限
   - 创建话题额度
   - 优先客服
   - 推荐加权百分比
4. **等级历史**: 最近等级变更记录（升级/降级时间、原因）
5. **衰减说明入口**: "了解经验衰减规则"链接，跳转衰减说明页

**交互逻辑**:
- 进入页面时调用 `GET /api/v1/content/user/growth/summary` 获取成长概览
- 等级权益调用 `GET /api/v1/content/user/growth/level/benefits` 获取当前等级权益配置
- 等级历史调用 `GET /api/v1/content/user/growth/level-history` 获取变更记录
- 升级时通过 WebSocket 或轮询触发展示升级祝贺弹窗（全屏动画 + "恭喜您升级到 LV.X！"）

#### P-07 衰减说明页

**路由**: `/user/growth/decay-info`
**入口**: 等级与成长页 > "了解经验衰减规则"链接

**页面结构**:
1. **规则说明区**: 图文并茂展示衰减规则：
   - 连续 30 天未登录后开始衰减
   - 衰减速率说明（每天 X%）
   - 7 天降级保护期机制
   - 活跃后停止衰减并恢复等级
2. **个人状态区**（已登录用户）:
   - 当前衰减状态（正常/衰减中/保护期）
   - 距离开始衰减还有多少天
   - 保护期剩余天数（如在保护期中）
3. **FAQ 区域**: 常见问题解答

**交互逻辑**:
- 进入页面时调用 `GET /api/v1/content/user/growth/decay-state` 获取个人衰减状态
- 衰减规则说明从配置或静态内容获取

### 5.2 数据流

```
用户行为事件（登录/点赞/评论/发布等）
  ↓ 后端统一奖励编排服务处理
  ↓
前端展示奖励提示（ToastReward 组件）
  - 调用轮询/WebSocket 获取最新积分/经验值
  - 或在行为完成后从响应中获取奖励信息

用户进入勋章页
  → API-01 获取勋章分类列表
  → 渲染分类 Tab + 勋章卡片网格
  → 点击勋章 → API-02 获取详情 → 弹窗展示

用户佩戴勋章
  → API-04 获取已拥有勋章
  → API-03 获取当前佩戴
  → 用户选择 → API-05 保存
  → 后续主页/帖子/评论读取 API-03 展示佩戴

用户查看积分明细
  → API-06 获取余额概览
  → API-07 获取明细列表（分页）
  → 筛选时重新调用 API-07

用户兑换商品
  → API-08 获取商品列表
  → API-06 检查余额
  → 确认 → API-09 创建订单
  → 成功后刷新 API-06 余额

用户查看等级
  → API-13 获取成长概览
  → API-14 获取等级权益
  → API-15 获取变更历史
```

## 6. 状态管理

### 6.1 Pinia Store 设计

| Store 名称 | 职责 | 核心状态 |
|-----------|------|---------|
| useBadgeStore | 勋章相关状态 | catalog, wornBadges, selectedBadge, loading |
| usePointStore | 积分相关状态 | balance, ledger, todayEarned, todayRemainingCap, filters, loading |
| useGrowthStore | 成长等级相关状态 | summary, levelBenefits, levelHistory, decayState, loading |
| useExchangeStore | 兑换商城相关状态 | goods, orderResult, loading |

### 6.2 状态流转规则

- **缓存策略**: 勋章分类、等级权益等低频变更数据在页面级缓存，离开页面清空
- **实时性**: 积分余额、经验值在涉及变更的交互后立即刷新（兑换、解锁后）
- **奖励提示**: 通过全局事件总线或 Pinia 全局 store 接收奖励事件，ToastReward 组件订阅并展示
- **跨页面一致性**: 佩戴勋章变更后，通过事件通知其他展示区域（主页、帖子）刷新

## 9. 性能要求

### 9.1 加载性能

| 指标 | 目标值 | 说明 |
|------|-------|------|
| 勋章页面首屏加载 | < 2s | 勋章图标使用懒加载和 CDN 缓存 |
| 积分明细首屏加载 | < 1.5s | 首页只加载 20 条，滚动加载更多 |
| 兑换商城首屏加载 | < 2s | 商品图片使用缩略图，懒加载 |
| 等级页面首屏加载 | < 1.5s | 等级图标使用 CSS/SVG 渲染 |

### 9.2 运行性能

| 指标 | 目标值 | 说明 |
|------|-------|------|
| 奖励提示展示延迟 | < 300ms | 行为完成后快速反馈 |
| 勋章佩戴保存响应 | < 1s | 立即反馈展示更新 |
| 积分兑换响应 | < 2s | 包含事务确认 |
| 勋章动态特效 | 不阻塞页面渲染 | 使用 requestIdleCallback 加载 |

### 9.3 优化策略

- **勋章图标**: 使用 WebP 格式 + CDN，动态特效勋章使用 Lottie 按需加载
- **积分明细**: 虚拟滚动处理大量列表数据
- **兑换商品图片**: 使用缩略图 + 懒加载
- **API 缓存**: 勋章分类、等级权益等低频数据使用 HTTP 缓存或 Pinia 缓存
- **代码分割**: 兑换商城、功能解锁等低频页面使用路由懒加载
- **减少重渲染**: 勋章列表使用 `v-memo` 或 `shallowRef` 优化
