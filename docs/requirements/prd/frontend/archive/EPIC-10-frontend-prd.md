# EPIC-10 - 圈子核心 前端 PRD

## 1. 概述

### 1.1 背景

内容社区系统当前缺乏基于兴趣聚合的垂直社区能力，用户只能浏览全局内容流，缺少结构化的兴趣分组和归属感。本 PRD 为圈子核心功能（EPIC-10）定义前端实现需求，覆盖圈子创建与设置、成员管理、圈子搜索三大特性。

### 1.2 目标

- 提供完整的圈子创建、设置与管理前端界面
- 实现多方式加入圈子的用户交互流程
- 提供圈子搜索与浏览发现能力
- 支持 PC 端和移动端响应式布局

### 1.3 技术栈约束

- 基于现有 JeecgBoot Vue3 前端框架
- 使用 Vue 3 + TypeScript + Ant Design Vue
- 遵循项目现有的组件、路由、状态管理规范

### 1.4 范围

| 包含 | 不包含 |
|------|--------|
| 圈子创建表单与设置页 | 内容置顶/精华 |
| 成员加入/退出交互 | 公告/@成员 |
| 角色管理与成员治理 UI | 审核机制工作台 |
| 圈子搜索与结果展示 | 数据统计面板 |
| 响应式布局适配 | 成长激励体系 |

### 1.5 里程碑

| 阶段 | 时间 | 交付内容 |
|------|------|----------|
| Milestone 1 | 第 1-2 周 | 圈子创建与设置页面 |
| Milestone 2 | 第 2-3 周 | 成员管理页面与治理操作 |
| Milestone 3 | 第 3-4 周 | 搜索页面、全量联调与测试 |

## 3. 页面设计

### 3.1 页面列表

| 页面 | 路由 | 说明 |
|------|------|------|
| 圈子列表页 | `/circle` | 展示公开圈子列表，支持搜索入口 |
| 圈子创建页 | `/circle/create` | 圈子创建表单 |
| 圈子详情页 | `/circle/:id` | 圈子主页，展示圈子信息、成员、内容 |
| 圈子设置页 | `/circle/:id/settings` | 编辑圈子信息（仅创建者） |
| 成员管理页 | `/circle/:id/members` | 成员列表与治理操作 |
| 圈子搜索结果页 | `/circle/search` | 搜索结果展示 |

### 3.2 页面详情

#### 3.2.1 圈子列表页 `/circle`

**功能描述**: 展示公开圈子列表，提供搜索入口和创建入口。

**页面元素**:
- 顶部搜索栏：关键词输入框 + 搜索按钮
- 创建圈子按钮（已登录用户可见）
- 圈子卡片列表：展示图标、名称、简介摘要、成员数、分类标签
- 加入状态标识：已加入显示"已加入"标签，未加入显示"加入"按钮
- 分页组件
- 空状态：无圈子时展示引导创建的空状态插画

**交互逻辑**:
- 点击搜索栏跳转至搜索结果页
- 点击"加入"按钮根据加入方式触发不同交互（直接加入/弹窗输入密码/提示申请审核）
- 点击卡片进入圈子详情页
- 下拉加载更多或点击分页

#### 3.2.2 圈子创建页 `/circle/create`

**功能描述**: 圈子创建表单，支持填写基础信息、设置隐私类型和加入方式。

**页面元素（分步表单）**:

**第一步 - 基础信息**:
- 圈子名称输入框（必填，最大 30 字符，唯一性实时校验）
- 圈子简介文本域（必填，最大 200 字符）
- 图标上传（可选，1:1 比例，JPG/PNG/WebP，最大 2MB，建议 200x200）
- 封面图上传（可选，16:9 比例，JPG/PNG/WebP，最大 5MB，建议 750x422）
- 分类标签选择（可选，支持多选）

**第二步 - 隐私与加入方式**:
- 隐私类型单选：公开 / 私有 / 密码保护
- 加入方式单选（根据隐私类型联动）：
  - 公开：直接加入 / 申请审核
  - 私有：申请审核 / 邀请加入
  - 密码保护：密码加入（显示密码输入框）
- 密码输入框（仅密码保护类型显示，BCrypt 加密传输）

**第三步 - 确认与提交**:
- 信息预览摘要
- 提交按钮

**交互逻辑**:
- 名称输入失焦时触发唯一性校验，重复时显示红色提示
- 敏感词检测失败时显示"内容包含敏感词，需人工审核"提示
- 必填项未填时高亮对应字段并阻止提交
- 图标/封面图上传前校验格式、比例和大小
- 提交成功后跳转至圈子详情页
- 提交失败时保留已填内容，显示错误提示

#### 3.2.3 圈子详情页 `/circle/:id`

**功能描述**: 圈子主页，展示圈子信息和操作入口。

**页面元素**:
- 封面图区域（无封面时使用默认渐变背景）
- 圈子图标、名称、简介
- 成员数、分类标签、隐私类型标识
- 操作按钮区：
  - 非成员：加入按钮（根据加入方式展示不同交互）
  - 已加入普通成员：退出按钮
  - 创建者/版主：设置按钮、成员管理按钮
- 内容区域（本 Epic 暂为占位，EPIC-11 实现）

**交互逻辑**:
- 加入按钮点击：
  - 直接加入：调用 API，成功后刷新页面状态
  - 密码保护：弹出密码输入弹窗，验证通过后加入
  - 申请审核：弹出确认弹窗，提交后提示"申请已提交"
  - 邀请加入：提示"该圈子仅限邀请加入"
- 退出按钮：二次确认弹窗，确认后退出
- 创建者不可退出，提示"创建者不可退出圈子，请先转让或解散圈子"
- 已是成员时按钮显示"已加入"，点击提示"您已是圈子成员"
- 圈子满员时加入按钮置灰，提示"圈子已满员，无法加入"

#### 3.2.4 圈子设置页 `/circle/:id/settings`

**功能描述**: 创建者编辑圈子信息（名称不可修改）。

**页面元素**:
- 圈子名称（只读展示，不可修改）
- 简介文本域（可编辑）
- 图标上传（可替换）
- 封面图上传（可替换）
- 分类标签选择（可修改）
- 隐私类型选择（可修改，展示变更影响说明）
- 加入方式选择（可修改）
- 保存按钮

**交互逻辑**:
- 仅创建者可访问，非创建者访问时跳转回详情页
- 名称字段灰显且显示"名称不可修改"提示
- 保存时校验敏感词和格式规范
- 保存成功后显示成功提示

#### 3.2.5 成员管理页 `/circle/:id/members`

**功能描述**: 展示成员列表，支持角色管理和治理操作。

**页面元素**:
- 顶部统计栏：总成员数、当前页筛选条件
- 筛选区域：按角色筛选（全部/创建者/版主/成员）、按状态筛选（正常/禁言/已移除）
- 成员列表（表格形式，PC 端）/ 成员卡片列表（移动端）：
  - 用户头像、昵称
  - 角色标签（创建者/版主/成员）
  - 状态标签（正常/禁言中-显示剩余时间/已移除）
  - 加入时间
  - 操作列（下拉菜单）：
    - 设为版主 / 取消版主（仅创建者可见）
    - 禁言（选择时长：1小时/24小时/7天/永久）
    - 解除禁言
    - 移除成员
    - 查看治理记录
- 分页组件

**交互逻辑**:
- 角色变更、禁言、移除操作需二次确认弹窗
- 禁言弹窗需选择时长和填写原因（选填）
- 创建者角色不可变更，操作列不显示变更选项
- 版主不可操作其他版主，操作列屏蔽相关选项
- 治理记录弹窗：按时间倒序展示操作历史（操作者、动作、原因、时间）
- 操作成功后刷新列表

#### 3.2.6 圈子搜索结果页 `/circle/search`

**功能描述**: 展示搜索结果。

**页面元素**:
- 顶部搜索栏（保留搜索关键词，支持修改后重新搜索）
- 搜索结果列表：与圈子列表页卡片样式一致
- 空结果状态：显示"无搜索结果"，提供"浏览公开圈子"入口
- 搜索服务异常状态：显示"搜索暂时不可用"，保留浏览入口

**交互逻辑**:
- 进入页面自动执行搜索
- 修改关键词后回车或点击搜索按钮触发新搜索
- 结果卡片交互与列表页一致
- 仅展示公开类型圈子

### 5.2 数据流

#### 圈子创建流程

```
用户填写表单
  → 前端校验（必填项、格式、图片规范）
  → 名称唯一性校验（失焦调用 check-name API）
  → 提交创建 API（POST /content/circle）
  → 后端敏感词检测 + 名称唯一性校验
  → 成功 → 跳转圈子详情页
  → 失败 → 显示错误提示（名称重复/敏感词/其他错误）
```

#### 加入圈子流程

```
用户点击"加入"
  → 前端判断加入方式
  → 直接加入：调用 join API → 成功刷新页面
  → 密码保护：弹出密码输入弹窗 → 用户输入密码 → 调用 join API（带密码）→ 成功/密码错误提示
  → 申请审核：弹出确认弹窗 → 调用 join API → 提示"申请已提交"
  → 邀请加入：直接提示"该圈子仅限邀请加入"
  → 异常处理：已是成员/满员/黑名单 → 显示对应提示
```

#### 成员管理流程

```
创建者/版主进入成员管理页
  → 加载成员列表（GET /members，支持筛选参数）
  → 执行操作（角色变更/禁言/移除）
  → 前端二次确认弹窗
  → 调用对应 API
  → 成功 → 刷新列表 + 成功提示
  → 失败 → 显示错误提示（权限不足等）
```

#### 搜索流程

```
用户输入关键词 → 点击搜索/回车
  → 跳转搜索结果页（携带 keyword 参数）
  → 调用搜索 API（GET /content/circle/search?keyword=xxx）
  → 成功 → 渲染结果列表
  → 无结果 → 展示空状态 + 浏览入口
  → 服务异常 → 展示降级提示 + 浏览入口
```

### 5.3 核心数据结构

```typescript
// 圈子
interface CircleVO {
  id: string;
  name: string;
  description: string;
  iconUrl: string;
  coverUrl: string;
  category: string;
  privacyType: 'PUBLIC' | 'PRIVATE' | 'PASSWORD';
  joinType: 'DIRECT' | 'APPROVAL' | 'INVITE' | 'PASSWORD';
  creatorId: string;
  memberCount: number;
  maxMemberCount: number;
  status: string;
  createTime: string;
}

// 创建请求
interface CircleCreateReq {
  name: string;
  description: string;
  iconUrl?: string;
  coverUrl?: string;
  category?: string;
```

## 6. 状态管理

### 6.1 全局状态（Pinia Store）

#### useCircleStore

```typescript
interface CircleStoreState {
  // 当前圈子详情
  currentCircle: CircleVO | null;
  // 当前用户在该圈子的角色
  currentRole: 'CREATOR' | 'MODERATOR' | 'MEMBER' | null;
  // 当前用户在该圈子的成员状态
  currentMemberStatus: 'ACTIVE' | 'MUTED' | 'REMOVED' | null;
  // 公开圈子列表缓存
  publicCircleList: CircleVO[];
  // 搜索结果缓存
  searchResults: CircleSearchResultVO[];
  // 加载状态
  loading: boolean;
}
```

**Actions**:
- `fetchCircleDetail(circleId)` - 获取圈子详情并更新当前角色/状态
- `fetchPublicCircleList(params)` - 获取公开圈子列表
- `searchCircles(keyword)` - 搜索圈子
- `joinCircle(circleId, req?)` - 加入圈子
- `leaveCircle(circleId)` - 退出圈子
- `clearSearchResults()` - 清除搜索缓存

### 6.2 页面级状态

各页面内部使用组件 `ref` / `reactive` 管理局部状态：

- **创建页**: 表单数据、当前步骤、校验状态、提交状态
- **成员管理页**: 筛选条件、成员列表、分页参数、操作弹窗状态
- **搜索结果页**: 搜索关键词、结果列表、加载/空/异常状态

### 6.3 缓存策略

- 圈子详情：页面级缓存，路由切换时清除
- 公开圈子列表：分页缓存，操作（加入/退出）后失效刷新
- 搜索结果：不缓存，每次搜索重新请求
- 成员列表：操作后刷新当前页

## 7. 交互设计

### 7.1 加入圈子交互矩阵

| 隐私类型 | 加入方式 | 用户交互 | 反馈 |
|----------|----------|----------|------|
| 公开 | 直接加入 | 点击"加入"按钮 | 按钮变为"已加入"，Toast 成功提示 |
| 公开 | 申请审核 | 点击"加入"按钮 | Toast "申请已提交，请等待审核" |
| 私有 | 申请审核 | 点击"加入"按钮 | Toast "申请已提交，请等待审核" |
| 私有 | 邀请加入 | 点击"加入"按钮 | Toast "该圈子仅限邀请加入" |
| 密码保护 | 密码加入 | 点击"加入" → 弹窗输入密码 → 确认 | 密码正确：加入成功；密码错误：弹窗内提示 |

### 7.2 操作确认交互

| 操作 | 确认方式 | 确认文案 |
|------|----------|----------|
| 退出圈子 | 二次确认弹窗 | "确定退出该圈子？退出后需重新申请加入" |
| 移除成员 | 二次确认弹窗 | "确定移除该成员？移除后可重新申请加入" |
| 禁言成员 | 专用弹窗（时长+原因） | 选择时长，填写原因（选填），确认执行 |
| 变更角色 | 二次确认弹窗 | "确定将 [用户名] 设为版主/取消版主？" |

### 7.3 错误反馈

| 场景 | 反馈方式 | 文案 |
|------|----------|------|
| 名称已存在 | 表单字段下方红色提示 | "该圈子名称已存在，请修改" |
| 敏感词触发 | 表单字段下方橙色提示 | "内容包含敏感词，需人工审核" |
| 必填项缺失 | 表单字段高亮 + 提示 | "请填写 [字段名]" |
| 权限不足 | Toast 错误提示 | "权限不足，仅创建者可管理角色" |
| 创建者退出 | Toast 提示 | "创建者不可退出圈子，请先转让或解散圈子" |
| 圈子满员 | 按钮置灰 + Tooltip | "圈子已满员，无法加入" |
| 已是成员 | Toast 提示 | "您已是圈子成员" |
| 黑名单用户 | Toast 提示 | "无法加入该圈子" |
| 搜索服务异常 | 页面状态展示 | "搜索暂时不可用" + 浏览入口 |

### 7.4 表单校验规则

| 字段 | 规则 |
|------|------|
| 圈子名称 | 必填，最大 30 字符，唯一性校验（失焦触发，防抖 500ms） |
| 简介 | 必填，最大 200 字符 |
| 图标 | 可选，JPG/PNG/WebP，1:1 比例，最大 2MB |
| 封面图 | 可选，JPG/PNG/WebP，16:9 比例，最大 5MB |
| 密码 | 密码保护类型必填，最小 6 字符，最大 20 字符 |

## 8. 响应式设计

### 8.1 断点定义

| 断点 | 宽度范围 | 设备类型 |
|------|----------|----------|
| xs | < 576px | 手机竖屏 |
| sm | 576px - 767px | 手机横屏 |
| md | 768px - 991px | 平板 |
| lg | 992px - 1199px | 小桌面 |
| xl | >= 1200px | 大桌面 |

最低适配宽度：375px。

### 8.2 各页面响应式策略

#### 圈子列表页
- **PC (>= 992px)**: 卡片网格布局，每行 3-4 个卡片
- **平板 (768-991px)**: 卡片网格布局，每行 2 个卡片
- **手机 (< 768px)**: 单列卡片列表，搜索栏置顶固定

#### 圈子创建页
- **PC**: 表单居中展示，最大宽度 640px，左右留白
- **手机**: 表单全宽展示，分步导航改为顶部步骤条，图片上传区域自适应缩放

#### 圈子详情页
- **PC**: 封面图区域高度 300px，信息区横向布局（左侧信息 + 右侧操作）
- **手机**: 封面图区域高度 200px，信息区纵向堆叠，操作按钮底部固定

#### 成员管理页
- **PC**: 表格形式展示成员列表，操作列为下拉菜单
- **手机**: 卡片形式展示成员，每张卡片包含头像、昵称、角色标签和操作按钮

#### 搜索结果页
- 布局策略与圈子列表页一致

### 8.3 移动端特殊处理

- 表单字段和操作按钮不得横向溢出
- 弹窗在移动端使用全屏抽屉（Drawer）替代居中弹窗
- 长列表使用虚拟滚动或分页加载
- 图片上传支持相机拍照和相册选择

## 9. 性能要求

### 9.1 前端性能指标

| 指标 | 目标值 |
|------|--------|
| 首屏加载时间（FCP） | < 2s（WiFi）/ < 3s（4G） |
| 最大内容绘制（LCP） | < 2.5s |
| 累积布局偏移（CLS） | < 0.1 |
| 交互响应延迟（INP） | < 200ms |

### 9.2 接口响应要求

| 接口 | P95 响应时间 |
|------|-------------|
| 圈子搜索 | < 500ms |
| 创建/更新圈子 | < 800ms |
| 加入/退出圈子 | < 800ms |
| 角色变更/禁言/移除 | < 800ms |
| 成员列表查询 | < 500ms |

### 9.3 前端优化策略

- **路由懒加载**: 圈子模块按页面级别拆分路由，按需加载
- **图片优化**: 图标和封面图使用 WebP 格式优先，支持 CDN 加载和缩略图
- **防抖/节流**: 搜索输入防抖 300ms，名称唯一性校验防抖 500ms，滚动加载节流
- **虚拟列表**: 成员列表在数据量 > 100 条时启用虚拟滚动
- **骨架屏**: 圈子列表页、详情页、成员列表使用骨架屏占位
- **接口缓存**: 圈子详情页缓存当前圈子信息，避免重复请求
- **错误降级**: 搜索服务不可用时展示降级 UI，不影响其他功能
