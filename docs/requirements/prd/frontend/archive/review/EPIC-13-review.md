# EPIC-13 圈子成长激励 — 前端 PRD 审核报告

> **审核人**: John (Product Manager)
> **审核日期**: 2026-06-02
> **审核视角**: 产品管理 — 用户价值、优先级、业务对齐

## 总体评价

PRD 结构完整，页面设计和组件拆分清晰，API 接口定义详尽，技术细节充分。但从产品管理角度看，存在几个关键问题：**用户价值假设未经验证**、**成功指标缺乏基线数据和测量方案**、**部分用户旅程缺失**、**优先级标注为"低"但内容量达 3 周工期**。整体质量为中等偏上，需要在业务价值论证和范围精简上做调整后方可进入开发。

## 优点

1. **成功指标量化明确**: 30 天内日均发帖量提升 20%、7 日留存提升 15%、30% 活跃成员获徽章 — 这些指标可衡量、有时限，体现了数据驱动思维。
2. **范围定义清晰**: "本期包含"与"本期不包含"边界明确，有效控制了范围蔓延风险。
3. **组件复用考虑充分**: 明确列出了复用组件来源（Page、Description、Modal、CountTo 等），避免重复造轮子。
4. **空状态设计完整**: 每个页面都考虑了空状态和引导入口，降低用户迷失风险。
5. **待确认问题与默认假设**: 主动列出 5 个待确认问题和 7 个默认假设，体现了工程严谨性。
6. **响应式设计覆盖三端**: 桌面/平板/移动端的断点规则和适配要点具体可执行。

## 问题与建议

### 1. 🔴 成功指标缺乏基线数据和测量方案

PRD 提出"日均发帖量提升 20%"和"7 日留存率提升 15%"，但未说明：
- 当前基线是多少？（当前日均发帖量？当前 7 日留存率？）
- 如何归因于成长体系而非其他因素？是否需要 A/B 测试或灰度发布？
- 测量工具是什么？埋点方案在哪里？

**建议**: 补充基线数据，明确数据采集方式（埋点事件清单），并说明是否需要灰度发布来隔离变量。如果无法做 A/B 测试，至少需要定义"上线前后对比"的时间窗口和排除条件。

### 2. 🔴 经验值/贡献值扣除规则未定义

US-13.2.1 验收标准只定义了获得规则（发帖 +10、评论 +3、加精 +30/+50），但测试要点中提到了"经验值回退：内容删除/撤回/违规后对应经验值扣减"和"徽章撤销：内容违规/删除/退出圈子后徽章撤销"。

这些扣除/撤销规则在用户故事和 API 设计中完全未定义：
- 删除内容后经验值是否扣减？扣减多少？是否会导致等级下降？
- 退出圈子后经验值是否清零？重新加入后是否恢复？
- 徽章撤销后是否可以重新获得？
- 如果经验值被扣减导致排名变化，排行榜如何处理？

**建议**: 在 US-13.2.1 中补充扣除规则的验收标准，明确边界条件。这是产品规则定义，不是技术细节。

### 3. 🔴 优先级标注为"低"与内容量矛盾

史诗标注优先级为"低"，但预计周期 3 周，包含 4 个页面、10 个新组件、4 个 API 对接、完整的状态管理和缓存策略。这在低优先级任务中属于重度投入。

**建议**: 要么将优先级调整为"中"以匹配实际投入，要么做范围裁剪 — 例如将徽章墙（P2）和排行榜（P2）推迟到下一期，本期只做圈子等级展示和个人成长信息页（P1），将工期压缩到 1.5 周。

### 4. 🟡 Jobs-to-be-Done 分析缺失

PRD 列出了用户角色和使用场景，但未深入分析用户的根本需求（Job-to-be-Done）：

- **圈子成员的 JTBD 是什么？** 是"获得社交认可"还是"有明确目标驱动参与"？前者需要社交分享能力（如分享徽章到朋友圈），后者需要目标可视化（如进度提醒）。PRD 两者都未涉及。
- **圈子创建者的 JTBD 是什么？** 是"了解社区健康度"还是"找到运营抓手"？当前设计只提供了信息展示，没有运营建议或行动指引。

**建议**: 补充每个角色的 JTBD 分析，确保功能设计真正解决用户问题而非仅展示数据。

### 5. 🟡 缺少社交分享和外部激励闭环

成长体系的核心价值在于**可感知、可炫耀**。当前设计完全局限于圈子内部：
- 徽章无法分享到个人主页、朋友圈或外部社交平台
- 排行榜仅在圈子内可见，无法在个人主页展示成就
- 等级提升没有社交传播机制

这意味着成长体系的激励效果完全依赖用户主动访问圈子页面查看，激励触达率会很低。

**建议**: 至少支持将徽章和排名展示在用户个人主页，让其他用户可见。这是最低成本的社交激励。

### 6. 🟡 徽章种类过少，激励梯度不足

目前仅定义了 4 种徽章：持续创作者、优质贡献者、活跃参与者、圈内新星。对于一个 3 周工期的成长体系来说，这个数量偏少：

- 没有里程碑型徽章（如"第 100 篇帖子"、"获得 1000 个点赞"）
- 没有社交型徽章（如"邀请 5 人加入"、"被 10 人关注"）
- 没有时间型徽章（如"连续 30 天活跃"、"周年纪念"）
- "圈内新星"的达成条件在徽章墙中标注为"近 7 天经验值排名前 10"，但在 US-13.3.1 中未定义

**建议**: 补充"圈内新星"的验收标准到 US-13.3.1。考虑增加 2-3 种徽章以形成更完整的激励梯度，特别是里程碑型徽章对长期留存更有价值。

### 7. 🟡 每日经验值上限 100 点的设计依据未说明

US-13.2.1 规定每日经验值上限 100 点，按当前规则（发帖 10 + 评论 3）计算，需要发 10 帖或评论 33 次才能达到上限。这个上限是否合理？

- 如果目标是防止刷经验，上限太高（10 帖已经很多）
- 如果目标是鼓励持续参与，上限太低（活跃用户很容易触达）
- 上限值是否会影响"日均发帖量提升 20%"的目标？

**建议**: 说明上限值的设计依据，并评估其对成功指标的影响。考虑是否需要根据用户等级动态调整上限。

### 8. 🟡 排行榜缺乏反作弊和公平性设计

排行榜直接按经验值/贡献值/发帖数排名，但未考虑：
- 刷帖行为：低质量内容大量发布是否应该获得排名？
- 历史累积优势：累计排行对新成员不公平，可能打击新用户积极性
- 并列排名处理：多人数值相同时如何排序？

**建议**: 考虑引入质量加权（如精华帖权重更高），或在"本周/本月"排行中重置数据以给新用户机会。至少在 PRD 中明确并列排名的排序规则。

### 9. 🟢 连续参与进度的"有效参与行为"未精确定义

US-13.2.2 多次提到"有效参与行为"，但未定义具体包含哪些行为。是仅限发帖和评论？还是也包括点赞、浏览、分享？

- 如果太宽泛（包含浏览），激励效果弱
- 如果太严格（仅发帖和评论），用户可能觉得门槛高

**建议**: 明确定义"有效参与行为"的范围，建议至少包含发帖、评论、点赞三种行为。

### 10. 🟢 圈子创建者和版主的差异化功能不足

目标用户中列出了圈子创建者和版主，但 PRD 中几乎没有为他们设计差异化功能：
- 创建者只能看到等级信息，没有运营建议
- 版主"查看圈子成长数据，辅助运营决策" — 但具体能看到什么数据？与普通成员有何不同？
- 没有创建者/版主专属的成长数据面板

**建议**: 如果创建者和版主的使用场景与普通成员相同，可以从目标用户中移除以简化范围。如果确实需要差异化功能，应补充具体设计。

## 缺失用户旅程

1. **新用户首次加入圈子的成长引导旅程**: 新成员加入圈子后，如何发现成长体系？是否有新手引导或首次进入的引导弹窗？PRD 中个人成长页的入口是"圈子详情页「我的成长」"，但新用户可能根本不知道这个入口的存在。

2. **用户从"了解规则"到"采取行动"的旅程**: 用户看到等级进度后，如何知道具体该做什么来提升？当前设计展示了"距下一等级还需: 成员 +20, 内容 +50"，但没有直接的行动入口（如"邀请好友"、"发帖"按钮）。

3. **用户等级下降或徽章撤销时的体验旅程**: 测试要点提到了经验值回退和徽章撤销，但 PRD 未设计等级下降时的 UI 状态、撤销通知的文案和时机、以及用户如何恢复。

4. **跨圈子成长对比旅程**: 用户可能加入多个圈子，如何在不同圈子间切换查看成长信息？当前设计是按 circleId 独立展示，但没有全局视图。

5. **用户流失前的挽留旅程**: 当用户连续参与进度中断（如连续 3 天未活跃），是否有提醒机制？PRD 只设计了空状态展示，没有主动触达。

## 优先级建议

基于用户价值和实现成本的权衡，建议调整优先级：

| 优先级 | 功能 | 理由 |
|--------|------|------|
| P0（必须有） | 圈子等级展示（详情页内嵌） | 最基础的成长感知，实现成本低 |
| P0（必须有） | 个人成长概览（经验值、贡献值、排名） | 核心激励数据，用户价值高 |
| P1（应该有） | 连续参与进度 | 促进留存的关键机制 |
| P1（应该有） | 徽章系统（精简版，2-3 种） | 成就激励，但可先精简种类 |
| P2（可以有） | 排行榜 | 锦上添花，可推迟到下期 |
| P2（可以有） | 徽章墙完整版 | 与精简版徽章系统配合即可 |

**建议**: 本期聚焦 P0 + P1，工期压缩至 2 周。P2 功能放入下一期迭代，与用户反馈结合再决定是否调整。

## 总结

EPIC-13 前端 PRD 在技术设计层面质量较高，组件拆分、API 定义、状态管理、响应式设计等细节完备。但从产品管理角度，存在三个核心问题需要解决后方可进入开发：

1. **补充成功指标的基线数据和测量方案**，否则上线后无法验证成长体系是否有效。
2. **定义经验值扣除和徽章撤销的产品规则**，这是影响用户体验的关键边界条件。
3. **精简范围或调整优先级**，当前"低优先级 + 3 周工期"的组合不合理，建议聚焦 P0/P1 功能压缩至 2 周。

此外，建议补充 JTBD 分析、社交分享能力、新用户引导旅程，以提升成长体系的实际激励效果。成长体系的本质是行为心理学产品，数据展示只是手段，关键是让用户"想要"参与而非"知道"自己参与了多少。
