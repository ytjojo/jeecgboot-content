# EPIC-02 个人资料与主页个性化 -- 前端 PRD 审核报告

> **审核人**: Sally (UX Designer)
> **审核日期**: 2026-06-02
> **审核视角**: 用户体验 -- 交互设计、可用性、响应式

## 总体评价

该 PRD 在功能定义和状态管理方面较为完整，覆盖了编辑资料、头像上传、隐私设置、主页定制、认证展示和历史记录六大模块，且对边界情况和错误处理有较好的规划。但从用户体验角度审视，存在若干交互细节缺失、用户引导不足、以及移动端体验考虑不够深入的问题，需要在开发前补充完善。

## 优点

- 状态反馈设计表（7.2 节）清晰列出了各类场景对应的反馈方式和组件选型，减少了开发歧义
- 频率限制的交互处理（置灰 + hover 提示 + 顶部剩余次数提示条）兼顾了透明度和易用性
- 响应式适配在 4.3 节按页面逐一给出了 PC/平板/移动端的布局方案，且移动端有特殊处理（如 ActionSheet 替代 Select、全屏裁剪弹窗、增大触摸热区）
- 头像裁剪弹窗支持滚轮缩放和滑块缩放双模式，覆盖了桌面和触屏场景
- 认证标识组件封装考虑了多认证并存的优先级排序逻辑
- 未保存修改离开时的确认弹窗（4.4 节）和危险操作二次确认机制设计合理
- 4 个核心交互流程（7.1 节）以流程图形式呈现，覆盖了成功和多种失败路径

## 问题与建议

1. **[高] 编辑资料页缺少"放弃修改"与"保存"的交互优先级说明。** 页面顶部导航栏和底部操作栏都有"保存"和"取消"按钮，但未说明两处按钮的行为是否一致。如果用户在页面中间位置，顶部和底部按钮的视觉权重和可达性不同。建议明确：移动端底部固定操作栏优先，顶部仅作为视觉引导；或者统一为一处操作区域，避免歧义。

2. **[高] 主页模块拖拽排序在移动端的交互方案不完整。** 8.3 节提到"移动端拖拽区域增大触摸热区（最小 44px）"，但未描述移动端拖拽的具体手势交互。vuedraggable 在移动端默认使用 touch 事件，长按拖拽 vs 点击拖拽手柄的行为差异很大。建议补充：移动端采用长按（300ms）触发拖拽模式，拖拽过程中列表项增加阴影提升反馈，同时支持键盘无障碍操作。

3. **[高] 可见性设置缺少"批量操作"能力。** 隐私设置页（3.7 节）有 10 个字段需要逐一设置可见性，当用户想将大部分字段设为同一级别时操作成本很高。建议增加"一键全部设为"快捷操作，或在页面顶部提供"默认可见性"设置，新字段自动继承默认值。

4. **[中] 头像裁剪弹窗缺少预览效果。** F2 裁剪弹窗只有裁剪框和缩放控制，但用户无法预览裁剪后在个人主页、评论区、消息列表等场景的实际显示效果。建议在裁剪弹窗右侧或底部增加小尺寸圆形/方形预览，模拟头像在不同场景的呈现。

5. **[中] 主题色设置缺少无障碍对比度校验。** F4 允许用户自定义主题色，但未提及对比度校验。用户选择浅色主题色可能导致白色文字不可见，深色主题色可能导致深色背景上的文字不可读。建议：在选择主题色后自动计算与白色/黑色文字的对比度，不满足 WCAG AA 标准时给出警告或自动调整文字颜色。

6. **[中] 认证标识在昵称过长时的截断策略未说明。** F6 提到"认证标识紧跟昵称右侧显示，与昵称同行"，但当昵称接近 20 字符上限时，加上多个认证标识（最多 7 种），在移动端可能导致换行或溢出。建议补充：昵称超长时的截断策略（省略号 + Tooltip 显示完整昵称），以及认证标识过多时的折叠方案（如显示前 2 个 + "+N" 徽标）。

7. **[中] 历史记录的"恢复"操作缺少视觉对比。** F9 历史记录列表中，用户难以判断恢复后的头像/昵称与当前值的差异。建议：头像历史增加"当前"标签标识当前使用项，昵称历史中当前昵称高亮或标记，帮助用户做出判断。

8. **[中] 主页预览区（F4）在移动端折叠后的"预览"入口不够直观。** 4.3 节提到移动端"预览区折叠，点击'预览'展开全屏预览"，但未描述"预览"按钮的位置和样式。建议放在背景图/主题色设置区域下方，使用带图标的文字按钮，让用户明确感知可以预览效果。

9. **[低] 编辑资料页表单字段缺少"可选/必填"的统一标注规范。** F1 只提到昵称标红星号，但简介、性别、生日等选填字段没有任何标识。建议统一遵循：必填字段标红星号，选填字段不标或标注"选填"，保持一致性。同时建议在表单标题旁或表单顶部添加"标 * 为必填"的说明。

10. **[低] 频率限制的提示条位置和样式不够统一。** 编辑资料页在"表单顶部"显示剩余次数提示条，隐私设置页在频率受限时也是"黄色提示条"，但两处的触发时机和展示逻辑不同（一个是页面加载时始终显示，一个是触发限制后才显示）。建议统一为：页面加载时显示剩余次数（绿色/正常），接近上限时变为黄色警告，达到上限时变为红色并禁用操作。

11. **[低] 审核中状态下编辑资料页的"返回"行为未说明。** 当资料处于待审核状态时，表单字段不可编辑，但用户仍可进入该页面。如果用户此时点击"返回"，是否需要弹出确认框？由于表单本身不可编辑（isDirty 始终为 false），应直接返回，但建议在 PRD 中显式说明，避免开发实现不一致。

12. **[低] 历史记录页缺少"加载更多"或分页方案。** 虽然最多 20 条数据量不大，但如果后续调整保留数量上限，当前一次性加载全部数据的方式可能不够灵活。建议预留分页或"加载更多"的扩展能力。

## 交互设计建议

- **表单保存的即时反馈增强**: 编辑资料保存成功后返回个人中心，用户需要在新页面才能看到变更效果。建议保存成功后在当前页面短暂展示"before/after"对比（如头像变化的过渡动画），让用户获得即时成就感。
- **头像上传的拖拽支持**: 除了点击触发文件选择，建议支持将图片文件直接拖拽到头像区域，降低操作步骤。
- **隐私设置的可视化辅助**: 纯文字的四级可见性选项（公开/仅关注者/互关可见/仅自己）认知负担较高。建议在每个选项旁增加小型图标（如地球、人群、双向箭头、锁），PRD 中已有 emoji 图标说明，建议确认在 Ant Design Vue Select 组件中如何实现为自定义 Option 渲染。
- **主页模块配置的实时预览**: 当前方案是修改后保存再查看效果。建议参考主页设置页的实时预览区，在模块配置页面也增加主页预览缩略图，模块排序变化时实时反映到预览中。
- **骨架屏的差异化设计**: PRD 多处提到骨架屏，但不同页面的数据结构差异大（表单 vs 列表 vs 配置面板），建议为每种页面类型设计对应的骨架屏形状，而非使用通用骨架屏，提升加载感知的真实感。
- **移动端底部操作栏的安全区域适配**: 移动端固定在屏幕底部的操作按钮需要适配 iPhone 底部安全区域（Safe Area），避免按钮被 Home Indicator 遮挡。

## 总结

该 PRD 功能覆盖面广、状态流转清晰、API 设计规范，是一份质量较高的前端需求文档。主要改进方向集中在：移动端拖拽交互的细化、隐私设置的操作效率优化、认证标识的空间适配策略、以及主题色的无障碍校验。建议在开发启动前补充上述高优先级问题的方案，中低优先级问题可在迭代中逐步完善。
