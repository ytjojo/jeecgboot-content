# EPIC-08 - 反馈与支持 前端 PRD

## 1. 概述

### 1.1 背景

内容社区当前缺少便捷的举报入口、透明的申诉流程和高效的客服支持。用户遇到违规内容时无法及时举报，受到处罚后没有公平的申诉渠道，遇到问题时也缺乏自助和人工支持。本 PRD 定义前端需要实现的反馈与支持体系，包括举报系统、申诉系统、帮助中心和客服通道四大模块。

### 1.2 目标

- 建立完善的用户反馈、举报申诉和客服支持前端界面
- 提供直观易用的举报、申诉流程交互
- 实现帮助中心自助服务，降低客服压力
- 高等级用户（LV.15+）在客服通道中享受优先体验

### 1.3 技术栈

- 框架：Vue 3 + TypeScript
- UI 组件库：Ant Design Vue
- 状态管理：Pinia
- 路由：Vue Router 4
- HTTP 请求：Axios
- 实时通信：WebSocket（客服对话）

### 1.4 依赖关系

- EPIC-01（用户认证）：用户需先登录才能使用举报、申诉、客服功能
- EPIC-03（积分成长等级体系）：高等级用户（LV.15+）在客服队列中享受优先权
- EPIC-09（用户状态与治理）：处罚通知中包含申诉入口，处罚撤销后恢复用户状态

### 1.5 里程碑

| 里程碑 | 时间 | 内容 |
|--------|------|------|
| M1 | Week 2 | 举报系统（举报表单 + 举报列表 + 举报详情） |
| M2 | Week 4 | 申诉系统 + 帮助中心 |
| M3 | Week 6 | 客服通道（智能客服 + 人工客服 + 会话历史） |

## 2. 用户故事

### 2.1 举报系统用户故事

**US-8.1.1: 举报违规内容**
- 作为注册用户，我希望对违规用户/内容/评论发起举报，选择举报类型并上传证据，以便参与社区治理
- 验收标准：
  - 点击"举报"按钮弹出举报表单
  - 可选择举报类型（色情、暴力、诈骗、骚扰等），每种类型显示说明文字
  - 可上传图片/视频证据（最大 10MB）
  - 提交成功后显示"举报已提交，我们将尽快处理"并返回举报编号
  - 同一用户对同一对象只能举报一次，重复举报提示"您已举报过该内容，请勿重复举报"

**US-8.1.2: 查看举报处理进度**
- 作为注册用户，我希望跟踪举报的处理进度，以便了解举报是否被处理
- 验收标准：
  - "我的举报"页面展示所有举报记录列表
  - 每条记录显示举报对象、举报类型、提交时间和当前状态（待处理/处理中/已处理）
  - 已处理的举报显示处理结果（已删除内容/已警告用户/举报不成立）
  - 处理完成后收到系统通知
  - 对处理结果不满意可点击"申诉"进入申诉流程

### 2.2 申诉系统用户故事

**US-8.2.1: 对处罚发起申诉**
- 作为注册用户，我希望对被删除内容、被封禁账户、被扣除积分/勋章等处罚发起申诉，以便获得复核机会
- 验收标准：
  - 处罚通知中包含"申诉"入口
  - 点击"申诉"后填写申诉理由并提交，生成申诉编号
  - "我的申诉"页面展示申诉列表和处理状态
  - 审核中的申诉显示"审核中"及预计处理时间
  - 审核完成后收到通知（维持原判/撤销处罚）
  - 同一事件最多申诉 3 次，超过提示"您已申诉多次，请耐心等待最终结果"

### 2.3 帮助中心用户故事

**US-8.3.1: 访问帮助中心与搜索**
- 作为用户，我希望访问帮助中心（常见问题、使用指南、功能更新日志），以便自助解决问题
- 验收标准：
  - 帮助中心页面展示常见问题分类（账号安全、举报申诉、隐私设置等）
  - 支持关键词搜索，响应时间 <500ms
  - 搜索无结果时显示"未找到相关文章"并提示联系客服
  - 文章详情页展示步骤说明和截图
  - 支持"有用/无用"反馈，标记无用时提示"是否联系客服？"

**US-8.3.2: 查看功能更新日志**
- 作为用户，我希望查看功能更新日志，以便了解平台最新变化
- 验收标准：
  - 更新日志页面按时间倒序展示版本更新记录
  - 每个版本展示版本号、更新日期、新增功能、优化内容和修复问题
  - 支持搜索功能名称查找相关更新记录
  - 有新版本发布时，首次登录提示"查看最新版本更新"

### 2.4 客服通道用户故事

**US-8.4.1: 智能客服与人工客服**
- 作为用户，我希望有客服通道（智能客服 + 人工客服入口），高等级用户可优先，以便获得支持
- 验收标准：
  - 点击"客服"入口首先启动智能客服机器人
  - 智能客服无法解决时可点击"转人工"
  - LV.15+ 用户在人工客服队列中优先，预计等待时间 <2 分钟
  - 普通用户预计等待时间 <5 分钟
  - 人工客服接入后可进行实时对话
  - 会话结束后邀请用户对服务评分

**US-8.4.2: 客服会话历史记录**
- 作为用户，我希望查看客服会话历史记录，以便回顾之前的咨询内容
- 验收标准：
  - "客服记录"页面展示按时间倒序排列的会话列表
  - 可查看单次会话的完整对话记录
  - 可基于历史会话点击"继续咨询"创建新会话
  - 超过 30 天的会话提示"历史记录仅保留 30 天"

### 3.2 页面详情

#### 3.2.1 举报弹窗（ReportModal）

**触发方式**：在内容卡片、评论区、用户主页点击"举报"按钮

**布局结构**：
- 标题：举报违规内容
- Step 1 - 选择举报类型：
  - 类型列表（单选）：色情低俗、暴力血腥、诈骗欺诈、骚扰辱骂、虚假信息、侵权内容、其他
  - 每种类型显示简要说明文字
- Step 2 - 填写举报详情：
  - 举报描述（文本域，必填，最大 500 字）
  - 证据上传区（可选）：
    - 支持拖拽上传图片/视频
    - 支持点击上传
    - 单文件最大 10MB
    - 最多上传 5 个文件
    - 显示上传进度条
    - 已上传文件可预览/删除
- Step 3 - 确认提交：
  - 展示已填写的举报摘要
  - "提交举报"按钮

**状态交互**：
- 提交中：按钮显示 loading 状态
- 提交成功：显示成功提示"举报已提交，我们将尽快处理"，附举报编号，自动关闭弹窗
- 提交失败：显示错误提示，可重试
- 重复举报：显示"您已举报过该内容，请勿重复举报"，关闭弹窗

**设计要点**：
- 弹窗宽度 520px
- 步骤条（Step）引导用户完成举报流程
- 举报类型选择使用卡片式单选，而非下拉框
- 证据上传区域支持拖拽，有虚线边框提示

#### 3.2.2 我的举报（ReportListPage）

**布局结构**：
- 页面标题：我的举报
- 筛选栏：
  - 状态筛选：全部 / 待处理 / 处理中 / 已处理（Tab 切换）
  - 时间范围：最近 7 天 / 最近 30 天 / 全部
- 举报列表：
  - 卡片式列表，每条记录包含：
    - 举报对象标题/摘要
    - 举报类型标签
    - 提交时间
    - 状态标签（待处理-橙色 / 处理中-蓝色 / 已处理-绿色）
    - "查看详情"链接
- 空状态：暂无举报记录
- 分页：每页 10 条

#### 3.2.3 举报详情（ReportDetailPage）

**布局结构**：
- 面包屑导航：我的举报 > 举报详情
- 举报信息卡片：
  - 举报编号
  - 举报类型
  - 举报描述
  - 证据列表（可预览图片/播放视频）
  - 提交时间
- 处理进度时间线：
  - 待处理 → 处理中 → 已处理
  - 每个节点显示时间
- 处理结果（已处理时显示）：
  - 结果说明（已删除内容 / 已警告用户 / 举报不成立等）
  - 处理时间
  - 处理人（可选显示）
- 操作区：
  - "申诉"按钮（仅已处理且对结果不满意时显示）
  - "返回列表"按钮

#### 3.2.4 申诉弹窗（AppealModal）

**触发方式**：
- 从处罚通知中的"申诉"入口点击触发

**布局结构**：
- 标题：发起申诉

  - 处罚类型
  - 处罚原因
  - 处罚时间
- 申诉理由填写：
  - 文本域（必填，最大 1000 字）
  - 附件上传（可选，支持图片，最大 10MB，最多 3 张）
- 申诉次数提示：
  - "本次为第 X 次申诉（最多 3 次）"
- 提交按钮

**状态交互**：
- 提交成功：显示"申诉已提交，审核结果将在 48 小时内通知您"，附申诉编号
- 超过次数限制：显示"您已申诉多次，请耐心等待最终结果"，禁用提交

#### 3.2.5 我的申诉（AppealListPage）

- 页面标题：我的申诉

- 申诉列表（卡片式）：

## 4. 组件设计

### 4.1 通用组件

#### 4.1.1 StatusTag 状态标签组件

**用途**：统一展示举报/申诉的状态标签

**Props**：
- `status`: 'pending' | 'processing' | 'processed' | 'approved' | 'rejected' | 'reviewing'
- `type`: 'report' | 'appeal'

**样式映射**：
- 待处理（pending）：橙色标签
- 处理中（processing）：蓝色标签
- 已处理（processed）：绿色标签
- 审核中（reviewing）：蓝色标签
- 已通过（approved）：绿色标签
- 已驳回（rejected）：红色标签

#### 4.1.2 EvidenceUploader 证据上传组件

**用途**：举报和申诉中的文件上传

**Props**：
- `accept`: 接受的文件类型（图片/视频）
- `maxSize`: 单文件最大大小（默认 10MB）
- `maxCount`: 最大上传数量（默认 5）
- `type`: 使用场景（'report' | 'appeal'）

**功能**：
- 支持拖拽上传
- 支持点击上传
- 显示上传进度
- 文件类型校验
- 文件大小校验
- 已上传文件预览（图片缩略图/视频封面）
- 已上传文件删除

#### 4.1.3 Timeline 进度时间线组件

**用途**：展示举报/申诉的处理进度

**Props**：
- `steps`: 时间线步骤数组
- `current`: 当前步骤索引

**功能**：
- 步骤节点（圆点 + 连接线）
- 每个步骤显示标题、描述、时间
- 已完成步骤高亮，未完成步骤灰色

#### 4.1.4 ChatBubble 聊天气泡组件

**用途**：客服对话中的消息气泡

**Props**：
- `message`: 消息内容
- `sender`: 'user' | 'bot' | 'agent'
- `timestamp`: 消息时间
- `avatar`: 头像 URL
- `name`: 发送者名称

### 5.2 数据流

#### 5.2.1 举报数据流

```
用户点击举报
  → ReportModal 打开
  → 用户选择举报类型、填写描述、上传证据
  → 证据文件上传到 OSS，返回 URL
  → 调用 check-duplicate API 检查重复
  → 若重复，显示提示，结束
  → 若不重复，调用 create-report API
  → 成功后显示举报编号，关闭弹窗
  → 用户可在"我的举报"查看进度
  → 举报状态变更时通过通知系统推送消息
```

#### 5.2.2 申诉数据流

```
用户收到处罚通知
  → 通知中点击"申诉"
  → AppealModal 打开，展示处罚信息
  → 调用 count API 查询已申诉次数
  → 若已达 3 次，显示提示，禁用提交
  → 若未达限制，用户填写理由、上传附件
  → 调用 create-appeal API
  → 成功后显示申诉编号
  → 用户可在"我的申诉"查看审核进度
  → 审核结果通过通知系统推送
```

#### 5.2.3 帮助中心数据流

```
用户进入帮助中心
  → 调用 categories API 加载分类列表
  → 调用 hot-articles API 加载热门文章
  → 用户可点击分类查看分类下文章
  → 用户可输入关键词搜索（调用 search API）
  → 用户点击文章进入详情（调用 article detail API）
  → 用户反馈"有用/无用"（调用 feedback API）
  → 点击"无用"后提示联系客服
```

#### 5.2.4 客服对话数据流

```
用户点击"客服"入口
  → 打开 ChatPage，初始化 WebSocket 连接
  → 系统启动智能客服模式
  → 用户发送消息 → 调用 send API → 智能客服回复
  → 用户点击"转人工" → 调用 transfer API
  → 系统显示排队界面，轮询 queue-status API
  → 人工客服接入 → WebSocket 推送状态变更
  → 实时对话：用户发送消息 → WebSocket 推送 → 客服回复
  → 会话结束 → WebSocket 推送 session_end → 弹出评分弹窗
  → 用户评分 → 调用 rate API
```

## 7. 交互设计

### 7.1 举报交互流程

1. **触发举报**：用户在内容/评论/用户页面点击"举报"按钮
2. **选择类型**：弹出举报弹窗，卡片式展示举报类型选项，选中高亮
3. **填写详情**：输入举报描述，上传证据（拖拽或点击）
4. **确认提交**：展示举报摘要，点击"提交举报"
5. **提交反馈**：
   - 成功：Toast 提示"举报已提交，我们将尽快处理"+ 举报编号，弹窗自动关闭
   - 重复：Toast 提示"您已举报过该内容，请勿重复举报"
   - 失败：Toast 提示错误信息，可重试
6. **查看进度**：进入"我的举报"页面，点击查看详情，进度时间线展示
7. **结果通知**：举报处理完成后，系统通知弹窗 + 站内消息

### 7.2 申诉交互流程

1. **触发申诉**：从处罚通知点击"申诉"入口
2. **查看处罚**：弹窗展示处罚信息（只读）
3. **填写理由**：输入申诉理由，上传附件（可选）
4. **次数检查**：显示"本次为第 X 次申诉（最多 3 次）"，超过限制时禁用提交
5. **提交申诉**：点击"提交申诉"
6. **提交反馈**：
   - 成功：Toast 提示"申诉已提交，审核结果将在 48 小时内通知您"
   - 超限：Toast 提示"您已申诉多次，请耐心等待最终结果"
7. **查看审核**：进入"我的申诉"页面，审核中状态显示预计处理时间
8. **审核结果**：通知弹窗告知结果（通过/驳回 + 原因）

### 7.3 帮助中心交互流程

1. **进入帮助中心**：导航菜单或底部链接进入
2. **浏览分类**：点击分类卡片查看分类下文章列表
3. **搜索文章**：输入关键词，实时搜索（防抖 300ms），结果高亮关键词
4. **查看文章**：点击文章进入详情页，阅读步骤说明和截图
5. **文章反馈**：
   - 点击"有用"：按钮变为已选中状态，显示"感谢您的反馈"
   - 点击"无用"：弹出确认框"这篇文章对您没有帮助，是否联系客服？"

### 7.4 客服对话交互流程

1. **进入客服**：点击"客服"入口，打开客服对话页
2. **智能客服**：机器人欢迎消息 + 快捷回复按钮
3. **对话交互**：
   - 用户输入消息，按 Enter 或点击发送
   - 机器人回复支持富文本（可点击链接）
   - 快捷回复按钮点击后自动发送
4. **转人工**：
   - 点击"转人工"按钮
   - 切换为排队等待界面，显示队列位置和预计时间
   - LV.15+ 用户显示"优先服务"标识
5. **人工对话**：
   - 人工客服接入，顶部状态栏切换为"人工客服中"
   - 实时对话（WebSocket）
   - 显示客服头像和昵称
6. **会话结束**：
   - 客服结束会话，WebSocket 推送 session_end
   - 弹出评分弹窗（1-5 星 + 文字评价）
   - 提交评分后显示"感谢您的评价"

### 7.5 通用交互规范

- **加载状态**：所有列表页首次加载显示骨架屏（Skeleton），翻页显示 Loading
- **空状态**：列表为空时显示空状态插图 + 引导文字
- **错误处理**：网络错误显示 Toast 提示，支持重试
- **防重复提交**：所有提交按钮在请求期间禁用，显示 Loading 状态
- **表单校验**：实时校验 + 提交时校验，错误信息就近显示
- **分页**：统一使用 Ant Design Vue 的 Pagination 组件
- **确认操作**：重要操作（如提交举报、提交申诉）需二次确认

## 9. 性能要求

### 9.1 性能指标

| 指标 | 目标值 | 说明 |
|------|--------|------|
| 举报提交响应 | < 1 秒 | 从点击提交到显示成功提示 |
| 申诉提交响应 | < 1 秒 | 从点击提交到显示成功提示 |
| 帮助中心搜索 | < 500ms | 从输入到显示搜索结果 |
| 智能客服响应 | < 2 秒 | 从发送消息到收到回复 |
| 客服排队时间 | < 5 分钟（普通）/ < 2 分钟（优先） | 从转人工到客服接入 |
| 页面首屏加载 | < 2 秒 | 所有页面首屏内容渲染完成 |

### 9.2 优化策略

#### 9.2.1 加载优化
- **路由懒加载**：所有页面组件使用动态 import 按需加载
- **图片懒加载**：帮助文章中的图片使用 Intersection Observer 懒加载
- **虚拟列表**：举报/申诉列表超过 100 条时使用虚拟滚动
- **骨架屏**：所有列表页和详情页使用骨架屏提升感知速度

#### 9.2.2 请求优化
- **请求防抖**：帮助中心搜索输入防抖 300ms
- **请求缓存**：帮助分类列表、热门文章缓存 5 分钟
- **分页加载**：列表数据分页加载，每页 10-15 条
- **重复请求取消**：切换筛选条件时取消前一个未完成的请求

#### 9.2.3 渲染优化
- **组件按需引入**：Ant Design Vue 组件按需引入
- **长列表优化**：超过 50 条记录使用虚拟列表
- **图片压缩**：上传证据前在前端压缩图片（质量 08，最大边 1920px）
- **WebSocket 复用**：客服对话复用单个 WebSocket 连接

#### 9.2.4 文件上传优化
- **分片上传**：超过 5MB 的文件使用分片上传
- **断点续传**：上传中断后支持断点续传
- **前端压缩**：图片在上传前进行压缩，减少上传时间

### 9.3 监控指标

- **FCP（首次内容绘制）**：< 1.5 秒
- **LCP（最大内容绘制）**：< 2.5 秒
- **FID（首次输入延迟）**：< 100ms
- **CLS（累积布局偏移）**：< 0.1
