## Why

内容社区当前缺少完善的用户关注体系和多维度内容订阅能力，用户难以高效管理信息流和发现感兴趣的内容。现有系统无法支持关注分组、特别关注、多源订阅等社交功能，导致用户粘性不足、内容发现效率低下。构建社交关系与内容订阅功能可以显著提升用户互动率和内容消费效率，目标是平均关注数提升40%、关注流点击率>30%、订阅转化率>25%、用户留存率提升20%。

## What Changes

**关注体系增强**
- From: 无关注功能或基础关注功能
- To: 完整的用户关注体系，包括关注/取消关注、特别关注、关注分组管理
- Reason: 用户需要管理关注关系，区分重要关注和普通关注
- Impact: 新增关注相关API和UI组件，非破坏性变更

**内容订阅系统**
- From: 无订阅功能
- To: 多维度内容订阅能力，支持专题/话题/栏目/频道等内容源订阅
- Reason: 用户需要订阅感兴趣的内容源，获取更新通知
- Impact: 新增订阅相关API和UI组件，非破坏性变更

**信息流优化**
- From: 无关注流和订阅流
- To: 基于关注和订阅关系的内容流展示，支持动态类型筛选和特别关注置顶
- Reason: 用户需要高效浏览关注对象和订阅源的更新内容
- Impact: 新增信息流API和UI组件，非破坏性变更

**批量管理功能**
- From: 无批量操作能力
- To: 支持批量取消关注、批量移动分组、批量取消订阅等操作
- Reason: 用户需要高效管理大量关注和订阅关系
- Impact: 新增批量操作API和UI组件，非破坏性变更

## Success Criteria

- 用户能成功关注/取消关注其他用户，按钮状态即时切换
- 特别关注设置/取消正常，星标状态切换正确，特别关注用户动态置顶展示
- 关注分组管理功能完整，支持默认分组和自定义分组的增删改查
- 关注列表分页加载正常，搜索和分组筛选结果正确
- 订阅/取消订阅操作正常，暂停/恢复订阅功能正确
- 订阅通知配置保存正常，渠道/频率/免打扰设置生效
- 订阅广场展示正常，分类浏览和搜索结果正确
- 所有页面在PC/平板/手机三种宽度下布局正确
- 关注流首次加载<2s，关注/订阅操作响应<500ms

## Non-Goals

- 独立支付结算和付费订阅（仅保留状态扩展点）
- 复杂机器学习推荐算法（本期使用可解释规则）
- 重构拉黑与屏蔽语义（仅在查询中尊重现有状态）
- 迁移 `/content/user/*` 路径到 `/api/v1/content/user/*`
- 国际化支持（默认中文界面）

## Capabilities

### New Capabilities

- `user-follow-system`: 用户关注体系，包括关注/取消关注、特别关注、关注分组管理、关注列表、特别关注列表、关注推荐、批量管理关注等功能
- `content-subscription`: 内容订阅系统，包括订阅内容源、暂停/恢复订阅、订阅通知配置、订阅广场、订阅流、订阅统一管理等功能
- `social-feed`: 社交信息流，包括关注流和订阅流的内容展示、动态类型筛选、特别关注置顶等功能

### Modified Capabilities

（无现有能力需要修改，本次为全新功能开发）

## Impact

**前端代码影响**
- 新增用户关注相关页面和组件：关注列表、特别关注列表、关注流、关注推荐、批量管理
- 新增内容订阅相关页面和组件：订阅广场、订阅流、订阅管理、通知配置
- 新增Pinia store管理关注和订阅状态
- 新增API接口调用封装
- 新增响应式布局适配

**后端API影响**
- 新增用户关注相关API：关注/取消关注、特别关注设置、关注分组管理、关注列表查询、关注推荐
- 新增内容订阅相关API：订阅/取消订阅、暂停/恢复订阅、订阅通知配置、订阅广场查询、订阅流查询
- 新增批量操作API：批量取消关注、批量移动分组、批量取消订阅

**依赖影响**
- 依赖现有用户系统（用户信息、头像、昵称等）
- 依赖现有内容系统（专题、话题、栏目等内容源）
- 依赖现有通知系统（站内通知、推送通知）
- 依赖Ant Design Vue 4组件库

**系统影响**
- 新增数据库表：user_follow、user_follow_group、user_subscribe、subscribe_notification_config等
- 新增缓存需求：关注关系缓存、订阅关系缓存、推荐结果缓存
- 新增性能考虑：关注流聚合查询优化、大关注量用户降级策略
