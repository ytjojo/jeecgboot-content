## 三、WARNING 级别问题 (2个)

#### W-001: batchDeleteFilterRules参数传递方式与后端约定不一致
- **位置**: `api/content/filterRule.ts:20-21`
- **严重级别**: WARNING
- **描述**: batchDeleteFilterRules将ruleIds放在body中（data），userId放在params中。design约定"所有写操作参数通过@RequestParam传递"，后端可能期望ruleIds也在query params中（逗号分隔）
- **影响**: 批量删除屏蔽词功能可能失败
- **修复建议**: 确认后端参数接收方式，必要时改为`params: { userId, ruleIds: ruleIds.join(',') }`

#### W-002: MuteListItemVO字段不足无法支持4个Tab
- **位置**: `api/content/mute.ts:28-35`
- **严重级别**: WARNING
- **描述**: MuteListItemVO仅包含用户屏蔽相关字段（userId/nickname/avatar），缺少话题屏蔽（topicId/topicName/topicCover）和内容类型屏蔽（contentTypeCode/contentTypeName）字段
- **影响**: 屏蔽话题Tab和屏蔽内容类型Tab无法正确渲染
- **修复建议**: 扩展MuteListItemVO添加topic和contentType相关可选字段
