## WARNING 级别漂移/问题 (2个)

#### W-001: batchDeleteFilterRules参数传递方式不一致
- **位置**: `api/content/filterRule.ts:20-21`
- **严重级别**: WARNING
- **描述**: 所有其他POST操作（blockUser/unblockUser/muteUser/unmuteUser/addFilterRule/deleteFilterRule/recordNotInterested）均将参数放在params（query）中，唯独batchDeleteFilterRules将ruleIds放在data（body）中。design D3约定"参数通过@RequestParam传递"
- **影响**: 若后端未使用@RequestBody接收数组，批量删除会失败
- **修复建议**: 确认后端接收方式，统一参数传递约定。若后端支持@RequestBody接收数组，更新design D3文档说明"批量数组操作使用@RequestBody"；否则改为params传递

#### W-002: MuteListItemVO字段不足无法支持多类型屏蔽Tab
- **位置**: `api/content/mute.ts:28-35`
- **严重级别**: WARNING
- **描述**: tasks 6.5-6.9要求屏蔽列表展示4个Tab（用户/话题/内容类型/临时屏蔽），但MuteListItemVO仅包含用户屏蔽字段（userId/nickname/avatar/mutedAt/muteType/expiresAt），缺少topicId/topicName（话题屏蔽）、contentTypeCode/contentTypeName（内容类型屏蔽）字段
- **影响**: 话题屏蔽Tab和内容类型屏蔽Tab无法正确渲染
- **修复建议**: 扩展MuteListItemVO或定义多态类型，支持不同屏蔽类型的数据结构
