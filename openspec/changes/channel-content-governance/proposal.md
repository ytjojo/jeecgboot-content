## Why

频道已具备基础架构（EPIC-20）和隐私成员规则（EPIC-21），但缺少内容如何进入频道、谁能发布、何时审核、发布后如何治理的完整机制。当前频道无法控制内容质量，没有发布权限分层、没有审核流程、没有防刷屏策略，导致频道内容质量无法保障。本变更建立频道内容发布与治理的完整能力，让频道从"可加入的社区单元"升级为"可持续产出、可治理的主题空间"。

## What Changes

**频道发布权限模型**
- From: 无频道级发布权限控制
- To: 支持四种发布权限模型——仅管理员可发布、所有成员可发布、公开投稿、先审后发；不同角色和身份的用户在不同权限模型下获得不同的发布结果
- Impact: 新增能力，非破坏性

**内容审核机制**
- From: 无审核流程
- To: 支持待审区管理，审核人可逐条或批量通过/拒绝投稿，拒绝需填写原因并通知提交者
- Impact: 新增能力，非破坏性

**定时发布与发布限额**
- From: 无定时发布、无防刷屏
- To: 支持定时发布（到达时间重新校验规则），支持每小时/每日发布上限和内容字数下限
- Impact: 新增能力，非破坏性

**频道内容治理**
- From: 无频道级内容管理
- To: 支持置顶、精华、删除（含回收站30天保留）、移出频道、编辑协助，关键操作记录治理日志
- Impact: 新增能力，非破坏性

**频道公告**
- From: 无公告能力
- To: 支持频道顶部公告，频道主/管理员可发布、编辑、删除公告，支持富文本预览
- Impact: 新增能力，非破坏性

**已发布内容添加到频道**
- From: 内容只能在发布时选择频道
- To: 支持将已发布内容添加到频道，覆盖系统频道添加、个人/组织频道添加、频道主添加他人作品三种场景
- Impact: 新增能力，非破坏性

## Success Criteria

- 频道发布成功率（合法请求）>= 99%
- 发布权限准确率（核心场景）100%
- 定时发布准时率（1分钟内完成）>= 99.9%
- 内容治理可追溯率（关键操作记录完整）100%
- 回收站恢复有效性（30天内）>= 99%
- 发布限额拦截率 >= 95%

## Non-Goals

- 频道创建与所有权管理（EPIC-20）
- 频道隐私与成员管理（EPIC-21）
- 推荐发现（EPIC-23）
- 频道数据统计（EPIC-24）
- 平台级内容安全审核策略
- 完整创作器能力

## Capabilities

### New Capabilities

- `channel-publishing`: 频道内容发布核心能力，包括发布权限模型（仅管理员/所有成员/公开投稿/先审后发）、频道选择组件、多频道发布、定时发布、发布限额与防灌水
- `channel-content-moderation`: 频道内容审核能力，包括待审区管理、逐条/批量审核、审核结果通知、超时提醒
- `channel-content-governance`: 频道内容治理能力，包括置顶、精华标记、删除与回收站、移出频道、编辑协助、治理日志
- `channel-announcements`: 频道公告能力，包括公告发布、编辑、删除、富文本预览与展示
- `channel-add-existing-content`: 已发布内容添加到频道能力，包括系统频道添加、个人/组织频道添加、频道主添加他人作品

## Impact

- **后端模块**: jeecg-module-content 新增频道发布、审核、治理、公告相关的 Entity、Mapper、Service、Controller
- **数据库**: 新增频道发布权限配置表、待审区表、定时发布任务表、发布限额配置表、内容治理记录表、回收站表、公告表等
- **API**: 新增频道发布、审核、治理、公告、添加内容相关 REST API
- **前端**: 新增频道选择组件、待审区页面、内容治理页面、公告编辑组件、发布结果反馈组件
- **依赖**: 依赖 EPIC-20 频道基础架构、EPIC-21 成员角色与禁言规则、内容发布服务、通知系统
