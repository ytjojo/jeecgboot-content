## Why

内容社区当前缺少便捷的举报入口、透明的申诉流程和高效的客服支持。用户遇到违规内容时无法及时举报，受到处罚后没有公平的申诉渠道，遇到问题时也缺乏自助和人工支持。这导致违规行为得不到有效遏制，用户体验和信任度下降。建立完善的反馈、举报与客服支持体系是社区治理的基础设施，需要优先建设。

## What Changes

**举报系统**
- 新增：用户可对违规用户、内容、评论发起举报，选择举报类型并上传证据（图片/视频，最大 10MB）
- 新增：同一用户对同一对象只能举报一次，防止重复举报
- 新增：用户可查看举报处理进度（待处理/处理中/已处理）和处理结果
- 新增：举报处理完成后自动通知用户

**申诉系统**
- 新增：用户可对被删除内容、被封禁账户、被扣除积分/勋章等处罚发起申诉
- 新增：申诉最多允许 3 次，审核需双人复核
- 新增：申诉成功后自动撤销处罚并恢复用户状态
- 新增：所有申诉操作生成不可篡改的审计日志

**帮助中心**
- 新增：提供常见问题分类（账号、隐私、举报、积分等）和全文搜索
- 新增：帮助文章支持"有用/无用"反馈，优化排序
- 新增：功能更新日志按版本展示，支持搜索

**客服通道**
- 新增：智能客服机器人作为首选入口，支持转人工客服
- 新增：高等级用户（LV.15+）在人工客服队列中享受优先权
- 新增：客服会话历史记录，保留 30 天

## Success Criteria

- 举报提交响应时间 <1 秒，处理时效 <24 小时
- 申诉提交响应时间 <1 秒，审核时效 <48 小时，申诉满意度 >80%
- 帮助中心搜索响应时间 <500ms，自助解决率 >60%
- 智能客服响应时间 <2 秒，人工客服排队时间 <5 分钟（普通用户）/<2 分钟（高等级用户）
- 客服响应时间 <5 分钟

## Non-Goals

- 独立工单系统
- 电话客服
- 智能客服 NLP 引擎的训练和优化（使用现有引擎）
- 帮助中心 CMS 的完整实现（使用现有 CMS 框架）

## Capabilities

### New Capabilities

- `content-reporting`: 举报违规用户/内容/评论，选择举报类型，上传证据，防止重复举报
- `report-tracking`: 查看举报处理进度和结果，处理完成通知
- `penalty-appeal`: 对处罚发起申诉，填写理由，跟踪进度，最多 3 次
- `appeal-review`: 申诉审核与恢复，双人复核，审计日志，自动撤销处罚
- `help-center`: 帮助中心，常见问题分类，全文搜索，文章反馈
- `changelog`: 功能更新日志，按版本展示，支持搜索
- `smart-customer-service`: 智能客服机器人入口，支持转人工
- `priority-service`: 高等级用户在客服队列中享受优先权
- `service-history`: 客服会话历史记录，30 天保留

### Modified Capabilities

（无现有能力需要修改）

## Impact

- **后端模块**: jeecg-module-content 需新增举报、申诉、帮助中心、客服相关实体和服务
- **数据库**: 新增举报表、申诉表、帮助文章表、客服会话表等
- **API**: 新增举报、申诉、帮助中心、客服相关 REST API
- **前端**: 新增举报弹窗、申诉页面、帮助中心页面、客服对话组件
- **依赖**: EPIC-01（用户认证）、EPIC-03（等级体系）、EPIC-09（用户状态与治理）
