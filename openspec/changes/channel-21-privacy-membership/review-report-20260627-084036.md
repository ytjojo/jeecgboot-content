## 三、问题详情

### BLOCK 级问题 (5个)

**BLOCK-01 [接口契约]**: design.md未定义具体API契约
- **位置**: design.md File Structure章节
- **描述**: 仅列出Controller文件名，无具体端点、HTTP方法、请求体、响应体、错误码定义
- **影响**: 前后端无法精确对齐接口
- **修复建议**: 补充API契约表格

**BLOCK-02 [API命名规范]**: API路径使用单数名词和RPC风格，不符合RESTful规范
- **位置**: ChannelMemberController:25 `/api/v1/content/channel/member`
- **描述**: 
  - 使用单数`channel/member`而非复数`channels/{id}/members`
  - 动作作为路径后缀`/join/free`、`/join/apply`、`/leave`、`/assign-role`
  - 不符合项目规范`/api/v1/content/{resources}`复数资源
- **影响**: API风格不一致，难以理解和维护
- **修复建议**: 统一为`/api/v1/content/channels/{channelId}/members/...`风格

**BLOCK-03 [API命名规范]**: Controller中大量使用@PostMapping处理查询（如/applications/pending是GET但语义上是查询）
- **位置**: ChannelMemberController
- **描述**: 部分GET/POST混用，查询类接口应该用GET
- **修复建议**: 查询用@GetMapping，写操作用@PostMapping

**BLOCK-04 [边界覆盖]**: 缺少黑名单优先级高于邀请的Scenario
- **位置**: specs/channel-blacklist/spec.md
- **描述**: 未明确"用户在黑名单中但持有有效邀请"场景的处理逻辑
- **影响**: 可能出现黑名单用户通过邀请绕过限制
- **修复建议**: 补充Scenario明确黑名单检查在邀请校验之前

**BLOCK-05 [边界覆盖]**: 缺少批量治理操作的部分失败处理Scenario
- **位置**: specs/channel-member-removal/spec.md 等
- **描述**: 批量移除/禁言操作中部分目标无效（如不存在/权限不足/已被处理）时的行为未定义
- **影响**: 批量操作可能全部失败或静默跳过问题项
- **修复建议**: 定义批量操作返回每个条目的成功/失败状态
