## Why

频道模块当前缺乏隐私控制、订阅机制和成员治理能力，所有频道内容对所有人可见，用户无法主动订阅频道，频道主无法管理成员角色和违规用户。这限制了频道从"内容容器"升级为"可运营、可参与、可治理的社区单元"。现在需要在前端补齐这些能力，支撑频道的隐私分级、订阅运营和成员治理场景。

## What Changes

**频道隐私与加入方式**
- 新增频道隐私设置页面（公开/私有 Radio.Group 切换，含影响确认弹窗）
- 新增加入方式配置（自由/审核/邀请三种模式，含审核配置项和邀请创建 Drawer）
- 系统频道隐私选项锁定为公开，展示提示信息

**订阅机制**
- 新增频道主页订阅/取消订阅按钮（含状态机：未订阅、已订阅、待审核、冷却期、黑名单）
- 采用乐观更新策略，点击后立即更新 UI，失败时回滚
- 新增订阅列表管理页面（分组、搜索、提醒控制、取消订阅）

**申请加入与审核**
- 新增私有频道申请加入表单 Modal（申请理由必填，10-200字）
- 新增待审队列页面（单条/批量批准/拒绝，超时高亮，批量接口契约）

**成员管理与治理**
- 新增成员列表页面（角色筛选、搜索、排序，JVxeTable 支持批量操作）
- 新增角色分配、移除成员、禁言/解除禁言功能（含二次确认 Modal）
- 新增黑名单管理页面（加入/移出黑名单）
- 新增治理操作日志页面（筛选、详情 Drawer）

**状态管理**
- 新增 `useChannelContext` composable，按 channelId 隔离管理频道上下文，避免多频道切换时数据串台

## Success Criteria

- 频道主可在设置页配置隐私状态和加入方式，切换时有影响说明确认
- 用户可在频道主页完成订阅/取消订阅/申请加入操作，按钮状态机正确响应所有场景
- 管理员可在成员管理页完成角色分配、移除、禁言、黑名单操作，支持批量操作
- 待审队列支持单条和批量审核，批量接口采用单次请求，返回逐条处理结果
- 订阅列表页支持分组管理、搜索、提醒控制
- 所有操作遵循乐观更新策略，失败时正确回滚并展示错误提示
- `useChannelContext` composable 按 channelId 隔离，切换频道时数据不串台
- 页面首屏加载 < 2s，核心操作 P95 <= 500ms

## Non-Goals

- 频道创建与所有权管理（EPIC-20 范围）
- 内容发布权限详细规则（EPIC-22 范围）
- 频道推荐与发现算法（EPIC-23 范围）
- 完整付费频道闭环
- 平台级黑名单联动
- 治理日志导出功能（P2，本期不实现）

## Capabilities

### New Capabilities

- `channel-privacy-settings`: 频道隐私状态设置（公开/私有）和加入方式配置（自由/审核/邀请），包含系统频道锁定逻辑和邀请创建管理
- `channel-subscription`: 频道订阅/取消订阅功能，含乐观更新策略、状态机、订阅列表管理（分组、搜索、提醒控制）
- `channel-join-application`: 私有频道申请加入流程，含申请表单、申请状态查询、待审队列页面（单条/批量审核）
- `channel-member-management`: 成员列表展示、角色分配、移除成员、禁言/解除禁言功能，含批量操作和权限控制
- `channel-governance`: 黑名单管理和治理操作日志，含加入/移出黑名单、操作记录查询和详情展示
- `channel-context-composable`: useChannelContext composable，按 channelId 隔离管理频道上下文、用户关系、权限判断

### Modified Capabilities

（无现有 spec 需要修改）

## Impact

- **前端路由**：新增频道设置页、成员管理页、待审队列页、订阅列表页、黑名单页、治理日志页等路由
- **API 对接**：新增约 25 个 API 接口调用（订阅、成员、黑名单、邀请、隐私、治理日志）
- **状态管理**：新增 `useChannelContext` composable，影响频道路由守卫和页面组件
- **组件依赖**：使用项目现有 Form、Table、Modal、Drawer、JVxeTable 等组件，无新外部依赖
- **响应式适配**：所有新增页面需支持桌面端和移动端响应式布局
