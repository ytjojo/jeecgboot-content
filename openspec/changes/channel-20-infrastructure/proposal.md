## Why

内容社区缺少统一的频道基础架构，无法让平台官方、个人创作者和认证组织以一致的规则创建和管理频道。当前没有频道类型定义、创建流程、审核状态机、编辑/转让/删除能力，导致后续的内容发布、推荐发现、订阅治理等模块缺乏稳定的数据基础。EPIC-20 作为频道体系的首个里程碑，需要先建立可信的内容组织单元，为 EPIC-21~24 提供前置能力。

## What Changes

**频道数据模型**
- 新增频道基础数据表，支持 system/personal/organization 三种类型
- 类型字段创建后不可直接修改，组织频道强制绑定组织 ID

**系统官方频道创建**
- 平台管理员可在后台创建系统频道，直接进入 Active 状态（无需内容安全审核）
- 系统频道强制公开、不可转让、不可删除、不参与用户频道名称唯一性校验

**个人/组织频道创建**
- 普通用户创建个人频道，需满足基础账号要求，名称在用户频道范围内唯一
- 已认证组织管理员创建组织频道，自动绑定组织且不可改绑
- 创建后进入 PendingReview 状态，审核通过后变为 Active

**频道信息编辑与所有权管理**
- 非关键字段保存后立即生效，关键字段（名称、简介、图标、封面、分类）修改需重新审核
- 个人频道支持转让（需双方确认），组织频道仅可在组织管理员间转移
- 删除需满足前置条件（内容清理、订阅结清），确认后进入 7 天冷静期，期间可撤销

## Success Criteria

- 三类频道创建流程完整可用，合法创建请求成功率 >= 99%
- 用户频道范围内名称唯一性校验准确，重名误放行数 = 0
- 内容安全审核 24 小时内完成比例 >= 95%
- 关键操作审计记录覆盖率 = 100%
- 创建成功频道的类型、状态、分类、归属、基础展示信息完整率 = 100%

## Non-Goals

- 频道隐私与加入规则（EPIC-21）
- 频道内容发布与治理（EPIC-22）
- 频道推荐与发现（EPIC-23）
- 频道数据统计与生命周期管理（EPIC-24）
- 付费订阅能力（本期仅预留删除校验字段）

## Capabilities

### New Capabilities

- `channel-data-model`: 频道基础数据模型，定义三类频道的统一表结构、类型枚举、状态枚举和差异化字段
- `channel-creation`: 频道创建能力，覆盖系统频道创建、个人频道创建、组织频道创建的完整流程和校验规则
- `channel-review`: 频道审核状态机，管理 PendingReview/Active/Rejected 等状态流转和审核结果通知
- `channel-edit`: 频道信息编辑能力，区分关键字段与非关键字段的编辑规则和审核策略
- `channel-ownership`: 频道所有权管理，覆盖转让流程（个人/组织差异化）和删除流程（冷静期机制）

### Modified Capabilities

（无已有 capability 需修改）

## Impact

- **数据库**: 新增频道主表、频道审核记录表、频道操作审计表
- **后端 API**: 新增频道 CRUD 接口、审核接口、转让/删除接口
- **依赖系统**: 用户认证系统（账号状态校验）、组织认证系统（组织权限校验）、内容安全审核服务、通知服务
- **前端页面**: 用户端频道创建页、频道管理页；后台频道管理页、审核队列页
