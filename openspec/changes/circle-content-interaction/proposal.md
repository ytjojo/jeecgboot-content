## Why

MVP 版本圈子仅支持基础内容发布与浏览，缺乏内容质量管控（置顶/精华）、圈子内互动能力（公告/@成员）和安全保障（审核机制）。圈子运营者无法突出优质内容、传达重要信息，成员之间缺乏有效互动方式，违规内容和恶意用户得不到及时处理。在 circle-core 基础设施稳定运行后，需要补齐这些能力以提升圈子运营效率和安全性。

## What Changes

**内容管理增强**
- 新增内容置顶能力，置顶内容始终排列在非置顶内容之前，多条置顶按最近置顶时间排序
- 新增内容精华标记能力，精华内容在列表、详情页有特殊展示标识
- 权限控制：仅圈子版主和创建者可操作，普通成员操作返回"权限不足"

**圈子互动功能**
- 新增圈子公告能力，公告展示在圈子顶部，同一圈子同一时间仅展示一条生效公告
- 新增圈子内 @成员能力，发布内容或评论时可提及圈内成员，被@成员收到通知
- @成员列表仅展示当前圈子内成员，已退出圈子的用户不发送通知
- 通知发送不阻塞内容发布主流程，失败通知支持后续补偿

**圈子审核机制**
- 新增加入申请审核流程，管理员可批准/拒绝申请，拒绝时通知用户
- 新增内容举报处理流程，管理员可删除内容/忽略举报/禁言用户
- 加入申请超过 3 天未处理自动提醒管理员
- 所有审核操作可追溯（操作人、时间、类型、对象、结果），记录保留≥180 天

## Success Criteria

- 上线后 4 周内，内容置顶/精华功能被 >20% 的活跃圈子使用
- 举报首次处理响应时间 P95 <24 小时
- 需要审核的加入申请、触发审核的内容和成员举报均进入可处理状态，审核覆盖率 100%
- @成员通知 P95 送达时间 <5 秒
- 普通成员不可执行管理操作，无权限操作返回明确提示

## Non-Goals

- 数据统计与分析
- 推荐功能
- 成长激励体系
- 内容审核 AI 自动化（仅支持人工审核）

## Capabilities

### New Capabilities
- `circle-content-pin-featured`: 内容置顶与精华管理，包含置顶排序、精华标识、权限控制、状态切换
- `circle-announcement`: 圈子公告管理，包含公告发布、有效期、替换展示、权限控制
- `circle-mention`: 圈子内 @成员功能，包含成员列表查询、提及解析、异步通知发送
- `circle-join-review`: 加入申请审核，包含申请列表、批准/拒绝操作、超时提醒、审核日志
- `circle-content-report`: 内容举报处理，包含举报提交、举报列表、删除/忽略/禁言操作、处理结果通知

### Modified Capabilities
（无已有能力需要修改）

## Impact

**后端代码**
- 在 jeecg-module-content 模块中新增内容管理增强功能（置顶/精华字段扩展）
- 新增公告 Entity、Mapper、Service、Controller
- 新增提及解析服务，与通知系统集成
- 新增审核相关 Entity（加入申请审核记录、举报记录）、Service、Controller
- 新增定时任务：超时申请提醒、禁言到期自动解除（如 circle-core 未实现）

**API**
- 新增内容置顶/精华 API（PUT /circle-content/{id}/pin、/featured）
- 新增圈子公告 CRUD API
- 新增 @成员候选列表 API 和提及解析服务
- 新增加入申请审核 API（GET /pending、PUT /approve、PUT /reject）
- 新增举报处理 API（POST /report、GET /reports、PUT /handle）

**数据库**
- circle_content 表新增 is_pinned、pinned_at、is_featured、featured_at 字段
- 新增 circle_announcement 表（公告）
- 新增 circle_join_request 表（加入申请审核记录）
- 新增 circle_report 表（举报记录）

**依赖**
- circle-core（圈子基础能力，已实现）
- 通知系统（@成员通知、审核结果通知）
- 敏感词检测服务（内容审核触发）
