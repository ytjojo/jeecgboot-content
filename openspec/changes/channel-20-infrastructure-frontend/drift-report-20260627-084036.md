### 1.1 API 路径漂移

| spec 中定义路径 | 代码中实际路径 | 文件位置 | 漂移类型 | 严重级别 |
|----------------|---------------|---------|---------|---------|
| `POST /api/v1/content/channels/transfer/{transferId}/confirm` | `POST /api/v1/content/channels/transfer/{transferId}/confirm` (拼接后缺少 `{id}/`) | `src/api/content/channel/index.ts:76` | ❌ 路径错误 | CRITICAL |
| `POST /api/v1/content/channels/transfer/{transferId}/reject` | `POST /api/v1/content/channels/transfer/{transferId}/reject` (拼接后缺少 `{id}/`) | `src/api/content/channel/index.ts:80` | ❌ 路径错误 | CRITICAL |
| `GET /api/v1/content/admin/channels/list` (后台列表) | 调用 `getChannelList` (用户端列表) | `src/views/content/channel/admin/index.vue:156` | ❌ API 方法用错 | CRITICAL |
| `GET /api/v1/content/channel/review/detail/{id}` | 未封装此方法 | `src/api/content/channel/index.ts` | ❌ API 缺失 | CRITICAL |
| （文档说路由是 `/api/v1/content/channel/*`） | 实际路由跳转使用 `/content/channel/*` | 所有 vue 文件 | ⚠️ 文档路径错误 | WARNING |

**问题说明**:
1. `confirmTransfer` 和 `rejectTransfer` 使用 `Api.detail + 'transfer/' + transferId` 拼接，`Api.detail` 的值是 `/api/v1/content/channels/`，所以直接拼接 `transfer/` 会导致路径变成 `/api/v1/content/channels/transfer/...`，正确应该是 `/api/v1/content/channels/{id}/transfer/...`
2. 后台管理页面错误调用用户端列表 API，无法获取全量频道
3. 审核详情 API 完全缺失，无法获取 diff 数据

## 三、漂移问题汇总

### CRITICAL 级漂移（必须修复）

1. **API 路径拼接错误**: confirmTransfer/rejectTransfer 缺少 channelId 路径段
2. **后台 API 用错**: admin 页面调用用户端列表 API，缺少 getAdminChannelList
3. **API 缺失**: 缺少 getReviewDetail 方法
4. **组件缺失**: ReviewDiffViewer 完全不存在
5. **核心逻辑空实现**: TransferConfirmModal 的用户搜索是 TODO
6. **功能模拟实现**: ChannelForm 图片上传未对接真实接口，使用本地 blob URL

### WARNING 级漂移（应当修复）

1. **编辑承载方式不符**: spec 要求 Drawer，实际直接在 Tab 内
2. **表格组件选型不符**: spec 要求 JVxeTable，实际使用 a-table
3. **错误提示笼统**: catch 块未展示后端具体错误信息
4. **coolingDays 硬编码**: 未动态计算冷静期剩余天数
5. **rejectReason 未获取**: 审核拒绝原因硬编码为空
6. **转让历史 key 错误**: 使用 item.id 而非 item.transferId
7. **名称唯一性校验未实现**: 创建/编辑时的实时名称校验缺失
8. **关键字段审核标识缺失**: ChannelForm 无浅黄色背景和橙色 Tag
9. **缺少"退回修改"审核操作**: review 页面只有通过/拒绝
10. **审核列表缺少等待时长列和超时高亮**: 管理员无法快速识别超时审核

### SUGGESTION 级建议（可选优化）

1. 考虑将异步操作封装到 Store
2. 路由跳转使用 name 而非硬编码 path
3. 频道分类改为接口获取
4. 批量审核使用并发请求
5. 拆分 ReviewActionPanel 为独立组件
6. 统一组件目录组织

## 五、审核结论

**状态**: CRITICAL_DRIFT_DETECTED

检测到严重的代码-文档漂移和架构问题：

1. **阻断性问题**: 3 个 API 路径/方法问题会导致运行时错误，2 个核心功能（用户搜索、图片上传、diff 对比）是 TODO/模拟状态完全不可用
2. **架构偏离**: 编辑承载方式（Drawer vs 直接展示）、表格组件（JVxeTable vs a-table）与 spec 决策不符
3. **体验问题**: 错误处理简单、关键功能标识缺失、loading 状态不统一

**修复优先级**:
1. 立即修复 CRITICAL 级 API 路径错误和缺失方法
2. 补全空实现的核心功能（用户搜索、上传、ReviewDiffViewer）
3. 对齐与 spec 不符的架构决策（Drawer、表格、审核操作）
4. 优化 WARNING 级体验问题
