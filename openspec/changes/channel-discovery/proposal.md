## Why

随着频道数量增长，用户无法高效发现感兴趣的频道。当前系统缺少分类体系、个性化推荐、排行榜和搜索能力，导致用户只能通过直接链接或订阅关系找到频道，新用户冷启动体验差，优质频道得不到曝光。EPIC-20/21/22 已完成频道基础架构、隐私订阅和内容治理，EPIC-23 需要在此基础上建立完整的发现链路，为后续 EPIC-24 统计看板提供数据支撑。

## What Changes

**平台分类体系**
- From: 无统一分类标准，频道缺乏组织维度
- To: 平台运营维护最多 4 级树形分类，频道必须选择 1 个主分类和最多 3 个副分类；系统频道可归属特殊分类
- Impact: 非破坏性变更，新增分类管理后台和频道创建/编辑中的分类选择

**频道内标签**
- From: 频道内无内容细分标签机制
- To: 频道管理员可创建最多 20 字符的自定义标签，用于频道内内容分类和筛选
- Impact: 非破坏性变更，新增标签管理功能

**个性化推荐**
- From: 无推荐能力
- To: 基于用户画像、订阅历史、浏览行为生成推荐，支持不感兴趣反馈和冷启动兜底
- Impact: 新增推荐模块和反馈机制

**排行榜**
- From: 无排行榜
- To: 热门频道榜（日/周/月）、新晋频道榜（30 天内增长）、系统频道榜，每日更新
- Impact: 新增榜单模块和定时更新任务

**编辑精选**
- From: 无精选机制
- To: 平台运营可标记频道为编辑精选，配置推荐语和有效期
- Impact: 新增精选管理后台和展示模块

**分类浏览**
- From: 无分类浏览入口
- To: 用户可按分类树浏览可发现频道，支持排序和分页
- Impact: 新增分类浏览页面

**频道搜索**
- From: 无搜索能力
- To: 关键词搜索覆盖名称、描述、标签，支持多条件筛选和排序，P99 <= 200ms
- Impact: 新增搜索功能和索引

## Success Criteria

1. 频道搜索 P99 响应时间 <= 200ms
2. 推荐频道点击率 > 15%
3. 搜索结果相关性满意度 >= 85%
4. 排行榜每日更新延迟 <= 5 分钟
5. 新建公开频道主分类选择率 = 100%
6. 标签筛选可用率 >= 99%
7. 私有/隐藏/冻结/限制公开曝光频道不进入任何公开发现入口

## Non-Goals

- 推荐模型训练方案和搜索基础设施选型（后续独立评估）
- 平台级违规内容识别（由 EPIC-22 治理模块覆盖）
- 频道创建流程（EPIC-20 已完成）
- 成员管理（EPIC-21 已完成）
- 数据统计看板（EPIC-24 负责）

## Capabilities

### New Capabilities

- `channel-taxonomy`: 平台分类体系管理，包括分类树 CRUD、特殊分类、分类启停、频道分类选择
- `channel-tags`: 频道内标签管理，包括标签 CRUD、标签验证、标签与内容关联
- `channel-recommendations`: 个性化频道推荐，包括推荐算法调用、推荐理由、不感兴趣反馈、冷启动兜底
- `channel-rankings`: 频道排行榜，包括热门榜/新晋榜/系统榜、活跃度计算、定时更新
- `channel-editorial-picks`: 编辑精选频道，包括精选标记、推荐语、有效期、状态联动
- `channel-category-browse`: 分类浏览，包括分类树导航、频道列表、排序分页
- `channel-search`: 频道搜索，包括关键词搜索、多条件筛选、结果排序、空状态引导

## Impact

- **后端**: 新增分类、标签、推荐、榜单、精选、搜索相关实体、Service、Controller 和定时任务
- **数据库**: 新增分类表、标签表、推荐记录表、榜单快照表、精选配置表、搜索索引
- **前端**: 新增分类管理、标签管理、推荐模块、排行榜、编辑精选、分类浏览、搜索等页面和组件
- **API**: 新增分类、标签、推荐、榜单、精选、搜索相关 REST API
- **依赖**: 依赖 EPIC-20 频道基础信息、EPIC-21 隐私订阅规则、EPIC-22 内容活跃信号
- **定时任务**: 榜单每日更新、推荐结果刷新、精选有效期检查
