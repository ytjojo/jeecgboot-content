## 三、API前缀验证（/sys/ vs /content/）

**当前API前缀**: `/content/user/support/`
**分析**:
- support功能属于用户支持/客服体系，在JeecgBoot中可能有两种归属：
  1. 内容社区模块 → `/content/` 前缀（当前实现）
  2. 系统管理模块 → `/sys/` 前缀（如系统帮助中心、系统通知）
- 举报/申诉/客服通常属于平台治理功能，可能需要/sys/前缀
- 帮助中心和更新日志可能属于内容模块

**建议**: 与后端确认API路径前缀，避免联调时大面积修改。
