## Why

内容社区目前缺少用户举报入口、处罚申诉流程和自助客服支持体系。用户发现违规内容后无法便捷举报，被处罚后无法获得复核机会，遇到问题时只能依赖人工客服。这导致社区治理效率低下、用户投诉率高、客服压力大。本变更补齐举报、申诉、帮助中心、更新日志和客服通道五大能力，提升社区自治水平和用户支持体验。

## What Changes

**新增举报系统**
- 新增举报表单弹窗组件（ReportModal），支持选择举报类型、上传证据、防重复校验
- 新增我的举报列表页，支持筛选、查看详情、撤回、跳转申诉

**新增申诉系统**
- 新增申诉提交页，支持关联处罚记录、填写理由、上传附件、次数限制（最多3次）
- 新增我的申诉列表页，支持筛选、查看详情、撤回

**新增帮助中心**
- 新增帮助中心首页，支持分类浏览和全文搜索
- 新增帮助文章详情页，支持 Markdown 渲染和有用/无用反馈

**新增更新日志**
- 新增更新日志页，按时间线展示版本记录，支持搜索
- 首次登录有新版本时弹出提示

**新增客服通道**
- 新增客服对话页，支持智能客服机器人 + 转人工排队 + 实时对话
- 新增客服历史记录页，支持查看历史会话和继续咨询
- 新增服务评分弹窗
- 高等级用户（LV.15+）优先排队

**新增状态管理**
- 新增 useFeedbackStore，管理举报、申诉、客服会话状态

## Success Criteria

1. 用户可通过内容详情页的"举报"按钮提交举报，包含类型选择和证据上传
2. 用户可在"我的举报"页面查看所有举报记录和处理进度
3. 用户可对处罚决定发起申诉，最多3次，系统正确限制次数
4. 帮助中心搜索响应时间 <500ms，支持关键词高亮
5. 客服对话支持智能客服和转人工，消息发送延迟 <500ms
6. 高等级用户（LV.15+）在客服排队中获得优先位置
7. 所有页面支持响应式布局（PC/平板/移动端）
8. 所有页面首屏加载时间 <2秒

## Non-Goals

- 独立工单系统
- 电话客服
- 智能客服 NLP 引擎训练（使用现有引擎）
- 帮助中心 CMS 完整后端（仅前端搜索接口对接）
- 申诉审计日志的前端生成（仅展示）

## Capabilities

### New Capabilities

- `report-system`: 举报系统 -- 举报表单弹窗、举报列表页、举报详情、防重复校验、撤回功能
- `appeal-system`: 申诉系统 -- 申诉提交页、申诉列表页、申诉详情、次数限制、撤回功能
- `help-center`: 帮助中心 -- 分类浏览、全文搜索、文章详情、有用/无用反馈
- `changelog`: 更新日志 -- 版本列表、时间线展示、搜索、新版本提示
- `customer-service`: 客服通道 -- 智能客服对话、转人工排队、实时对话、服务评分、会话历史
- `feedback-store`: 状态管理 -- useFeedbackStore，管理举报、申诉、客服会话状态

## Impact

**新增页面路由**:
- `/user/reports` -- 我的举报列表
- `/user/appeals` -- 我的申诉列表
- `/user/appeals/create` -- 申诉提交页
- `/help` -- 帮助中心
- `/help/article/:id` -- 帮助文章详情
- `/changelog` -- 更新日志
- `/customer-service` -- 客服对话
- `/user/service-history` -- 客服历史记录

**新增 API 对接**: 21个 API 接口，覆盖举报、申诉、帮助中心、更新日志、客服会话等

> **后端依赖说明**（2026-06-04 验证）: 21 个 API 中仅 7 个在后端控制器中已存在，5 个服务层已实现但未暴露 HTTP 端点，12 个完全缺失。详见 `verification-review.md` 和 `backend-issues.md`。

**新增依赖**:
- 复用现有 `defHttp` 封装
- 复用现有 `Upload` 组件
- 复用现有 `useUserStore`、`usePermissionStore`
- WebSocket 连接（客服实时对话）

**新增组件**: ReportModal、AppealForm、ChatPanel、ChatMessage、RatingModal、HelpSearch、ChangelogTimeline、VersionCard、ArticleFeedback（共9个业务组件）

**菜单权限**: 需要在个人中心菜单中新增"我的举报"、"我的申诉"、"客服记录"菜单项，帮助中心和更新日志需在顶部导航或页脚添加入口
