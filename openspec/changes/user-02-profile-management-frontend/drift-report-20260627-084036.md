## WARNING 级别漂移/问题 (2个)

#### W-001: viewerUserId参数被标记为必填但应为可选
- **位置**: `api/content/profile/index.ts:28`
- **严重级别**: WARNING
- **描述**: `getProfileDetail(ownerUserId: string, viewerUserId: string)` 第二个参数必填，但design.md:117标注viewerUserId为可选。在查看他人主页场景（尤其是未登录用户查看），viewerUserId应该允许为undefined
- **影响**: 未登录场景无法调用该接口查看他人资料
- **修复建议**: 
  ```typescript
  export const getProfileDetail = (ownerUserId: string, viewerUserId?: string) =>
    defHttp.get<ContentUserProfileVO>({
      url: Api.detail,
      params: viewerUserId ? { ownerUserId, viewerUserId } : { ownerUserId },
    });
  ```

#### W-002: privacy-settings spec字段名与实际代码/types不一致（文档漂移）
- **位置**: `specs/privacy-settings/spec.md:26-34` vs `api/content/profile/types.ts:157-181` vs `views/content/profile/privacy/index.vue:155-178`
- **严重级别**: WARNING
- **描述**: spec列出的字段名与types.ts/实际UI实现存在系统性偏差：
  - spec有bioVisibility，代码无
  - spec有homepageBackgroundVisibility/themeColorVisibility/homepageModuleVisibility，代码为homepageVisibility/dynamicVisibility
  - spec有certificationVisibility/verificationBadgesVisibility(复数)，代码为verificationBadgeVisibility(单数)/contactBadgeVisibility
  - spec有profileCompletionVisibility/profileReviewStatusVisibility/recentActivityVisibility/showMutualFollowersCount/showRecentActivityHighlight，代码为browseHistoryVisibility/likeActivityVisibility/favoriteVisibility
- **影响**: spec无法作为测试依据；新人阅读spec会产生严重误解
- **修复建议**: 以design D7和types.ts/后端Req为准重写specs/privacy-settings/spec.md中的字段列表
