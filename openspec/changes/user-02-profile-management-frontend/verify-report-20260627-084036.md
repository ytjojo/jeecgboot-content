## WARNING 级别问题 (3个)

#### W-001: getProfileDetail的viewerUserId参数为必填但design定义为可选
- **位置**: `api/content/profile/index.ts:28`
- **严重级别**: WARNING
- **描述**: `getProfileDetail(ownerUserId: string, viewerUserId: string)` 第二个参数必填，但查看他人主页时viewerUserId可能为undefined（未登录场景）
- **影响**: 未登录用户查看他人主页时TypeScript编译报错或运行时参数缺失
- **修复建议**: 改为 `getProfileDetail(ownerUserId: string, viewerUserId?: string)`

#### W-002: 隐私页面未渲染onlineStatusVisible布尔字段
- **位置**: `api/content/profile/types.ts:172` vs `views/content/profile/privacy/index.vue`
- **严重级别**: WARNING
- **描述**: types.ts中定义了onlineStatusVisible为boolean（注释为"旧字段兼容保留"），但隐私页面UI中没有提供该字段的Switch控件，也未在hydrateFromVo中读取
- **影响**: 如果后端返回该字段，前端无法修改或展示；标记为兼容保留但实际未处理
- **修复建议**: 确认是否需要UI入口，若为纯兼容字段可在注释中明确说明不需要UI

#### W-003: spec文件隐私字段与实际实现不一致（文档问题）
- **位置**: `specs/privacy-settings/spec.md:26-34`
- **严重级别**: WARNING
- **描述**: spec列出的15+2个隐私字段名与design/types/实际代码不一致（bioVisibility、themeColorVisibility等不存在；contactBadgeVisibility缺失等）
- **影响**: 按spec进行测试会产生大量失败用例；新人按spec理解字段会产生混淆
- **修复建议**: 更新spec以匹配design D7和实际后端字段
