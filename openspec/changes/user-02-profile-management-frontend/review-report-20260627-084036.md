## 问题列表

### BLOCK 级别问题 (0个)

无阻断性问题。

### FLAG 级别问题 (4个)

#### F-001: 隐私设置spec字段名与design.md/types.ts/实现严重不一致
- **位置**:
  - `specs/privacy-settings/spec.md:28-34` (spec定义)
  - `design.md:89-95` (D7决策)
  - `api/content/profile/types.ts:157-181` (实际类型)
  - `views/content/profile/privacy/index.vue:155-178` (实际实现)
- **严重级别**: FLAG
- **描述**: spec文件列出的15+2个隐私字段与design D7/types.ts/实际实现存在大量不一致：

  | spec中字段 | design/types/实现中字段 | 状态 |
  |-----------|----------------------|------|
  | bioVisibility (基础资料5) | 不存在 | ❌ spec多了 |
  | homepageBackgroundVisibility | homepageVisibility | ❌ 名称不同 |
  | themeColorVisibility | 不存在 | ❌ spec多了 |
  | homepageModuleVisibility | dynamicVisibility | ❌ 名称不同 |
  | certificationVisibility | verificationBadgeVisibility | ❌ 名称不同 |
  | verificationBadgesVisibility(复数) | verificationBadgeVisibility(单数) | ❌ 拼写不同 |
  | profileCompletionVisibility | 不存在 | ❌ spec多了 |
  | profileReviewStatusVisibility | 不存在 | ❌ spec多了 |
  | recentActivityVisibility | browseHistoryVisibility/likeActivityVisibility/favoriteVisibility | ❌ 完全不同 |
  | showMutualFollowersCount | 不存在 | ❌ spec多了 |
  | showRecentActivityHighlight | 不存在 | ❌ spec多了 |
  | 不存在 | contactBadgeVisibility | ❌ spec缺少 |
  | 不存在 | onlineStatusVisible(boolean) | ❌ spec缺少(design中存在) |

- **影响**: 按spec实现会产生大量错误字段，后端无法识别；文档自相矛盾
- **修复建议**: 统一以design.md D7和types.ts/后端Req为准，修正specs/privacy-settings/spec.md中的字段名和分组

#### F-002: profile-editing spec声明update返回Result\<String\>，实际应为Result\<ContentUserProfileVO\>
- **位置**: `specs/profile-editing/spec.md:12` vs `design.md:8,118` vs `api/content/profile/index.ts:36`
- **严重级别**: FLAG
- **描述**: spec中描述"后端返回 `Result<String>`（'更新成功'）"，但proposal.md(行8)、design.md(行113-118)明确说明4个POST端点（含/update）返回`Result<ContentUserProfileVO>`，代码也按`ContentUserProfileVO`类型定义
- **影响**: 如果按spec理解为返回字符串，前端不会使用返回的VO更新本地状态，导致数据不一致
- **修复建议**: 修正spec中update接口返回类型为`Result<ContentUserProfileVO>`

#### F-003: getProfileDetail的viewerUserId参数被定义为必填，但design中为可选
- **位置**: `api/content/profile/index.ts:28` vs `design.md:117`
- **严重级别**: FLAG
- **描述**: design.md API矩阵中标注viewerUserId为可选(`viewerUserId(可选)`)，但代码定义为必填参数`getProfileDetail(ownerUserId: string, viewerUserId: string)`
- **影响**: 在查看他人主页场景（不需要viewerUserId或viewerUserId可能未登录）时无法调用
- **修复建议**: 将viewerUserId改为可选参数：`getProfileDetail(ownerUserId: string, viewerUserId?: string)`

#### F-004: tasks.md声明创建5个独立API文件，实际合并为1个
- **位置**: `tasks.md:11-21` vs `api/content/profile/index.ts`
- **严重级别**: FLAG
- **描述**: tasks 2.1-2.5要求分别创建`api/profile.ts`、`api/homepage.ts`、`api/verification.ts`、`api/privacy.ts`、`api/profile-history.ts`，实际所有API合并在`api/content/profile/index.ts`中
- **影响**: 低（代码功能不受影响，但任务声明与实际不符）
- **修复建议**: 更新tasks.md说明已合并为单一文件，或拆分文件（推荐合并方案，功能内聚合理）
