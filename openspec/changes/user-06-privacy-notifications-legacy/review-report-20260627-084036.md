## 三、问题详情

### BLOCK 级问题 (2个)

**BLOCK-01 [一致性]**: 表名定义与非legacy版本及实际代码不一致
- **位置**: design.md 第103-104行 vs V3.9.1_58__content_privacy_notifications.sql
- **描述**: 
  - design.md Migration Plan定义表名为 `content_user_notification_delivery_log`
  - 实际SQL和非legacy版本表名为 `content_notification_audit_log`
  - design.md定义表名为 `content_third_party_authorization`
  - 实际SQL和非legacy版本表名为 `content_user_third_party_auth`
- **影响**: 如果按此plan实现，会创建与现有实现不同名的表，导致表冲突或重复表
- **修复建议**: 统一表名为实际已使用的 `content_notification_audit_log` 和 `content_user_third_party_auth`

**BLOCK-02 [数据表审核]**: Flyway脚本版本号引用与文件结构存在混淆
- **位置**: design.md File Structure 第80-81行
- **描述**: 
  - design.md引用回滚脚本名为 `V3.9.1_58__content_privacy_notifications_rollback.sql`
  - 实际回滚脚本名为 `R3.9.1_58__content_privacy_notifications_rollback.sql`（Flyway规范用R前缀表示undo）
- **影响**: 按plan创建的文件名不符合Flyway undo脚本命名规范
- **修复建议**: 统一为Flyway规范命名 `R3.9.1_58__content_privacy_notifications_rollback.sql`

### FLAG 级问题 (5个)

**FLAG-01 [接口契约]**: 第三方授权撤销未明确使用HTTP方法
- **位置**: plan.md Step 4.3 vs ContentUserThirdPartyAuthController.java
- **描述**: spec未明确撤销授权使用DELETE还是POST方法，实际代码使用DELETE
- **建议**: 在spec中明确HTTP方法约定，DELETE用于资源撤销符合RESTful规范

**FLAG-02 [接口契约]**: plan.md Source Artifacts路径引用错误（引用epic-06而非user-06）
- **位置**: plan.md 第15-21行
- **描述**: plan.md中Source Artifacts路径写的是 `openspec/changes/epic-06-privacy-notifications/`，但实际路径是 `openspec/changes/user-06-privacy-notifications-legacy/`
- **建议**: 修正路径引用为实际change路径

**FLAG-03 [API命名规范]**: File Structure中Controller类名未明确是否新增独立Controller
- **位置**: design.md File Structure 第72-73行
- **描述**: plan提到扩展ContentUserSettingsController并新增ContentUserThirdPartyAuthController，与实际实现一致，但未明确API路径是否使用/sys/前缀
- **建议**: 明确API路径前缀规范（实际使用/api/v1/content/user/）

**FLAG-04 [跨端一致性]**: 本change无配对frontend，与非legacy版本的关系不明确
- **位置**: change整体
- **描述**: user-06-privacy-notifications有配对的user-06-privacy-notifications-frontend，但legacy版本未说明与前端配对的关系
- **建议**: 在proposal.md中明确本legacy change与非legacy版本的关系（是否为替代/备选方案）

**FLAG-05 [数据表审核]**: 第三方授权表字段与实际实现存在差异
- **位置**: design.md Decision 4 vs 实际ContentUserThirdPartyAuth.java
- **描述**: design.md提到"应用名、授权范围、授权时间、状态和令牌标识"，但实际实体还包含openId/unionId/nickname/avatar/rawDataJson等OAuth标准字段
