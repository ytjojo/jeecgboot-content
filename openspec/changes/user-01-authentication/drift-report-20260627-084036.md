## 四、架构分层审核

### 分层依赖方向检查

```
预期: Controller → BizService → Service → Mapper
实际: Controller → BizService → (Service + Mapper 混合)
```

| 层次 | 文件 | 依赖注入 | 跳层调用 |
|------|------|---------|---------|
| Controller | ContentAuthController.java | ContentAuthBizService, IContentDeviceSessionService | ✅ 基本合规，注入1个Service+1个Biz |
| Controller | ContentRiskControlController.java | IContentRiskControlBizService | ✅ 仅注入Biz |
| Controller | ContentAccountCancellationController.java | IContentAccountCancellationBizService | ✅ 仅注入Biz |
| **Biz** | **ContentAuthBizServiceImpl.java** | **credentialMapper, accountMapper, profileMapper, notificationSettingMapper, thirdPartyAuthMapper, deviceSessionMapper, passwordHistoryMapper (7个Mapper直接注入)** | ❌ **CRITICAL 架构违规** |
| Biz | ContentRiskControlBizServiceImpl.java | (待检查) | - |
| Biz | ContentAccountCancellationBizServiceImpl.java | cancellationRequestMapper, accountMapper (2个Mapper) | ❌ 直接注入Mapper |
| Service | 各Service接口 | extends IService<T> | ✅ |
| Mapper | 各Mapper接口 | extends BaseMapper<T> | ✅ |

### 架构问题详情

**CRITICAL**:
- ContentAuthBizServiceImpl 直接注入 7 个 Mapper
  - 位置: ContentAuthBizServiceImpl.java:65-107
  - 注入清单: ContentUserCredentialMapper, ContentUserAccountMapper, ContentUserProfileMapper, ContentUserNotificationSettingMapper, ContentUserThirdPartyAuthMapper, ContentUserDeviceSessionMapper, ContentUserPasswordHistoryMapper
  - 影响: Biz层直接操作Mapper，跳过了Service层封装，事务边界不清晰，业务逻辑与数据访问耦合
  - 建议: 为每个Mapper对应创建/注入Service接口，Biz层只依赖Service

- ContentAccountCancellationBizServiceImpl 直接注入 2 个 Mapper
  - 位置: ContentAccountCancellationBizServiceImpl.java:49-52
  - 影响: 同样违反分层，但规模较小
  - 建议: 注入IContentCancellationRequestService和IContentUserAccountService

**WARNING**:
- 认证相关代码分散在 auth 包和 user 包中
  - auth包: Content*Account/Credential/PasswordHistory/RiskEvent/CancellationRequest
  - user包: ContentUserThirdPartyAuth, ContentUserProfile, ContentUserDeviceSession等
  - 影响: 包边界不清晰，同一领域概念分散在两个包下
  - 建议: 明确auth和user的包边界，或将认证相关实体统一到auth包

- 第三方登录参数使用@RequestParam而非@RequestBody
  - 位置: ContentAuthController.java:102-111
  - 影响: POST请求参数暴露在URL中，rawJson等长参数可能超出URL长度限制
  - 建议: 创建ContentAuthThirdPartyLoginReq DTO，用@RequestBody接收
