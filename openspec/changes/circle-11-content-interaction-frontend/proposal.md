## Why

circle-core MVP 圈子缺乏内容质量管控、成员互动方式和违规内容处理能力。运营者无法突出优质内容、传达重要信息；成员之间缺少互动提及机制；管理员无法审核加入申请和处理举报。本变更补齐这些能力，使圈子具备基本的运营和治理功能。

## What Changes

- 新增内容置顶/精华功能：支持对圈子内容执行置顶/取消置顶、标记/取消精华，置顶内容排序在前，精华内容展示标识
- 新增圈子公告功能：支持发布圈子公告，顶部展示，同圈子仅一条生效，支持有效期管理
- 新增 @成员功能：发布内容/评论时可 @圈内成员，触发异步通知
- 新增加入申请审核功能：管理员可审核用户加入申请（批准/拒绝），支持超时提醒和批量操作
- 新增内容举报处理功能：成员可举报违规内容，管理员可处理举报（删除内容/忽略/禁言用户）

## Capabilities

### New Capabilities

- `content-pin-featured`: 内容置顶与精华标记功能，包括列表排序、标识展示、权限控制
- `circle-announcement`: 圈子公告发布与展示功能，包括单条生效、有效期、顶部展示
- `mention-member`: 圈子内 @成员功能，包括候选列表、提及解析、通知触发
- `join-request-review`: 加入申请审核功能，包括申请列表、批准/拒绝、超时提醒
- `content-report`: 内容举报处理功能，包括举报提交、处理操作、结果通知

### Modified Capabilities

（无，本次为新增功能，不修改已有 spec）

> **注意**: `circle-announcement` spec 已创建。公告发布/查询/删除接口后端均已实现。@成员查询复用圈子成员列表接口 `GET /api/v1/content/circle/member/list`。

## Impact

- **前端新增页面/组件**：
  - 圈子内容列表页增强（置顶排序、精华标识、公告栏）
  - 内容详情页增强（置顶/精华标识和操作）
  - 公告发布弹窗 + 公告展示栏组件
  - @成员选择器组件
  - 加入申请审核页
  - 举报处理页 + 举报提交弹窗
- **新增 API 对接**：置顶/精华切换（2个）、公告发布/查询/删除（3个）、@成员查询（复用圈子成员列表接口）、申请审核列表/批准/拒绝/待审核列表（4个）、举报提交/列表/删除内容/忽略/禁言（5个），共 14 个接口
- **新增 Store**：`useCircleInteractionStore`（公告状态、@成员列表、审核统计）
- **依赖**：circle-core 已有的内容列表、成员管理、圈子详情页面
