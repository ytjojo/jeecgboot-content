## WARNING 级别问题 (5个)

#### W-001: 11个API接口路径与design/后端不一致，将导致联调失败
- **位置**:
  - `api/content/auth/index.ts:65` confirmEmail
  - `api/content/auth/index.ts:68` loginSmsCode
  - `api/content/account/security.ts:42-55` 10个绑定/设备/重置密码接口
  - `api/content/account/cancellation.ts:21` cancelCancellation
  - `api/content/account/security.ts:56` anomalyList
- **严重级别**: WARNING
- **描述**: 共约11个API路径与design.md定义不一致，部分是前缀错误（/auth/ vs /account-security/），部分是具体路径名错误
- **影响**: 前后端联调时这些接口全部返回404，对应功能不可用
- **修复建议**: 逐一对照design.md修正API路径

#### W-002: getCaptchaImage使用POST方法，design定义为GET
- **位置**: `api/content/auth/captcha.ts:19`
- **严重级别**: WARNING
- **描述**: `getCaptchaImage`使用`defHttp.post`，但design.md:44定义为`GET /api/v1/content/auth/captcha/image`
- **影响**: 图形验证码获取失败
- **修复建议**: 与后端确认Method，若后端为GET则改为`defHttp.get`

#### W-003: 参数字段命名 phone vs mobile 不一致
- **位置**: `api/content/auth/index.ts:4,17,33,41` 等
- **严重级别**: WARNING
- **描述**: 前端所有字段使用`phone`命名（SmsCodeParams.phone, RegisterMobileParams.phone等），后端spec使用`mobile`术语
- **影响**: 请求参数字段名不匹配，后端无法正确接收手机号参数
- **修复建议**: 统一字段命名，前端改为`mobile`或后端适配`phone`

#### W-004: resetPassword参数设计过于简化
- **位置**: `api/content/account/security.ts:93-94`
- **严重级别**: WARNING
- **描述**: resetPassword使用`smsOrEmailCode`单字段和`type`区分，后端可能期望更明确的参数结构（如step1验证标识token + step2新密码）
- **影响**: 密码找回流程可能无法正确走完三步流程
- **修复建议**: 对照后端Req/Vo结构调整参数

#### W-005: 缺少修改密码API在密码找回流程中的调用
- **位置**: tasks.md 7.7提到"修改密码"操作入口
- **严重级别**: WARNING
- **描述**: changePassword API存在，但密码找回流程(specs/password-recovery)使用resetPassword（未登录状态），已登录修改密码场景在account-security页面需要确认是否已实现入口
- **影响**: 低，主要是已登录用户修改密码功能需验证
- **修复建议**: 确认account-security页面是否已实现修改密码弹窗
