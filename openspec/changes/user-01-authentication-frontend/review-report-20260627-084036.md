## 问题列表

### BLOCK 级别问题 (0个)

### FLAG 级别问题 (6个)

#### F-001: API路径前缀严重不一致 — 账号安全接口
- **位置**: `design.md:47-69`（文档定义） vs 实现 `api/content/account/security.ts:42-59`
- **严重级别**: FLAG
- **描述**: design.md 明确定义账号安全接口前缀为 `/api/v1/content/account-security/*`，但实现中大量接口使用 `/api/v1/content/auth/*` 前缀，涉及：
  - bindPhone: 代码为 `/api/v1/content/auth/bind/phone`，设计为 `/api/v1/content/account-security/bind/mobile`
  - rebindPhone: 代码为 `/api/v1/content/auth/rebind/phone`，设计为 `/api/v1/content/account-security/rebind/mobile`
  - unbindPhone: 代码为 `/api/v1/content/auth/unbind/phone`，设计为 `/api/v1/content/account-security/unbind/mobile`
  - bindEmail/rebindEmail/unbindEmail: 同上前缀错误
  - bindThirdParty/unbindThirdParty: 同上前缀错误
  - devices/deviceRevoke: 代码为 `/api/v1/content/auth/devices`，设计为 `/api/v1/content/account-security/devices`
  - resetPassword: 代码为 `/api/v1/content/auth/password/reset`，设计为 `/api/v1/content/account-security/password/reset`
- **影响**: 前后端联调时所有这些接口会404，无法正常工作
- **修复建议**: 统一修正为 `/api/v1/content/account-security/` 前缀，字段名 `phone` → `mobile` 与后端对齐

#### F-002: 验证码登录API路径不一致
- **位置**: `design.md:39` vs `api/content/auth/index.ts:68`
- **严重级别**: FLAG
- **描述**: design.md定义验证码登录路径为 `/api/v1/content/auth/login/sms-code`，实现中为 `/api/v1/content/auth/login/sms`
- **影响**: 验证码登录功能无法调用
- **修复建议**: 修正为 `/api/v1/content/auth/login/sms-code`，与后端对齐

#### F-003: 邮箱确认API路径和Method不一致
- **位置**: `design.md:36` vs `api/content/auth/index.ts:65,82`
- **严重级别**: FLAG
- **描述**: design.md定义邮箱注册确认为 `GET /api/v1/content/auth/register/email/confirm`，实现中为 `POST /api/v1/content/auth/confirm-email`
- **影响**: 邮箱验证链接点击后无法正确回调验证
- **修复建议**: 修正路径为 `/api/v1/content/auth/register/email/confirm`，Method需与后端确认（通常邮件链接为GET）

#### F-004: 图形验证码HTTP Method不一致
- **位置**: `design.md:44` vs `api/content/auth/captcha.ts:14,19`
- **严重级别**: FLAG
- **描述**: design.md定义获取图形验证码为 `GET /api/v1/content/auth/captcha/image`，实现中使用 `defHttp.post`
- **影响**: 可能导致后端无法正确处理（GET用于获取图片资源是更合理的语义）
- **修复建议**: 确认后端Method，若后端为GET则改为 `defHttp.get`

#### F-005: 注销取消API路径不一致
- **位置**: `design.md:78` vs `api/content/account/cancellation.ts:21,32`
- **严重级别**: FLAG
- **描述**: design.md定义取消注销为 `POST /api/v1/content/account-cancellation/revoke`，实现中为 `/api/v1/content/account-cancellation/cancel`
- **影响**: 冷静期内取消注销功能无法调用
- **修复建议**: 修正为 `/api/v1/content/account-cancellation/revoke`

#### F-006: 异常通知列表API路径不一致
- **位置**: `design.md:66` vs `api/content/account/security.ts:56`
- **严重级别**: FLAG
- **描述**: design.md定义为 `GET /api/v1/content/account-security/anomaly-notifications`，实现中为 `/api/v1/content/account-security/anomaly/list`
- **影响**: 消息中心异常通知列表无法加载
- **修复建议**: 修正为 `/api/v1/content/account-security/anomaly-notifications`，与后端对齐

- **影响**: 前后端字段映射可能出错

#### A-005: 验证码冷却组件未独立封装
- **位置**: `tasks.md:30` (任务4.3)

- **影响**: 代码复用性降低
- **修复建议**: 确认是否已封装为独立组件，若内联实现建议抽取
