# 内容社区认证系统差距分析报告

> 分析日期: 2026-05-28
> 对比对象: tasks.md 8 大模块 vs 现有系统（平台层 + 内容模块）

---

## 总览

| 模块 | 已有 | 缺失 | 覆盖率 |
|------|------|------|--------|
| 1. 基础结构与数据迁移 | 2 张表 + entity/mapper | content.auth 包、凭证表、密码历史表、风险事件表、注销申请表 | ~15% |
| 2. 验证码与外部通道适配 | 平台层有图片验证码 + SMS | 内容模块无适配接口、无邮件验证、无冷却机制 | ~20% |
| 3. 注册与登录 | 平台层有完整登录；内容模块有注册编排 | 内容模块无登录逻辑、无邮箱注册确认流 | ~40% |
| 4. 会话与设备管理 | 有 device_session 表基础结构 | 无 token jti、无设备上限、无黑名单、无下线拦截 | ~10% |
| 5. 账号绑定与密码找回 | 内容模块有 bind/unbind/reset 编排 | 无密码历史校验、无换绑双向验证 | ~50% |
| 6. 风控与异常登录 | 平台层有 5 次锁定机制 | 内容模块无任何风控、无风险事件表 | ~5% |
| 7. 账号注销 | **已完整实现**（冷静期 + 取消 + 完成） | 冷静期不可配置（固定 7 天）、无数据匿名化 | ~75% |
| 8. API 收口与验收 | Controller 有基本入参校验 | 无 `/api/v1/` 路径、无 Knife4j 注解、无错误码常量 | ~10% |

**整体预估覆盖率: ~28%**

## 现有平台层能力（可复用/参考）

| 能力 | 平台实现 | 内容模块可复用程度 |
|------|----------|-------------------|
| JWT Token 生成/验证 | `JwtUtil.sign()` / `JwtUtil.getUsername()` | 直接复用 |
| AES 密码解密 | `AesEncryptUtil.resolvePassword()` | 直接复用 |
| 密码加密比对 | `PasswordUtil.encrypt()` | 直接复用 |
| 图片验证码 | `LoginController.randomImage()` + `RandImageUtil` | 参考实现 |
| SMS 验证码 | `LoginController.sms()` + `DySmsHelper` | 参考实现 |
| Redis 缓存 token | `LoginController` 中 Redis 操作 | 直接复用模式 |
| 登录失败锁定 | 5 次 / 10 分钟 | 参数需调整为 tasks.md 要求 |
| Spring Security 配置 | `SecurityConfig` | 需扩展内容模块路径 |
| 用户网关 | `SystemUserAccountGateway` | 已有，需扩展登录能力 |
