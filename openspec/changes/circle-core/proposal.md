## Why

内容社区系统缺乏基于兴趣聚合的垂直社区能力，用户无法围绕特定主题进行深度交流。当前用户只能浏览全局内容流，缺少结构化的兴趣分组和归属感。构建圈子核心基础设施作为 MVP，可以快速验证"兴趣聚合 → 深度交流 → 用户留存"的价值假设，为后续内容互动、推荐和成长激励奠定基础。

## What Changes

**圈子创建与设置**
- 新增圈子创建能力，支持名称、简介、图标、封面图、分类标签等基础属性
- 新增隐私类型配置（公开/私有/密码保护）和加入方式设置（直接加入/申请审核/邀请/密码）
- 圈子名称唯一性校验和敏感词检测

**圈子成员管理**
- 新增成员加入/退出流程，支持多种加入方式
- 新增角色体系（创建者 > 版主 > 普通成员），仅创建者可变更版主角色
- 新增成员禁言（1小时/24小时/7天/永久）和移除能力，禁言到期自动解除
- 治理操作可追溯（操作者、对象、动作、原因、时间，保留≥180天）

**圈子搜索**
- 新增圈子搜索能力，支持按名称和简介关键词模糊匹配
- 私有圈子不展示在搜索结果中
- 搜索响应时间 P95 < 500ms

## Success Criteria

- 圈子创建成功率 > 95%
- 用户加入圈子转化率 > 30%
- 圈子搜索响应时间 P95 < 500ms
- MVP 上线后 4 周内创建且通过合规校验的圈子数达到 50 个
- 核心操作（创建/加入/退出/角色变更/禁言/移除）P95 响应时间 < 800ms
- 单圈子成员上限 10,000 人

## Non-Goals

- 内容置顶/精华、公告/@成员
- 审核机制（人工审核流程）
- 数据统计与分析
- 成长激励体系
- 圈子推荐算法

## Capabilities

### New Capabilities
- `circle-creation`: 圈子创建与设置，包含基础属性填写、隐私类型配置、加入方式设置、名称唯一性校验和敏感词检测
- `circle-member-management`: 圈子成员管理，包含加入/退出流程、角色体系（创建者/版主/成员）、禁言与移除、治理操作追溯
- `circle-search`: 圈子搜索，包含关键词模糊匹配、隐私类型过滤、搜索结果展示

### Modified Capabilities
（无已有能力需要修改）

## Impact

**后端代码**
- 新增圈子模块（jeecg-module-content），包含 Entity、Mapper、Service、Controller
- 新增数据库表：circle（圈子）、circle_member（成员）、circle_governance_log（治理日志）
- 依赖现有用户账号系统和内容发布系统

**API**
- 新增圈子 CRUD API
- 新增成员管理 API（加入/退出/角色变更/禁言/移除）
- 新增圈子搜索 API

**依赖**
- 用户账号系统（已稳定运行）
- 搜索服务（需确认部署状态）
- 敏感词检测服务
